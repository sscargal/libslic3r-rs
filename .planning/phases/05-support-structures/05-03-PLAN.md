  <action>
Create `bridge.rs` implementing bridge detection per user decision — combined approach with all three criteria required.

1. `BridgeRegion` struct:
   - `contour: ValidPolygon` — the bridge region polygon
   - `span_direction: f64` — angle in radians of the primary bridge span direction
   - `span_length: f64` — estimated length of the longest unsupported span in mm
   - `layer_index: usize`
   - `z: f64`

2. `is_bridge_candidate(region: &ValidPolygon, below_contours: &[ValidPolygon], min_span_mm: f64) -> Option<BridgeRegion>`:
   - **Criterion 1 — Angle threshold:** Compute the bounding box of the region. If the region is an overhang (already detected), check if it is near-horizontal. For FDM bridging, the surface normal must be within 10 degrees of horizontal (i.e., the angle from vertical is >= 80 degrees). Since we are working with 2D slices, this criterion is effectively: the overhang region exists on a layer where the layer below has support on both sides.
   - **Criterion 2 — Endpoint support:** The region must have supported material on at least two opposing sides. Check by computing the intersection of the region's bounding box edges with the below_contours. If the below_contours border the region on at least 2 sides (left/right or top/bottom), this criterion is met. Implement by checking if expanding the below_contours by a small amount overlaps with the region's convex hull edges on opposite sides.
   - **Criterion 3 — Minimum span length:** The width of the unsupported gap must be >= `min_span_mm` (default 5mm per user decision). Estimate span by measuring the maximum distance across the region perpendicular to the nearest supported edge.
   - If all 3 criteria met, return `Some(BridgeRegion)` with computed span direction and length. Otherwise `None`.

3. `detect_bridges(overhang_regions: &[ValidPolygon], below_contours: &[ValidPolygon], layer_index: usize, z: f64, min_span_mm: f64) -> (Vec<BridgeRegion>, Vec<ValidPolygon>)`:
   - For each overhang region, check if it is a bridge candidate.
   - Return tuple: (detected bridges, remaining non-bridge overhangs).
   - This separates bridge regions from regular overhang regions so they get different treatment.

4. `compute_bridge_infill_angle(bridge: &BridgeRegion) -> f64`:
   - Compute the optimal infill angle for the bridge: perpendicular to the span direction.
   - Bridge lines should run perpendicular to the gap to minimize unsupported span per line.
   - Return angle in radians.
