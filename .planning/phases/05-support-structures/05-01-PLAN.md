<task type="auto">
  <name>Task 2: Overhang detection with hybrid layer-diff and raycast validation</name>
  <files>

  <action>
Create `detect.rs` implementing the hybrid overhang detection per user decision (layer-by-layer analysis for speed, downward raycast validation for accuracy).

1. `detect_overhangs_layer(current_contours, below_contours, overhang_angle_deg, layer_height, extrusion_width) -> Vec<ValidPolygon>`:
   - Convert overhang angle from degrees to radians.
   - Compute max horizontal offset: `layer_height * tan(overhang_angle_rad)`. This is how far the current layer can extend beyond the layer below before needing support.
   - If below_contours is empty, return empty vec (first layer is supported by bed).
   - Expand below contours outward by `max_offset` using `slicecore_geo::offset_polygons` with `JoinType::Miter`.
   - Compute overhang regions via `slicecore_geo::polygon_difference(current_contours, &expanded_below)`.
   - Return resulting polygons (empty if no overhangs).

2. `validate_overhangs_raycast(candidate_regions, mesh, layer_z, sample_spacing_mm) -> Vec<ValidPolygon>`:
   - For each candidate region, sample points on a grid within the region's bounding box (spacing = sample_spacing_mm, default ~2mm).
   - For each sample point that is inside the region (use `slicecore_geo::point_in_polygon`), cast a ray downward using `mesh.bvh().intersect_ray(origin, direction, vertices, indices)` with origin at (x, y, layer_z) and direction (0, 0, -1).
   - If the ray hits the model surface within a configurable distance (e.g., 5 * layer_height), the point is actually supported by internal geometry — mark it as not needing support.
   - If >50% of sample points in a region are internally supported, remove the entire region.
   - Return only validated overhang regions.

3. `filter_small_regions(regions, min_area_mm2, extrusion_width) -> Vec<ValidPolygon>`:
   - For each region, compute area in mm^2 using `slicecore_geo::signed_area_i64` converted via COORD_SCALE.
   - Discard regions with area < `extrusion_width^2` (unprintable).
   - Keep regions with area >= `min_area_mm2` unchanged.
   - Regions between extrusion_width^2 and min_area_mm2: keep as-is (they will become thin support pillars naturally when sliced into toolpaths). Per research recommendation for two-tier filtering.

4. `detect_all_overhangs(layers, mesh, config) -> Vec<Vec<ValidPolygon>>`:
   - Convenience function that runs detection across all layers.
   - For each layer (skipping layer 0 — first layer on bed):
     a. Run `detect_overhangs_layer` with layer N vs layer N-1.
     b. Run `validate_overhangs_raycast` if regions found (skip if no overhangs for performance).
     c. Run `filter_small_regions`.
   - Returns per-layer overhang region vectors.

Include `#[cfg(test)] mod tests` with:
- Test that a simple overhang at 60 degrees (beyond 45 threshold) is detected.
- Test that a vertical wall (0 degrees from vertical) produces no overhangs.

- Test that overhang detection with identical layers above and below produces no overhangs.

Use existing math primitives: `slicecore_math::{mm_to_coord, coord_to_mm, IPoint2, Point3, Vec3, COORD_SCALE}`, `slicecore_geo::{ValidPolygon, polygon_difference, offset_polygons, signed_area_i64, point_in_polygon, JoinType}`, `slicecore_mesh::TriangleMesh`.
