Create `overhang_perimeter.rs` implementing SUPP-07: 4-tier angle control.

Per research (Open Question #4): overhang tiers are print parameter adjustments, not support placement. They affect speed and fan for perimeters near overhangs that DON'T have support.

1. `OverhangTier` enum: `None`, `Mild` (0-25%), `Moderate` (25-50%), `Steep` (50-75%), `Severe` (75-100%)
   - Percentage refers to the position between vertical (0% overhang) and horizontal (100% overhang).
   - 0-25%: nearly vertical, minimal adjustment needed.
   - 25-50%: moderate overhang, slow down slightly.
   - 50-75%: steep overhang, significant slowdown + more fan.
   - 75-100%: severe overhang (near bridge territory), maximum adjustments.

2. `classify_overhang_tier(overhang_angle_from_vertical: f64) -> OverhangTier`:
   - Convert the local overhang angle to a tier.
   - angle_from_vertical = 0 → `None` (vertical wall, no overhang)
   - angle_from_vertical 0-22.5° → `None` (negligible)
   - angle_from_vertical 22.5-45° → `Mild`
   - angle_from_vertical 45-67.5° → `Moderate`
   - angle_from_vertical 67.5-90° → `Steep`
   - angle_from_vertical >= support_threshold → `None` (these get support instead)

3. `overhang_speed_factor(tier: OverhangTier) -> f64`:
   - `None`: 1.0 (no change)
   - `Mild`: 0.9 (10% slower)
   - `Moderate`: 0.75 (25% slower)
   - `Steep`: 0.5 (50% slower)
   - `Severe`: 0.35 (65% slower)

4. `overhang_fan_override(tier: OverhangTier, base_fan: u8) -> u8`:
   - `None`: base_fan (no change)
   - `Mild`: max(base_fan, 180) (~70%)
   - `Moderate`: max(base_fan, 220) (~86%)
   - `Steep`: 255 (100%)
   - `Severe`: 255 (100%)

5. `classify_perimeter_overhangs(perimeter_contours: &[ValidPolygon], below_contours: &[ValidPolygon], layer_height: f64) -> Vec<OverhangTier>`:
   - For each perimeter contour, estimate the overhang angle relative to the layer below.
   - Compare each contour with the below layer to estimate how much the perimeter extends beyond the support below.
   - Return a tier per contour.

6. `auto_select_support_type(overhang_regions: &[Vec<ValidPolygon>], extrusion_width: f64) -> SupportType`:
   - Per user decision: small contact areas -> tree, large areas -> traditional.
   - Compute total overhang area across all layers.
   - Compute average region area. If average < 10 * extrusion_width^2 (small isolated overhangs), return `SupportType::Tree`.
   - If average >= 10 * extrusion_width^2 (large flat overhangs), return `SupportType::Traditional`.
   - Mixed: if both large and small regions exist, return `SupportType::Auto` (both types used per-region).

Create helper `assemble_support_toolpath(support_regions: &[SupportRegion], config: &PrintConfig, layer_z: f64, layer_height: f64) -> Vec<ToolpathSegment>`:
- For each support region:
  - Generate infill lines (body or interface density).
  - Convert to ToolpathSegments with:
    - FeatureType::Support for body, FeatureType::SupportInterface for interface.

    - Feedrate: bridge.speed * 60.0.
    - E-value: computed with bridge.flow_ratio multiplier.
- Return all support segments (printed AFTER model perimeters/infill per standard ordering).
