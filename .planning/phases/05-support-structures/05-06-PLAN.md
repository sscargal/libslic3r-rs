  <name>Task 1: Volume modifiers and mesh-based enforcer/blocker system</name>

  <action>
Create `override_system.rs` implementing the manual support override system.

1. `VolumeShape` enum: `Box`, `Cylinder`, `Sphere` — geometric primitive types.

2. `OverrideRole` enum: `Enforcer`, `Blocker` — whether the modifier adds or removes support.

3. `VolumeModifier` struct:
   - `shape: VolumeShape`
   - `role: OverrideRole`
   - `center: (f64, f64, f64)` — center position in mm
   - `size: (f64, f64, f64)` — dimensions in mm (width, depth, height for box; radius, radius, height for cylinder; radius, radius, radius for sphere)
   - `rotation: f64` — rotation around Z in degrees (for box/cylinder alignment)
   - Derive Serialize, Deserialize for config/API use.

4. `volume_modifier_at_z(modifier: &VolumeModifier, z: f64) -> Option<ValidPolygon>`:
   - Computes the 2D cross-section of the volume modifier at height z.
   - `Box`: if z is within [center.z - size.z/2, center.z + size.z/2], return a rectangle polygon. Apply rotation if specified.
   - `Cylinder`: if z is within height range, return a circle polygon (approximated with 32 vertices).
   - `Sphere`: if z is within [center.z - radius, center.z + radius], return a circle polygon with radius = sqrt(R^2 - (z - center.z)^2).
   - Return None if z is outside the volume.

5. `MeshOverride` struct:
   - `mesh: TriangleMesh` — the enforcer or blocker mesh
   - `role: OverrideRole`
   - `sliced_regions: Vec<Vec<ValidPolygon>>` — pre-sliced regions per layer (computed once)

6. `slice_override_mesh(mesh: &TriangleMesh, role: OverrideRole, layer_heights: &[(f64, f64)]) -> MeshOverride`:
   - Slice the enforcer/blocker mesh at each layer height using `slicecore_slicer::slice_at_height`.
   - Store the resulting contour polygons per layer.
   - This allows mesh-based regions to be applied per-layer.

7. `apply_overrides(auto_support: &mut [Vec<ValidPolygon>], enforcers: &[MeshOverride], blockers: &[MeshOverride], volume_modifiers: &[VolumeModifier], layer_heights: &[(f64, f64)]) -> Vec<ConflictWarning>`:
   - For each layer:
     a. Apply enforcers first: `polygon_union(auto_support[layer], enforcer_regions[layer])` — adds support where enforcers specify.
     b. Apply volume modifier enforcers: union with volume cross-sections at this Z.
     c. Apply blockers: `polygon_difference(support[layer], blocker_regions[layer])` — removes support where blockers specify.
     d. Apply volume modifier blockers: difference with volume cross-sections at this Z.
   - Blocker always wins (applied after enforcer) per user decision default ("Warn on conflicts").
   - Return list of ConflictWarning for cases where blockers removed auto-detected critical support.

**mod.rs** — Add `pub mod override_system;` and add `enforcers: Vec<MeshOverride>`, `blockers: Vec<MeshOverride>`, `volume_modifiers: Vec<VolumeModifier>` fields to SupportConfig or add a separate `SupportOverrides` struct.

Include tests:
- Test box volume modifier at Z within range produces rectangle polygon.
- Test box volume modifier at Z outside range produces None.
- Test sphere cross-section at equator is maximum radius, at poles is zero.
- Test enforcer union: support region grows when enforcer covers an area without auto-support.
- Test blocker difference: support region shrinks when blocker overlaps auto-support.
- Test blocker wins over enforcer: if both overlap same region, result has no support there.

<task type="auto">
  <name>Task 2: Conflict detection and smart merge</name>
  <files>

  <action>
Create `conflict.rs` implementing conflict detection and resolution.

1. `ConflictWarning` struct:
   - `layer_index: usize`
   - `conflict_type: ConflictType` — enum: `BlockerRemovesCritical`, `EnforcerDuplicatesAuto`, `OverlapConflict`
   - `message: String` — human-readable description
   - `affected_area_mm2: f64` — area of the conflicting region

2. `detect_conflicts(auto_support: &[Vec<ValidPolygon>], overridden_support: &[Vec<ValidPolygon>], overhang_regions: &[Vec<ValidPolygon>]) -> Vec<ConflictWarning>`:
   - Compare auto-generated support with post-override support.
   - For each layer, check if blockers removed support that was under a critical overhang (overhang angle > 60 degrees or large overhang area).
   - Generate `BlockerRemovesCritical` warnings for dangerous removals.
   - Generate `EnforcerDuplicatesAuto` info for enforcers that exactly match auto-support (no effect).

3. `smart_merge(auto_support: &[Vec<ValidPolygon>], enforcer_regions: &[Vec<ValidPolygon>], blocker_regions: &[Vec<ValidPolygon>], overhang_regions: &[Vec<ValidPolygon>]) -> (Vec<Vec<ValidPolygon>>, Vec<ConflictWarning>)`:
   - Per user decision "Smart merge" mode: combines auto + manual intelligently.
   - Start with auto support.
   - Add enforcer regions via union.
   - For blockers overlapping critical overhangs: keep minimal support (reduce density instead of fully removing). Use polygon_intersection to find overlap, keep a reduced version.
   - For blockers on non-critical areas: remove fully.
   - Return merged regions and warnings about what was adjusted.

**mod.rs** — Add `pub mod conflict;`.

Include tests:
- Test conflict detection: blocker removing support under steep overhang generates BlockerRemovesCritical warning.
- Test conflict detection: blocker removing support under mild overhang generates no warning.
- Test smart merge: blocker on critical overhang keeps reduced support instead of removing.
- Test smart merge: blocker on non-critical area fully removes support.
