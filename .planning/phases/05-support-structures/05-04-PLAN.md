  <name>Task 1: TreeNode data structure and branch merging logic</name>
  <files>
    crates/slicecore-engine/src/support/tree_node.rs
    crates/slicecore-engine/src/support/mod.rs
  </files>
  <action>
Create `tree_node.rs` with the tree support data structures and algorithms.

1. `TreeNode` struct:
   - `position: (f64, f64)` — XY position in mm
   - `z: f64` — Z height in mm
   - `radius: f64` — radius at this node in mm
   - `children: Vec<usize>` — indices into a flat node arena (not recursive pointers — follows project pattern)
   - `parent: Option<usize>` — index of parent node
   - `is_contact: bool` — true if this node touches the overhang surface
   - `is_root: bool` — true if this node is on the build plate (z=0)

2. `TreeSupportArena` struct:
   - `nodes: Vec<TreeNode>` — flat arena of all nodes
   - Methods: `add_node(node) -> usize`, `get_node(idx) -> &TreeNode`, `get_node_mut(idx) -> &mut TreeNode`
   - `children_of(idx) -> &[usize]`

3. `compute_taper(base_radius: f64, tip_radius: f64, z: f64, total_height: f64, method: TaperMethod) -> f64`:
   - `Linear`: `tip_radius + (base_radius - tip_radius) * (1.0 - z / total_height)` — wider at bottom
   - `Exponential`: `tip_radius + (base_radius - tip_radius) * (1.0 - (z / total_height)).powi(2)` — rapid widening near base
   - `LoadBased`: width proportional to estimated load above. At each Z, count contact points above and scale radius proportionally. `tip_radius + (base_radius - tip_radius) * (contacts_above as f64 / total_contacts as f64).sqrt()`
   - `Auto`: default to `Linear`

4. `merge_nearby_branches(arena: &mut TreeSupportArena, merge_distance: f64, max_diameter: f64)`:
   - For each pair of root nodes (on build plate), if distance < `merge_distance`:
     a. Merge into one trunk positioned at the midpoint.
     b. Children of both original roots become children of merged trunk.
     c. Verify merged trunk radius doesn't exceed `max_diameter / 2`.
     d. If it would, skip merge.
   - Use greedy nearest-neighbor: sort all root pairs by distance, merge closest first.
   - Per research: merge within `merge_distance_factor * trunk_diameter` (default 3x), minimum 5mm.

5. `compute_branch_angle(parent_pos: (f64, f64), child_pos: (f64, f64), parent_z: f64, child_z: f64) -> f64`:
   - Returns angle in degrees between trunk vertical and branch direction.
   - Used to enforce minimum 15-degree divergence per research.

**mod.rs** — Add `pub mod tree_node;` and `pub mod tree;`.

Include tests:
- Test linear taper: radius at z=0 equals base_radius, at z=height equals tip_radius.
- Test exponential taper: radius at z=0 equals base_radius, decreases faster near top.
- Test merge: two roots within merge distance produce one merged root.
- Test merge: two roots beyond merge distance remain separate.
- Test branch angle computation: vertical branch = 0 degrees, 45-degree offset = 45 degrees.

<task type="auto">
  <name>Task 2: Bottom-up tree growth and per-layer slicing</name>
  <files>
    crates/slicecore-engine/src/support/tree.rs
  </files>
  <action>
Create `tree.rs` implementing the bottom-up tree growth algorithm.

1. `extract_contact_points(overhang_regions: &[Vec<ValidPolygon>], layers: &[SliceLayer], extrusion_width: f64) -> Vec<(f64, f64, f64)>`:
   - From overhang regions, extract representative contact points (x, y, z) where tree branches need to reach.
   - Sample points along overhang region boundaries at `extrusion_width * 3` spacing.
   - Each point represents a place where a tree branch tip must arrive.

2. `grow_tree(contact_points: &[(f64, f64, f64)], model_contours_per_layer: &[Vec<ValidPolygon>], layer_heights: &[f64], config: &TreeSupportConfig, extrusion_width: f64) -> TreeSupportArena`:
   - **Step 1**: For each contact point, create a contact node at (x, y, z).
   - **Step 2**: Project each contact point straight down to find the trunk base position. If the model is in the way (polygon_difference check), offset the trunk position laterally until clear.
   - **Step 3**: Create root node at (base_x, base_y, 0.0) on build plate.
   - **Step 4**: Connect root to contact via intermediate nodes at each layer height.
   - **Step 5**: At each intermediate layer, check for collision with model contours using polygon_difference (branch circle at that layer vs model contours). If collision, adjust branch position laterally.
   - **Step 6**: Run `merge_nearby_branches` to combine nearby trunks.
   - **Step 7**: Apply taper (compute_taper) to all nodes.

3. `apply_branch_style(arena: &mut TreeSupportArena, style: TreeBranchStyle)`:
   - `Geometric`: straight line segments between nodes (already the default from grow_tree).
   - `Organic`: insert intermediate nodes with Bezier-curve-like interpolation between branch points. Add control points that create smooth curves.
   - `Auto`: use geometric for flat overhang surfaces (low curvature), organic for curved surfaces. Estimate surface curvature from contact point distribution.

4. `slice_tree_to_layers(arena: &TreeSupportArena, layer_count: usize, layer_heights: &[f64]) -> Vec<Vec<ValidPolygon>>`:
   - For each layer, find all tree nodes at that Z height (within layer_height/2 tolerance).
   - For each node, generate a circular polygon with the node's radius.
   - Union all circles at each layer into support region polygons.
   - Return per-layer support regions from the tree.

5. `generate_tree_supports(overhang_regions: &[Vec<ValidPolygon>], layers: &[SliceLayer], config: &SupportConfig, extrusion_width: f64) -> Vec<Vec<SupportRegion>>`:
   - Main entry point combining all steps.
   - Extract contact points, grow tree, apply style, slice to layers, generate infill per layer.
   - Use same support infill generation as traditional (from Plan 02) on the tree-produced per-layer regions.

Include tests:
- Test that a single contact point above the bed produces a trunk from bed to contact point.
- Test that two nearby contact points share a merged trunk.
- Test that tree nodes do not overlap with a model contour (collision avoidance).
- Test that slice_tree_to_layers produces circular polygons at the correct Z heights.
- Test geometric vs organic: organic has more intermediate nodes than geometric.
  </action>
