  <action>
Create `interface.rs` implementing support interface layers.

1. `Material` enum: `Pla`, `Petg`, `Abs`, `Tpu`, `Nylon`, `Generic` — with serde support.

2. `MaterialDefaults` struct with method `for_material(material: Material) -> MaterialGaps`:
   - Returns `MaterialGaps { z_gap: f64, xy_gap: f64 }` with material-specific defaults:
     - PLA: z_gap=0.2mm, xy_gap=0.4mm (per research)
     - PETG: z_gap=0.25mm, xy_gap=0.4mm
     - ABS: z_gap=0.2mm, xy_gap=0.4mm
     - TPU: z_gap=0.3mm, xy_gap=0.5mm
     - Nylon: z_gap=0.25mm, xy_gap=0.4mm
     - Generic: z_gap=0.2mm, xy_gap=0.4mm

3. `identify_interface_layers(support_regions: &[Vec<ValidPolygon>], model_contours: &[Vec<ValidPolygon>], interface_layer_count: u32) -> Vec<bool>`:
   - For each layer with support regions, determine if it is an interface layer.
   - Interface layers are the topmost `interface_layer_count` layers of each support column (layers closest to the model surface).
   - A support layer is "near the model" if the layer above it has model contours that overlap with the support region. Specifically: if at layer N there is support, check if layer N+1 or N+2 (within interface_layer_count) has model contours. If yes, layer N is an interface layer.
   - Also identify bottom interface layers: the bottommost `interface_layer_count` layers of each support column (near the build plate or near model surface below). These are less critical but improve adhesion.
   - Return per-layer boolean flags (true = interface layer, false = body layer).

4. `apply_z_gap(support_regions: &mut [Vec<ValidPolygon>], model_contours: &[Vec<ValidPolygon>], z_gap_mm: f64, layer_height: f64)`:
   - Remove the topmost N layers of support where N = ceil(z_gap / layer_height).
   - This creates a physical gap between the top of the support and the bottom of the model.
   - For partial gaps (z_gap not an exact multiple of layer_height), remove the top layer if z_gap > 0.5 * layer_height.

5. `generate_interface_infill(interface_regions: &[ValidPolygon], density: f64, pattern: InterfacePattern, layer_index: usize, extrusion_width: f64) -> Vec<InfillLine>`:
   - Generate dense infill for interface layers.
   - `Rectilinear` (default per Claude's discretion): use existing rectilinear infill at interface density (80% default).
   - `Grid`: cross-hatched at interface density.
   - `Concentric`: inward offset rings following region contour (similar to brim generation pattern).
   - Interface density is much higher than body density (80% vs 15% per research).

6. `apply_quality_preset(config: &mut SupportConfig)`:
   - If `config.quality_preset` is Some, apply the preset values:
     - Low: density=0.10, interface_density=0.50, z_gap=0.30, interface_layers=1
     - Medium: density=0.15, interface_density=0.80, z_gap=0.20, interface_layers=2
     - High: density=0.20, interface_density=1.0, z_gap=0.15, interface_layers=3
   - Preset overrides individual settings. This function should be called early in pipeline.
