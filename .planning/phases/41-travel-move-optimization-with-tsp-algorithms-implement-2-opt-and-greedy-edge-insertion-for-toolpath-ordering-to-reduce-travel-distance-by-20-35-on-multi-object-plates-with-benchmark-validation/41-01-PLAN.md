2. **TspNode struct:**
```rust
/// A node in the TSP problem representing a printable element.
#[derive(Debug, Clone)]
pub struct TspNode {
    /// Entry point (where the nozzle arrives to start printing).
    pub entry: Point2,
    /// Exit point (where the nozzle is after printing this element).
    pub exit: Point2,
    /// Whether this path can be reversed (open paths like infill lines).
    pub reversible: bool,
    /// Index back to the original element for reordering.
    pub original_index: usize,
}
```

3. **Tour struct** holding ordered node indices and reversed flags:
```rust
#[derive(Debug, Clone)]
pub struct Tour {
    /// Ordered indices into the TspNode slice.
    order: Vec<usize>,
    /// Whether each node in the tour is reversed (only meaningful for reversible nodes).
    reversed: Vec<bool>,
}
```
With methods: `len()`, `total_distance(matrix, nodes)` (uses sqrt only here), `to_permutation()` returning `Vec<(usize, bool)>` of (original_index, reversed).

4. **DistanceMatrix struct** using flat `Vec<f64>` with squared distances:
- `new(nodes: &[TspNode])` -- compute n*n matrices for normal and reversed-entry
- `dist_sq(from, to)` -- squared distance from exit of `from` to entry of `to`
- `dist_sq_reversed(from, to)` -- squared distance from exit of `from` to reversed-entry (i.e., exit) of `to`
- Store actual (non-squared) distances in the matrix for correct 2-opt delta computation. Use `(dx*dx + dy*dy).sqrt()` during matrix construction (n^2 sqrts is acceptable at construction time).

5. **nearest_neighbor function:**
- Takes `&DistanceMatrix`, `&[TspNode]`, `start_pos: Point2` -> `Tour`
- Iterates: for each unvisited node, compute min of dist-to-entry and dist-to-reversed-entry (for reversible). Pick closest. Mark visited. Update current_pos to exit (or reversed exit).
- Start from `start_pos` (compute initial distances to all nodes).

6. **greedy_edge_insertion function:**
- Takes `&DistanceMatrix`, `&[TspNode]` -> `Tour`
- Generate all candidate edges (i, j, distance, j_reversed) including reversed variants for reversible nodes.
- Sort by distance ascending.
- Use UnionFind (implemented inline as private struct) to detect cycles.
- Track in-degree and out-degree per node (max 1 each).
- Add edges greedily. Final step: connect path fragments into a single tour.

7. **UnionFind private struct** with `new(n)`, `find(&mut self, x) -> usize`, `union(&mut self, x, y) -> bool`, `connected(&mut self, x, y) -> bool`. Path compression + union by rank.

8. **two_opt_improve function:**
- Takes `&mut Tour`, `&DistanceMatrix`, `&[TspNode]`, `max_iterations: u32`
- On each pass, try all (i, j) pairs where j > i+1. Compute delta of reversing tour[i+1..=j].
- Delta computation must account for reversible node direction flips within the reversed segment.
- If delta < -1e-9, apply the reversal (reverse order of indices, flip reversed flags for reversible nodes in the segment).
- Stop when no improvement found or max_iterations reached.
- For n <= 30, when called from Auto mode, 2-opt both initial tours. For n > 30, only 2-opt the shorter one.

9. **optimize_tour public function:**
```rust
pub fn optimize_tour(
    nodes: &[TspNode],
    start_pos: Point2,
    config: &TravelOptConfig,
) -> Vec<(usize, bool)>
```
- Returns Vec of (original_index, reversed) tuples in optimized order.
- Short-circuits for n <= 1.
- For n == 2: compare both orderings and return shorter.
- Dispatches based on `config.algorithm`:
  - `Auto`: build NN tour and greedy tour, pick shorter. For n <= 30, 2-opt both and pick best. For n > 30, 2-opt only the shorter.
  - `NearestNeighbor`: NN + 2-opt
  - `GreedyEdgeInsertion`: greedy + 2-opt
  - `NearestNeighborOnly`: NN only
  - `GreedyOnly`: greedy only
