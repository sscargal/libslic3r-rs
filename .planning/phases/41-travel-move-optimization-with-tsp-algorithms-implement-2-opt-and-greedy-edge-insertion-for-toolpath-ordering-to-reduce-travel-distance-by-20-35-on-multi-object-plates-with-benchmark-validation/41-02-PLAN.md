Modify `assemble_layer_toolpath` in toolpath.rs to use the TSP optimizer for each feature group. The key change: instead of iterating perimeters/gap_fills/infill in their original order, build TspNode lists and call `optimize_tour` to get optimized ordering.

1. Add import at top of toolpath.rs:
```rust
use crate::travel_optimizer::{optimize_tour, TspNode};
```

2. **Perimeters section** (replaces the `for contour_perims in perimeters` block starting at line ~178):

Before emitting perimeter segments, build a flat list of all polygon entries across all contours/shells. Each polygon becomes a TspNode:
- For each contour_perims -> for each shell -> for each polygon: compute seam_pt, create TspNode with entry=seam_pt, exit=seam_pt (closed path), reversible=false, original_index = flat index.
- Store the polygon data alongside: (contour_idx, shell_idx, polygon_idx, seam_idx, polygon reference).
- Call `optimize_tour(&perim_nodes, current_pos.unwrap_or(Point2::new(0.0, 0.0)), &config.travel_opt)` if enabled and len > 1.
- Iterate the optimized order, emitting segments for each polygon using the existing polygon-to-segments logic.
- If travel_opt disabled, fall back to original sequential order.

The perimeter assembly logic (seam selection, scarf joint, e-value computation) stays the same -- only the VISIT ORDER changes.

Important: seam selection must happen BEFORE TspNode creation so the seam point is known for the entry/exit.

3. **Gap fill section** (replaces the `for gap_path in gap_fills` block starting at line ~283):

Build TspNode list from gap fill paths:
- entry = first point of gap_path (converted to mm), exit = last point of gap_path (converted to mm)
- reversible = true (gap fills are open paths)
- Call optimize_tour if enabled and len > 1.
- When a gap fill path is reversed, iterate its points in reverse order.

4. **Infill section** (replaces the `nearest_neighbor_order` call at line ~353):

Replace the existing `nearest_neighbor_order()` call with TSP optimizer:
- Build TspNode from each InfillLine: entry=start.to_mm(), exit=end.to_mm(), reversible=true
- Call optimize_tour if enabled and len > 1.
- When reversed flag is set, swap start/end of the infill line.
- If travel_opt disabled, fall back to existing `nearest_neighbor_order()` for backward compatibility.

5. **Travel distance tracking** (for TravelOptStats):

Add a helper function or inline computation that calculates total travel distance for a given segment ordering. This is used to compare baseline vs optimized:
- Before optimization: compute sum of travel distances in original order
- After optimization: the actual travel distances are in the emitted segments
- Return both values from `assemble_layer_toolpath` (expand the return tuple to include baseline and optimized travel distances per layer).

Change the return type from `(LayerToolpath, Option<IPoint2>)` to `(LayerToolpath, Option<IPoint2>, f64, f64)` where the last two are baseline_travel_mm and optimized_travel_mm for this layer.

IMPORTANT constraints:
- Feature group order MUST be preserved: perimeters first, then gap fill, then infill
- Wall order within each contour MUST be preserved (inner-first vs outer-first)
- Seam point selection MUST happen before optimizer sees the polygon
- The optimizer only reorders WHICH polygon/path to visit next, not segments within a polygon

6. Update all call sites of `assemble_layer_toolpath` in engine.rs to handle the expanded return type. Accumulate per-layer travel distances into running totals and populate `TravelOptStats` on the `SliceResult` at the end.
