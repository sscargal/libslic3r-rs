**anthropic.rs** -- AnthropicProvider:
- Struct holds: `client: reqwest::Client`, `api_key: SecretString`, `model: String`
- Constructor: `new(api_key: SecretString, model: String, timeout: std::time::Duration)`
- Key differences from OpenAI:
  - Endpoint: POST `https://api.anthropic.com/v1/messages`
  - Auth: `x-api-key` header (NOT Bearer)
  - Version header: `anthropic-version: 2023-06-01`
  - System prompt: top-level `system` field in request body (NOT a message)
  - Response: `content` is array of blocks `[{type: "text", text: "..."}]`
  - No direct response_format equivalent -- use explicit "respond only with JSON" in system prompt
- Internal types:
  - `AnthropicRequest { model, max_tokens, system: String, messages, temperature: Option<f32> }`
  - `AnthropicResponse { content: Vec<AnthropicContent>, model, stop_reason, usage: AnthropicUsage }`
  - `AnthropicContent { content_type: String (renamed from "type"), text: String }`
  - `AnthropicUsage { input_tokens: u32, output_tokens: u32 }`
- Implement AiProvider:
  - Map system_prompt to top-level system field
  - Only include user/assistant messages (not system messages)
  - Extract first text block from content array
  - Map stop_reason "end_stop" -> Stop, "max_tokens" -> Length
  - `capabilities()`: structured_output=true, streaming=true, max_context_tokens=200_000
  - `name()`: "anthropic"

**mod.rs** -- Provider module and factory:
```rust
pub mod openai;
pub mod anthropic;
pub mod ollama;

pub use openai::OpenAiProvider;
pub use anthropic::AnthropicProvider;
pub use ollama::OllamaProvider;

use crate::{AiConfig, AiError, AiProvider, ProviderType};
use secrecy::SecretString;
use std::time::Duration;

pub fn create_provider(config: &AiConfig) -> Result<Box<dyn AiProvider>, AiError> {
    let timeout = Duration::from_secs(config.timeout_secs);
    match config.provider {
        ProviderType::OpenAi => {
            let api_key = config.api_key.clone()
                .ok_or(AiError::MissingApiKey("OpenAI"))?;
            Ok(Box::new(OpenAiProvider::new(api_key, config.model.clone(), config.base_url.clone(), timeout)))
        }
        ProviderType::Anthropic => {
            let api_key = config.api_key.clone()
                .ok_or(AiError::MissingApiKey("Anthropic"))?;
            Ok(Box::new(AnthropicProvider::new(api_key, config.model.clone(), timeout)))
        }
        ProviderType::Ollama => {
            Ok(Box::new(OllamaProvider::new(config.base_url.clone(), config.model.clone(), timeout)))
        }
    }
}
```

Note: SecretString does not impl Clone by default. Use `SecretString::new(api_key.expose_secret().to_string())` or store api_key as `Arc<SecretString>` if Clone is needed. Check secrecy 0.10 API -- it may have changed. If SecretString doesn't clone, accept owned SecretString in constructors and have create_provider build new SecretStrings from expose_secret().

**ollama.rs** -- OllamaProvider:
- Struct holds: `client: reqwest::Client`, `base_url: String`, `model: String`
- Constructor: `new(base_url: Option<String>, model: String, timeout: std::time::Duration)`
  - Default base_url: "http://localhost:11434"
  - No API key needed (local service)
- Key differences from OpenAI/Anthropic:
  - Endpoint: POST `{base_url}/api/chat`
  - No auth headers required
  - `stream: false` in request body (we don't want streaming)
  - System prompt sent as a message with role "system" (like OpenAI)
  - JSON structured output: `format` field with JSON schema value (not a type string)
  - Response: single `message` object (not choices array), plus optional `eval_count`/`prompt_eval_count`
- Internal types:
  - `OllamaRequest { model, messages: Vec<OllamaMessage>, stream: bool, format: Option<serde_json::Value>, options: Option<OllamaOptions> }`
  - `OllamaMessage { role: String, content: String }`
  - `OllamaOptions { temperature: f32, num_predict: u32 }` (num_predict = max tokens)
  - `OllamaResponse { message: OllamaResponseMessage, model: String, done: bool, eval_count: Option<u32>, prompt_eval_count: Option<u32> }`
  - `OllamaResponseMessage { role: String, content: String }`
- Implement AiProvider:
  - Map system_prompt + messages into OllamaMessage vec (system first, then user/assistant)
  - Set stream=false always
  - For Json ResponseFormat: set format to `serde_json::json!({"type": "object"})`. For JsonSchema: pass the schema value directly as format.
  - Map eval_count to completion_tokens, prompt_eval_count to prompt_tokens (use 0 if None)
  - FinishReason: always Stop if `done=true`, Other("incomplete") if done=false
  - `capabilities()`: structured_output=true, streaming=false (we always use non-streaming), max_context_tokens=32_000
  - `name()`: "ollama"
