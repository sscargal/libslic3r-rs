Key choices:
- reqwest with `rustls-tls` (not native-tls) for pure Rust TLS, matching project's pure-Rust philosophy
- reqwest `default-features = false` to avoid pulling in native-tls
- async-trait for dyn AiProvider dispatch (native async fn in trait doesn't support dyn dispatch)
- secrecy 0.10 with serde feature for SecretString deserialization
- tokio with minimal features (rt + macros), not full

Do NOT add slicecore-mesh or slicecore-math dependencies yet -- those come in Plan 03 (geometry analysis).

<task type="auto">
  <name>Task 2: Implement core types, AiProvider trait, config, and error handling</name>
  <files>crates/slicecore-ai/src/lib.rs, crates/slicecore-ai/src/error.rs, crates/slicecore-ai/src/config.rs, crates/slicecore-ai/src/provider.rs, crates/slicecore-ai/src/types.rs</files>
  <action>
Create 5 source files:

**error.rs** -- AiError enum using thiserror:
- `HttpError(reqwest::Error)` -- network/connection failures
- `ProviderError { status: u16, body: String }` -- non-2xx HTTP response
- `ParseError(reqwest::Error)` -- response body parsing failure
- `InvalidJsonResponse(String)` -- LLM returned non-JSON or invalid JSON
- `EmptyResponse` -- no choices/content in response
- `MissingApiKey(&'static str)` -- provider requires API key but none configured
- `RuntimeError(String)` -- tokio runtime creation failure
- `ValidationError(String)` -- response value out of valid range
- All variants should have human-readable Display messages.

**types.rs** -- Request/response types:
- `CompletionRequest { system_prompt: String, messages: Vec<Message>, temperature: f32, max_tokens: u32, response_format: Option<ResponseFormat> }`
- `CompletionResponse { content: String, model: String, usage: Usage, finish_reason: FinishReason }`
- `Message { role: Role, content: String }`
- `Role` enum: System, User, Assistant
- `ResponseFormat` enum: Json, JsonSchema(serde_json::Value)
- `Usage { prompt_tokens: u32, completion_tokens: u32 }`
- `FinishReason` enum: Stop, Length, Other(String)
- `ProviderCapabilities { supports_structured_output: bool, supports_streaming: bool, max_context_tokens: u32 }`
- All types derive Debug, Clone, Serialize, Deserialize where appropriate.

**provider.rs** -- AiProvider trait:
```rust
#[async_trait::async_trait]
pub trait AiProvider: Send + Sync {
    async fn complete(&self, request: &CompletionRequest) -> Result<CompletionResponse, AiError>;
    fn capabilities(&self) -> ProviderCapabilities;
    fn name(&self) -> &str;
}
```

**config.rs** -- AiConfig:
- `ProviderType` enum: OpenAi, Anthropic, Ollama (with serde snake_case rename)
- `AiConfig { provider: ProviderType, model: String, api_key: Option<SecretString>, base_url: Option<String>, timeout_secs: u64 }` with Default impl
- Default: provider = Ollama, model = "llama3.2", timeout_secs = 30, api_key = None, base_url = None
- Implement custom Debug for AiConfig that redacts api_key (shows "[REDACTED]" if Some, "None" if None)
- `AiConfig::from_toml(toml_str: &str) -> Result<Self, toml::de::Error>` for TOML parsing
- Add `toml = "0.8"` to dependencies if needed for AiConfig TOML parsing

**lib.rs** -- Public API:
```rust
pub mod config;
pub mod error;
pub mod provider;
pub mod types;

pub use config::{AiConfig, ProviderType};
pub use error::AiError;
pub use provider::AiProvider;
pub use types::{
    CompletionRequest, CompletionResponse, FinishReason, Message,
    ProviderCapabilities, ResponseFormat, Role, Usage,
};
```

Important: Do NOT implement any providers yet (that's Plan 02). Do NOT add geometry types (that's Plan 03). This plan only creates the abstractions.
