3. **ProfileCommand enum** with `#[derive(Subcommand)]`:
   - `Clone { source: String, name: String, #[arg(long)] force: bool, #[arg(long)] r#type: Option<String>, #[arg(long)] profiles_dir: Option<PathBuf> }`
   - `Set { name: String, key: String, value: String, #[arg(long)] profiles_dir: Option<PathBuf> }`
   - `Get { name: String, key: String, #[arg(long)] profiles_dir: Option<PathBuf> }`
   - `Reset { name: String, key: String, #[arg(long)] profiles_dir: Option<PathBuf> }`
   - `Edit { name: String, #[arg(long)] profiles_dir: Option<PathBuf> }`
   - `Validate { name: String, #[arg(long)] profiles_dir: Option<PathBuf>, #[arg(long)] json: bool }`
   - `Delete { name: String, #[arg(long)] yes: bool, #[arg(long)] profiles_dir: Option<PathBuf> }`
   - `Rename { old_name: String, new_name: String, #[arg(long)] profiles_dir: Option<PathBuf> }`
   - `List { #[arg(long)] vendor: Option<String>, #[arg(long, value_name = "TYPE")] profile_type: Option<String>, #[arg(long)] material: Option<String>, #[arg(long)] vendors: bool, #[arg(long)] profiles_dir: Option<PathBuf>, #[arg(long)] json: bool }` (alias for top-level list-profiles)
   - `Show { id: String, #[arg(long)] raw: bool, #[arg(long)] profiles_dir: Option<PathBuf> }` (alias for top-level show-profile)
   - `Search { query: String, #[arg(short, long, default_value = "20")] limit: usize, #[arg(long)] profiles_dir: Option<PathBuf>, #[arg(long)] json: bool }` (alias for top-level search-profiles)
   - `Diff(crate::diff_profiles_command::DiffProfilesArgs)` (alias for top-level diff-profiles)

   Each variant should have a `///` doc comment matching what the top-level CONTEXT.md specifies.

4. **`pub fn run_profile_command(cmd: ProfileCommand) -> Result<(), anyhow::Error>`** dispatcher:
   - Match on each variant, calling the appropriate handler function
   - For Clone: call `cmd_clone()`
   - For Set/Get/Reset/Edit/Validate/Delete/Rename: call stub functions that return `anyhow::bail!("profile {} not yet implemented", "set"|"get"|etc.)` -- Plan 02 will implement these
   - For List/Show/Search/Diff: delegate to existing top-level handlers (these will be wired in Plan 02)

5. **`fn is_valid_profile_name(name: &str) -> bool`:**
   - Returns true if: not empty, length <= 128, all chars are ASCII alphanumeric or `-` or `_`, does not start with `-`
   - This is the regex equivalent of `^[a-zA-Z0-9_][a-zA-Z0-9_-]{0,127}$`

6. **`fn try_resolve_any(resolver: &ProfileResolver, query: &str, type_hint: Option<&str>) -> Result<ResolvedProfile, anyhow::Error>`:**
   - If `type_hint` is `Some(t)`, call `resolver.resolve(query, t)` directly and map errors with `anyhow`
   - If `type_hint` is `None`, try each type in `["machine", "filament", "process"]`:
     - On `Ok(resolved)` -> collect into `matches: Vec<ResolvedProfile>`
     - On `Err(ProfileError::NotFound { .. })` -> continue
     - On `Err(ProfileError::TypeMismatch { .. })` -> continue
     - On other `Err(e)` -> return `Err(e.into())`
   - If `matches.len() == 0`: `anyhow::bail!("Profile '{}' not found in any type (machine, filament, process)", query)`
   - If `matches.len() == 1`: return `Ok(matches.remove(0))`
   - If `matches.len() > 1`: `anyhow::bail!("Ambiguous profile '{}': found in types {}. Use --type to disambiguate.", query, types_list)`

7. **`fn user_profiles_base_dir() -> Result<PathBuf, anyhow::Error>`:**
   - Returns `home::home_dir()?.join(".slicecore/profiles")` or errors if home dir not found

8. **`fn cmd_clone(source: &str, new_name: &str, force: bool, type_hint: Option<&str>, profiles_dir: Option<&Path>) -> Result<(), anyhow::Error>`:**
   - Validate `new_name` with `is_valid_profile_name()`, bail with: `"Invalid profile name '{new_name}'. Use only letters, numbers, hyphens, and underscores (must not start with hyphen)."`
   - Create resolver: `ProfileResolver::new(profiles_dir)`
   - Resolve source: `try_resolve_any(&resolver, source, type_hint)?`
   - Load source config: `PrintConfig::from_file(&resolved.path)?`
   - Serialize to TOML: `toml::to_string_pretty(&config)?`
   - Build metadata header:
     ```
     # Custom profile cloned from {source}
     [metadata]
     name = "{new_name}"
     is_custom = true
     inherits = "{source}"
     clone_source = "{resolved.path.display()}"
     ```
   - Determine destination: `user_profiles_base_dir()?.join(&resolved.profile_type).join(format!("{new_name}.toml"))`
   - Create parent dirs: `std::fs::create_dir_all(dest.parent().unwrap())?`
   - Check conflict: if `dest.exists() && !force` -> `anyhow::bail!("Profile '{new_name}' already exists at {dest}. Use --force to overwrite or choose a different name.")`
   - Write file: `std::fs::write(&dest, format!("{metadata}\n{toml_body}"))?`
   - Print success: `println!("Created custom profile '{new_name}' at {}", dest.display())`
   - Print next steps:
     ```
     println!("\nNext steps:");
     println!("  slicecore profile show {new_name}");
     println!("  slicecore profile set {new_name} <key> <value>");
     println!("  slicecore profile edit {new_name}");
     ```

9. **Inline `#[cfg(test)] mod tests`:**
   - `test_valid_profile_names`: assert `is_valid_profile_name("my-pla")`, `is_valid_profile_name("PLA_Basic_v2")`, `is_valid_profile_name("a")`
   - `test_invalid_profile_names`: assert `!is_valid_profile_name("")`, `!is_valid_profile_name("-starts-dash")`, `!is_valid_profile_name("has spaces")`, `!is_valid_profile_name("path/../traversal")`, `!is_valid_profile_name("has.dots")`
   - `test_valid_name_length_boundary`: 128-char name valid, 129-char name invalid
