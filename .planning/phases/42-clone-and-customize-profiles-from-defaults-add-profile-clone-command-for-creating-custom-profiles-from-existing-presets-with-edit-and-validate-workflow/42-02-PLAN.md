**1. `fn cmd_set(name: &str, key: &str, value: &str, profiles_dir: Option<&Path>) -> Result<(), anyhow::Error>`:**
- Resolve profile with `try_resolve_any(&resolver, name, None)?`
- Check source is `ProfileSource::User`, otherwise bail with: `"Cannot modify {source} profile '{name}'. Clone it first:\n  slicecore profile clone {name} my-{name}"`
- Look up key in setting registry: `slicecore_engine::setting_registry().get(&SettingKey(key.to_string()))`
- If key not found, use `registry.search(key)` to get top 3 suggestions, bail with: `"Unknown setting key '{key}'. Did you mean: {suggestions}?"` or `"Unknown setting key '{key}'"` if no suggestions
- Parse the TOML file as `toml::Value` (NOT through `PrintConfig` to preserve metadata): `let contents = std::fs::read_to_string(&resolved.path)?; let mut doc: toml::Value = toml::from_str(&contents)?;`
- Navigate the dotted key path (e.g., `speed.perimeter`) through the `toml::Value::Table` tree:
  - Split key on `.`, walk through tables, creating intermediate tables if needed
  - Set the final leaf to the parsed value (try `value.parse::<f64>()` for numbers, `value.parse::<bool>()` for bools, otherwise string)
- Write back: `std::fs::write(&resolved.path, toml::to_string_pretty(&doc)?)?`
- Print: `"Set {key} = {value} in profile '{name}'"`

**2. `fn cmd_get(name: &str, key: &str, profiles_dir: Option<&Path>) -> Result<(), anyhow::Error>`:**
- Resolve profile with `try_resolve_any`
- Read TOML as `toml::Value`
- Navigate dotted key path to find value
- If found: print value (using `Display` for the `toml::Value`)
- If not found: bail with `"Key '{key}' not found in profile '{name}'"`

**3. `fn cmd_reset(name: &str, key: &str, profiles_dir: Option<&Path>) -> Result<(), anyhow::Error>`:**
- Resolve target profile (must be User source)
- Read target TOML as `toml::Value`
- Extract `inherits` from `[metadata]` table: `doc.get("metadata").and_then(|m| m.get("inherits")).and_then(|v| v.as_str())`
- If no `inherits` field: bail with `"Profile '{name}' has no inherited source. Cannot reset to original value."`
- Resolve the inherited source profile using `try_resolve_any`
- Read source TOML as `toml::Value`
- Navigate key path in source to get original value
- If key not found in source: bail with `"Key '{key}' not found in source profile '{inherits}'"`
- Set key in target to source value, write back
- Print: `"Reset {key} in profile '{name}' to inherited value from '{inherits}'"`

**4. `fn cmd_validate(name: &str, json_output: bool, profiles_dir: Option<&Path>) -> Result<(), anyhow::Error>`:**
- Resolve profile with `try_resolve_any`
- Load config via `PrintConfig::from_file(&resolved.path)?`
- Convert to JSON: `serde_json::to_value(&config)?`
- Validate: `let issues = slicecore_engine::setting_registry().validate_config(&config_json);`
- Partition issues by severity into errors and warnings
- If `json_output`: print `serde_json::to_string_pretty(&issues)?`
- Otherwise, print each issue with severity prefix:
  - Errors: `"ERROR: [{key}] {message}"`
  - Warnings: `"WARNING: [{key}] {message}"`
  - Info: `"INFO: [{key}] {message}"`
- Print summary: `"{N} error(s), {M} warning(s)"` on stderr
- If no issues: `"Profile '{name}' is valid"`

**5. Add helper `fn navigate_toml_path<'a>(doc: &'a toml::Value, key: &str) -> Option<&'a toml::Value>`:**
- Split key on `.`, walk through tables
- Return `None` if any intermediate key missing

**6. Add helper `fn navigate_toml_path_mut<'a>(doc: &'a mut toml::Value, key: &str) -> Option<&'a mut toml::Value>`:**

**7. Add helper `fn parse_toml_value(value: &str) -> toml::Value`:**

- Try `value.parse::<bool>()` -> `toml::Value::Boolean`

**1. `fn cmd_edit(name: &str, profiles_dir: Option<&Path>) -> Result<(), anyhow::Error>`:**
- Resolve profile (must be User source, bail with clone suggestion if library/builtin)
- Detect editor: check `$VISUAL`, then `$EDITOR`, then fall back to `"nano"`, then `"vi"`:
  ```rust
  let editor = std::env::var("VISUAL")
      .or_else(|_| std::env::var("EDITOR"))
      .unwrap_or_else(|_| "nano".to_string());
  ```
- Spawn editor: `std::process::Command::new(&editor).arg(&resolved.path).status()?`
- Check exit status: if not success, print warning `"Editor exited with non-zero status"` to stderr but continue
- Check file still exists: if not, bail with `"Profile file was deleted during editing"`
- Validate TOML syntax: attempt `toml::from_str::<toml::Value>(&std::fs::read_to_string(&resolved.path)?)?`
- If TOML parse fails: print warning to stderr `"Warning: TOML syntax error: {err}. File saved but may need fixing."` (save regardless per CONTEXT.md decision)
- If TOML parses ok: run schema validation and print any warnings/errors as informational output
- Print: `"Profile '{name}' updated at {path}"`

**2. `fn cmd_delete(name: &str, yes: bool, profiles_dir: Option<&Path>) -> Result<(), anyhow::Error>`:**
- Resolve profile
- If not `ProfileSource::User`: bail with `"Cannot delete {source} profile '{name}'. Only user profiles can be deleted."`
- Print file path: `"Will delete: {resolved.path.display()}"`
- If not `yes`: bail with `"Use --yes to confirm deletion, or abort"`
- Remove file: `std::fs::remove_file(&resolved.path)?`
- Print: `"Deleted profile '{name}'"`

**3. `fn cmd_rename(old_name: &str, new_name: &str, profiles_dir: Option<&Path>) -> Result<(), anyhow::Error>`:**
- Validate `new_name` with `is_valid_profile_name()` (same validation as clone)
- Resolve `old_name` (must be User source)
- Build new path: same directory as old, but with `{new_name}.toml` filename
- Check new path doesn't exist: bail if it does with `"Profile '{new_name}' already exists at {new_path}"`
- Read file content, parse as `toml::Value`, update `metadata.name` to `new_name` if metadata section exists
- Write updated content to new path
- Remove old file: `std::fs::remove_file(&resolved.path)?`
- Print: `"Renamed profile '{old_name}' to '{new_name}' at {new_path}"`
