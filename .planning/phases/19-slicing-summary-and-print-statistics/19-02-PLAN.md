**In Cargo.toml:** Add `comfy-table = "7"` to `[dependencies]`.

**Create `crates/slicecore-cli/src/stats_display.rs`** with the following functions:

Import `slicecore_engine::{PrintStatistics, FeatureStatistics, StatisticsSummary, GcodeMetrics, TimePrecision, StatsSortOrder}`.

1. **`format_time(seconds: f64, precision: TimePrecision) -> String`**:
   - Compute hours, minutes, remaining seconds
   - Format based on precision:
     - Seconds: "38m18s", "1h15m30s", "45s"
     - Deciseconds: "38m18.3s"
     - Milliseconds: "38m18.312s"
   - Omit hours if 0, omit minutes if both hours and minutes are 0

2. **`format_length(mm: f64) -> String`**:
   - If mm >= 1000.0: format as meters with 2 decimals ("3.87m")
   - Otherwise: format as mm with 1 decimal ("450.2mm")

3. **`format_filament(mm: f64, g: f64) -> String`**:
   - Combine length and weight: "{length} / {weight:.2}g"
   - Per user decision: always show both length and weight

4. **`sort_features(features: &[FeatureStatistics], order: StatsSortOrder) -> Vec<&FeatureStatistics>`**:
   - Default: return in existing order (already in default_feature_order from Plan 01)
   - TimeDesc: sort by time_seconds descending
   - FilamentDesc: sort by filament_mm descending
   - Alphabetical: sort by feature name alphabetically

5. **`format_ascii_table(stats: &PrintStatistics, precision: TimePrecision, sort_order: StatsSortOrder) -> String`**:
   - Build output string with two sections:

   **Summary section** (plain text, before the table):
   ```
   === Slicing Statistics ===
   Total time:     {formatted_time}
   Print time:     {formatted_time}
   Filament:       {length} / {weight}g
   Cost:           {cost:.2}
   Layers:         {count}
   Segments:       {count}
   Travel:         {distance}
   Retractions:    {count} ({distance})
   Z-hops:         {count} ({distance})
   ```

   **Per-feature breakdown table** using comfy-table:
   - Use `ContentArrangement::Dynamic` for auto-sizing
   - Header: ["Feature", "Time", "% Total", "% Print", "Filament", "Weight", "% Fil"]
   - One row per feature (sorted per sort_order)
   - After model features, add a separator row then "-- Model total --" row
   - After support features, add "-- Support total --" row (only if any support features have non-zero time)
   - Final row: "-- Overall total --"
   - Zero values display as "0s", "0.0mm", "0.0%", "0.00g" (not blank -- per user decision)

   **G-code metrics section** (after table):
   ```
   Retractions:    {count} ({total_distance})
   Unretractions:  {count}
   Z-hops:         {count} ({total_distance})
   Wipes:          {count} ({total_distance})
   ```

6. **`format_csv(stats: &PrintStatistics, sort_order: StatsSortOrder) -> String`**:
   - Per user decision: standardized column names, one row per feature, no section headers
   - Header row: `feature,time_s,time_pct_total,time_pct_print,filament_mm,filament_g,filament_pct_total,filament_pct_print`
   - One data row per feature with values formatted to appropriate precision
   - No quoting needed (feature names don't contain commas)
   - Hand-written (no csv crate -- per research recommendation)

7. **`format_json(stats: &PrintStatistics) -> String`**:
   - Simply serialize PrintStatistics to pretty JSON via serde_json::to_string_pretty
   - The type is already Serialize

Include tests:
- format_time with various durations (0s, 45s, 3m20s, 1h15m30s) at all three precision levels
- format_length at both mm and m thresholds
- format_filament produces expected "3.87m / 11.73g" format
- format_csv produces correct header and data rows
- format_ascii_table produces non-empty output with expected sections

4. **Statistics display logic:**

```rust
// 10. Display statistics.
if let Some(ref statistics) = result.statistics {
    if !quiet {
        let stats_output = match stats_format {
            "csv" => stats_display::format_csv(statistics, sort_order),
            "json" => stats_display::format_json(statistics),
            _ => stats_display::format_ascii_table(statistics, time_precision_enum, sort_order),
        };

        // When structured output (--json/--msgpack) is active, stats go to stderr
        if json_output || msgpack_output {
            eprintln!("{}", stats_output);
        } else {
            println!("{}", stats_output);
        }
    }

    // Save to file if requested (regardless of quiet)
    if let Some(ref file_path) = stats_file {
        let stats_output = match stats_format {
            "csv" => stats_display::format_csv(statistics, sort_order),
            "json" => stats_display::format_json(statistics),
            _ => stats_display::format_ascii_table(statistics, time_precision_enum, sort_order),
        };
        if let Err(e) = std::fs::write(file_path, &stats_output) {
            eprintln!("Warning: Failed to write statistics to '{}': {}", file_path.display(), e);
        }
    }
}
```

5. **JSON output integration (step 9):**

   - Build a serde_json::Value from the existing `to_json()` output
   - Add a "statistics" field containing the serialized statistics

   When `json_output` is true and `json_no_stats`:
   - Use existing `to_json()` behavior (no statistics in JSON)

   Implementation approach:
   ```rust
   if json_output {
       match slicecore_engine::output::to_json(&result, &print_config) {
           Ok(json_str) => {
               if !json_no_stats {
                   if let Some(ref statistics) = result.statistics {
                       // Parse base JSON, inject statistics, re-serialize
                       let mut value: serde_json::Value = serde_json::from_str(&json_str).unwrap();
                       value["statistics"] = serde_json::to_value(statistics).unwrap();
                       println!("{}", serde_json::to_string_pretty(&value).unwrap());
                   } else {
                       println!("{}", json_str);
                   }
               } else {
                   println!("{}", json_str);
               }
           }
           Err(e) => { /* existing error handling */ }
       }
   }
   ```
