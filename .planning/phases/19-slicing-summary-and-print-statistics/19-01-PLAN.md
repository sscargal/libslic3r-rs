  <name>Task 1: PrintStatistics types and per-feature computation from toolpaths</name>
  <files>crates/slicecore-engine/src/statistics.rs, crates/slicecore-engine/src/lib.rs</files>
  <action>
Create `crates/slicecore-engine/src/statistics.rs` with the following types and functions:

**Types (all derive Debug, Clone, Serialize, Deserialize):**

1. `TimePrecision` enum: `Seconds`, `Deciseconds`, `Milliseconds` -- configurable time display precision (per user decision).

2. `StatsSortOrder` enum: `Default`, `TimeDesc`, `FilamentDesc`, `Alphabetical` -- configurable feature sort order (per user decision).

3. `StatisticsSummary` struct:
   - `total_time_seconds: f64` -- from trapezoid estimate
   - `print_time_seconds: f64` -- total minus travel/retract/prepare time
   - `total_filament_mm: f64`, `total_filament_m: f64`, `total_filament_g: f64`, `total_filament_cost: f64`
   - `total_travel_distance_mm: f64`
   - `total_segments: u64`
   - `layer_count: usize`
   - `model_time_seconds: f64` -- time for model features only (excludes support)
   - `support_time_seconds: f64` -- time for support features only
   - `model_filament_mm: f64`, `support_filament_mm: f64`
   - `model_filament_g: f64`, `support_filament_g: f64`

4. `FeatureStatistics` struct:
   - `feature: String` -- human-readable display name (e.g., "Outer wall", "Inner wall", "Sparse infill")
   - `feature_type: String` -- machine-readable key (e.g., "outer_wall", "inner_wall")
   - `time_seconds: f64`
   - `time_pct_total: f64` -- percentage of total time (including travel, retract, everything)
   - `time_pct_print: f64` -- percentage of print time (only actual printing features, excludes travel/retract)
   - `filament_mm: f64`, `filament_m: f64`, `filament_g: f64`
   - `filament_pct_total: f64`, `filament_pct_print: f64`
   - `segment_count: u64`
   - `distance_mm: f64`
   - `is_support: bool` -- for support subtotaling
   - `display: bool` -- for future GUI integration (always true for now)

5. `GcodeMetrics` struct:
   - `retraction_count: u32`, `total_retraction_distance_mm: f64`
   - `unretraction_count: u32`
   - `wipe_count: u32`, `total_wipe_distance_mm: f64` -- will be 0 until wipe-on-retraction is implemented
   - `z_hop_count: u32`, `total_z_hop_distance_mm: f64`
   - `total_move_count: u64`

6. `PrintStatistics` struct:
   - `summary: StatisticsSummary`
   - `features: Vec<FeatureStatistics>`
   - `gcode_metrics: GcodeMetrics`

**Display name mapping function** `feature_display_name(feature: FeatureType) -> (&'static str, &'static str)`:
Returns (human_name, machine_key) pairs. Map existing FeatureType variants to user-facing names per user decision:
- `OuterPerimeter` -> ("Outer wall", "outer_wall")
- `InnerPerimeter` -> ("Inner wall", "inner_wall")
- `SolidInfill` -> ("Internal solid infill", "internal_solid_infill")
- `SparseInfill` -> ("Sparse infill", "sparse_infill")
- `Skirt` -> ("Skirt", "skirt")
- `Brim` -> ("Brim", "brim")
- `GapFill` -> ("Gap infill", "gap_infill")
- `VariableWidthPerimeter` -> ("Variable width wall", "variable_width_wall")
- `Support` -> ("Support", "support")
- `SupportInterface` -> ("Support interface", "support_interface")
- `Bridge` -> ("Bridge", "bridge")
- `Ironing` -> ("Ironing", "ironing")
- `PurgeTower` -> ("Purge tower", "purge_tower")
- `Travel` -> ("Travel", "travel")

**Feature ordering function** `default_feature_order() -> Vec<FeatureType>`:
Returns features in logical print sequence order per user decision: OuterPerimeter, InnerPerimeter, VariableWidthPerimeter, SparseInfill, SolidInfill, Bridge, GapFill, Ironing, Support, SupportInterface, PurgeTower, Skirt, Brim, Travel.

**is_support_feature(feature: FeatureType) -> bool**: Returns true for Support, SupportInterface, PurgeTower.

**Per-feature computation function** `compute_toolpath_statistics(layer_toolpaths: &[LayerToolpath], config: &PrintConfig) -> HashMap<FeatureType, FeatureAccumulator>`:
- Iterate all segments across all layers
- For each segment, accumulate into a HashMap keyed by FeatureType:
  - `segment_count += 1`
  - `distance_mm += seg.length()`
  - `filament_mm += seg.e_value` (will be 0 for Travel)
  - `time_seconds += seg.length() / (seg.feedrate / 60.0)` (naive per-segment time, used for relative proportions)
- Use a private `FeatureAccumulator` struct with Default impl

**G-code metrics extraction** `extract_gcode_metrics(commands: &[GcodeCommand]) -> GcodeMetrics`:
- Iterate commands tracking state:
  - `Retract { distance, .. }` -> increment retraction_count, add distance to total_retraction_distance_mm, set `just_retracted = true`
  - `Unretract { .. }` -> increment unretraction_count
  - `RapidMove { z: Some(z), x: None, y: None, .. }` when `just_retracted && z > cur_z` -> increment z_hop_count, add (z - cur_z) to total_z_hop_distance_mm
  - Track `cur_z` from all Z-bearing moves
  - Count all LinearMove and ArcMove* commands as total_move_count
  - Reset `just_retracted = false` on any non-z-hop command
  - Wipe count/distance = 0 (infrastructure ready but no wipe moves yet -- per research pitfall 4)

**Main assembly function** `compute_statistics(layer_toolpaths: &[LayerToolpath], gcode_commands: &[GcodeCommand], time_estimate: &PrintTimeEstimate, filament_usage: &FilamentUsage, config: &PrintConfig) -> PrintStatistics`:
1. Call `compute_toolpath_statistics()` for per-feature accumulators
2. Call `extract_gcode_metrics()` for G-code metrics
3. Compute naive total time = sum of all per-feature time_seconds
4. Compute scaling factor = `time_estimate.total_seconds / naive_total_time` (use 1.0 if naive is 0)
5. Scale each feature's time_seconds by the scaling factor (so per-feature times sum to trapezoid total)
6. Compute print_time = total_time - travel_time - (retraction_count * 0.5) (matching estimation.rs overhead)
7. For each feature in `default_feature_order()`:
   - Get accumulator (or default with all zeros if feature unused -- per user decision "show all features even if unused")
   - Compute percentages: time_pct_total = (scaled_time / total_time) * 100, time_pct_print = (scaled_time / print_time) * 100 (0 for travel/retract features)
   - Compute filament_m = filament_mm / 1000.0
   - Compute filament_g from filament_mm using config.filament_diameter, config.filament_density (same formula as filament.rs)
   - Compute filament_pct_total and filament_pct_print (print excludes Travel, which has 0 filament anyway)
   - Set is_support based on is_support_feature()
8. Add "Retract" and "Unretract" as virtual features from gcode_metrics with time = retraction_count * 0.5s (retraction overhead from estimation.rs), and zero filament. Wipe metrics similarly.
9. Compute summary totals including model_time/support_time/model_filament/support_filament subtotals
10. Return assembled PrintStatistics

**Add Eq, Hash derives to FeatureType** if not already present (needed for HashMap key). Check existing derives -- it already has `PartialEq, Eq` and toolpath.rs uses it. Add `Hash` derive to FeatureType in toolpath.rs.
