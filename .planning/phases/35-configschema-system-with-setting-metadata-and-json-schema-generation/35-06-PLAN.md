1. Create `crates/slicecore-config-schema/src/json_schema.rs`:

Implement `impl SettingRegistry`:

```rust
pub fn to_json_schema(&self) -> serde_json::Value
```

This generates JSON Schema 2020-12 with nested structure matching the Rust struct hierarchy.

Algorithm:
- Start with root object: `{ "$schema": "https://json-schema.org/draft/2020-12/schema", "$id": "https://slicecore.dev/config-schema.json", "title": "SliceCore Print Configuration", "type": "object" }`
- Build nested `properties` from dotted keys: "speed.perimeter" becomes `properties.speed.properties.perimeter`
- For each leaf property, generate JSON Schema type from ValueType:
  - Float -> `{ "type": "number" }`
  - Int -> `{ "type": "integer" }`
  - Bool -> `{ "type": "boolean" }`
  - String -> `{ "type": "string" }`
  - Percent -> `{ "type": "number", "minimum": 0, "maximum": 100 }`
  - FloatVec -> `{ "type": "array", "items": { "type": "number" } }`
  - Enum { variants } -> `{ "type": "string", "enum": ["variant1", "variant2", ...] }`
- Add constraints: Range { min, max } -> `"minimum"`, `"maximum"`
- Add default_value as `"default"`
- Add custom x- extensions:
  - `"x-tier"`: tier as u8
  - `"x-category"`: category as lowercase string
  - `"x-units"`: units string (if present)
  - `"x-display-name"`: display_name
  - `"x-affects"`: array of affected key strings
  - `"x-affected-by"`: array of affecting key strings
  - `"x-tags"`: array of tag strings
  - `"x-since-version"`: version string
  - `"x-deprecated"`: deprecation message (if present)
  - `"x-description"`: description (in addition to JSON Schema "description" field)
  - `"x-depends-on"`: depends_on constraint info (if present)

For nested structure building, create a helper `fn insert_nested(root: &mut serde_json::Value, key_parts: &[&str], property_schema: serde_json::Value)` that creates intermediate `{ "type": "object", "properties": {} }` nodes.

For `$defs`, generate sub-schemas for each intermediate object (speed, cooling, etc.) using the struct name as the def key. Use `$ref` in the root properties to reference `$defs`.

1. Create `crates/slicecore-config-schema/src/search.rs`:

```rust
impl SettingRegistry {
    /// Search settings by case-insensitive substring match across key, display_name,
    /// description, and tags. Returns results ranked by match quality.
    pub fn search(&self, query: &str) -> Vec<&SettingDefinition> {
        let query_lower = query.to_lowercase();
        let mut results: Vec<(&SettingDefinition, u8)> = self.definitions
            .values()
            .filter_map(|def| {
                let mut score = 0u8;
                // Exact key segment match (highest priority)
                if def.key.0.to_lowercase().contains(&query_lower) {
                    score += 4;
                }
                // Display name match
                if def.display_name.to_lowercase().contains(&query_lower) {
                    score += 3;
                }
                // Tag match
                if def.tags.iter().any(|t| t.to_lowercase().contains(&query_lower)) {
                    score += 2;
                }
                // Description match (lowest priority)
                if def.description.to_lowercase().contains(&query_lower) {
                    score += 1;
                }
                if score > 0 { Some((def, score)) } else { None }
            })
            .collect();
        results.sort_by(|a, b| b.1.cmp(&a.1).then(a.0.key.0.cmp(&b.0.key.0)));
        results.into_iter().map(|(def, _)| def).collect()
    }
}
```

use std::sync::LazyLock;
use slicecore_config_schema::{SettingRegistry, HasSettingSchema};
use crate::config::PrintConfig;

static GLOBAL_REGISTRY: LazyLock<SettingRegistry> = LazyLock::new(|| {
    let mut registry = SettingRegistry::new();

    // Register all setting definitions from PrintConfig (recursively includes all sub-structs)
    for def in PrintConfig::setting_definitions("") {
        registry.register(def);
    }

    // Extract default values from PrintConfig::default() via serde serialization
    let defaults = serde_json::to_value(PrintConfig::default()).unwrap_or_default();
    registry.populate_defaults(&defaults);

    // Compute affected_by as inverse of affects
    registry.compute_affected_by();

    registry
});

/// Returns the global setting registry containing metadata for all config fields.
pub fn setting_registry() -> &'static SettingRegistry {
    &GLOBAL_REGISTRY
}
```

4. Add `populate_defaults` method to SettingRegistry in `registry.rs`:

/// Populate default values from a serialized config JSON object.
/// Matches dotted keys against nested JSON paths.
pub fn populate_defaults(&mut self, defaults_json: &serde_json::Value) {
    for def in self.definitions.values_mut() {
        // Split key by "." and traverse JSON object
        let parts: Vec<&str> = def.key.0.split('.').collect();
        let mut current = defaults_json;
        let mut found = true;
        for part in &parts {
            match current.get(part) {
                Some(next) => current = next,
                None => { found = false; break; }
            }
        }
        if found {
            def.default_value = current.clone();
        }
    }
}
```
