```rust
// SettingKey - newtype around String for dotted path keys
#[derive(Debug, Clone, PartialEq, Eq, Hash, Serialize, Deserialize, PartialOrd, Ord)]
pub struct SettingKey(pub String);

// Tier - 5-level progressive disclosure
#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord, Hash, Serialize, Deserialize)]
#[repr(u8)]
pub enum Tier {
    AiAuto = 0,
    Simple = 1,
    Intermediate = 2,
    Advanced = 3,
    Developer = 4,
}

// SettingCategory - ~16 variants for UI grouping
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
pub enum SettingCategory {
    Quality, Speed, LineWidth, Cooling, Retraction, Support, Infill,
    Adhesion, Advanced, Machine, Filament, Acceleration, PostProcess,
    Timelapse, MultiMaterial, Calibration,
}

// ValueType - discriminated union of setting value types
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum ValueType {
    Bool, Int, Float, String, Percent,
    Enum { variants: Vec<EnumVariant> },
    FloatVec,
}

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct EnumVariant {
    pub value: String,      // snake_case variant name
    pub display: String,    // human-readable display name
    pub description: String,
}

// Constraint - validation rules
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum Constraint {
    Range { min: f64, max: f64 },
    DependsOn { key: SettingKey, condition: String },
}

// SettingDefinition - full metadata for one setting
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SettingDefinition {
    pub key: SettingKey,
    pub display_name: String,
    pub description: String,
    pub tier: Tier,
    pub category: SettingCategory,
    pub value_type: ValueType,
    pub default_value: serde_json::Value,
    pub constraints: Vec<Constraint>,
    pub affects: Vec<SettingKey>,
    pub affected_by: Vec<SettingKey>,
    pub units: Option<String>,
    pub tags: Vec<String>,
    pub since_version: String,
    pub deprecated: Option<String>,
}
```

```rust
use std::collections::BTreeMap;
use crate::types::{SettingDefinition, SettingKey, Tier, SettingCategory};

/// Central registry of all setting definitions.
#[derive(Debug, Default)]
pub struct SettingRegistry {
    definitions: BTreeMap<SettingKey, SettingDefinition>,
}
```

Implement these methods on `SettingRegistry`:

1. `pub fn new() -> Self` - empty registry
2. `pub fn register(&mut self, def: SettingDefinition)` - insert definition by key (panic on duplicate key with descriptive message)
3. `pub fn get(&self, key: &SettingKey) -> Option<&SettingDefinition>` - lookup by key
4. `pub fn get_by_str(&self, key: &str) -> Option<&SettingDefinition>` - convenience lookup
5. `pub fn all(&self) -> impl Iterator<Item = &SettingDefinition>` - iterate all definitions (sorted by key via BTreeMap)
6. `pub fn len(&self) -> usize` and `pub fn is_empty(&self) -> bool`
7. `pub fn compute_affected_by(&mut self)` - for each definition, scan all OTHER definitions' `affects` lists. If definition A's affects contains key B, then add A's key to B's affected_by. This builds the inverse graph.
8. `pub fn filter_by_tier(&self, max_tier: Tier) -> Vec<&SettingDefinition>` - return definitions with tier <= max_tier
9. `pub fn filter_by_category(&self, category: SettingCategory) -> Vec<&SettingDefinition>` - return definitions matching category
10. `pub fn validate_integrity(&self) -> Vec<String>` - check that all keys referenced in `affects` and `constraints` (DependsOn) actually exist in the registry. Return list of error messages for dangling references.

Add unit tests in a `#[cfg(test)] mod tests` block:
