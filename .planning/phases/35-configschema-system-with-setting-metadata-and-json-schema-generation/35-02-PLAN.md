3. Create `crates/slicecore-config-derive/src/lib.rs`:
```rust
//! Derive macro for the ConfigSchema setting metadata system.

mod parse;
mod codegen;

use proc_macro::TokenStream;

/// Derive `HasSettingSchema` for config structs and enums.
///
/// # Struct attributes
/// - `#[setting(category = "Speed")]` - default category for all fields
///
/// # Field attributes
/// - `#[setting(tier = 1, description = "...", units = "mm/s")]`
/// - `#[setting(min = 0.0, max = 100.0)]`
/// - `#[setting(affects = ["key1", "key2"])]`
/// - `#[setting(depends_on = "other.key")]`
/// - `#[setting(skip)]` - exclude from schema
/// - `#[setting(flatten)]` - delegate to sub-struct
///
/// # Enum variant attributes
/// - `#[setting(display = "Human Name", description = "...")]`
#[proc_macro_derive(SettingSchema, attributes(setting))]
pub fn derive_setting_schema(input: TokenStream) -> TokenStream {
    let input = syn::parse_macro_input!(input as syn::DeriveInput);
    match codegen::expand_setting_schema(&input) {
        Ok(tokens) => tokens.into(),
        Err(err) => err.to_compile_error().into(),
    }
}
```

Create `crates/slicecore-config-derive/src/codegen.rs` implementing `pub fn expand_setting_schema(input: &syn::DeriveInput) -> syn::Result<proc_macro2::TokenStream>`.

The function must handle two cases:

**Case 1: Struct (syn::Data::Struct)**

For each named field:
1. If `#[setting(skip)]` -> skip entirely
2. If `#[setting(flatten)]` -> generate a delegation call:
   ```rust
   {
       let child_prefix = if prefix.is_empty() {
           "{field_prefix}".to_string()
       } else {
           format!("{prefix}.{field_prefix}")
       };
       defs.extend(<FieldType as ::slicecore_config_schema::HasSettingSchema>::setting_definitions(&child_prefix));
   }
   ```
   where `field_prefix` is from `prefix = "custom"` attribute or the field name itself.

3. Otherwise -> generate a SettingDefinition literal:
   - `key`: `SettingKey(format!("{prefix}.{field_name}"))` (if prefix empty, just field_name)
   - `display_name`: from `display_name = "..."` attribute, or auto-generated from field name via snake_case to Title Case (split on `_`, capitalize each word, join with space)
   - `description`: from `description = "..."` attribute, or empty string
   - `tier`: from `tier = N` attribute, or default to `Tier::Developer` (4)
   - `category`: from field-level `category = "..."`, or struct-level `category = "..."`, or `SettingCategory::Advanced` default
   - `value_type`: inferred from Rust type:
     - `f64` -> `ValueType::Float`
     - `f32` -> `ValueType::Float`
     - `bool` -> `ValueType::Bool`
     - `i32`, `u32`, `i64`, `u64`, `usize` -> `ValueType::Int`
     - `String` -> `ValueType::String`
     - `Vec<f64>` -> `ValueType::FloatVec`
     - `Option<T>` -> recurse on T (mark as optional via default_value = null)
     - Any other type -> check if it implements HasSettingSchema (enum type), generate `ValueType::Enum { variants: vec![] }` as placeholder (actual variants filled at runtime would require calling the enum's method, but for schema purposes, use a simpler approach: just mark as `ValueType::String` for now and let the enum's own derive handle it)
   - For type inference, use a helper function `infer_value_type(ty: &syn::Type) -> proc_macro2::TokenStream` that matches on the type path's last segment identifier string.
   - `default_value`: `::serde_json::Value::Null` (actual defaults extracted at runtime from `Default::default()`)
   - `constraints`: build from `min`/`max` -> `Constraint::Range { min, max }`, and `depends_on` -> `Constraint::DependsOn { key, condition }`
   - `affects`: from `affects = [...]` attribute, mapped to `SettingKey::new("...")`
   - `affected_by`: always `vec![]` (computed later by registry)
   - `units`: from `units = "..."` attribute
   - `tags`: from `tags = [...]` attribute
   - `since_version`: from `since_version = "..."` attribute or `"0.1.0"` default
   - `deprecated`: from `deprecated = "..."` attribute

**Case 2: Enum (syn::Data::Enum)**

Generate `HasSettingSchema` impl that returns a single `SettingDefinition` with `ValueType::Enum { variants }`.
For each variant:
- `value`: variant name converted to snake_case
- `display`: from `#[setting(display = "...")]` or auto-generated from variant name (CamelCase to spaced words)
- `description`: from `#[setting(description = "...")]` or empty string

The enum's `setting_definitions` returns `vec![]` (enums don't register themselves as individual settings -- they provide variant metadata when referenced by a struct field). Instead, provide a separate method or associated function that returns the `ValueType::Enum` for use by struct fields.

Actually, better approach: add a separate trait method or helper. For enums, implement a helper that struct codegen can call to get the ValueType. Simplest: make the enum's `setting_definitions()` return empty vec, and add a standalone function approach.

Simplest working approach: For struct fields whose type is an enum that derives SettingSchema, the struct codegen should generate `ValueType::String` (since the enum value is a string in JSON). The actual enum variant metadata is available via `<EnumType as HasSettingSchema>::setting_definitions("")` which returns a definition with the enum's variants. But for the field-level definition, just use `ValueType::String`.

For enum derive, generate `setting_definitions` that returns one `SettingDefinition` with:
- key based on prefix
- `value_type: ValueType::Enum { variants: vec![...] }` populated from variant attributes

This way, when a struct has `pub wall_order: WallOrder`, the struct codegen generates a basic field definition with `ValueType::String`, and the caller can look up the enum's full variant list from the registry if needed by calling `WallOrder::setting_definitions("")`.

**Type inference function:** Create `fn infer_value_type(ty: &syn::Type) -> proc_macro2::TokenStream` that:
- Extracts the last path segment's ident as a string
- Matches against known type names: "f64"|"f32" -> Float, "bool" -> Bool, "i32"|"u32"|"i64"|"u64"|"usize" -> Int, "String" -> String
- For "Vec" -> check inner type, if f64 -> FloatVec, else String
- For "Option" -> recurse on inner type
- For anything else (enums, custom types) -> String
