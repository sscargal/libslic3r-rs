    Add CSG operation plugin traits to `crates/slicecore-plugin-api/src/traits.rs` following the same pattern as `InfillPatternPlugin`:

    ```rust
    /// Plugin trait for custom CSG operations.
    ///
    /// Plugins implementing this trait can create custom mesh primitives
    /// and apply boolean operations via the plugin system.
    pub trait CsgOperationPlugin: Send + Sync + Debug {
        /// Plugin name for identification.
        fn name(&self) -> &str;

        /// Create a custom mesh primitive.
        /// Returns vertices and triangle indices.
        fn create_primitive(&self, params: &CsgPrimitiveParams) -> Result<(Vec<[f64; 3]>, Vec<[u32; 3]>), String>;

        /// Apply a custom boolean operation.
        /// Takes two meshes as vertices+indices pairs, returns result mesh.
        fn apply_boolean(
            &self,
            a_vertices: &[[f64; 3]],
            a_indices: &[[u32; 3]],
            b_vertices: &[[f64; 3]],
            b_indices: &[[u32; 3]],
            operation: &str,
        ) -> Result<(Vec<[f64; 3]>, Vec<[u32; 3]>), String>;
    }

    /// Parameters for custom primitive generation.
    #[derive(Clone, Debug, Serialize, Deserialize)]
    pub struct CsgPrimitiveParams {
        pub primitive_type: String,
        pub dimensions: Vec<f64>,
        pub segments: u32,
    }
    ```

    Use plain arrays `[f64; 3]` and `[u32; 3]` in the plugin API (not Point3/TriangleMesh) to keep the plugin boundary simple and FFI-friendly. The plugin host converts between TriangleMesh and these arrays.

    Add doc comments with `/// # Examples` showing how a plugin would implement the trait.
