  <action>
Add the public boolean API to `csg/mod.rs`:

1. Internal `fn mesh_boolean(a: &TriangleMesh, b: &TriangleMesh, op: BooleanOp) -> Result<(TriangleMesh, CsgReport), CsgError>` implementing the full pipeline:
   a. Auto-repair inputs: call `repair::repair(a.vertices().to_vec(), a.indices().to_vec())`, map errors to `CsgError::RepairFailedA/B`
   b. BVH broad-phase via `compute_intersections(&repaired_a, &repaired_b)`
   c. Split intersected triangles via `split_all_intersected_triangles`
   d. Classify non-intersected triangles via `classify_connected_component`
   e. Select triangles based on operation:
      - Union: outside_A + outside_B
      - Intersection: inside_A + inside_B
      - Difference: outside_A + inside_B (flip winding on B triangles!)
      - XOR: (outside_A + inside_B_flipped) + (outside_B + inside_A_flipped)
   f. Merge selected triangles into output vertex/index arrays. When merging, propagate per-triangle attributes from source mesh/triangle.
   g. Cleanup: remove zero-area triangles, merge near-duplicate vertices (within 1e-10), ensure consistent winding via `repair::normals::fix_normal_directions`
   h. Build CsgReport with counts
   i. Construct output TriangleMesh, return `Ok((mesh, report))`

2. Public API functions (thin wrappers):
   - `pub fn mesh_union(a: &TriangleMesh, b: &TriangleMesh) -> Result<(TriangleMesh, CsgReport), CsgError>`
   - `pub fn mesh_difference(a: &TriangleMesh, b: &TriangleMesh) -> Result<(TriangleMesh, CsgReport), CsgError>`
   - `pub fn mesh_intersection(a: &TriangleMesh, b: &TriangleMesh) -> Result<(TriangleMesh, CsgReport), CsgError>`
   - `pub fn mesh_xor(a: &TriangleMesh, b: &TriangleMesh) -> Result<(TriangleMesh, CsgReport), CsgError>`

3. Re-export all five functions from `csg/mod.rs` and from `lib.rs`.

CRITICAL: For the difference operation, flip winding order (reverse index order) on "inside B" triangles before adding to output. This prevents inside-out normals (RESEARCH.md Pitfall 4).

Write integration tests in `crates/slicecore-mesh/tests/csg_boolean.rs`. Use primitives from Plan 01 to construct test cases. Include a JSON serialization test for CsgReport.

  <action>
Add to `csg/mod.rs`:

`pub fn mesh_union_many(meshes: &[&TriangleMesh]) -> Result<(TriangleMesh, CsgReport), CsgError>` — divide-and-conquer approach:
1. If 0 meshes, return `CsgError::EmptyResult { operation: "union_many".into() }`
2. If 1 mesh, clone it and return with empty report
3. If 2 meshes, delegate to `mesh_union`
4. Otherwise: split slice in half, recursively union each half, then union the two halves. This is O(n log n) rather than O(n^2) per RESEARCH.md Pitfall 6.

Aggregate CsgReport across recursive calls (sum triangle counts, concatenate warnings).

Re-export `mesh_union_many` from csg/mod.rs and lib.rs.

Add tests to the existing `csg_boolean.rs` test file.
