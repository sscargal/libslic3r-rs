    2. Create `crates/slicecore-mesh/src/csg/boolean.rs`:

       Internal pipeline function:
       ```
       fn mesh_boolean(a: &TriangleMesh, b: &TriangleMesh, op: BooleanOp, options: &CsgOptions) -> Result<(TriangleMesh, CsgReport), CsgError>
       ```
       Steps:
       a. Start timer for CsgReport.duration_ms
       b. Auto-repair both inputs using `crate::repair::repair()`. Map errors to CsgError::RepairFailedA/B. Count repairs in report.
       c. Call `compute_intersection_curves(&repaired_a, &repaired_b)` to get IntersectionResult
       d. Retriangulate both meshes along intersection curves using `retriangulate_mesh`
       e. Classify all triangles of retriangulated mesh A against mesh B, and vice versa
       f. Select triangles based on operation:
          - Union: A.OUTSIDE + B.OUTSIDE
          - Intersection: A.INSIDE + B.INSIDE
          - Difference: A.OUTSIDE + B.INSIDE (flip winding on B's INSIDE triangles)
          - XOR: (A.OUTSIDE + B.INSIDE with flipped winding) + (B.OUTSIDE + A.INSIDE with flipped winding)
       g. Merge selected triangles into a new vertex/index list (re-index vertices, remove duplicates within merge tolerance ~1e-10)
       h. Remove degenerate (zero-area) triangles from output
       i. Fix winding order consistency using existing repair::normals logic if available, or manual check
       j. Construct output TriangleMesh. Map errors to CsgError::ResultConstruction.
       k. If `options.validate_output`, check manifold: every edge shared by exactly 2 triangles. If not, return CsgError::NonManifoldResult.
       l. Compute volume and surface_area for CsgReport
       m. Populate CsgReport fields and return

       Public API functions (thin wrappers):
       - `pub fn mesh_union(a: &TriangleMesh, b: &TriangleMesh) -> Result<(TriangleMesh, CsgReport), CsgError>` -- calls mesh_boolean with Union and default options
       - `pub fn mesh_difference(a: &TriangleMesh, b: &TriangleMesh) -> Result<(TriangleMesh, CsgReport), CsgError>`
       - `pub fn mesh_intersection(a: &TriangleMesh, b: &TriangleMesh) -> Result<(TriangleMesh, CsgReport), CsgError>`
       - `pub fn mesh_xor(a: &TriangleMesh, b: &TriangleMesh) -> Result<(TriangleMesh, CsgReport), CsgError>`
       - `pub fn mesh_union_many(meshes: &[&TriangleMesh]) -> Result<(TriangleMesh, CsgReport), CsgError>`
         - Sequential left-fold: result = union(meshes[0], meshes[1]), then union(result, meshes[2]), etc.
         - Accumulate all sub-reports into one combined CsgReport
         - Error if fewer than 2 meshes provided

       Also provide versions taking CsgOptions:
       - `pub fn mesh_union_with(a: &TriangleMesh, b: &TriangleMesh, options: &CsgOptions) -> Result<(TriangleMesh, CsgReport), CsgError>`
       - (similarly for difference, intersection, xor)

    3. Update `csg/mod.rs`:

    1. `test_union_overlapping_boxes` -- Two boxes overlapping by 50%. Result should be watertight, have fewer triangles than sum of inputs (shared faces removed), volume between max(vol_a, vol_b) and vol_a + vol_b.

    2. `test_union_non_overlapping_boxes` -- Two boxes that don't touch. Result triangle count equals sum of inputs. Volume equals sum.

    3. `test_union_touching_boxes` -- Two boxes sharing a face. Result should merge into one watertight box. Volume equals sum. Triangle count less than sum (shared face removed).

    4. `test_difference_overlapping_boxes` -- Large box minus small box. Result is watertight, has a rectangular hole. Volume = vol_large - vol_overlap.

    5. `test_difference_no_overlap` -- A minus B where B is completely outside A. Result equals A unchanged.

    6. `test_intersection_overlapping_boxes` -- Result is the overlap region. Volume less than either input. Watertight.

    7. `test_intersection_no_overlap` -- Result should be CsgError::EmptyResult.

    8. `test_xor_overlapping_boxes` -- Result is both non-overlapping parts. Volume = vol_a + vol_b - 2 * vol_overlap.

    9. `test_union_many_four_boxes` -- Four boxes in a 2x2 grid, all overlapping slightly. Result is one watertight mesh. Volume less than 4 * individual volume.

    10. `test_coplanar_touching_cubes` -- Two cubes sharing an exact face. Union should produce a single elongated box. Difference should produce a valid result.

    11. `test_boolean_report_fields` -- Verify CsgReport has correct input/output triangle counts, non-zero volume, non-zero surface_area, duration_ms > 0.

    12. `test_union_sphere_box` -- Sphere partially embedded in box. Result is watertight, volume between max(sphere, box) and sphere+box.
