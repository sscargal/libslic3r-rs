```rust
/// Plugin trait for mesh boolean/CSG operations.
///
/// Implementations can create meshes (primitives), apply CSG operations,
/// or perform custom mesh manipulation (hollowing, lattice generation, cutting).
pub trait CsgPlugin: Send + Sync + Debug {
    /// Returns the plugin name.
    fn name(&self) -> &str;

    /// Returns a description of what this plugin does.
    fn description(&self) -> &str;

    /// Apply a custom mesh operation.
    ///
    /// Takes one or more input meshes and returns a modified mesh.
    /// The `params` map allows plugin-specific configuration.
    fn apply(
        &self,
        inputs: &[&TriangleMesh],
        params: &std::collections::HashMap<String, String>,
    ) -> Result<(TriangleMesh, CsgReport), CsgError>;

    /// Returns the list of operations this plugin supports.
    fn supported_operations(&self) -> Vec<String>;
}
```

3. If slicecore-plugin-api uses abi_stable for InfillPatternPlugin, follow the same pattern for CsgPlugin (sabi_trait macro etc.). Read the existing InfillPatternPlugin implementation carefully and mirror the approach exactly.

4. Add doc comments explaining the plugin lifecycle: discovery -> load -> call apply() with parameters -> receive output mesh.

Note: Do NOT create an example plugin in this task — that would expand scope. Just define the trait interface.
