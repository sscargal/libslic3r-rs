1. Create `csg/perturb.rs` with symbolic perturbation:
   - `pub(crate) fn perturbed_orient3d(pa: [f64; 3], pb: [f64; 3], pc: [f64; 3], pd: [f64; 3], pa_idx: usize, pd_idx: usize) -> f64` — calls `robust::orient3d` first; if result is exactly 0.0 (coplanar), applies simulation of simplicity tie-breaking based on vertex indices (lower index wins, returns small positive/negative based on index comparison). This ensures deterministic handling of coplanar configurations.

2. Create `csg/intersect.rs` with the intersection curve computation pipeline:

   Data types:
   - `pub(crate) struct IntersectionPoint { pub position: Point3, pub mesh_a_tri: usize, pub mesh_b_tri: usize, pub edge_key: EdgeKey }` — a point where mesh A and mesh B intersect
   - `pub(crate) struct EdgeKey(pub u32, pub u32)` — canonical edge key (min, max vertex indices)
   - `pub(crate) struct IntersectionSegment { pub start: usize, pub end: usize, pub tri_a: usize, pub tri_b: usize }` — indices into intersection point registry
   - `pub(crate) struct IntersectionRegistry { points: Vec<IntersectionPoint>, segments: Vec<IntersectionSegment>, point_map: HashMap<(usize, usize, EdgeKey), usize> }` — canonical point registry preventing T-junctions (CRITICAL per RESEARCH.md Pitfall 3)

   Functions:
   - `pub(crate) fn compute_intersections(mesh_a: &TriangleMesh, mesh_b: &TriangleMesh) -> Result<IntersectionRegistry, CsgError>` — the main entry point:
     a. Build BVH for mesh B
     b. For each triangle in mesh A, query mesh B's BVH for AABB overlaps
     c. For each candidate pair, compute exact intersection segment using `robust::orient3d` for all point-plane classifications
     d. Use `IntersectionRegistry::get_or_insert_point()` to ensure each intersection point is computed ONCE and referenced by index (prevents T-junctions)
     e. Return the registry with all intersection points and segments

   - `fn triangle_triangle_intersection(va: [Point3; 3], vb: [Point3; 3], tri_a_idx: usize, tri_b_idx: usize, registry: &mut IntersectionRegistry) -> Result<bool, CsgError>` — computes the intersection segment between two triangles using Moller's algorithm enhanced with `robust::orient3d` for all orientation tests. Uses `perturbed_orient3d` when coplanar. Returns true if an intersection segment was added.

   The intersection algorithm: classify vertices of triangle B against plane of triangle A (using orient3d), then classify vertices of triangle A against plane of triangle B. If all vertices of either triangle are on the same side of the other's plane, no intersection. Otherwise, compute the line of intersection of the two planes and project both triangles' intersection intervals onto this line. The overlap of the intervals is the intersection segment.

3. Register both submodules in `csg/mod.rs`: `pub(crate) mod intersect; pub(crate) mod perturb;`

All types need doc comments. The IntersectionRegistry is the critical data structure — it must be well-documented explaining WHY canonical point storage prevents T-junctions.

1. Create `csg/retriangulate.rs` with ear-clipping constrained retriangulation:

   - `pub(crate) struct SplitResult { pub triangles: Vec<[Point3; 3]>, pub source_tri: usize, pub source_mesh: MeshLabel }` — result of splitting one triangle along intersection points
   - `pub(crate) enum MeshLabel { A, B }` — which input mesh a triangle came from
   - `pub(crate) fn split_triangle(vertices: [Point3; 3], constraints: &[(Point3, Point3)], source_tri: usize, source_mesh: MeshLabel) -> Vec<SplitResult>` — splits a triangle along one or more intersection segments that cross it. Uses ear-clipping for the sub-polygons created by the cuts. Each output triangle inherits the `source_tri` and `source_mesh` from the parent.

   The ear-clipping algorithm:
   a. Insert constraint points on triangle edges (compute barycentric coords to find which edge each point lies on)
   b. Build a polygon from the original triangle vertices plus inserted constraint points
   c. The constraint segments divide this polygon into sub-polygons
   d. Triangulate each sub-polygon using ear-clipping (sufficient for convex sub-polygons per RESEARCH.md)
   e. Maintain winding order consistent with parent triangle

   - `pub(crate) fn split_all_intersected_triangles(mesh_a: &TriangleMesh, mesh_b: &TriangleMesh, registry: &IntersectionRegistry) -> (Vec<SplitResult>, Vec<SplitResult>)` — processes all triangles that have intersection segments, returns split results for both meshes.

2. Create `csg/classify.rs` with ray-casting triangle classification:

   - `pub(crate) enum Classification { Inside, Outside }`
   - `pub(crate) fn classify_triangle(centroid: &Point3, other_mesh: &TriangleMesh) -> Classification` — casts a ray from centroid in +Z direction, counts intersections with other mesh using BVH-accelerated ray casting. Odd = Inside, Even = Outside. If ray hits vertex/edge exactly (degenerate), perturb ray direction and retry (up to 3 attempts with different directions: +Z, +X, +Y).
   - `pub(crate) fn classify_connected_component(component_tris: &[usize], mesh: &TriangleMesh, other_mesh: &TriangleMesh) -> Classification` — classifies one representative triangle from the component, then propagates to all triangles in the component (per RESEARCH.md: all triangles in a connected component of non-intersected triangles have the same classification).

3. Register both submodules in `csg/mod.rs`: `pub(crate) mod retriangulate; pub(crate) mod classify;`

Focus on correctness over performance. Ensure winding order is preserved through retriangulation (RESEARCH.md Pitfall 4).
