  <action>
1. Create `csg/split.rs` with:

`pub fn mesh_split_at_plane(mesh: &TriangleMesh, normal: &Vec3, distance: f64, cap: bool) -> Result<(Option<(TriangleMesh, CsgReport)>, Option<(TriangleMesh, CsgReport)>), CsgError>`

This is an OPTIMIZED path (not general CSG through mesh_difference with plane primitive):
a. Classify each vertex as above/below/on the plane using `robust::orient3d`
b. Triangles fully above -> positive half. Fully below -> negative half.
c. Straddling triangles: compute intersection points with plane, split into sub-triangles, assign to appropriate half.
d. If `cap` is true: collect the intersection polygon (the cut boundary) and triangulate it using ear-clipping to create cap faces for both halves (one with normal pointing up, one pointing down).
e. Build TriangleMesh for each non-empty half. Return `None` for empty halves (e.g., plane doesn't intersect mesh).
f. Populate CsgReport for each half.

2. Create `csg/offset.rs` with:

`pub fn mesh_offset(mesh: &TriangleMesh, distance: f64) -> Result<(TriangleMesh, CsgReport), CsgError>`

Vertex-normal displacement approach:
a. Compute per-vertex normals: for each vertex, average the normals of all adjacent faces, weighted by the face angle at that vertex.
b. Normalize each vertex normal.
c. Displace each vertex: `new_pos = old_pos + distance * vertex_normal`
d. Rebuild mesh with new vertices, same indices.
e. If distance is negative (shrink), the winding may need no flip since we're displacing inward.
f. Add warning to CsgReport if mesh has sharp concave features (angle between adjacent normals > 120 degrees) — per RESEARCH.md Pitfall 5.
g. Run basic cleanup (remove zero-area triangles from collapsed thin features).

3. Register both modules in `csg/mod.rs`. Re-export `mesh_split_at_plane` and `mesh_offset`.

4. Write tests in `crates/slicecore-mesh/tests/csg_split_hollow.rs`.

Create `csg/hollow.rs` with:

```rust
pub struct DrainHole {
    pub position: Point3,
    pub direction: Vec3,
    pub diameter: f64,
    pub depth: f64,
}

pub fn hollow_mesh(
    mesh: &TriangleMesh,
    wall_thickness: f64,
    drain_hole: Option<&DrainHole>,
) -> Result<(TriangleMesh, CsgReport), CsgError>
```

Implementation:
1. Validate wall_thickness > 0, return error otherwise.
2. Create inner shell: `mesh_offset(mesh, -wall_thickness)` (negative = inward).
3. Compute hollow shell: `mesh_difference(mesh, &inner_shell)`.
4. If drain_hole is provided:
   a. Generate a cylinder primitive at `drain_hole.position`, oriented along `drain_hole.direction`, with radius = diameter/2 and height = depth.
   b. Transform cylinder to correct position/orientation using existing transform functions.
   c. Subtract cylinder from hollow shell: `mesh_difference(&shell, &cylinder)`.
5. Aggregate CsgReport from all operations.
6. Return final mesh and report.

Register in `csg/mod.rs`, re-export `hollow_mesh` and `DrainHole`.

Add tests to existing `csg_split_hollow.rs` file.
