  <action>
    1. Create `crates/slicecore-mesh/src/csg/split.rs`:

       Types:
       - `SplitPlane { normal: Vec3, offset: f64 }` -- analytical plane (normal dot point = offset)
         - Constructor: `SplitPlane::new(normal: Vec3, offset: f64)` normalizes the normal vector
         - Convenience: `SplitPlane::xy(z: f64)` -> horizontal plane at z height, `SplitPlane::xz(y: f64)`, `SplitPlane::yz(x: f64)`
       - `SplitResult { above: TriangleMesh, below: TriangleMesh, report: CsgReport }`
       - `SplitOptions { cap: bool }` with `Default` (cap = true)

       Main function:
       - `mesh_split_at_plane(mesh: &TriangleMesh, plane: &SplitPlane, options: &SplitOptions) -> Result<SplitResult, CsgError>`
         - Classify each vertex as above/below/on the plane using `robust::orient3d` (construct 3 reference points on the plane to use orient3d)
         - For triangles fully above -> add to above mesh
         - For triangles fully below -> add to below mesh
         - For triangles crossing the plane: compute intersection points where edges cross the plane, split triangle into sub-triangles, assign each to above/below
         - If `options.cap == true`: collect the intersection polygon (the cross-section outline), triangulate it using ear-clipping, add to both halves (with opposite winding) to make both halves watertight
         - Compute CsgReport with triangle counts, volumes
       - This is an OPTIMIZED path -- does NOT go through the general boolean pipeline. Direct plane classification is much faster than creating a huge plane mesh and doing mesh_difference.

    2. Create `crates/slicecore-mesh/src/csg/offset.rs`:
       - `mesh_offset(mesh: &TriangleMesh, distance: f64) -> Result<(TriangleMesh, CsgReport), CsgError>`
         - Compute per-vertex normals: average of adjacent face normals, weighted by the angle at that vertex
         - Displace each vertex by `distance * vertex_normal` (positive = outward/grow, negative = inward/shrink)
         - Recompute normals for the new mesh
         - Run repair to fix any self-intersections introduced at concave features
         - Add warning to CsgReport if self-intersections were detected and repaired
         - Positive distance = outward offset (grow), negative = inward offset (shrink)

    3. Update `csg/mod.rs` to add `pub mod split; pub mod offset;` and re-export key types/functions.

    Doc comments with examples on all public items.
  </action>

  </verify>

  <files>

  <action>
    1. Create `crates/slicecore-mesh/src/csg/hollow.rs`:

       Types:
       - `DrainHole { position: Point3, direction: Vec3, diameter: f64, tapered: bool }`
         - `Default`: position at mesh bottom center, direction = -Z, diameter = 3.0mm, tapered = false
       - `HollowOptions { wall_thickness: f64, drain_hole: Option<DrainHole> }`
         - No Default -- wall_thickness is mandatory

       Main function:
       - `hollow_mesh(mesh: &TriangleMesh, options: &HollowOptions) -> Result<(TriangleMesh, CsgReport), CsgError>`
         - Step 1: Create inner shell via `mesh_offset(mesh, -options.wall_thickness)`
         - Step 2: Flip inner shell winding (reverse triangle index order) so normals point inward
         - Step 3: Use `mesh_difference(mesh, &inner_shell)` to create the hollow shell
         - Step 4: If `drain_hole` is Some:
           a. Create a cylinder (or tapered cone if `tapered == true`) primitive at the specified position/direction with the specified diameter
           b. Use `mesh_difference(&shell, &drain_cylinder)` to punch the drain hole
         - Step 5: Populate CsgReport with volume reduction info:
           - Original volume, hollow volume, percentage saved
           - Include in `warnings` if wall thickness > 50% of smallest dimension (likely self-intersection)

    2. Update `csg/mod.rs` to add `pub mod hollow;` and re-export.

    3. Create `crates/slicecore-mesh/tests/csg_split_hollow.rs`:

       Split tests:
       - `test_split_box_at_midpoint` -- Split a box in half. Each half is watertight. Combined volume = original volume. Each half has correct bounding box (half height).
       - `test_split_box_uncapped` -- Split with cap=false. Each half is NOT watertight (open cross-section). Triangle count equals original split triangles only.
       - `test_split_sphere_at_equator` -- Split sphere at z=0. Two hemispheres, each watertight, each with approximately half the volume.
       - `test_split_no_intersection` -- Plane doesn't intersect mesh. One half has full mesh, other half is empty (CsgError or empty mesh).
       - `test_split_at_arbitrary_angle` -- Diagonal plane. Both halves watertight.

       Hollow tests:
       - `test_hollow_box` -- Hollow a box with 2mm walls. Result is watertight. Volume < original. Volume approximately = original - inner_box_volume.
       - `test_hollow_sphere` -- Hollow a sphere. Result is watertight hollow sphere.
       - `test_hollow_with_drain_hole` -- Hollow box with drain hole. Result is watertight except for the drain hole opening (or still watertight if drain connects properly). Triangle count > hollow without drain.
       - `test_hollow_thin_wall_warning` -- Very thick wall relative to mesh size. CsgReport should contain a warning.

       Offset tests:
       - `test_offset_box_positive` -- Positive offset on a box. Result box is larger in all dimensions. Volume > original.
       - `test_offset_box_negative` -- Negative offset (shrink). Result box is smaller.
