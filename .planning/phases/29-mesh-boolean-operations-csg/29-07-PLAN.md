  <action>
1. Ensure slicecore-cli/Cargo.toml has slicecore-mesh dependency (should already via slicecore-engine, but check).

2. Add `Csg` variant to the `Commands` enum in main.rs:

```rust
/// Perform CSG (boolean) operations on mesh files
Csg {
    /// Boolean operation: union, difference, intersection, xor
    #[arg(value_parser = ["union", "difference", "intersection", "xor"])]
    operation: String,

    /// First input mesh file (STL, 3MF, or OBJ)
    input1: PathBuf,

    /// Second input mesh file (STL, 3MF, or OBJ)
    input2: PathBuf,

    /// Output mesh file (format inferred from extension: .stl, .3mf, .obj)
    #[arg(short, long)]
    output: PathBuf,

    /// Output CsgReport as JSON to stdout
    #[arg(long)]
    json: bool,
}
```

3. Add handler in main match:
   a. Load both input meshes using slicecore-fileio
   b. Parse operation string to BooleanOp enum
   c. Call the appropriate mesh_* function
   d. Infer output format from file extension (.stl -> STL binary, .3mf -> 3MF, .obj -> OBJ)
   e. Export using slicecore-fileio export functions
   f. If `--json`, serialize CsgReport to stdout using `serde_json::to_string_pretty`
   g. Print summary to stderr: "CSG {operation}: {input_tris_a} + {input_tris_b} -> {output_tris} triangles"
   h. Handle errors with human-readable messages using anyhow context

4. Also add `split` and `hollow` as sub-operations for the csg subcommand:
   - `slicecore csg split input.stl --plane 0,0,1,50 -o upper.stl` (normal_x,normal_y,normal_z,distance)
   - `slicecore csg hollow input.stl --thickness 2.0 -o output.stl`

   These can be implemented as separate subcommands under a `Csg` subcommand group, or as operation values alongside union/difference/intersection/xor. Choose the cleanest clap structure.
  </action>
