  <action>
    1. Create `crates/slicecore-render/src/shading.rs`:
       - `pub(crate) fn shade_vertex(normal: Vec3f, light_dir: Vec3f, ambient: f32) -> f32`
         Returns: ambient + max(0, dot(normal, light_dir)) * (1.0 - ambient)
       - Light direction constant: normalized (0.577, 0.577, 0.577) = upper-right-front
       - Default ambient: 0.2

    2. Create `crates/slicecore-render/src/rasterizer.rs`:
       - `pub(crate) fn rasterize_triangle(fb: &mut Framebuffer, v0/v1/v2: screen-space vertices with z and shaded color)`
       - Screen-space vertex struct: `pub(crate) struct ScreenVertex { x: f32, y: f32, z: f32, r: f32, g: f32, b: f32 }`
       - Algorithm:
         a. Compute edge function area = edge_function(v0, v1, v2). If area <= 0 (CW/degenerate), skip (back-face cull).
         b. Compute bounding box of triangle, clamp to framebuffer dimensions.
         c. For each pixel in bbox: compute barycentric coords (w0, w1, w2) via edge functions.
         d. If all >= 0, pixel is inside. Interpolate z and color using barycentric weights (divide by area).
         e. Call fb.set_pixel(x, y, z, color_rgba) where alpha = 255.
       - `fn edge_function(a, b, point) -> f32` = (point.x - a.x) * (b.y - a.y) - (point.y - a.y) * (b.x - a.x)

    3. Create `crates/slicecore-render/src/png_encode.rs`:
       - `pub(crate) fn encode_png(width: u32, height: u32, pixels: &[[u8; 4]]) -> Vec<u8>`
       - Use png crate: Encoder::new, set ColorType::Rgba, BitDepth::Eight, Compression::Fast
       - Flatten &[[u8; 4]] to &[u8] via unsafe pointer cast (safe because [u8; 4] is repr(C)-compatible) or use bytemuck-style manual flatten: `unsafe { std::slice::from_raw_parts(pixels.as_ptr() as *const u8, pixels.len() * 4) }`
       - Framebuffer stores top-to-bottom row order (matching PNG convention, no flip needed)

    4. Create `crates/slicecore-render/src/pipeline.rs`:
       - `pub(crate) fn render_to_framebuffer(mesh: &TriangleMesh, angle: &CameraAngle, config: &ThumbnailConfig) -> Framebuffer`
       - Algorithm:
         a. Handle degenerate mesh (0 triangles): return framebuffer with only background color
         b. Compute vertex normals via camera::compute_vertex_normals
         c. Build view + projection matrices via camera::build_camera
         d. Compute MVP = projection * view (no model matrix, mesh is in world space)
         e. Viewport transform: NDC [-1,1] -> screen [0, width/height], flip Y
         f. For each triangle:
            - Transform 3 vertices by MVP to clip space
            - Perspective divide (w=1 for ortho, but keep pattern general)
            - Viewport transform to screen coordinates
            - Shade 3 vertices using shading::shade_vertex (apply model_color * intensity)
            - Call rasterizer::rasterize_triangle
         g. Return framebuffer

    5. Update `crates/slicecore-render/src/lib.rs` -- complete public API:

