  <name>Task 1: 3MF thumbnail embedding and G-code thumbnail comment formatting</name>
  <files>
    crates/slicecore-render/Cargo.toml,
    crates/slicecore-render/src/gcode_embed.rs,
    crates/slicecore-render/src/lib.rs,

    crates/slicecore-gcode-io/Cargo.toml,
    crates/slicecore-engine/src/config.rs
  </files>
  <behavior>
    - save_mesh_to_writer_with_thumbnail writes 3MF that contains Metadata/thumbnail.png entry
    - format_gcode_thumbnail_block produces "; thumbnail begin WxH SIZE\n; base64lines...\n; thumbnail end"
    - Base64 lines are max 78 chars each, prefixed with "; "
    - GcodeDialect::Bambu skips G-code thumbnails (3MF only)
    - PrintConfig.thumbnail_resolution defaults to [300, 300]
    - thumbnail_resolution field roundtrips through TOML serialization
  </behavior>
  <action>
    1. Add `base64 = "0.22"` dependency to `crates/slicecore-render/Cargo.toml` (or workspace Cargo.toml as workspace dep).

    2. Create `crates/slicecore-render/src/gcode_embed.rs`:
       - `pub fn format_gcode_thumbnail_block(thumbnail: &Thumbnail, format: ThumbnailFormat) -> String`
       - `pub enum ThumbnailFormat { PrusaSlicer, Creality }` -- PrusaSlicer uses `; thumbnail begin`, Creality uses `; png begin`
       - Algorithm: base64-encode png_data, split into 78-char lines, prefix each with `; `, wrap in begin/end tags with WxH and byte size
       - `pub fn thumbnail_format_for_dialect(dialect_name: &str) -> Option<ThumbnailFormat>` -- "marlin"|"klipper"|"reprap" -> PrusaSlicer format, "bambu" -> None (3MF only)

    3. Update `crates/slicecore-render/src/lib.rs` to add:
       - `pub mod gcode_embed;` (make it public for cross-crate use)
       - Re-export: `pub use gcode_embed::{format_gcode_thumbnail_block, ThumbnailFormat, thumbnail_format_for_dialect};`

    4. Update `crates/slicecore-fileio/src/export.rs`:
       - Add `pub fn save_mesh_with_thumbnail(mesh: &TriangleMesh, path: &Path, thumbnail_png: Option<&[u8]>) -> Result<(), FileIOError>` -- works like save_mesh but inserts thumbnail into Model.attachments at "Metadata/thumbnail.png" before writing
       - Also add `pub fn save_mesh_to_writer_with_thumbnail<W: Write + Seek>(mesh: &TriangleMesh, writer: W, format: ExportFormat, thumbnail_png: Option<&[u8]>) -> Result<(), FileIOError>` -- same but to writer. For non-3MF formats, thumbnail is ignored.
       - In the internal `triangle_mesh_to_model`, after creating the model, if thumbnail_png is Some, insert into `model.attachments` with key "Metadata/thumbnail.png" and set `object.thumbnail = Some("Metadata/thumbnail.png".to_string())`
       - Add `slicecore-render` is NOT a dependency of fileio. The fileio functions just accept raw `&[u8]` PNG bytes.

    5. Create `crates/slicecore-gcode-io/src/thumbnail.rs`:
       - `pub fn write_thumbnail_comments<W: Write>(writer: &mut W, png_data: &[u8], width: u32, height: u32, format: &str) -> Result<(), std::io::Error>`
       - format: "prusaslicer" or "creality" to select begin/end tags
       - Base64-encode, split lines, write as comments
       - This takes raw bytes (no dependency on slicecore-render)
       - Add `base64 = "0.22"` dependency to slicecore-gcode-io

    6. Update `crates/slicecore-gcode-io/src/lib.rs`:
       - Add `pub mod thumbnail;`
       - Re-export: `pub use thumbnail::write_thumbnail_comments;`

    7. Update `crates/slicecore-engine/src/config.rs`:
       - Add to PrintConfig: `pub thumbnail_resolution: [u32; 2],` with `#[serde(default = "default_thumbnail_resolution")]`
       - `fn default_thumbnail_resolution() -> [u32; 2] { [300, 300] }`

    8. Add unit tests:
       - fileio: save_mesh_with_thumbnail for 3MF produces larger output than without (thumbnail data present)
       - gcode_embed: format_gcode_thumbnail_block output contains "thumbnail begin", base64 lines, "thumbnail end"
       - gcode_embed: lines are <= 80 chars ("; " + 78 base64 chars)
       - config: thumbnail_resolution defaults and TOML round-trip
  </action>

    2. Add `Thumbnail` subcommand to the Commands enum in main.rs:
       ```rust
       /// Generate thumbnail preview images from a mesh file.
       Thumbnail {
           /// Input mesh file (STL, 3MF, OBJ)
           input: String,
           /// Output directory or file path (default: input filename with .png extension)
           #[arg(short, long)]
           output: Option<String>,
           /// Camera angles to render (comma-separated: front,back,left,right,top,isometric)
           /// Default: isometric only
           #[arg(long, default_value = "isometric")]
           angles: String,
           /// Resolution WxH (e.g., "300x300", "220x124", "640x480")
           /// Default: 300x300
           #[arg(long, default_value = "300x300")]
           resolution: String,
           /// Background color as hex (e.g., "transparent", "FFFFFF", "000000")
           /// Default: transparent
           #[arg(long, default_value = "transparent")]
           background: String,
           /// Model color as hex (e.g., "C8C8C8", "FF0000")
           /// Default: C8C8C8 (light gray)
           #[arg(long, default_value = "C8C8C8")]
           color: String,
       }
       ```

    3. Add `--thumbnails` flag to the existing Slice subcommand:
       ```rust
       /// Embed thumbnail images in output (3MF or G-code).
       #[arg(long)]
       thumbnails: bool,
       ```

    4. Implement `cmd_thumbnail(input, output, angles, resolution, background, color)`:
       - Load mesh via slicecore_fileio::load_mesh
       - Parse angles string into Vec<CameraAngle> (split on comma, map to enum variants, "all" = CameraAngle::all())
       - Parse resolution string "WxH" into (u32, u32)
       - Parse background: "transparent" -> [0,0,0,0], hex -> [r,g,b,255]
       - Parse color hex string -> [u8; 3]
       - Build ThumbnailConfig, call render_mesh
       - For single angle: write PNG to output path (default: input.png)
       - For multiple angles: write each as {input}_{angle}.png in output dir

    5. Update Slice command handler (cmd_slice) to handle `--thumbnails`:
       - After loading mesh, if thumbnails=true:
         a. Build ThumbnailConfig from PrintConfig.thumbnail_resolution
         b. Call render_mesh with single isometric angle
         c. If output is 3MF: use save_mesh_with_thumbnail (but since slice output is G-code, this applies if we add 3MF slice output later; for now, embed in G-code)
         d. If G-code output: prepend thumbnail comment block to G-code output using appropriate dialect format. Use thumbnail_format_for_dialect to determine comment format (skip for Bambu).
       - The thumbnail is injected before the start G-code in the output

    6. Add match arm for Commands::Thumbnail in main:
       ```rust
       Commands::Thumbnail { input, output, angles, resolution, background, color } => {
           cmd_thumbnail(&input, output.as_deref(), &angles, &resolution, &background, &color)
       }
       ```
