<task type="auto">
  <name>Task 1: BVH AABB overlap query and enhanced self-intersection detection</name>
  <files>
    crates/slicecore-mesh/src/bvh.rs
    crates/slicecore-mesh/src/repair/intersect.rs
    crates/slicecore-mesh/src/repair.rs
  </files>
  <action>
1. **Add `query_aabb_overlaps()` to BVH** (`bvh.rs`):
   - Add a new public method `pub fn query_aabb_overlaps(&self, query: &BBox3) -> Vec<usize>` that returns all triangle indices whose AABBs overlap with the given query AABB.
   - Implement via recursive traversal similar to `query_plane_recursive`, but testing AABB-AABB overlap instead of Z-range containment. Two AABBs overlap when: `a.min.x <= b.max.x && a.max.x >= b.min.x && a.min.y <= b.max.y && a.max.y >= b.min.y && a.min.z <= b.max.z && a.max.z >= b.min.z`.

2. **Add `find_intersecting_pairs()` to `intersect.rs`**:
   - New public function: `pub fn find_intersecting_pairs(vertices: &[Point3], indices: &[[u32; 3]]) -> Vec<(usize, usize)>`.
   - Uses BVH broad-phase: build BVH, for each triangle compute its AABB, query BVH for overlapping triangles, then narrow-phase test with existing `triangles_intersect()`.
   - Only collect pairs where `i < j` and `!shares_vertex()`.
   - Return `Vec<(usize, usize)>` of intersecting pair indices.

3. **Add `intersection_z_range()` to `intersect.rs`**:
   - New public function: `pub fn intersection_z_range(vertices: &[Point3], indices: &[[u32; 3]], pairs: &[(usize, usize)]) -> Option<(f64, f64)>`.
   - Iterates all triangle vertices involved in intersecting pairs and returns the `(z_min, z_max)` span.
   - Returns `None` if pairs is empty.

4. **Update `detect_self_intersections()`** to use BVH broad-phase internally (replacing the O(n^2) loop with `query_aabb_overlaps`). Keep the same return type (usize count) for backward compatibility.

5. **Update `RepairReport`** in `repair.rs`:
   - Add fields:
     - `pub intersecting_pairs: Vec<(usize, usize)>` -- triangle pair indices
     - `pub self_intersections_resolvable: bool` -- true when intersections detected (will be resolved at slice time)
     - `pub intersection_z_range: Option<(f64, f64)>` -- Z-band affected
   - Update `repair()` function: call `find_intersecting_pairs()` instead of (or in addition to) `detect_self_intersections()`, populate the new fields, set `self_intersections_detected = pairs.len()`, set `self_intersections_resolvable = !pairs.is_empty()`, compute z_range from pairs.

6. **Add unit tests** for the new functions:
   - `find_intersecting_pairs()` on the existing test cases (separated triangles = empty, crossing triangles = 1 pair).
   - `intersection_z_range()` with known Z values.
   - `query_aabb_overlaps()` on a simple mesh.
  </action>
  <verify>
    `cargo test -p slicecore-mesh` -- all existing + new tests pass. `cargo clippy -p slicecore-mesh` -- no warnings.
  </verify>
  <done>
    `find_intersecting_pairs()` returns correct pair indices, `intersection_z_range()` returns correct Z span, `detect_self_intersections()` uses BVH acceleration, RepairReport has new fields populated by repair().
  </done>
</task>

  <action>
1. **Create `crates/slicecore-slicer/src/resolve.rs`**:
   - Add module-level doc comment explaining per-slice contour resolution via Clipper2 polygon union.
   - Implement `pub fn resolve_contour_intersections(contours: &[ValidPolygon]) -> Vec<ValidPolygon>`:
     - If contours has 0 or 1 elements, return `contours.to_vec()` (no resolution needed).
     - Otherwise, call `polygon_union(contours, &[])` from `slicecore_geo` to self-union all contours, which merges overlapping regions and resolves self-intersecting boundaries.
     - On error from polygon_union (should be rare), fall back to returning the original contours unchanged (log the issue but don't panic).
     - Return the cleaned contours.
   - Note: This is a "self-union" operation -- unioning all contours with an empty clip set causes Clipper2 to clean up all overlaps and self-intersections in the subject set.

2. **Register the module in `lib.rs`**:
   - Add `pub mod resolve;` to lib.rs.
   - Add `pub use resolve::resolve_contour_intersections;` to the re-exports.

3. **Add unit tests** in resolve.rs:
   - Test with two overlapping square contours (partially overlapping ValidPolygons) -- result should be a single merged polygon.
   - Test with non-overlapping contours -- result should preserve both.
   - Test with single contour -- returned unchanged.
   - Test with empty input -- returns empty.
  </action>

  </verify>
