<objective>
Wire contour resolution into the slicing pipeline so self-intersecting meshes automatically produce clean contours.

Purpose: Connects the detection (Plan 01 RepairReport) to the resolution (Plan 01 resolve_contour_intersections) through the engine pipeline. When the repair report indicates self-intersections, the slicer applies polygon union on each layer's contours to resolve overlaps. This is the "deferred resolution" pattern from the research -- detect at repair time, resolve at slice time.

Output: Updated engine pipeline that runs repair, checks for intersections, and conditionally applies contour resolution during slicing. Test meshes with self-intersections that demonstrate the full flow.
</objective>

  <files>

  <action>
1. **Add `slice_at_height_resolved()` to `contour.rs`**:
   - New public function: `pub fn slice_at_height_resolved(mesh: &TriangleMesh, z: f64) -> Vec<ValidPolygon>`.
   - Identical to `slice_at_height()` but applies `resolve_contour_intersections()` from the resolve module on the contours before returning.
   - This is the version called when the mesh is known to have self-intersections.

2. **Add `slice_mesh_resolved()` and `slice_mesh_adaptive_resolved()` to `layer.rs`**:
   - Mirror of `slice_mesh()` and `slice_mesh_adaptive()` but using `slice_at_height_resolved()` instead of `slice_at_height()`.
   - These are the entry points the engine calls when self-intersections are detected.

3. **Register new re-exports in `lib.rs`**:
   - Add `pub use contour::slice_at_height_resolved;`
   - Add `pub use layer::{slice_mesh_resolved, slice_mesh_adaptive_resolved};`

4. **Wire into engine pipeline** (`engine.rs`):
   - In the `slice_to_writer_with_events()` method, before step 1 (slicing), run mesh repair on the input mesh to get a RepairReport. The engine currently does NOT run repair (CLI does it before engine). Instead of running repair, add a simpler approach:
     - Add a `needs_contour_resolution: bool` field to Engine (set during construction or via a method).
     - Add a public method `pub fn set_needs_contour_resolution(&mut self, needs: bool)` that sets the flag.
     - In step 1 (mesh slicing), check the flag. If true, use `slice_mesh_resolved()` / `slice_mesh_adaptive_resolved()` instead of the regular versions.
     - Emit a SliceEvent::Warning when contour resolution is active: "Applying per-slice contour union to resolve self-intersecting geometry".
   - **Alternative (preferred) approach:** Rather than a field on Engine, check if the mesh has self-intersections at slice time. Add a lightweight helper: call `detect_self_intersections()` from slicecore_mesh (already public) on the mesh in the engine, and if count > 0, use the resolved slicing path. This is simpler and automatic. Cache the result to avoid running detection twice.

   Choose the preferred approach: detect at slice time, use resolved path if needed. This is fully transparent to the user.

5. **Update engine's slice_to_writer_with_events()**:
   - Before step 1, add a new step "0c. Self-intersection check":
     ```
     let has_self_intersections = slicecore_mesh::repair::intersect::detect_self_intersections(
         mesh.vertices(), mesh.indices()
     ) > 0;
     ```
   - If true, emit a warning event and use resolved slice functions in step 1.
   - In step 1, branch on `has_self_intersections`: use `slice_mesh_resolved` / `slice_mesh_adaptive_resolved` when true, regular versions when false.

6. **Add unit tests**:
   - Test that a clean mesh uses the regular path (no performance penalty).
   - Test that a self-intersecting mesh (two overlapping cubes combined into one TriangleMesh) produces contours with contour union applied (the overlapping region is merged into one contour instead of two overlapping contours).
  </action>

  </verify>
  <done>
    Self-intersecting meshes are automatically detected at slice time and their contours are resolved via polygon union. Clean meshes skip the resolution step. Engine emits a warning when resolution is active.
  </done>
