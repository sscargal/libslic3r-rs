Create `crates/slicecore-engine/tests/phase12_integration.rs` with tests for each success criterion:

**Helper functions:**
- `make_two_overlapping_cubes() -> TriangleMesh`: Two 10mm cubes overlapping by 5mm in X and Y, sharing the Z range. Creates 24 triangles with known self-intersections.
- `make_three_overlapping_cubes() -> TriangleMesh`: Three cubes in a chain (A overlaps B, B overlaps C) for more complex intersection patterns. ~36 triangles.
- `make_large_overlapping_mesh(n: usize) -> TriangleMesh`: Generates n overlapping cube pairs arranged in a grid for performance testing. Produces ~24*n triangles.

**SC1: Self-intersection resolution uses Clipper2 boolean union**
- `sc1_clipper2_union_resolves_overlapping_contours`:
  - Create two_overlapping_cubes mesh.
  - Slice at a Z-height in the overlap region.
  - Call `resolve_contour_intersections()` on the raw contours.
  - Assert the resolved contours are non-empty.
  - Assert the resolved contour total area is approximately equal to the union area (not the sum of individual areas).
  - Verify the resolved contours are valid (no self-intersecting polygon edges).

**SC2: RepairReport shows before/after metrics**
- `sc2_repair_report_shows_intersection_metrics`:
  - Create two_overlapping_cubes from raw vertices and indices.
  - Run `repair()` on them.
  - Assert `report.self_intersections_detected > 0`.
  - Assert `report.intersecting_pairs` is non-empty with valid triangle indices.
  - Assert `report.self_intersections_resolvable == true`.
  - Assert `report.intersection_z_range.is_some()` and the Z range covers the overlap region.
- `sc2_clean_mesh_has_zero_intersections`:
  - Create a single cube (clean mesh).
  - Run `repair()`.
  - Assert `report.self_intersections_detected == 0`.
  - Assert `report.intersecting_pairs.is_empty()`.
  - Assert `report.self_intersections_resolvable == false`.

**SC3: Real-world-like self-intersecting models repair and slice successfully**
- `sc3_two_overlapping_cubes_slices_end_to_end`:
  - Create two_overlapping_cubes mesh.
  - Slice with `Engine::slice()`.
  - Assert no error.
  - Assert G-code is non-empty.
  - Assert layer_count > 0.
- `sc3_three_overlapping_cubes_slices_end_to_end`:
  - Create three_overlapping_cubes mesh.
  - Slice with `Engine::slice()`.
  - Assert no error.
  - Assert G-code is non-empty.
- `sc3_offset_sphere_shell_slices_successfully`:
  - Create a programmatic mesh with two offset icosphere shells (2x subdivided, one offset by half-radius) that produce self-intersections.
  - If constructing an icosphere is too complex, use two offset boxes with rotation to create non-axis-aligned intersections.
  - Slice with Engine::slice().
  - Assert no error.

**SC4: Repaired mesh output passes validation**
- `sc4_resolved_contours_are_valid_polygons`:
  - Create two_overlapping_cubes mesh.
  - For 5 Z-heights in the overlap region, call `slice_at_height_resolved()`.
  - Assert each layer has at least 1 contour.
  - Assert each contour is a valid ValidPolygon (it already is by type, but verify area > 0 and point count >= 3).
  - Verify no contour has zero area (degenerate).
- `sc4_resolved_contours_have_correct_winding`:
  - For the resolved contours above, verify outer boundaries are CCW (positive area) and holes are CW (negative area) -- matching the project's winding convention.

**SC5: Performance -- resolution completes in <5 seconds for <10k triangles**
- `sc5_performance_under_5_seconds`:
  - Create a mesh with ~5000-8000 triangles using `make_large_overlapping_mesh()`.
  - Time the full pipeline: repair + detect intersections + slice + resolve.
  - Assert total time < 5 seconds.
  - Print the elapsed time for diagnostics.
  - Use `std::time::Instant` for measurement.
  - If <10k triangles is hard to construct with overlapping cubes, use ~400 cube pairs (400 * 24 = 9600 triangles).
