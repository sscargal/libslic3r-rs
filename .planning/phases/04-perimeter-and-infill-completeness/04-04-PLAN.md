  <action>
    Implement honeycomb (hexagonal grid) infill in `infill/honeycomb.rs`.

    **Algorithm (zigzag polyline approach -- most common in slicers):**

    Honeycomb is generated as a series of zigzag polylines that form hexagonal boundaries when stacked. The pattern alternates between two row types that together create the hex grid.

    1. Compute hex cell geometry from density:
       - `spacing = line_width / density` (same as rectilinear)
       - `hex_height = spacing * sqrt(3)` (height of one hex cell)
       - `hex_half_width = spacing` (half the width of one hex cell)

    2. Generate zigzag rows:
       - Two row types alternate vertically (even/odd rows):
         - Even rows: zigzag going right (\_/\_/\_/)
         - Odd rows: zigzag going right but offset by half a period (shifted phase)
       - Each row is a series of connected line segments forming a zigzag at +-60 degrees from horizontal.
       - Row spacing = hex_height / 2.

    3. For each zigzag row across the bounding box:
       - Generate the zigzag polyline points.
       - Convert each polyline segment to integer coordinates.
       - Clip each segment against the infill region using the scanline intersection approach from rectilinear (find entry/exit points against polygon edges).

    4. Simplified approach for Phase 4:
       - Generate horizontal lines at hex_height spacing.
       - Generate diagonal lines at +60 degrees connecting alternating endpoints.
       - Generate diagonal lines at -60 degrees connecting the remaining endpoints.
       - Clip all lines against infill_region.
       - This produces the hex pattern when viewed together.

    Alternative simpler implementation:
    - Use three sets of parallel lines at 0, 60, and 120 degrees.
    - Each set uses spacing = spacing * 2 (doubled because three sets combine).
    - Clip all three sets against infill_region.
    - This produces a triangular grid, not hexagonal. To make it hexagonal, use offset patterns.

    **Recommended: Zigzag approach.**
    - For each row y: generate a zigzag polyline where the x coordinates alternate between x and x + hex_half_width at vertical intervals of hex_height/2.
    - Segment each zigzag into individual InfillLine segments.
    - Clip each segment against the infill region.

    ```rust
        infill_region: &[ValidPolygon],
        density: f64,
        layer_index: usize,
        line_width: f64,
    ) -> Vec<InfillLine>
    ```

    Note: layer_index is used to shift the phase of the honeycomb pattern between layers (even/odd layers shift by half a period). This creates proper interlocking.

    - Density 0% returns empty.

    Implement cubic infill in `infill/cubic.rs`.

    **Algorithm:**
    Cubic infill creates a 3D interlocking cube structure by cycling through three angles across layers with Z-dependent phase offsets.

    1. The pattern uses three angles: 0, 60, and 120 degrees, cycling every 3 layers:
       - Layer 0, 3, 6, ...: angle = 0 degrees
       - Layer 1, 4, 7, ...: angle = 60 degrees
       - Layer 2, 5, 8, ...: angle = 120 degrees

    2. Z-dependent phase offset:
       - Each layer's lines are shifted by an offset proportional to the Z height.
       - `offset = (layer_z * frequency) % spacing` where frequency controls the cube periodicity.
       - This creates the 3D interlocking structure when viewed in cross-section.

    3. For the selected angle, generate parallel lines at the computed spacing + offset, then clip against the infill region.

    4. Reuse the rectilinear line generation but with rotation:
       - For angle 0: use horizontal lines (same as rectilinear).
       - For angle 60 or 120: rotate all polygon points by -angle, generate horizontal lines, then rotate line endpoints back by +angle.
       - The rotation approach avoids implementing diagonal line clipping from scratch.

    **Rotation helper:**
    ```rust
    fn rotate_point(pt: IPoint2, angle_rad: f64, center: IPoint2) -> IPoint2 {
        let dx = (pt.x - center.x) as f64;
        let dy = (pt.y - center.y) as f64;
        let cos_a = angle_rad.cos();
        let sin_a = angle_rad.sin();
        IPoint2::new(
            center.x + (dx * cos_a - dy * sin_a).round() as Coord,
            center.y + (dx * sin_a + dy * cos_a).round() as Coord,
        )
    }
    ```

    ```rust
    pub fn generate(
        infill_region: &[ValidPolygon],
        density: f64,
        layer_index: usize,
        layer_z: f64,
        line_width: f64,
    ) -> Vec<InfillLine>
    ```

    Note: cubic needs `layer_z` for the phase offset (unlike rectilinear which only needs layer_index).

    - Layer 0, layer 1, and layer 2 produce lines at different angles.
    - Lines are within bounding box of infill region.
    - Z-offset shift: same layer index but different Z produces shifted line positions.
    - Empty region returns empty.
