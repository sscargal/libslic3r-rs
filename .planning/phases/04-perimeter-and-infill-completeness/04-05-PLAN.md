Implement Gyroid infill pattern (INFILL-04) using TPMS implicit surface evaluation and marching squares contour extraction.

Purpose: Gyroid is the strongest infill pattern per unit of material, based on a triply periodic minimal surface (TPMS). Unlike grid-based patterns, gyroid creates smooth curves that distribute stress evenly. It's the preferred pattern for structural parts in modern slicing.

Output: Working gyroid infill generator that produces smooth Z-dependent curves clipped to the infill region.
</objective>

  <action>
    Create `infill/gyroid.rs` with the gyroid pattern implementation.

    **Gyroid TPMS function:**
    ```
    f(x, y, z) = cos(x)*sin(y) + cos(y)*sin(z) + cos(z)*sin(x)
    ```
    At a fixed Z = z_layer, this becomes a 2D implicit function:
    ```
    f(x, y) = cos(x)*sin(y) + cos(y)*sin(z_layer) + cos(z_layer)*sin(x)
    ```
    The iso-contour at f(x,y) = 0 gives the infill lines for that layer.

    **Algorithm:**

    1. Compute frequency from density:
       - `spacing = line_width / density`
       - `freq = 2.0 * PI / spacing` (one full gyroid period per spacing distance)

    2. Determine sampling grid:
       - Compute bounding box of infill_region (in mm, using coord_to_mm).
       - Grid step = line_width / 2.0 (fine enough to capture all features).
       - Create a 2D grid of f(x,y) values where x and y are scaled by freq.

    3. Marching squares to extract iso-contour at threshold = 0:
       - For each grid cell (4 corners), classify each corner as positive (f > 0) or negative (f <= 0).
       - The 4 binary corner states form a 4-bit index (0-15) identifying one of 16 marching squares cases.
       - For each case, generate 0, 1, or 2 line segments connecting the edges where the iso-contour crosses.
       - Edge intersection: linearly interpolate between the two corner values to find the crossing point.

    **Marching squares case table (standard):**
    Cases 0 and 15: no contour (all same sign).
    Cases 1,2,4,8,7,11,13,14: one line segment.
    Cases 3,6,9,12: one line segment.
    Cases 5,10: ambiguous (saddle point) -- resolve by checking center value; produces two segments.

    4. Convert marching squares output segments to InfillLine:
       - Each segment is in mm coordinates. Convert to IPoint2 via mm_to_coord.
       - Clip: only keep segments whose midpoint is inside the infill_region (point-in-polygon test using the existing `point_in_polygon` function from slicecore-geo, or simply check that both endpoints are inside). Alternatively, generate segments only within the bbox and filter.

    5. Better clipping approach: generate the full marching squares grid within the infill region's bounding box, then filter segments. For segments partially outside, the clipping against polygon edges can be done by checking intersection points. For Phase 4, a simpler approach: keep segments where BOTH endpoints are inside the infill region (test each endpoint with point-in-polygon). This may lose some edge segments but produces correct infill.

    ```rust
    pub fn generate(
        infill_region: &[ValidPolygon],
        density: f64,
        layer_index: usize,
        layer_z: f64,
        line_width: f64,
    ) -> Vec<InfillLine>
    ```

    **Implementation details:**
    - The `gyroid_value(x: f64, y: f64, z: f64) -> f64` function computes the TPMS value.
    - The marching squares grid uses floating-point coordinates internally, converting to IPoint2 only at the output stage.
    - Grid resolution: max(ceil((bbox_width / grid_step)), 1) cells in each direction.
    - For very small regions, the grid may have few cells -- that's fine.

    **Tests:**
    - 20mm square at 20% density produces infill lines.
    - Different Z heights produce different line patterns (3D surface varies with Z).
    - Lines are within or near the bounding box of the infill region.
    - Density 0% returns empty.
    - Empty region returns empty.
    - Pattern contains curved/non-axis-aligned segments (unlike rectilinear).
    - Deterministic: same inputs produce same output.
    - Marching squares unit test: a known 2x2 grid with specific values produces expected segments.
