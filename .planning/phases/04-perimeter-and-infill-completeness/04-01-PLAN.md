    **Grid infill (INFILL-02)** in `infill/grid.rs`:

    Grid = rectilinear at BOTH 0 and 90 degrees on the SAME layer (crosshatch pattern).
    ```rust
    pub fn generate(
        infill_region: &[ValidPolygon],
        density: f64,
        layer_index: usize,
        line_width: f64,
    ) -> Vec<InfillLine> {
        // Generate horizontal lines (0 degrees)
        let mut lines = rectilinear::generate(infill_region, density, 0.0, line_width);
        // Generate vertical lines (90 degrees) on the same layer
        lines.extend(rectilinear::generate(infill_region, density, 90.0, line_width));
        lines
    }
    ```
    NOTE: Grid uses half the density for each direction (density/2) so the combined density matches the target. Otherwise at density=0.2, grid would produce twice the material of rectilinear at 0.2. Correct formula: each direction uses `density` as-is but the spacing is doubled (i.e., the user-specified density accounts for crosshatch). Actually, the standard convention is: grid at density D produces same material as rectilinear at density D, because grid provides strength in both directions simultaneously. Keep density as-is for each direction -- the user picks a lower density for grid since it's stronger.

    Tests for Grid:
    - 20mm square at 20% density produces lines in both horizontal AND vertical directions.
    - Grid produces more total lines than rectilinear at same density (since both directions).
    - Lines include both same-y (horizontal) and same-x (vertical) segments.
    - Empty region returns empty.

    **Monotonic infill (INFILL-08)** in `infill/monotonic.rs`:

    Monotonic = rectilinear but all lines are printed in ONE direction (left-to-right). This eliminates ridges from bidirectional printing on top surfaces.

    Algorithm:
    1. Generate rectilinear scanlines (same as rectilinear pattern).
    2. Sort all lines so they are printed strictly left-to-right (start.x < end.x for horizontal) or top-to-bottom (start.y < end.y for vertical).
    3. Group adjacent lines and order them by sweep direction. For horizontal lines: sort by Y ascending, then ensure each line goes from left to right (if end.x < start.x, swap start/end).
    4. The key difference from rectilinear: rectilinear alternates direction (serpentine), monotonic always goes the same direction.

    ```rust
    pub fn generate(
        infill_region: &[ValidPolygon],
        density: f64,
        layer_index: usize,
        line_width: f64,
    ) -> Vec<InfillLine> {
        let angle = alternate_infill_angle(layer_index);
        let mut lines = rectilinear::generate(infill_region, density, angle, line_width);
        // Ensure all lines go in the same direction (monotonic)
        for line in &mut lines {
            if angle < 45.0 {
                // Horizontal: ensure left-to-right
                if line.start.x > line.end.x {
                    std::mem::swap(&mut line.start, &mut line.end);
                }
            } else {
                // Vertical: ensure bottom-to-top
                if line.start.y > line.end.y {
                    std::mem::swap(&mut line.start, &mut line.end);
                }
            }
        }
        lines
    }
    ```

    Tests for Monotonic:
    - 20mm square produces lines where all horizontal lines go left-to-right (start.x <= end.x).
    - Produces same number of lines as rectilinear (same density, different ordering).
    - Vertical layer (angle=90) has all lines going bottom-to-top.

    **Update dispatch in mod.rs:**
    - Wire `InfillPattern::Grid => grid::generate(...)` and `InfillPattern::Monotonic => monotonic::generate(...)`.
