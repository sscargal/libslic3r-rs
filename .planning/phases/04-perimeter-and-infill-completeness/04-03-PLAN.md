<objective>
Implement adaptive layer heights based on surface curvature (SLICE-02).

Purpose: Adaptive layers use thinner layers where surface curvature is high (smooth curves, overhangs) and thicker layers where the surface is flat or near-vertical. This improves visual quality on curved surfaces while maintaining fast print speed on flat areas -- potentially reducing print time 20-50% compared to uniformly thin layers.

Output: Working adaptive layer height algorithm integrated into the slicer and engine, with configurable parameters.
</objective>

  <action>
    Create `adaptive.rs` in slicecore-slicer with the adaptive layer height algorithm.

    **Main function:**
    ```rust
    pub fn compute_adaptive_layer_heights(
        mesh: &TriangleMesh,
        min_height: f64,
        max_height: f64,
        quality: f64,       // 0.0 = max speed (thicker layers), 1.0 = max quality (thinner)
        first_layer_height: f64,
    ) -> Vec<(f64, f64)>    // Returns (z_position, layer_height) pairs
    ```

    **Algorithm:**

    Step 1: Sample surface curvature at fine Z intervals.
    - Sample step = min_height / 2.0 (fine enough to detect curvature changes).
    - For each sample Z from first_layer_height/2 to mesh max Z:
      - Find all triangle faces that span this Z (triangle's min_z <= z <= max_z).
      - Compute the average |normal.z| component of these triangles.
      - `steepness = 1.0 - avg_abs_normal_z`. High steepness = surface nearly vertical = low curvature for layer height purposes. The key metric is the CHANGE in steepness between adjacent Z samples.
    - Compute curvature as: `curvature[i] = |steepness[i] - steepness[i-1]| / sample_step`. This measures how quickly the surface angle changes = the actual surface curvature that affects visual quality.

    Step 2: Map curvature to desired layer height.
    - `desired_height = max_height - (max_height - min_height) * clamp(curvature * quality_factor, 0.0, 1.0)`
    - `quality_factor` maps quality parameter: quality=0 -> very low sensitivity, quality=1 -> high sensitivity.
    - Clamp desired_height to [min_height, max_height].

    Step 3: Smooth height profile.
    - Enforce max 50% height change between adjacent layers: if layer[i] > 1.5 * layer[i-1], clamp to 1.5 * layer[i-1]. If layer[i] < layer[i-1] / 1.5, clamp to layer[i-1] / 1.5.
    - Run smoothing in both forward and backward passes (2 passes total) for balanced smoothing.

    Step 4: Generate actual (z, height) pairs.
    - Start with first layer at z = first_layer_height / 2, height = first_layer_height.
    - For subsequent layers: z = previous_z + previous_height/2 + current_height/2, using the smoothed desired heights sampled at each Z position.
    - Continue until z + height/2 > mesh max Z.

    **Helper functions:**
    - `sample_curvature_profile(mesh: &TriangleMesh, sample_step: f64) -> Vec<(f64, f64)>` -- returns (z, curvature) pairs.
    - `triangles_at_z(mesh: &TriangleMesh, z: f64) -> Vec<usize>` -- returns indices of triangles spanning Z. Use mesh AABB for quick rejection, then per-triangle min/max Z check.
    - `smooth_heights(heights: &mut [(f64, f64)], max_ratio: f64)` -- forward+backward smoothing pass.

    **Update lib.rs:**
    - Add `pub mod adaptive;` and re-export `compute_adaptive_layer_heights`.

    **Tests:**
    - Unit sphere mesh: layers near the equator (z ~ 0.5, high curvature) have thinner heights than layers near the poles (z ~ 0, z ~ 1, low curvature).
    - Flat box mesh: all layers have approximately max_height (no curvature variation).
    - Height smoothing: no adjacent layers differ by more than 50%.
    - First layer height is preserved regardless of curvature.
    - quality=0 produces mostly thick layers; quality=1 produces more thin layers.
    - Returns non-empty heights for valid mesh with positive dimensions.

    **Config additions to PrintConfig:**
    ```rust
    // --- Adaptive Layer Heights ---
    /// Enable adaptive layer heights based on surface curvature.
    pub adaptive_layer_height: bool,  // default: false
    /// Minimum layer height for adaptive layers (mm).
    pub adaptive_min_layer_height: f64,  // default: 0.05
    /// Maximum layer height for adaptive layers (mm).
    pub adaptive_max_layer_height: f64,  // default: 0.3
    /// Adaptive layer quality (0.0 = speed, 1.0 = quality).
    pub adaptive_layer_quality: f64,  // default: 0.5
    ```

    **layer.rs changes:**
    Add a new function:
    ```rust
    pub fn slice_mesh_adaptive(
        mesh: &TriangleMesh,
        heights: &[(f64, f64)],  // pre-computed adaptive heights
    ) -> Vec<SliceLayer>
    ```
    This takes pre-computed height pairs instead of computing uniform heights. For each (z, layer_height) pair, slice at that Z and return a SliceLayer. This reuses `slice_at_height` from the contour module.

    **engine.rs changes:**
    In `Engine::slice_to_writer()`:
    ```rust
    // 1. Compute layer heights (uniform or adaptive)
    let layers = if self.config.adaptive_layer_height {
        let heights = compute_adaptive_layer_heights(
            mesh,
            self.config.adaptive_min_layer_height,
            self.config.adaptive_max_layer_height,
            self.config.adaptive_layer_quality,
            self.config.first_layer_height,
        );
        slice_mesh_adaptive(mesh, &heights)
    } else {
        slice_mesh(mesh, self.config.layer_height, self.config.first_layer_height)
    };
    ```

    The rest of the pipeline (perimeters, infill, toolpath) already uses `layer.layer_height` from each SliceLayer, so they will automatically use the varying layer heights.

    **Backward compatibility:**
    - adaptive_layer_height defaults to false, so all existing behavior is preserved.
    - When disabled, the uniform slice_mesh path is used unchanged.

    **Tests:**
    - Engine with adaptive_layer_height=false produces same output as before.
    - Engine with adaptive_layer_height=true on a unit cube produces valid G-code.
    - Adaptive slicing produces layers with varying heights (not all the same).
    - Layer Z values are monotonically increasing.
    - TOML config parsing: adaptive fields can be set via TOML.
