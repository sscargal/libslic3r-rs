<objective>
Implement gap fill between perimeters (PERIM-04).

Purpose: After generating perimeter shells by inward offsetting, narrow regions may remain between the innermost perimeter and the infill boundary that are too narrow for a standard-width extrusion but too wide to leave empty. Gap fill detects these regions and generates thin extrusion paths to fill them, eliminating voids and producing solid walls.

Output: Working gap fill that detects narrow regions and generates appropriate thin extrusion paths.
</objective>

  <action>
    Create `gap_fill.rs` with gap detection and fill path generation.

    **Gap fill types:**
    ```rust
    /// A gap fill extrusion path (thin path along the center of a narrow gap).
    #[derive(Clone, Debug)]
    pub struct GapFillPath {
        /// Path points in integer coordinates.
        pub points: Vec<IPoint2>,
        /// Extrusion width at each segment (may vary along the path).
        pub width: f64,
    }
    ```

    **Main function:**
    ```rust
    pub fn detect_and_fill_gaps(
        perimeter_shells: &[PerimeterShell],
        inner_contour: &[ValidPolygon],
        original_contours: &[ValidPolygon],
        min_width: f64,    // minimum extrusion width in mm (default 0.1)
        max_width: f64,    // maximum gap width to fill in mm (default nozzle_diameter)
        line_width: f64,   // standard extrusion width in mm
    ) -> Vec<GapFillPath>
    ```

    **Algorithm:**

    1. **Compute the gap region:**
       - The gap is the area between the innermost perimeter shell and the infill boundary.
       - Compute: `gap_region = polygon_difference(innermost_shell_polygons, inner_contour)`
       - If the innermost shell doesn't exist (wall_count=0), there's no gap to fill.
       - Also check for gaps between the original contour and the outermost perimeter (external gap fill).

    2. **Filter gap regions:**
       - Only keep gap polygons whose width is between `min_width` and `max_width`.
       - Width estimation: the gap polygon's area divided by its perimeter length gives an approximate average width. More precisely, inset the gap polygon by `min_width/2` -- if the result is non-empty, the gap is wider than min_width at that point.
       - Filter out tiny gaps: skip gap polygons with area < `line_width * line_width * 2` (area threshold).

    3. **Generate centerline paths:**
       - For each gap polygon that passes the filter:
         a. Compute the medial axis / centerline. Simplified approach: offset the gap polygon inward by its approximate half-width. The resulting polygon (if non-empty) approximates the centerline.
         b. Even simpler: use the midpoints of the gap polygon's short edges as the path. For a long thin gap, the two long edges are roughly parallel. The centerline is the average of corresponding points on the two long edges.
         c. Simplest viable approach: For each gap polygon, compute the skeleton by repeatedly offsetting inward by small steps until collapse. Collect the collapse points as the centerline.

    4. **Convert to GapFillPath:**
       - Each centerline becomes a GapFillPath with width equal to the average gap width at that location.
       - Filter paths shorter than `2 * min_width` (too short to be useful).

    **Simplified Phase 4 approach (recommended):**
    Rather than full medial axis computation (which is Arachne territory), use a simple geometric heuristic:
    - For each gap polygon, find the two longest edges (these are roughly the "walls" of the gap).
    - If the polygon is roughly rectangular (2 long edges, 2 short edges), the centerline is the line connecting the midpoints of the short edges.
    - If the polygon is more complex, offset it inward by estimated_width/4 and use the result as the fill path.
    - Width = distance between the two long edges at each point.

    **Config additions:**
    ```rust
    /// Enable gap fill between perimeters.
    pub gap_fill_enabled: bool,         // default: true
    /// Minimum gap width to fill (mm).
    pub gap_fill_min_width: f64,        // default: 0.1
    ```

    **Tests:**
    - A rectangle with width = 1.5 * nozzle_width (narrower than 2 perimeters): gap fill detects the gap.
    - A 20mm square with standard 2 perimeters: gap fill returns empty or near-empty (no significant gaps on a simple square).
    - Gap fill paths have width >= min_width.
    - Gap fill paths are within the gap region.
    - Very tiny gaps (area < threshold) are filtered out.
    - Gap fill disabled returns empty.

    Modify `assemble_layer_toolpath()`:

    2. After perimeter segments but before infill, emit gap fill paths:
       a. For each GapFillPath, convert points to Point2 (mm coordinates).
       b. Generate ToolpathSegment for each consecutive point pair.
       c. Use the gap fill width for E-value computation (not the standard extrusion width).
       d. Use perimeter speed for gap fill segments.
       e. Insert travel moves between disconnected gap fill paths.

    **Engine integration:**
    In the per-layer processing loop:
    1. After generating perimeters, if `config.gap_fill_enabled`:
       ```rust
       let gap_fills = detect_and_fill_gaps(
           &perimeters[0].shells,
           &perimeters[0].inner_contour,
           &layer.contours,
           config.gap_fill_min_width,
           config.nozzle_diameter,
           config.extrusion_width(),
       );
       ```
    2. Pass `&gap_fills` to `assemble_layer_toolpath()`.
    3. When gap fill is disabled or returns empty, pass an empty slice.

    **Backward compatibility:**

    **Tests:**

    - Engine with gap_fill_enabled=false: identical to previous behavior.
