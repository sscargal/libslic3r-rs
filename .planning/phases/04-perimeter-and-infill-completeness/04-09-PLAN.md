<objective>
Implement Arachne variable-width perimeters (PERIM-02) using boostvoronoi for medial axis computation.

Purpose: Classic fixed-width perimeters leave gaps in thin walls (walls thinner than 2 * nozzle_width). Arachne solves this by computing the medial axis of the polygon and generating variable-width extrusion paths that perfectly fill the available space. This is the most impactful quality feature in modern slicers, default in PrusaSlicer since 2.5 and Cura since 5.0.

Output: Working Arachne perimeter generator that handles thin walls via medial axis variable-width paths.
</objective>

    **Create arachne.rs:**

    **Types:**
    ```rust
    /// A variable-width perimeter path.
    #[derive(Clone, Debug)]
    pub struct ArachnePerimeter {
        /// Path points in integer coordinates.
        pub points: Vec<IPoint2>,
        /// Width at each point (mm). Length = points.len().
        pub widths: Vec<f64>,
        /// True if this is the outermost perimeter.
        pub is_outer: bool,
    }

    /// Result of Arachne perimeter generation for a contour.
    #[derive(Clone, Debug)]
    pub struct ArachneResult {
        /// Variable-width perimeter paths.
        pub perimeters: Vec<ArachnePerimeter>,
        /// Inner contour for infill (same as classic perimeters).
        pub inner_contour: Vec<ValidPolygon>,
    }
    ```

    **Medial axis extraction:**
    ```rust
    fn compute_medial_axis(polygon: &ValidPolygon) -> Vec<MedialAxisSegment>
    ```

    Using boostvoronoi:
    1. Convert polygon points to boostvoronoi input format (line segments).
       - Each edge of the polygon becomes a line segment for the Voronoi computation.
       - Use integer coordinates (the polygon is already in IPoint2).
    2. Build the Voronoi diagram:
       ```rust
       let mut builder = boostvoronoi::builder::Builder::default();
       // Add line segments from the polygon
       for i in 0..points.len() {
           let p1 = points[i];
           let p2 = points[(i + 1) % points.len()];
           builder.insert_segment(p1.x as i32, p1.y as i32, p2.x as i32, p2.y as i32);
       }
       let voronoi = builder.build().expect("Voronoi build failed");
       ```
       NOTE: boostvoronoi uses i32 coordinates. Scale IPoint2 (i64) down to i32 range by dividing by a scale factor. Since IPoint2 uses COORD_SCALE=1_000_000, coordinates can be large. Scale: divide by 1000 to get micrometer-precision i32 coords (max ~2 billion micrometers = 2000mm range, sufficient).
    3. Extract internal Voronoi edges (edges between two line segments of the same polygon = medial axis).
       - Filter edges: keep only edges that are internal to the polygon (both endpoints are inside the polygon).
       - Each kept edge is a medial axis segment with distance-to-boundary at each endpoint.

    **Width computation:**
    For each medial axis point, the distance to the nearest polygon edge = half the local gap width.
    - `width_at_point = 2.0 * distance_to_nearest_edge`
    - Clamp to [min_extrusion_width, max_extrusion_width] where min=0.1mm, max=2*nozzle_width.

    **Width smoothing:**
    - Linearly interpolate widths over a minimum transition distance (2 * nozzle_width).
    - Filter out medial axis segments shorter than min_extrusion_width.

    **Tests:**
    - boostvoronoi compiles and produces a Voronoi diagram for a simple square.
    - Medial axis of a rectangle: produces a line along the center of the rectangle.
    - Width at medial axis of a 0.8mm-wide rectangle: approximately 0.8mm.
    - Width at medial axis of a 10mm-wide rectangle: approximately 10mm (but will be handled by classic perimeters, not Arachne).
    - Empty polygon returns empty medial axis.

    **Main Arachne function:**
    ```rust
    pub fn generate_arachne_perimeters(
        contours: &[ValidPolygon],
        config: &PrintConfig,
    ) -> Vec<ArachneResult>
    ```

    **Algorithm:**
    For each contour polygon:
    1. Compute the medial axis using `compute_medial_axis`.
    2. Determine which regions are "thin" (width < 2 * nozzle_width at medial axis):
       - Thin regions: generate a single variable-width perimeter path along the medial axis.
       - Standard regions: fall back to classic fixed-width perimeters (call existing `generate_perimeters`).
    3. For thin regions:
       a. The medial axis path IS the perimeter path.
       b. Width varies along the path (from the width computation above).
       c. Mark as outer perimeter if it replaces the outermost shell.
    4. For standard regions:
       a. Use classic offsetting (existing code).
       b. Generate shells as before.
    5. Combine variable-width paths and fixed-width shells into ArachneResult.
    6. Compute inner_contour: for thin regions, the inner contour is empty (the medial axis path fills the entire gap). For standard regions, compute as before.

    **ToolpathSegment update:**
    Add an optional `extrusion_width` field to ToolpathSegment:
    ```rust
    pub struct ToolpathSegment {
        // ... existing fields ...
        /// Extrusion width override in mm (None = use config default).
        pub extrusion_width: Option<f64>,
    }
    ```
    When `extrusion_width` is Some, use it for E-value computation instead of config.extrusion_width().

    **Toolpath assembly update:**
    When processing ArachnePerimeter paths:
    - For each segment, set `extrusion_width = Some(width_at_segment)`.
    - Compute E-value using the variable width.

    **Config addition:**
    ```rust
    /// Enable Arachne variable-width perimeters.
    pub arachne_enabled: bool,  // default: false
    ```

    **Engine update:**
    ```rust
    let perimeter_result = if self.config.arachne_enabled {
        // Use Arachne for variable-width perimeters
        let arachne = generate_arachne_perimeters(&layer.contours, &self.config);
        // Convert ArachneResult to the format expected by toolpath assembly
        // ... adapter code ...
    } else {
        generate_perimeters(&layer.contours, &self.config)
    };
    ```

    **G-code generation:**
    When ToolpathSegment has `extrusion_width = Some(w)`:
    - The G-code generator should use `w` for any width-dependent calculations.
    - G-code comment should indicate variable width: `; TYPE:Variable-width perimeter`.

    **Tests:**
    - A 0.8mm thin wall rectangle: Arachne generates a single perimeter filling the full width.
    - A 10mm wide rectangle: Arachne falls back to standard perimeters (no thin wall).
    - Variable-width segments have correct extrusion_width values.
    - E-values are computed using the variable width.
    - arachne_enabled=false: identical to classic perimeter behavior.
    - Engine with Arachne on thin-wall model: produces valid G-code.
