This is a key quality feature popularized by OrcaSlicer.

    ```rust
    #[derive(Debug, Clone, Serialize, Deserialize)]
    #[serde(default)]
    pub struct ScarfJointConfig {
        /// Enable scarf joint seam.
        pub enabled: bool,                    // default: false
        /// Apply to contours and/or holes.
        pub scarf_joint_type: ScarfJointType, // default: Contour
        /// Only apply on smooth perimeters (no sharp corners near seam).
        pub conditional_scarf: bool,          // default: false
        /// Speed during scarf region (mm/s, 0 = use wall speed).
        pub scarf_speed: f64,                 // default: 0.0 (use wall speed)
        /// Z offset at ramp start as fraction of layer height (0.0-1.0).
        pub scarf_start_height: f64,          // default: 0.5
        /// Apply scarf around entire wall (not just seam region).
        pub scarf_around_entire_wall: bool,   // default: false
        /// Horizontal length of the scarf ramp in mm.
        pub scarf_length: f64,                // default: 10.0
        /// Number of discrete steps in the ramp.
        pub scarf_steps: u32,                 // default: 10
        /// Extrusion flow ratio during scarf (1.0 = normal).
        pub scarf_flow_ratio: f64,            // default: 1.0
        /// Apply scarf to inner walls (not just outer).
        pub scarf_inner_walls: bool,          // default: false
        /// Use role-based wipe speed at seam.
        pub role_based_wipe_speed: bool,      // default: false
        /// Wipe speed at seam end (mm/s).
        pub wipe_speed: f64,                  // default: 0.0
        /// Enable inward wipe at seam close.
        pub wipe_on_loop: bool,               // default: false
    }

    #[derive(Debug, Default, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
    #[serde(rename_all = "snake_case")]
    pub enum ScarfJointType {
        #[default]
        Contour,
        ContourAndHole,
    }
    ```

    Add `scarf_joint: ScarfJointConfig` field to `PrintConfig` with default (disabled).

    **Create scarf.rs with the core algorithm:**

    ```rust
    pub fn apply_scarf_joint(
        perimeter_segments: &mut Vec<ToolpathSegment>,
        seam_index: usize,    // Index in segments where the seam occurs
        config: &ScarfJointConfig,
        layer_height: f64,
        layer_z: f64,
    )
    ```

    **Algorithm:**
    The scarf modifies toolpath segments near the seam point to create a gradual Z ramp:

    1. **Identify scarf region:** Find segments within `scarf_length` mm of the seam point (both before and after). Measure distance by accumulating segment lengths backward from seam (trailing ramp) and forward from seam (leading ramp).

    2. **Trailing ramp (before seam close):** The last `scarf_length` mm of the perimeter loop gradually decrease Z from layer_z to (layer_z - layer_height * scarf_start_height). Split segments in this region into `scarf_steps` sub-segments. For each sub-segment, linearly interpolate Z from layer_z down to the target.

    3. **Leading ramp (after seam start):** The first `scarf_length` mm of the perimeter loop gradually increase Z from (layer_z - layer_height * scarf_start_height) up to layer_z. Split segments similarly.

    4. **Flow adjustment:** Scale E values proportionally with the Z change. If Z is lower (ramp down), reduce flow. Formula: `e_adjusted = e_original * (current_z / layer_z) * scarf_flow_ratio`.

    5. **Segment splitting:** When a segment partially overlaps the scarf region, split it at the boundary point. The split creates two new segments: one with original Z/E, one with ramped Z/E.

    **Helper function:**
    ```rust
    fn split_segment_at_distance(
        segment: &ToolpathSegment,
        distance_from_start: f64,
    ) -> (ToolpathSegment, ToolpathSegment)
    ```
    Splits a segment at a point `distance_from_start` mm along it, producing two segments.

    **Tests:**
    - Apply scarf to a simple 4-segment square perimeter: segments near seam have Z values different from layer_z.
    - Trailing ramp: Z decreases smoothly before seam.
    - Leading ramp: Z increases smoothly after seam.
    - E values are adjusted proportionally with Z.
    - scarf_steps controls the number of sub-segments in the ramp.
    - With scarf disabled (enabled=false), segments are unchanged.
    - scarf_length=0 produces no modification.
