<objective>
Implement seam placement strategies (PERIM-05): aligned, random, rear, and smart hiding (nearest corner).

Purpose: Seam placement controls where each perimeter loop starts and ends. Different strategies produce visually different seam lines -- aligned creates a vertical seam line, random scatters seams, rear hides them at the back, and smart hiding finds concave corners. This is a critical visual quality feature.

Output: Working seam placement module with 4 strategies, integrated into perimeter toolpath assembly.
</objective>

  <action>
    Create `seam.rs` with seam placement strategies.

    **SeamPosition enum:**
    ```rust
    #[derive(Debug, Default, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
    #[serde(rename_all = "snake_case")]
    pub enum SeamPosition {
        /// Align seam points vertically across layers.
        #[default]
        Aligned,
        /// Deterministic pseudo-random placement.
        Random,
        /// Place seam at the rear (maximum Y) of the model.
        Rear,
        /// Smart hiding: prefer concave corners, then convex, with alignment bias.
        NearestCorner,
    }
    ```

    **Main function:**
    ```rust
    pub fn select_seam_point(
        polygon: &ValidPolygon,
        strategy: SeamPosition,
        previous_seam: Option<IPoint2>,
        layer_index: usize,
    ) -> usize
    ```
    Returns the index into polygon.points() where printing should start.

    **Strategy implementations:**

    1. **Aligned**: If `previous_seam` is Some, find the vertex closest to that point. If None, find the vertex with maximum Y (start at rear for first layer). This creates a vertical seam line across layers.

    2. **Random**: Use `layer_index` as a deterministic seed. Compute `index = simple_hash(layer_index) % num_vertices`. Use a simple integer hash (e.g., `layer_index.wrapping_mul(2654435761) >> 16`). MUST be deterministic: same layer_index = same result.

    3. **Rear**: Find the vertex with maximum Y coordinate. Break ties by choosing the vertex closest to `previous_seam` (if available) or the one with the smallest X.

    4. **NearestCorner (Smart Hiding)**: Score each vertex by:
       - Compute the angle at each vertex between its two adjacent edges.
       - Concave corners (angle > PI in CCW polygon) score highest (200.0 + angle).
       - Convex corners score their angle value.
       - Add alignment bonus: `50.0 / (1.0 + distance_to_previous_seam_in_mm)` if previous_seam exists.
       - Choose the vertex with the highest score.
       - Fallback: if all vertices have nearly equal angles (smooth curve, no sharp corners), fall back to Aligned strategy.

    **Helper functions:**
    - `compute_corner_angle(prev: IPoint2, curr: IPoint2, next: IPoint2) -> f64` -- returns angle at `curr` in radians [0, 2*PI).
    - `distance_squared_i64(a: IPoint2, b: IPoint2) -> i128` -- for distance comparison without sqrt.

    **Config update:**
    - Add `seam_position: SeamPosition` to `PrintConfig` with default `SeamPosition::Aligned`.

    **Tests:**
    - Aligned: same polygon on two layers with previous_seam returns same/nearby vertex.
    - Random: different layer indices produce different seam indices; same index produces same.
    - Rear: returns the vertex with maximum Y.
    - NearestCorner: on a polygon with one obvious concave corner, selects that corner.
    - NearestCorner: on a regular polygon (all angles equal), falls back to aligned behavior.
    - All strategies return a valid index (< polygon.points().len()).
