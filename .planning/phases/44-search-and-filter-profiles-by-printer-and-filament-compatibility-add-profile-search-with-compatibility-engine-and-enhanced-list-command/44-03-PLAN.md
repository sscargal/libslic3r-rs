<objective>
Add profile set management CLI commands and integrate sets + compatibility warnings into the slice workflow. Rename existing config-value `Set` variant to `Setting` to free the `set` subcommand name for profile sets per user decision.

Purpose: Users can save favorite machine+filament+process combos as named sets, use them with `--profile-set` in the slice command (or as default), and see compatibility warnings before slicing. CLI uses `profile set` (singular) as specified in CONTEXT.md. Note: the slice command uses `--profile-set` (not `--set`) to avoid collision with the existing `--set key=value` config override flag.
Output: ProfileSetCommand subcommand group under `profile set` in profile_command.rs, --profile-set flag + default set expansion in main.rs, pre-slice compatibility warnings in slice_workflow.rs, integration test stubs.

   // Pre-slice compatibility warnings
   if let (Some(ref machine_name), Some(ref filament_name)) = (&options.machine, &options.filament) {
       if let Ok(Some(index)) = slicecore_engine::profile_library::ProfileIndex::load(
           &slicecore_engine::profile_library::default_index_path(options.profiles_dir.as_deref())
       ) {
           let machine_entries: Vec<&ProfileIndexEntry> = index.profiles.iter()

               .find(|e| e.profile_type == "filament" && (e.id.contains(filament_name) || e.name.contains(filament_name)));

           if let Some(filament) = filament_entry {

               let report = CompatibilityInfo::compat_report(filament, &machine_entries, 300.0);
