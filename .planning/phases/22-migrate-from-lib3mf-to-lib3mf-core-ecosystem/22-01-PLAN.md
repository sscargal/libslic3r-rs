     Then: `ZipArchiver::new(cursor)` -> `find_model_path(&mut archiver)` -> `archiver.read_entry(&model_path)` -> `parse_model(Cursor::new(model_data))`

   - Error mapping stays the same pattern: `.map_err(|e| FileIOError::ThreeMfError(e.to_string()))`
   - The function signature `pub fn parse(data: &[u8]) -> Result<TriangleMesh, FileIOError>` does NOT change.

Replace `mesh.triangles.push(lib3mf::Triangle::new(v1, v2, v3))` with `mesh.add_triangle(v1, v2, v3)`. Replace `lib3mf::Object::new(1)` with Object struct construction + `model.resources.add_object(ResourceId(1), object)`.

For multi-object, since HashMap iteration order is nondeterministic, adjust assertions: assert total vertex count (6) and total triangle count (2) are correct.
