  <action>
    Modify `crates/slicecore-cli/Cargo.toml` to:

    1. **Update the `slicecore-engine` dependency** to enable both feature flags:
       ```toml
       slicecore-engine = { path = "../slicecore-engine", features = ["plugins", "ai"] }
       ```

    2. **Add `slicecore-ai` as a direct dependency** (needed for `AiConfig::from_toml()` and `create_provider()` in the CLI):
       ```toml
       slicecore-ai = { path = "../slicecore-ai" }
       ```

    3. **Add `slicecore-plugin` as a direct dependency** (needed for `PluginRegistry::new()` and `discover_and_load()` in the CLI):
       ```toml
       slicecore-plugin = { path = "../slicecore-plugin" }
       ```

    Do NOT add tokio as a dependency. The `Engine::suggest_profile()` method calls `suggest_profile_sync()` internally, which creates its own single-threaded tokio runtime. The CLI does not need tokio directly.

    Do NOT add any feature flags to the CLI crate itself. The features are on the engine dependency -- the CLI unconditionally includes all features since it is the user-facing binary.

    After editing, run:
    - `cargo build --bin slicecore` to verify compilation
    - `cargo test --package slicecore-cli` to verify existing tests still pass
    - `cargo clippy --package slicecore-cli -- -D warnings` to verify no new warnings
  </action>

    All three commands exit with code 0.
  </verify>
