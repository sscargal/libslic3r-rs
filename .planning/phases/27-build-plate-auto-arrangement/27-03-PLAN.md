<task type="auto" tdd="true">
  <name>Task 1: Auto-orient and bottom-left fill placer</name>
  <files>
    crates/slicecore-arrange/src/orient.rs,
    crates/slicecore-arrange/src/placer.rs
  </files>
  <behavior>
    - auto_orient with MinimizeSupport: a cube mesh returns identity (no rotation needed, all faces perpendicular to gravity); a tilted mesh with large overhangs returns a rotation that reduces overhang area
    - auto_orient with MaximizeFlatContact: returns orientation maximizing bottom-face area
    - place_parts on 220x220 bed: two 50mm cubes placed without overlap; parts sorted largest-first
    - place_parts: part exceeding bed returns it in unplaced list
    - place_parts with rotation: tries 8 orientations (45-deg increments) for non-locked parts
    - center_arrangement shifts all placements so group centroid aligns with bed center
  </behavior>
  <action>
    1. Create `src/orient.rs`:
       - `auto_orient(vertices: &[Point3], normals: &[Vec3], face_areas: &[f64], criterion: &OrientCriterion) -> (f64, f64, f64)`: Returns (rx, ry, rz) rotation angles in degrees.
       - Sample orientations: 30-degree increments around X and Y axes (12 x 12 = 144 candidates). For each candidate, rotate all normals and score.
       - `overhang_score(normals, face_areas, threshold_cos)`: Sum face areas where rotated_normal.dot(Z_UP) < cos(45 degrees). Lower is better.
       - `contact_score(normals, face_areas)`: Sum face areas where rotated_normal.z < -0.99 (nearly flat on bed). Higher is better.
       - `multi_criteria_score`: Weighted combination: support_weight * (1.0 - normalized_overhang) + contact_weight * normalized_contact.
       - Helper: `rotate_normal(normal: &Vec3, rx_rad: f64, ry_rad: f64) -> Vec3` using rotation matrices.
       - Unit tests: cube returns (0,0,0); asymmetric mesh selects orientation with less overhang.

    2. Create `src/placer.rs`:
       - `place_parts(parts: &[PreparedPart], bed: &[IPoint2], bed_margin: Coord, config: &ArrangeConfig) -> (Vec<Placement>, Vec<String>)`: Core bottom-left fill algorithm.
       - `PreparedPart` internal struct: id, footprint_variants (one per allowed rotation), area (for sorting), mesh_height, material.
       - Algorithm:
         a. Sort parts by area descending (largest-first).
         b. Compute inset bed boundary (bed offset inward by bed_margin).
         c. For each part, for each rotation variant:
            - Raster scan candidate positions at nozzle_diameter resolution (converted to Coord). Scan from min_y to max_y, min_x to max_x of inset bed.
            - At each position, translate footprint and check: (1) all vertices inside inset bed via point_in_bed, (2) no overlap with placed footprints via footprints_overlap.
            - Track best placement (first valid position for best-fit rotation = rotation minimizing wasted space, approximated as smallest bounding box of remaining free area).
         d. If no valid position found for any rotation, add to unplaced list.
         e. Record placed footprint for future collision checks.
       - `center_arrangement(placements: &mut [Placement], bed: &[IPoint2])`: Compute bounding box of all placements, compute offset to center on bed, shift all positions.
       - For performance: use bounding box pre-check before polygon intersection. Skip candidate positions where bounding boxes don't fit.
       - Unit tests: two cubes placed on 220x220 bed; oversized part returns unplaced; rotation-locked part only tries one orientation.
  </action>

<task type="auto" tdd="true">
  <name>Task 2: Multi-plate grouping, sequential support, and public API</name>
  <files>
    crates/slicecore-arrange/src/grouper.rs,
    crates/slicecore-arrange/src/sequential.rs,
    crates/slicecore-arrange/src/lib.rs
  </files>
  <behavior>
    - group_parts_by_material: parts with same material grouped together; multi-head printers skip material grouping
    - group_parts_by_height: when splitting across plates, similar-height parts grouped together
    - expand_for_gantry with Cylinder model: expands footprint by radius
    - expand_for_gantry with Rectangular model: expands footprint by half-width in X, half-depth in Y
    - order_back_to_front: sorts placements by Y coordinate descending (largest Y first)
    - validate_sequential: returns error if expanded footprints overlap
    - arrange() end-to-end: returns ArrangementResult with correct plates and placements
    - arrange_with_progress: calls progress callback during placement
  </behavior>
  <action>
    1. Create `src/grouper.rs`:
       - `group_by_material(parts: &[ArrangePart], multi_head: bool) -> Vec<Vec<usize>>`: Group part indices by material string. If multi_head is true or material_grouping disabled, return single group with all indices.
       - `group_by_height(parts: &[ArrangePart], indices: &[usize]) -> Vec<Vec<usize>>`: Sort indices by mesh_height, then split into groups where max_height / min_height < 2.0 (similar height heuristic). This minimizes total print time per plate.
       - `split_into_plates(parts: &[PreparedPart], bed: &[IPoint2], bed_margin: Coord, config: &ArrangeConfig) -> Vec<Vec<Placement>>`: Run place_parts. Any unplaced parts go to next plate. Repeat until all placed or genuinely too large.
       - Unit tests: 3 parts with 2 materials -> 2 groups; multi-head -> 1 group; height grouping clusters similar heights.

    2. Create `src/sequential.rs`:
       - `expand_for_gantry(footprint: &[IPoint2], model: &GantryModel) -> Vec<IPoint2>`: Expand footprint based on gantry model:
         * Cylinder: offset_polygon by radius (reuse expand_footprint with radius as spacing and zero brim/raft)
         * Rectangular: compute bounding box, expand by half-width in X, half-depth in Y, return rectangle
         * CustomPolygon: Minkowski sum approximation -- offset polygon by max radius of custom polygon (simplified v1)
         * None: return footprint unchanged
       - `validate_sequential(placements: &[Placement], gantry_model: &GantryModel) -> Result<(), ArrangeError>`: For each pair of placements, check that gantry-expanded footprints don't overlap. Return error with part IDs if they do.
       - `order_back_to_front(placements: &mut [Placement])`: Sort by Y coordinate of placement position descending (back of bed = higher Y first). Assign print_order = Some(index).
       - Unit tests: cylinder expansion increases footprint area; two well-separated parts pass validation; overlapping parts fail validation; back-to-front ordering correct.

    3. Update `src/lib.rs` with public API:
       - Add module declarations for orient, placer, grouper, sequential.
       - `pub fn arrange(parts: &[ArrangePart], config: &ArrangeConfig, bed_shape: &str, bed_x: f64, bed_y: f64) -> Result<ArrangementResult, ArrangeError>`:
         a. Parse bed shape (fall back to bed_from_dimensions if empty).
         b. If no parts, return error.
         c. For each part: auto-orient (if enabled and not locked), compute footprint, prepare rotation variants, expand for spacing/brim/raft.
         d. Group by material (if enabled and not multi-head).
         e. For each material group: group by height, then for each height group: split_into_plates.
         f. If sequential_mode: expand for gantry, validate, order back-to-front.
         g. If center_after_packing: center each plate's arrangement.
         h. Assemble ArrangementResult with all plates.
       - `pub fn arrange_with_progress(parts: &[ArrangePart], config: &ArrangeConfig, bed_shape: &str, bed_x: f64, bed_y: f64, progress: impl Fn(f64)) -> Result<ArrangementResult, ArrangeError>`:
         Same as arrange but calls progress(0.0..1.0) at key steps (after orient, after each part placed, after grouping).
       - Integration-level tests in lib.rs: arrange 3 cubes of different sizes on 220x220 bed, verify all placed on one plate; arrange 20 large parts, verify multi-plate; arrange with sequential_mode and verify print_order set.
  </action>
