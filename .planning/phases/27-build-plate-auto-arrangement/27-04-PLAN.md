  <action>
    1. Add `slicecore-arrange` dependency to `crates/slicecore-cli/Cargo.toml` (path = "../slicecore-arrange").

    2. Add `Arrange` variant to `Commands` enum:
       ```
       Arrange {
           /// Input mesh files to arrange
           #[arg(required = true)]
           input: Vec<PathBuf>,

           /// Print config file (TOML or JSON; optional)
           #[arg(short, long)]
           config: Option<PathBuf>,

           /// Bed shape string (e.g., "0x0,220x0,220x220,0x220"). Overrides config.
           #[arg(long)]
           bed_shape: Option<String>,

           /// Part spacing in mm (default: 2.0)
           #[arg(long, default_value = "2.0")]
           spacing: f64,

           /// Bed edge margin in mm (default: 5.0)
           #[arg(long, default_value = "5.0")]
           margin: f64,

           /// Rotation step in degrees (default: 45.0)
           #[arg(long, default_value = "45.0")]
           rotation_step: f64,

           /// Disable auto-orient
           #[arg(long)]
           no_auto_orient: bool,

           /// Enable sequential printing mode
           #[arg(long)]
           sequential: bool,

           /// Output as JSON (default behavior, but explicit flag for clarity)
           #[arg(long)]
           json: bool,
       }
       ```

    3. Add `--auto-arrange` flag to `Slice` variant:
       ```
       /// Auto-arrange input mesh(es) on bed before slicing
       #[arg(long)]
       auto_arrange: bool,
       ```

    4. Implement `cmd_arrange` function:
       - Load each input file via `slicecore_fileio::load_mesh()`.
       - Build ArrangeConfig from CLI flags and optional PrintConfig.
       - Build ArrangePart for each mesh (id = filename, vertices from mesh, mesh_height from bounding box).
       - Get bed_shape from CLI flag, or config.machine.bed_shape, or fall back to config.machine.bed_x/bed_y.
       - Call `slicecore_arrange::arrange()`.
       - Print ArrangementResult as JSON to stdout via serde_json::to_string_pretty.
       - If any unplaced parts, print warning to stderr.

    5. Wire `Commands::Arrange { .. }` match arm in main() to call cmd_arrange.

    6. For `--auto-arrange` on Slice: when flag is set and multiple input files provided (or even single file), build ArrangePart list, call arrange(), then apply the resulting transforms to meshes before slicing. For v1, print the arrangement plan and note that actual multi-mesh slicing with transforms is a future enhancement (the arrange output is the plan; applying it to slice requires multi-object engine support which exists via sequential.rs).
