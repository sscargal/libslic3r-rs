    - SC2: Twenty 80mm cubes exceed bed capacity and split across multiple plates

    - SC4: Auto-orient on a mesh with large overhangs produces lower overhang score than default orientation
    - SC5: Three parts with two different materials produce at least 2 material groups (when material grouping enabled)
    - SC6: Arrangement result serializes to valid JSON via serde_json
    - SC7: Empty parts list returns NoPartsProvided error
    - SC8: Single part larger than bed returns PartTooLargeForBed error
    - SC9: Bed shape parsing handles rectangular "0x0,220x0,220x220,0x220" format
    - SC10: center_after_packing places arrangement centroid near bed center
  </behavior>
  <action>
    Create `crates/slicecore-arrange/tests/integration.rs`:

    1. Helper: `make_cube_vertices(size_mm: f64, offset_x: f64, offset_y: f64) -> Vec<Point3>` -- generates 8 vertices of an axis-aligned cube at given offset. This is a synthetic mesh (just vertices, no triangles needed for arrangement since we only project to XY).

    2. Helper: `make_arrange_part(id: &str, vertices: Vec<Point3>, material: Option<&str>) -> ArrangePart` -- convenience builder.

    3. Helper: `default_config() -> ArrangeConfig` -- returns config with reasonable defaults for a 220x220 bed.

    4. Test `sc1_three_cubes_single_plate`:
       - Create three 50mm cubes as ArrangeParts.
       - Call arrange() with 220x220 bed.
       - Assert total_plates == 1.
       - Assert unplaced_parts is empty.
       - Assert all 3 placements exist on plate 0.
       - Assert no two placements overlap (positions are far enough apart given footprint sizes).

    5. Test `sc2_multi_plate_splitting`:
       - Create twenty 80mm cubes.
       - Call arrange() with 220x220 bed.
       - Assert total_plates > 1 (can't fit all on one plate).
       - Assert unplaced_parts is empty (all eventually placed).
       - Assert sum of placements across all plates == 20.

    6. Test `sc3_sequential_mode`:
       - Create two 50mm cubes.
       - Set sequential_mode = true, gantry_model = Cylinder { radius: 35.0 }.
       - Call arrange().
       - Assert all placements have print_order = Some(_).
       - Assert print_order values are sequential (0, 1).
       - Assert placement with higher Y coordinate has lower print_order (back-to-front).

    7. Test `sc4_auto_orient_reduces_overhangs`:
       - Create vertices representing a wedge/tilted surface with large overhangs.
       - Call arrange() with auto_orient = true, orient_criterion = MinimizeSupport.
       - Verify that the result has an orientation field set on the placement.
       - (Qualitative test: the auto-orient should have selected an orientation.)

    8. Test `sc5_material_grouping`:
       - Create 3 parts: 2 with material "PLA", 1 with material "ABS".
       - Set material_grouping = true, use a small bed (100x100) so they must split.
       - Assert parts with same material are on same plate.

    9. Test `sc6_json_serialization`:
       - Call arrange(), serialize result to JSON via serde_json::to_string_pretty.
       - Assert JSON is valid (parses back).
       - Assert JSON contains "plates", "total_plates", "unplaced_parts" keys.

    10. Test `sc7_empty_parts_error`:
        - Call arrange with empty parts vec.
        - Assert returns Err(ArrangeError::NoPartsProvided).

    11. Test `sc8_oversized_part`:
        - Create a 500mm cube on a 220x220 bed.
        - Assert the part appears in unplaced_parts (or returns specific error).

    12. Test `sc9_bed_shape_parsing`:
        - Use bed_shape = "0x0,220x0,220x220,0x220".
        - Call arrange with one small cube.
        - Assert placement succeeds within bed bounds.

    13. Test `sc10_centering`:
        - Place a single small cube on a large bed.
        - Assert placement position is near bed center (not at bottom-left corner).
        - Tolerance: within 10mm of bed center.

    Run full test suite to verify all pass.
