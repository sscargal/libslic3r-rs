<task type="auto" tdd="true">
  <name>Task 2: Footprint computation and collision detection</name>
  <files>crates/slicecore-arrange/src/footprint.rs</files>
  <behavior>
    - compute_footprint: projects 3D vertices to XY IPoint2 and returns convex hull; degenerate (collinear/single-point) falls back to bounding box rectangle
    - expand_footprint: offsets hull outward by half-spacing + brim_width + raft_margin; returns original if offset collapses
    - footprints_overlap: returns true when two ValidPolygon footprints have non-empty polygon_intersection
    - rotate_footprint: rotates hull around centroid by given angle; returns new hull (always from original, not accumulated)
    - footprint_area: returns area in mm^2 from Coord polygon
  </behavior>
  <action>
    Create `src/footprint.rs` with:

    1. `compute_footprint(vertices: &[Point3]) -> Vec<IPoint2>`: Project each Point3 to IPoint2 via COORD_SCALE, call convex_hull(). If result has fewer than 3 points (degenerate), compute bounding box and return its 4 corners as rectangle.

    2. `expand_footprint(hull: &[IPoint2], spacing_mm: f64, brim_width_mm: f64, raft_margin_mm: f64) -> Vec<IPoint2>`: Convert hull to ValidPolygon, compute total expansion = spacing_mm / 2.0 + brim_width_mm + raft_margin_mm, convert to Coord, call offset_polygon with JoinType::Round. If offset returns empty (collapse), return original hull. Unwrap result from Vec<ValidPolygon> back to Vec<IPoint2>.

    3. `footprints_overlap(a: &[IPoint2], b: &[IPoint2]) -> bool`: Convert both to ValidPolygon, call polygon_intersection. Return true if result is non-empty.

    4. `rotate_footprint(hull: &[IPoint2], angle_deg: f64) -> Vec<IPoint2>`: Compute centroid of hull. Rotate each point around centroid by angle. Return convex_hull() of rotated points (to clean up any precision issues).

    5. `footprint_area(hull: &[IPoint2]) -> f64`: Shoelace formula on Coord values, convert to mm^2 via COORD_SCALE^2.

    6. `centroid(hull: &[IPoint2]) -> IPoint2`: Average of all hull points.

    All functions get doc comments with examples. Unit tests cover: square mesh footprint, degenerate (vertical line) fallback, expansion with positive spacing, zero expansion passthrough, overlap detection (overlapping vs separated), 90-degree rotation correctness, area computation.
  </action>
