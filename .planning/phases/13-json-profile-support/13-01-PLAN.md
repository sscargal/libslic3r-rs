<objective>
Implement the profile import module with content-based format detection, OrcaSlicer/BambuStudio JSON field mapping, and unified config loading.

Purpose: Users need to load existing slicer profiles from OrcaSlicer and BambuStudio without manual format conversion. The field mapping translates upstream JSON schema (string values, array wrapping, nil sentinels, percentage strings) into our PrintConfig struct.

Output: `profile_import.rs` module with format detection, field mapping tables, and ImportResult; `config.rs` extended with `from_json()` and `from_file()` methods.

Create `crates/slicecore-engine/src/profile_import.rs` with all profile import logic:

**1. ConfigFormat enum and detection:**
```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ConfigFormat { Toml, Json }
```
- `pub fn detect_config_format(data: &[u8]) -> ConfigFormat` -- skip UTF-8 BOM (0xEF, 0xBB, 0xBF), find first non-whitespace byte, `{` = Json, everything else = Toml, empty = Toml.

**2. Value extraction helpers (private):**
- `fn extract_string_value(value: &serde_json::Value) -> Option<String>` -- handles String, Array[0] (first element for primary extruder), skips "nil" sentinel.
- `fn extract_f64(value: &serde_json::Value) -> Option<f64>` -- extracts string then parses.
- `fn extract_u32(value: &serde_json::Value) -> Option<u32>` -- same for integers.
- `fn extract_bool_from_string(value: &serde_json::Value) -> Option<bool>` -- "1"/"true" -> true, "0"/"false" -> false.
- `fn extract_percentage(value: &serde_json::Value) -> Option<f64>` -- strip `%` suffix, parse, divide by 100.0.

**3. ImportResult and ProfileMetadata structs:**
```rust
pub struct ImportResult {
    pub config: PrintConfig,
    pub mapped_fields: Vec<String>,
    pub unmapped_fields: Vec<String>,
    pub metadata: ProfileMetadata,
}

pub struct ProfileMetadata {
    pub name: Option<String>,
    pub profile_type: Option<String>,
    pub inherits: Option<String>,
}
```

**4. Field mapping functions:**
- `pub fn import_upstream_profile(value: &serde_json::Value) -> Result<ImportResult, EngineError>` -- iterates over JSON object keys, skips metadata fields (type, name, inherits, from, setting_id, instantiation, compatible_printers, compatible_printers_condition, filament_id, description, version, compatible_prints, compatible_prints_condition), extracts string values, calls `apply_field_mapping` for each, tracks mapped/unmapped fields.

- `fn apply_field_mapping(config: &mut PrintConfig, key: &str, value: &str) -> bool` -- large match statement mapping upstream field names to PrintConfig mutations. Return true if mapped, false if unknown.

**5. Enum mapping helpers (private):**
- `fn map_infill_pattern(value: &str) -> Option<InfillPattern>` -- "grid" -> Grid, "honeycomb" -> Honeycomb, "gyroid" -> Gyroid, "cubic" -> Cubic, "adaptivecubic" -> AdaptiveCubic, "lightning" -> Lightning, "monotonic" -> Monotonic, "zig-zag"|"rectilinear"|"line" -> Rectilinear. Unrecognized -> None (caller falls back to not mapping, logs as unmapped).
- `fn map_seam_position(value: &str) -> Option<SeamPosition>` -- "aligned" -> Aligned, "random" -> Random, "rear" -> Rear, "nearest" -> NearestCorner. Unrecognized -> None.
- `fn map_gcode_dialect(value: &str) -> Option<GcodeDialect>` -- "marlin" -> Marlin, "klipper" -> Klipper, "reprapfirmware"|"reprap" -> RepRapFirmware. Unrecognized -> None.

**Note on apply_field_mapping:** The string value has ALREADY been extracted from the JSON Value by `extract_string_value` in the caller (`import_upstream_profile`). The `apply_field_mapping` function receives a plain `&str`. For percentage fields, the function itself handles the `%` stripping. For enum fields, it calls the mapping helper. This means both scalar and array-wrapped values are handled uniformly by the caller.

**6. Update config.rs:**
Add two new methods to `impl PrintConfig`:
- `pub fn from_json(json_str: &str) -> Result<Self, EngineError>` -- parse to serde_json::Value, check for `"type"` field. If present, call `import_upstream_profile` and return `result.config`. If absent, attempt direct `serde_json::from_str::<PrintConfig>()` (native JSON format with same field names as TOML).
- `pub fn from_file(path: &std::path::Path) -> Result<Self, EngineError>` -- read file, call `detect_config_format`, dispatch to `from_toml` (with ConfigParse error mapping) or `from_json`.
- `pub fn from_json_with_details(json_str: &str) -> Result<ImportResult, EngineError>` -- same as from_json but returns full ImportResult for callers that want mapped/unmapped field reporting.

Update the doc comment on PrintConfig to mention `from_json` and `from_file`.
