<task type="auto">
  <name>Task 2: Integration tests with real upstream profiles</name>
  <files>
    tests/integration_profile_import.rs
  </files>
  <action>
Create `tests/integration_profile_import.rs` as a workspace-level integration test.

**Test organization:**

All tests that load files from `/home/steve/slicer-analysis/` must be gated with `#[ignore]` and a comment explaining they require the slicer-analysis directory to be present. These run manually with `cargo test --test integration_profile_import -- --ignored`.

Tests with synthetic JSON (inline in the test) run as normal tests.

**Required tests:**

1. `test_synthetic_process_profile_roundtrip` -- Create a synthetic OrcaSlicer process profile JSON string (inline), load via `PrintConfig::from_json`, verify specific field values. Use values from research: layer_height=0.2, wall_loops=2, sparse_infill_density="15%", outer_wall_speed=200, seam_position="aligned", default_acceleration=10000.

2. `test_synthetic_filament_profile_roundtrip` -- Create synthetic filament profile JSON (inline), verify nozzle_temp=220, bed_temp=55, filament_density=1.24, extrusion_multiplier=0.98, disable_fan_first_layers=1.

3. `test_synthetic_machine_profile_roundtrip` -- Create synthetic machine profile JSON (inline), verify nozzle_diameter=0.4, retract_length=0.8, gcode_dialect=Marlin, jerk_x=8.0.

4. `test_synthetic_nil_values_use_defaults` -- Profile with `"layer_height": "nil"` and `"wall_loops": "nil"` should use PrintConfig defaults (0.2 and 2 respectively).

5. `test_synthetic_mixed_array_scalar` -- Profile with mix of scalar strings and array-wrapped values.

6. `test_native_json_config` -- JSON with PrintConfig-matching field names and real numeric values (not strings). Verify direct deserialization works.

7. `test_toml_still_works` -- Write a temp TOML file, load via `PrintConfig::from_file`, verify values. Regression guard.

8. `test_import_result_reports_unmapped_fields` -- Load a profile with some known and some unknown fields. Verify `ImportResult.unmapped_fields` contains the unknown field names and `mapped_fields` contains the known ones. Use `PrintConfig::from_json_with_details`.

9. `#[ignore] test_real_orcaslicer_process_profile` -- Load an actual OrcaSlicer process profile from `/home/steve/slicer-analysis/OrcaSlicer/resources/profiles/BBL/process/`. Pick a common one (e.g., contains "0.20mm" in the name). Verify it loads without error and produces a config with `layer_height` close to 0.2.

10. `#[ignore] test_real_orcaslicer_filament_profile` -- Load an actual OrcaSlicer filament profile from the filament/ subdirectory. Verify it loads and `nozzle_temp > 0.0` (not default) or some other sanity check.

11. `#[ignore] test_real_bambustudio_profile` -- Load an actual BambuStudio profile from `/home/steve/slicer-analysis/BambuStudio/resources/profiles/BBL/`. Verify it loads without error.

12. `#[ignore] test_bulk_orcaslicer_profiles` -- Iterate over a directory of OrcaSlicer profiles (e.g., all files in a vendor's process/ directory), attempt to load each one, count successes/failures. Assert at least 80% load without error (some may have edge cases). Print summary of any failures for debugging.

**Profile file discovery for ignored tests:**
Use `std::fs::read_dir` to find `.json` files in the slicer-analysis directories. For the bulk test, use the BBL vendor directory which has the most profiles.

**Test dependencies:**
- `use slicecore_engine::{PrintConfig, ImportResult, ProfileMetadata};`
- `use slicecore_engine::profile_import::{detect_config_format, ConfigFormat};`
- `use std::io::Write;` for temp file tests

For temp file tests, use `tempfile` crate if available in workspace, otherwise use `std::env::temp_dir()` with unique names.
