**Main entry function:**
```rust
pub fn run_diff_profiles_command(args: &DiffProfilesArgs) -> Result<bool, Box<dyn std::error::Error>>
```
Returns `Ok(true)` if differences found, `Ok(false)` if identical.

Logic:
1. Validate args: if `--defaults` is false and `right` is None, return error "Provide two profiles or use --defaults"
2. Resolve left profile: check if it's a file path (Path::new(&args.left).exists()), if so load with `PrintConfig::from_toml()` or `from_json()`, else use `ProfileResolver::new(args.profiles_dir.as_deref()).resolve(&args.left, "process")`. Extract profile name for display. If ProfileResolver returns a `ProfileError` (including type mismatch), propagate the error with a descriptive message -- ProfileResolver enforces same-type comparison via the `expected_type` parameter, so both profiles are resolved with `"process"` which ensures they are the same type.
3. Resolve right profile: if `--defaults`, use `PrintConfig::default()` with name "defaults". Otherwise resolve like left, also with `expected_type: "process"`.
4. Call `diff_configs(&left_config, &right_config, &left_name, &right_name)` to get DiffResult. Note: diff_configs from plan 38-01 returns ALL entries with a `changed: bool` field on each entry.
5. Filter entries for display: if `--all` flag is NOT set, filter to only entries where `entry.changed == true`. If `--all` IS set, show all entries. No modification to profile_diff.rs needed -- the engine already provides the `changed` field.
6. Apply filters: if `--category` provided, parse each with `parse_category()` and filter entries to matching categories. If `--tier` provided, filter entries to matching or lower tier.
7. If `--quiet`: return `Ok(result.total_differences > 0)` with no output.
8. If `--json`: serialize the filtered entries and metadata to JSON with serde_json::to_string_pretty, print, return.
9. Otherwise: display table format.

**Table display function:**
```rust
fn display_table(result: &DiffResult, entries: &[&DiffEntry], verbose: bool, use_color: bool)
```
1. Print summary header:
   ```
   Comparing: {left_name} vs {right_name}
   {N} differences across {M} categories:
     Category1: X  |  Category2: Y  |  ...
   ```
2. Group entries by category (use BTreeMap<Option<SettingCategory>, Vec<&DiffEntry>>).
3. For each category group (skip empty in non-all mode):
   - Print category header: `\n  {CategoryName}\n  {dashes}`
   - Create comfy_table::Table with columns: "Setting", left_name, right_name (+ "Affects" if verbose)
   - For each entry: row is `["{display_name} ({key})", format_value(&left, &units), format_value(&right, &units)]`
   - If verbose, add affects column: comma-separated SettingKey list
   - If verbose, print description below each entry or as a fourth column
   - Print the table
4. For entries with category None, group under "Uncategorized".

**Color logic:**
- Determine `use_color` from `--color` flag: "always" -> true, "never" -> false, "auto" -> `std::io::stdout().is_terminal()`
- When `use_color` is true, use comfy-table Cell coloring (green for left value, red for right value in changed entries) or ANSI escape codes directly.

**File path detection for profile loading:**
- If path ends with `.toml`: `PrintConfig::from_toml(&std::fs::read_to_string(path)?)?`
- If path ends with `.json`: `PrintConfig::from_json(&std::fs::read_to_string(path)?)?`
- If path exists as file but unknown extension: try TOML first, then JSON
- Otherwise: treat as profile library name, use ProfileResolver
