**`optimize_greedy(z_samples, config) -> Vec<(f64, f64)>`:**

Algorithm:
1. First layer: always `(first_layer_height / 2.0, first_layer_height)`.
2. For each subsequent position, evaluate the next `lookahead` (default 5) Z samples.
3. At each candidate Z, compute the weighted objective score: `scores.combine(&config.weights)`.
4. If a feature demands a specific height at this Z (`feature_demanded_height.is_some()`), take the minimum of the demanded height and the objective-derived height.
5. Clamp the selected height to `[config.min_height, config.max_height]`.
6. Also clamp to `config.nozzle_diameter * 0.75` (max height hard limit per user decision).
7. Pick the height that minimizes total cost over the lookahead window. Cost function: sum of `|selected_height - ideal_height|` for each Z in the window where `ideal_height` is the weighted objective score.
8. Advance by the selected height and repeat.
9. Handle remaining mesh height (same logic as adaptive.rs: if remaining > min_h * 0.5, add final layer).

The lookahead window size is 5 layers (Claude's discretion). All iteration order is fixed (no parallelism in the inner loop) for determinism.

Add `pub mod optimizer;` to vlh/mod.rs.

**`optimize_dp(z_samples, config) -> Vec<(f64, f64)>`:**

Algorithm (1D lattice shortest path):
1. Discretize candidate heights: 15 linearly-spaced values between `config.min_height` and `min(config.max_height, config.nozzle_diameter * 0.75)`. Use exact floating-point computation (no random perturbation) for determinism.
2. Build DP table: `dp[z_idx][h_idx]` = (min_cost, predecessor_h_idx).
   - State space: `O(num_z_levels * 15)`.
   - Time: `O(num_z_levels * 15^2)` = `O(num_z_levels * 225)`.
3. For first Z level: cost = `|candidate_h - ideal_h|` where `ideal_h = z_samples[0].scores.combine(&weights)`, but first layer is forced to `first_layer_height` so only the candidate closest to `first_layer_height` gets zero cost, others get infinity.
4. Transition cost between adjacent Z levels: `transition_cost = |h_curr / h_prev - 1.0|` penalizes large ratio changes. If `h_curr / h_prev > 1.5` or `< 1/1.5`, transition is forbidden (infinity cost).
5. Per-Z cost: `|candidate_h - ideal_h|` where `ideal_h` is `min(scores.combine(&weights), feature_demanded_height.unwrap_or(f64::MAX))`.
6. Backtrack from the minimum-cost final state to recover the optimal height sequence.
7. Convert height indices back to actual (z, height) pairs, recomputing Z positions.

Memory: `O(num_z_levels * 15 * 2)` for the DP table (cost + predecessor). For 2000 layers: 2000 * 15 * 16 bytes = ~480 KB.

All comparisons use `f64::total_cmp` for deterministic tie-breaking.
