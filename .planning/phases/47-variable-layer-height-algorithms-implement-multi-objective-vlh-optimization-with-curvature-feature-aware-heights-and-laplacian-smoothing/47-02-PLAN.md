**FeatureMap struct:**
```rust
/// Sorted collection of feature detections with efficient Z-based lookup.
pub struct FeatureMap {
    detections: Vec<FeatureDetection>,  // sorted by z_min
}
```

**`build_feature_map(mesh, sliced_layers, config) -> FeatureMap`:**
- Takes the mesh and pre-sliced layer contours (from `slice_at_height` at fine Z intervals).
- For each pair of adjacent layers, run feature detection:
  1. **Overhang detection:** For each Z, compute triangle normal angles. For triangles with surface angle in `[config.overhang_angle_min, config.overhang_angle_max]`, create `FeatureDetection` with `FeatureType::Overhang { angle_deg }`. Demanded height = `config.min_height + (config.max_height - config.min_height) * (1.0 - sensitivity)` where `sensitivity = (angle - angle_min) / (angle_max - angle_min)` clamped to [0, 1]. Multiply by `config.feature_overhang_weight`.
  2. **Hole detection:** For each layer's contours, call `is_circular_hole(polygon, 0.5)`. If detected, create `FeatureDetection` with `FeatureType::Hole { diameter_mm }`. Demanded height = config.min_height. Weight: `config.feature_hole_weight`.
  3. **Thin wall detection:** Check if any contour has width < `config.thin_wall_threshold` by checking polygon bounding box width or using offset-based measurement. Create `FeatureDetection` with `FeatureType::ThinWall { width_mm }`. Demanded height scaled by `config.feature_thin_wall_weight`.
  4. **Bridge detection:** Use contour differences between adjacent layers to identify bridge candidates. Demanded height = config.min_height. Weight: `config.feature_bridge_weight`.

- After collecting all detections, extend each by margin: `z_min -= margin_mm` and `z_max += margin_mm` where `margin_mm = config.feature_margin_layers as f64 * config.min_height`.
- Sort detections by `z_min` using `f64::total_cmp` for determinism.

**NOTE:** The feature map operates on mesh geometry (triangle normals, AABB) and optionally sliced contours. For overhang angle detection, use triangle normals directly from the mesh (no dependency on slicecore-engine's `detect_overhangs_layer` which needs ValidPolygon contours). For holes and thin walls, if sliced contours are available, use them; otherwise, skip those detections (they become available when the full pipeline is wired in Plan 04).

For this plan, implement overhang detection from mesh normals directly (it's simpler and doesn't require contour data), and stub holes/bridges/thin walls with a TODO noting they need sliced contour data.

**Query functions:**

`query_stress_factor(feature_map, z) -> f64`:
- Binary search for all features overlapping z (z_min <= z <= z_max).
- For each overlapping feature, compute a stress contribution based on feature type and weight.
- Return max stress factor (most demanding wins), clamped to [0.0, 1.0].

`query_feature_demanded_height(feature_map, z) -> Option<f64>`:
- Binary search for overlapping features.
- Return the minimum demanded_height across all overlapping features (thinnest wins).
- Return None if no features overlap this Z.

Add `pub mod features;` to vlh/mod.rs.

**`laplacian_smooth(heights, pinned, lambda, iterations)`:**
```rust
/// 1D Laplacian smoothing on layer heights.
/// `heights`: mutable slice of (z, height) pairs.
/// `pinned`: boolean slice, true = anchor point (not moved during smoothing).
/// `lambda`: smoothing strength 0.0 (no change) to 1.0 (full move toward neighbor average).
/// `iterations`: number of smoothing passes (recommended 3-5).
///
/// Boundary elements (first and last) are never moved.
/// Uses uniform kernel (equal weight for left and right neighbors).
pub fn laplacian_smooth(
    heights: &mut [(f64, f64)],
    pinned: &[bool],
    lambda: f64,
    iterations: usize,
)
```

Implementation:
- For each iteration, snapshot current heights into a temporary Vec<f64>.
- For i in 1..len-1: if !pinned[i], compute `avg = (prev[i-1] + prev[i+1]) / 2.0`, then `heights[i].1 = prev[i] + lambda * (avg - prev[i])`.
- First and last elements are always skipped (boundary condition).
- Lambda clamped to [0.0, 1.0] at entry.

**`ratio_clamp(heights, max_ratio)`:**
```rust
/// Forward-backward ratio clamping (safety net after Laplacian smoothing).
/// Ensures no adjacent layers differ by more than max_ratio (default 1.5 = 50%).
/// This is the existing smooth_heights algorithm from adaptive.rs, extracted and reusable.
pub fn ratio_clamp(heights: &mut [(f64, f64)], max_ratio: f64)
```

Implementation: identical to existing `smooth_heights` in adaptive.rs -- forward pass then backward pass clamping each height relative to its neighbor.
