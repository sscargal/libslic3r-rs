**Part B: Write integration tests**

Create `crates/slicecore-engine/tests/integration_phase20.rs` with the following tests:

1. **test_expanded_json_field_coverage**: Load a real BambuStudio X1C profile JSON (0.20mm Standard process or similar), import via `import_upstream_profile`, verify that at least 50 fields are mapped (up from ~24 for process), and that sub-config fields (speeds, line_widths, cooling, machine, accel) have non-default values.

2. **test_expanded_ini_field_coverage**: Load a real PrusaSlicer profile (from the slicer-analysis directory), parse via the INI pipeline, verify that sub-config fields have non-default values.

3. **test_passthrough_storage**: Import a profile with known unmapped fields, verify they appear in config.passthrough BTreeMap.

4. **test_converted_toml_has_nested_sections**: Load a converted TOML profile from profiles/ directory, verify it contains `[speeds]`, `[line_widths]`, `[cooling]`, `[machine]`, `[retraction]`, `[accel]`, or `[filament]` sections (at least some of them for profiles that have non-default values).

5. **test_x1c_profiles_have_comprehensive_settings**: Load the BambuStudio X1C 0.4mm nozzle + Generic PLA + 0.20mm Standard converted TOML profiles, merge them into a single PrintConfig, verify that speeds.bridge > 0, line_widths.outer_wall > 0, machine.start_gcode is non-empty, retraction.length > 0, etc. This is the key comparison-readiness test.

6. **test_all_existing_tests_pass**: This is implicitly verified by running `cargo test --workspace`. No explicit test needed.

Gate real-profile tests with `#[ignore]` annotation for CI compatibility (they depend on external slicer-analysis directories).
