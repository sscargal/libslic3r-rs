<task type="auto">
  <name>Task 1: Migrate flat fields into sub-configs in config.rs</name>
  <files>crates/slicecore-engine/src/config.rs</files>
  <action>
Move the following existing flat fields INTO their respective sub-config structs. Remove them from the top-level PrintConfig struct body and add them to the sub-config struct instead. Update the PrintConfig Default impl to set defaults on the sub-config, and update the sub-config Default to include the migrated field defaults.

**Retraction fields -> RetractionConfig:**
- `retract_length: f64` -> `retraction.length: f64` (rename from retract_length to length since it's inside RetractionConfig)
- `retract_speed: f64` -> `retraction.speed: f64`
- `retract_z_hop: f64` -> `retraction.z_hop: f64`
- `min_travel_for_retract: f64` -> `retraction.min_travel: f64`

**Speed fields -> SpeedConfig:**
- `perimeter_speed: f64` -> `speeds.perimeter: f64`
- `infill_speed: f64` -> `speeds.infill: f64`
- `travel_speed: f64` -> `speeds.travel: f64`
- `first_layer_speed: f64` -> `speeds.first_layer: f64`

**Temperature fields -> FilamentPropsConfig:**
- `nozzle_temp: f64` -> `filament.nozzle_temp: f64`
- `bed_temp: f64` -> stays on PrintConfig (bed temp is not a filament property -- or move to a new TemperatureConfig?)
- `first_layer_nozzle_temp: f64` -> `filament.first_layer_nozzle_temp: f64`
- `first_layer_bed_temp: f64` -> stays on PrintConfig

Actually, on reflection, temperatures are a cross-cutting concern (not purely filament). Keep bed_temp, first_layer_bed_temp, nozzle_temp, and first_layer_nozzle_temp on PrintConfig for now. The filament sub-config already has nozzle_temperature_range_low/high and filament-specific overrides. Moving main temps would create confusion about which temp to use. **Leave temperature fields flat on PrintConfig.**

**Fan fields -> CoolingConfig:**
- `fan_speed: u8` -> `cooling.fan_speed: u8`
- `fan_below_layer_time: f64` -> `cooling.fan_below_layer_time: f64`
- `disable_fan_first_layers: u32` -> `cooling.disable_fan_first_layers: u32`

**Bed fields -> MachineConfig:**
- `bed_x: f64` -> `machine.bed_x: f64`
- `bed_y: f64` -> `machine.bed_y: f64`

**Filament fields -> FilamentPropsConfig:**
- `filament_diameter: f64` -> `filament.diameter: f64`
- `filament_density: f64` -> `filament.density: f64`
- `filament_cost_per_kg: f64` -> `filament.cost_per_kg: f64`

**Acceleration/jerk -> AccelerationConfig:**
- `print_acceleration: f64` -> `accel.print: f64`
- `travel_acceleration: f64` -> `accel.travel: f64`
- `jerk_x: f64` -> `machine.jerk_x: f64` (jerk is a machine property)
- `jerk_y: f64` -> `machine.jerk_y: f64`
- `jerk_z: f64` -> `machine.jerk_z: f64`
- `pressure_advance: f64` -> stays flat on PrintConfig (cross-cutting firmware tuning)
- `acceleration_enabled: bool` -> stays flat on PrintConfig

**Extrusion fields:**
- `extrusion_multiplier: f64` -> stays flat (used everywhere, cross-cutting)

**Keep extrusion_width() method** working by accessing nozzle_diameter (which stays flat on PrintConfig).

**Update SettingOverrides.merge_into()** to use new field paths:
- `config.perimeter_speed` -> `config.speeds.perimeter`
- `config.infill_speed` -> `config.speeds.infill`

**Update all config.rs tests** to use new field paths.
