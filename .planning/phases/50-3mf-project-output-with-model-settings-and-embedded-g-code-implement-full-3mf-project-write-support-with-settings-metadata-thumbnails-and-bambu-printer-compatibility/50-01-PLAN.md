Implementation steps:
1. Build the base model with meshes and per-object settings (reuse existing logic from `export_plate_to_3mf` -- extract the model-building code into a private helper `fn build_plate_model(meshes, object_configs) -> Result<Model, FileIOError>` and have both `export_plate_to_3mf` and `export_project_to_3mf` call it).
2. For each plate (0-indexed in the Vec, 1-indexed in file paths):
   - Insert G-code: `model.attachments.insert(format!("Metadata/plate_{}.gcode", plate_idx + 1), gcode_bytes.clone())`
   - Compute MD5 and insert: `model.attachments.insert(format!("Metadata/plate_{}.gcode.md5", plate_idx + 1), md5_hex.into_bytes())`. Use `md5::Md5` from the `md-5` crate: `use md5::{Md5, Digest}; let hash = Md5::digest(&gcode_bytes); let md5_hex = format!("{:x}", hash);`
   - Insert thumbnail (if Some): `model.attachments.insert(format!("Metadata/plate_{}.png", plate_idx + 1), png_bytes.clone())`
   - Insert plate metadata JSON: serialize `plate_metadata[i]` with `serde_json::to_string_pretty()`, insert at `Metadata/plate_{N}.json`
3. Insert settings configs using the builders from project_config.rs:
   - `model.attachments.insert("Metadata/process_settings.config".to_string(), build_process_settings_config(&project_options.process_settings).into_bytes())`
   - Same for filament_settings.config and machine_settings.config
4. Insert SliceCore native config: `model.attachments.insert("Metadata/slicecore_config.toml".to_string(), project_options.config_toml.as_bytes().to_vec())`
5. Insert project metadata: `model.attachments.insert("Metadata/project_settings.config".to_string(), build_project_metadata_config(&project_options.project_metadata).into_bytes())`
6. If AMS mapping present, serialize to JSON and insert at `Metadata/ams_mapping.json`
7. Call `model.write(&mut writer)`
