```rust
impl CascadeResolver {
    /// Resolves layer-range overrides (cascade layer 9) for a specific Z height.
    ///
    /// Takes the base per-object config (layers 1-8) and applies any matching
    /// layer-range overrides from the ObjectConfig. Returns a new Arc<PrintConfig>
    /// if any layer-range overrides match, or the original Arc if none match.
    ///
    /// Multiple matching ranges are applied in definition order (last wins for
    /// conflicting fields, matching CONTEXT.md "last-defined wins" rule).
    pub fn resolve_for_z(
        resolved: &ResolvedObject,
        object_config: &ObjectConfig,
        z: f64,
        layer_number: u32,
    ) -> Result<Arc<PrintConfig>, EngineError> {
        // Collect all matching layer-range overrides
        let matching: Vec<&LayerRangeOverride> = object_config
            .layer_overrides
            .iter()
            .filter(|lr| {
                if let Some((z_min, z_max)) = lr.z_range {
                    z >= z_min - 1e-6 && z <= z_max + 1e-6
                } else if let Some((layer_start, layer_end)) = lr.layer_range {
                    layer_number >= layer_start && layer_number <= layer_end
                } else {
                    false
                }
            })
            .collect();

        if matching.is_empty() {
            return Ok(Arc::clone(&resolved.config));
        }

        // Create a new composer starting from the resolved base config
        // Apply each matching layer-range override as layer 9
        let mut composer = ProfileComposer::from_config((*resolved.config).clone());
        for (i, lr) in matching.iter().enumerate() {
            let range_desc = if let Some((z_min, z_max)) = lr.z_range {
                format!("z:{z_min:.2}-{z_max:.2}")
            } else if let Some((start, end)) = lr.layer_range {
                format!("layers:{start}-{end}")
            } else {
                format!("range-{i}")
            };
            composer.add_table_layer(
                SourceType::LayerRangeOverride {
                    object_id: resolved.name.clone(),
                    range_desc,
                },
                lr.overrides.clone(),
            );
        }
        let composed = composer.compose()?;
        Ok(Arc::new(composed.config))
    }
}
```

pub fn from_plate_config(plate: PlateConfig, base_composed: ComposedConfig) -> Result<Self, EngineError> {
    let resolved = CascadeResolver::resolve_all(&plate, &base_composed)?;
    let first_config = resolved.first()
        .map(|r| Arc::clone(&r.config))

        resolved_objects: resolved,
        plate_config: Some(plate),
        #[cfg(feature = "plugins")]

Add `slice_plate` method. For each object at each Z height, call `CascadeResolver::resolve_for_z` to apply matching layer-range overrides (cascade layer 9) before slicing that layer. This ensures layer-range overrides take effect at the correct Z heights.

```rust
/// Slices all objects in the plate, returning per-object results.
///
/// For each object, resolves layer-range overrides (cascade layer 9) at each
/// Z height via CascadeResolver::resolve_for_z before processing that layer.
pub fn slice_plate(
    &self,
    meshes: &[&TriangleMesh],
    cancel: Option<CancellationToken>,
) -> Result<PlateSliceResult, EngineError> {
    let plate = self.plate_config.as_ref();
    let mut object_results = Vec::new();

    for (obj, mesh) in self.resolved_objects.iter().zip(meshes.iter()) {
        let object_config = plate.and_then(|p| p.objects.get(obj.index));
        let has_layer_overrides = object_config
            .map_or(false, |oc| !oc.layer_overrides.is_empty());

        if has_layer_overrides {
            // Object has layer-range overrides -- resolve per-Z during slicing.
            // Use the base per-object config (layers 1-8) and apply layer 9
            // at each Z height via resolve_for_z.
            let oc = object_config.unwrap();
            let obj_engine = Engine::new_with_layer_range_resolver(
                obj, oc, mesh, cancel.clone(),
            )?;
            object_results.push(obj_engine);
        } else {
            // No layer-range overrides -- use static per-object config.
            let obj_engine = Engine::new((*obj.config).clone());
            let result = obj_engine.slice(mesh, cancel.clone())?;
            object_results.push(ObjectSliceResult {
                name: obj.name.clone(),
                index: obj.index,
                result,
                copies: obj.copies,
            });
        }
    }
    Ok(PlateSliceResult { objects: object_results })
}
```

For the layer-range-aware path, implement a private helper that iterates Z heights and calls `CascadeResolver::resolve_for_z(obj, object_config, z, layer_number)` for each layer, using the returned config to drive that layer's slicing parameters. The exact integration depends on how the existing `slice()` method structures its layer loop -- read engine.rs carefully and inject the resolve_for_z call at the appropriate point (likely where per-layer config is read).

If the existing slice() method does not have a clean per-layer config injection point, consider:
1. Pre-computing the set of distinct configs across all Z ranges (often just 2-3 unique configs)
2. Grouping contiguous layers by their resolved config
3. Slicing each group with its config

This avoids re-resolving at every Z height when overrides typically span contiguous ranges.
