```rust
#[derive(Subcommand)]
pub enum OverrideSetCommands {
    /// List all override sets with field counts
    List {
        #[arg(long)]
        json: bool,
    },
    /// Show an override set with metadata
    Show {
        name: String,
        #[arg(long)]
        json: bool,
    },
    /// Create a new override set
    Create {
        name: String,
        /// Set key=value pairs (repeatable)
        #[arg(long = "set")]
        set_overrides: Vec<String>,
        /// Create from diff between two profiles
        #[arg(long)]
        from_diff: Option<Vec<String>>,
        #[arg(long)]
        json: bool,
    },
    /// Open an override set in $EDITOR
    Edit { name: String },
    /// Delete an override set
    Delete {
        name: String,
        #[arg(long)]
        force: bool,
    },
    /// Rename an override set
    Rename {
        old: String,
        new: String,
    },
    /// Compare two override sets side-by-side
    Diff {
        set_a: String,
        set_b: String,
        #[arg(long)]
        json: bool,
    },
}
```

**2. Override set storage:**
Use `dirs::home_dir()` to resolve `~/.slicecore/override-sets/`. Create directory if it doesn't exist. Each set is a `.toml` file in that directory.

Helper functions:
- `fn override_sets_dir() -> Result<PathBuf, CliError>` -- resolve and create dir
- `fn set_path(name: &str) -> Result<PathBuf, CliError>` -- path for a named set
- `fn load_set(name: &str) -> Result<toml::map::Map<String, toml::Value>, CliError>` -- load and parse
- `fn save_set(name: &str, table: &toml::map::Map<String, toml::Value>) -> Result<(), CliError>` -- save
- `fn list_sets() -> Result<Vec<String>, CliError>` -- list all set names
- `fn fuzzy_suggest(name: &str, available: &[String]) -> Option<String>` -- strsim suggestion

**3. Implement each command:**

- **list**: Read dir, for each .toml file load and count fields, display table (name, field count) or JSON array
- **show**: Load set, look up each field in SettingRegistry for metadata (description, units, override_safety), display table or JSON
- **create**: Parse --set key=value pairs using `parse_set_value` from profile_compose.rs, validate keys with `validate_set_key`, save to file. For --from-diff: load two profiles, diff them, save differing fields
- **edit**: Open `$EDITOR` on the set file, re-validate after editor closes, report errors
- **delete**: Check file exists, prompt confirmation unless --force, remove file
- **rename**: Check old exists, new doesn't exist, rename file
- **diff**: Load both sets, compute fields only in A, only in B, differing values, display side-by-side table or JSON

**4. Integrate into main.rs:**
Add `OverrideSet(OverrideSetCommands)` variant to `Commands` enum. Add `mod override_set;` and dispatch in main match.

```rust
        output: PathBuf,
    },

        /// Output 3MF file
        #[arg(short, long)]
        output: PathBuf,
    },
```

**2. Implement plate init:**
Generate a fully commented plate.toml template:
```toml
# SliceCore Plate Configuration
# See: slicecore plate --help

# process = "quality-preset"

# [default_overrides]
# Settings applied to ALL objects (cascade layer 7)
# infill_density = 0.2

# [override_sets.example]
# Named override set (reusable across objects)
# layer_height = 0.1
# wall_count = 4

[[objects]]
model = "model.stl"
# name = "Part Name"
# override_set = "example"
copies = 1

# [objects.transform]
# position = [0.0, 0.0, 0.0]
# rotation = [0.0, 0.0, 0.0]
# scale = [1.0, 1.0, 1.0]

# [objects.overrides]
# Inline overrides (applied after override_set)
# infill_density = 0.8

# [[objects.modifiers]]
# shape = "box"
# position = [0.0, 0.0, 0.0]
# size = [10.0, 10.0, 10.0]
# [objects.modifiers.overrides]
# infill_density = 1.0

# [[objects.layer_overrides]]
```
