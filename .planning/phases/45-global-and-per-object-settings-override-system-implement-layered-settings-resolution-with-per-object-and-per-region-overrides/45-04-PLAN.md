**Step 5: Update split_by_modifiers()**
The current implementation calls `overrides.merge_into(base)` to produce per-region PrintConfig. Replace with TOML-level merge:

1. Serialize `base: &PrintConfig` to `toml::Value::Table` using `toml::Value::try_from(base)`
2. For each region's overrides, call the existing `merge_layer()` function from profile_compose.rs to deep-merge the override table into the base table
3. Deserialize the merged table back to `PrintConfig` using `PrintConfig::deserialize(toml::Value::Table(merged))`
4. Last-defined modifier wins: process modifiers in definition order so later ones override earlier ones in overlapping regions

Add `use crate::profile_compose::merge_layer;` import.
