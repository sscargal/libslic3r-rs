} else if meta.path.is_ident("override_safety") {
    let value = meta.value()?;
    let lit: syn::LitStr = value.parse()?;
    let val = lit.value();
    if !["safe", "warn", "ignored"].contains(&val.as_str()) {
        return Err(meta.error(format!(
            "override_safety must be \"safe\", \"warn\", or \"ignored\", got \"{val}\""
        )));

    result.override_safety = Some(val);

**Step 1: Create OVERRIDE_SAFETY_MAP.md**
Create `designDocs/OVERRIDE_SAFETY_MAP.md` with a table mapping every PrintConfig field to its override_safety classification. Use domain knowledge to classify:

- **safe**: Fields that make sense per-object AND per-region. Examples: `layer_height`, `infill_density`, `infill_pattern`, `wall_count`, `top_solid_layers`, `bottom_solid_layers`, all `speeds.*` fields, `support_enabled`, `support_angle`, `ironing_enabled`, `seam_position`, `line_width`, `extrusion_multiplier`, `retraction.*` fields, `bridge_*` fields, `gap_fill_*` fields, `first_layer_*` speed/flow overrides.

- **warn**: Fields that are nonsensical per-region but allowed (machine-level concerns). Examples: `bed_temperature`, `nozzle_temperature` (per-object ok, per-region questionable), `fan_speed_*` (per-object ok, per-region odd), `skirt_*`, `brim_*`, `raft_*` (plate-level adhesion, not per-region), `sequential_*` fields.

- **ignored**: Machine properties that have no meaning per-object. Examples: `machine.*` fields (bed_x, bed_y, bed_z, nozzle_diameter, etc.), `gcode_flavor`, `start_gcode`, `end_gcode`, `plugin_dir`, `firmware_*`.

Format:
```markdown
# Override Safety Map

| Field | Safety | Reason |
|-------|--------|--------|
| layer_height | safe | Different layer heights per object/region |
| infill_density | safe | Core use case for modifier meshes |
| bed_temperature | warn | Machine-level, nonsensical per-region |
| machine.bed_x | ignored | Machine property, no per-object meaning |
...
```
