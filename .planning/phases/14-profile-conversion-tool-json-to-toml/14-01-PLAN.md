  <name>Task 1: Profile conversion module with selective TOML output and multi-file merge</name>
  <files>crates/slicecore-engine/src/profile_convert.rs, crates/slicecore-engine/src/lib.rs</files>
  <action>
Create `crates/slicecore-engine/src/profile_convert.rs` with the following:

1. **ConvertResult struct:**
   ```rust
   pub struct ConvertResult {
       pub toml_output: String,        // Generated TOML string
       pub mapped_count: usize,        // Number of fields mapped from source
       pub unmapped_fields: Vec<String>, // Fields with no PrintConfig equivalent
       pub source_name: Option<String>,  // Profile name from metadata
       pub source_type: Option<String>,  // Profile type from metadata
   }
   ```

2. **convert_to_toml(result: &ImportResult) -> ConvertResult:**
   - Build header comments: source name, type, inherits (if present), mapped/unmapped count.
   - Serialize `result.config` to `toml::Value::Table` and `PrintConfig::default()` to another `toml::Value::Table`.
   - Filter: keep only top-level keys where the value differs from the default. For nested tables (like `[ironing]`, `[support]`, etc.), only include the table if it differs from the default.
   - Use `toml::to_string_pretty()` on the filtered `toml::Value::Table` to produce the TOML body.
   - Append an unmapped fields section as TOML comments at the end (e.g., `# Unmapped fields from source (no equivalent in PrintConfig):\n# - bridge_speed\n# - gap_infill_speed`).
   - Return `ConvertResult` with the assembled output string and stats.
   - Handle float precision: round float values to 6 decimal places before comparison and output to avoid IEEE 754 artifacts (e.g., 0.15000000000000002).

3. **merge_import_results(results: &[ImportResult]) -> ImportResult:**
   - Start with `PrintConfig::default()`.
   - For each `ImportResult` in order: serialize its config to `toml::Value::Table`, serialize the default to `toml::Value::Table`, find keys where the result differs from default, and overlay those keys onto a running merged `toml::Value::Table`.
   - After merging all results, deserialize the merged table back into `PrintConfig`.
   - Collect all mapped_fields (deduplicated) and all unmapped_fields (deduplicated) across results.
   - For metadata: use the last result's metadata, or create a merged metadata with names joined by " + ".

4. **Helper: round_floats_in_value(value: &mut toml::Value):**
   - Recursively walk a `toml::Value` tree. For any `Value::Float(f)`, round to 6 decimal places: `(f * 1_000_000.0).round() / 1_000_000.0`.
   - This prevents TOML output like `infill_density = 0.15000000000000002`.

5. **Unit tests (in module):**
   - `test_convert_basic_process_profile`: Import a process JSON, convert to TOML, verify header comments present, verify TOML contains `layer_height` and `wall_count` but NOT `support` section (which would be at default).
   - `test_convert_selective_output`: Verify that converting a profile with only 3 mapped fields produces TOML with only those 3 fields (not 86 defaults).
   - `test_merge_two_profiles`: Import process + filament JSONs, merge, verify merged config has fields from both.
   - `test_float_rounding`: Verify that a percentage value like 15% (which becomes 0.15 in f64) outputs cleanly without floating-point noise.
   - `test_unmapped_fields_in_comments`: Verify unmapped fields appear as comments at end of output.

6. **Register module in lib.rs:**
   - Add `pub mod profile_convert;` to the module list.
   - Add re-exports: `pub use profile_convert::{convert_to_toml, merge_import_results, ConvertResult};`

<task type="auto">
  <name>Task 2: CLI convert-profile subcommand with conversion report</name>
  <files>crates/slicecore-cli/src/main.rs</files>
  <action>
Add a `ConvertProfile` subcommand to the CLI following the established pattern of existing subcommands:

1. **Add enum variant to Commands:**
   ```rust
   /// Convert OrcaSlicer/BambuStudio JSON profiles to native TOML format.
   ///
   /// Reads one or more JSON profile files, maps fields to PrintConfig,
   /// and outputs clean TOML with only the converted fields.
   ConvertProfile {
       /// Input JSON profile file(s) to convert.
       /// Multiple files are merged in order (e.g., process + filament + machine).
       #[arg(required = true)]
       input: Vec<PathBuf>,

       /// Output TOML file path (default: stdout).
       #[arg(short, long)]
       output: Option<PathBuf>,

       /// Show detailed conversion report on stderr.
       #[arg(short, long)]
       verbose: bool,
   },
   ```

2. **Add match arm in main():**
   ```rust
   Commands::ConvertProfile { input, output, verbose } => {
       cmd_convert_profile(&input, output.as_deref(), verbose)
   }
   ```

3. **Implement cmd_convert_profile:**
   - Import `slicecore_engine::{convert_to_toml, merge_import_results, ImportResult, PrintConfig}` and `slicecore_engine::profile_import::import_upstream_profile`.
   - For each input file:
     - Read file contents with `std::fs::read_to_string`.
     - Parse JSON with `serde_json::from_str::<serde_json::Value>`.
     - Call `import_upstream_profile(&value)` to get `ImportResult`.
     - If verbose, print per-file conversion summary to stderr.
   - If multiple input files, call `merge_import_results(&results)` to get a single merged `ImportResult`. If single file, use the one `ImportResult` directly.
   - Call `convert_to_toml(&final_result)` to get `ConvertResult`.
   - Output the TOML:
     - If `--output` specified: write to file.
     - Otherwise: print to stdout.
   - Print conversion summary to stderr:
     ```
     Converted "profile name" (type)
       Mapped: 12 fields
       Unmapped: 36 fields
       Output: stdout (or path)
     ```
   - If verbose, also list the mapped field names and unmapped field names on stderr.
   - On any error (file read, JSON parse, import), print error to stderr and exit(1).

4. **Update the after_help text in the Cli struct:**
   Add a section about the convert-profile subcommand:
   ```
   PROFILE CONVERSION:
     Convert OrcaSlicer/BambuStudio JSON profiles to native TOML:
       slicecore convert-profile profile.json > my_config.toml
       slicecore convert-profile process.json filament.json machine.json -o config.toml
     Multiple files are merged in order (later files override earlier ones for shared fields).
   ```
