<task type="auto">
  <name>Task 1: Integration tests for profile conversion round-trip and merge</name>
  <files>crates/slicecore-engine/tests/integration_profile_convert.rs</files>
  <action>
Create `crates/slicecore-engine/tests/integration_profile_convert.rs` with the following tests:

1. **test_round_trip_process_profile:**
   - Create a synthetic OrcaSlicer process JSON with ~10 fields (layer_height, wall_loops, sparse_infill_density as percentage, outer_wall_speed, travel_speed, seam_position, sparse_infill_pattern, top_shell_layers, bottom_shell_layers, initial_layer_print_height).
   - Import via `import_upstream_profile`.
   - Convert to TOML via `convert_to_toml`.
   - Parse the TOML output back via `PrintConfig::from_toml`.
   - Assert all mapped field values match the original import (within f64 epsilon for floats, exact for integers/enums).
   - This is the critical round-trip test proving data fidelity.

2. **test_round_trip_filament_profile:**
   - Create a synthetic filament JSON with array-wrapped values (nozzle_temperature, hot_plate_temp, filament_density, filament_diameter, filament_flow_ratio, filament_cost).
   - Import, convert, parse back. Assert all values round-trip correctly.

3. **test_round_trip_machine_profile:**
   - Create a synthetic machine JSON (nozzle_diameter, retraction_length, retraction_speed, z_hop, gcode_flavor as "klipper").
   - Import, convert, parse back. Assert GcodeDialect round-trips to Klipper.

4. **test_merge_process_and_filament:**
   - Create two JSONs: one process (layer_height, wall_loops, infill), one filament (temps, density).
   - Import both, merge via `merge_import_results`.
   - Convert merged result to TOML and parse back.
   - Assert the parsed config has fields from BOTH profiles (layer_height from process, nozzle_temp from filament).
   - Assert mapped_fields contains fields from both sources (deduplicated).

5. **test_merge_three_profiles:**
   - Create process + filament + machine JSONs.
   - Merge all three, convert to TOML, parse back.
   - Assert config has layer_height (process), nozzle_temp (filament), retraction_length (machine).

6. **test_selective_output_no_defaults:**
   - Import a JSON with only layer_height and wall_loops mapped.
   - Convert to TOML.
   - Assert the TOML body does NOT contain `fan_speed`, `support`, `ironing`, `scarf_joint`, `multi_material` or other default-only fields.
   - Assert TOML is concise (under 20 lines including comments).

7. **test_unmapped_fields_in_output:**
   - Import a JSON with known unmapped fields (e.g., `"bridge_speed": "50"`, `"gap_infill_speed": "200"`).
   - Convert to TOML.
   - Assert the output contains comment lines mentioning the unmapped field names.
   - Assert ConvertResult.unmapped_fields contains those field names.

8. **test_percentage_float_clean_output:**
   - Import a JSON with `"sparse_infill_density": "15%"` (produces 0.15 in f64, which may have IEEE 754 noise).
   - Convert to TOML.
   - Assert the TOML string contains `0.15` and NOT `0.15000000000000002` or similar noise.

9. **test_real_orcaslicer_profile_conversion (gated with #[ignore]):**
   - Load a real OrcaSlicer process profile from `/home/steve/slicer-analysis/OrcaSlicer/resources/profiles/BBL/process/`.
   - Import and convert to TOML.
   - Parse the TOML back via `PrintConfig::from_toml`.
   - Assert it does not error and produces a config with reasonable values (layer_height > 0, perimeter_speed > 0).

10. **test_real_multi_file_merge (gated with #[ignore]):**
    - Load a real OrcaSlicer process + filament + machine profile set from BBL resources.
    - Merge and convert to TOML.
    - Parse back and verify the merged config has populated fields from all three sources.

All tests use `slicecore_engine::profile_import::import_upstream_profile` for the import step and `slicecore_engine::profile_convert::{convert_to_toml, merge_import_results}` for conversion. Use `slicecore_engine::PrintConfig` for round-trip parsing.
