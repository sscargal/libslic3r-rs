    **Integration Tests (TEST-02):**

    Create `crates/slicecore-engine/tests/integration_pipeline.rs` with comprehensive end-to-end tests:

    1. `test_stl_to_gcode_calibration_cube`: Build a 20mm calibration cube mesh in-memory, create default PrintConfig, create Engine, call slice(), verify:
       - Result is Ok
       - layer_count > 0 (should be ~100 for 20mm at 0.2mm layers)
       - gcode is non-empty
       - gcode starts with G28 (home) command
       - gcode contains M104 or M109 (temperature) commands
       - gcode contains G1 commands with E values (extrusion)
       - gcode ends with M104 S0 (heater off)
       - estimated_time_seconds > 0
       - filament_usage length > 0

    2. `test_stl_to_gcode_with_custom_config`: Same cube but with custom config:
       - layer_height = 0.1 (should double layer count)
       - infill_density = 0.5
       - Verify layer count is approximately 2x the default config

    3. `test_stl_to_gcode_with_supports`: Build a T-shape model (box with overhang shelf), enable support:
       - Verify result contains support-related G-code comments (;TYPE: Support)
       - Verify more gcode output than without supports

    4. `test_stl_to_gcode_with_brim`: Cube with brim_width = 5.0:
       - Verify gcode contains brim extrusion on layer 0

    5. `test_mesh_repair_integration`: Build a mesh with intentionally degenerate triangles, repair it, then slice:
       - Verify repair succeeds and slice produces valid gcode

    6. `test_json_output_integration`: Slice a cube, call `to_json`, parse the JSON, verify fields:
       - layer_count matches
       - time_estimate has total_seconds > 0
       - filament_usage has length_mm > 0

    7. `test_event_system_integration`: Slice a cube with EventBus + CallbackSubscriber, verify:
       - At least one StageChanged event received
       - LayerComplete events received (count should match layer_count)
       - Complete event received at end
