    1. `SliceResult` in engine.rs -- add `#[derive(Serialize, Deserialize)]`. The `gcode: Vec<u8>` field should use `#[serde(skip)]` (users get gcode from the file, metadata from JSON). Any function pointer or non-serializable fields should also use `#[serde(skip)]`.
    2. `FeatureType` enum in toolpath.rs -- add `#[derive(Serialize, Deserialize)]`
    3. `ToolpathSegment` in toolpath.rs -- add `#[derive(Serialize, Deserialize)]`
    4. `LayerToolpath` in toolpath.rs -- add `#[derive(Serialize, Deserialize)]`
    5. `ContourPerimeters` and `PerimeterShell` in perimeter.rs -- add if they are pub
    6. `SupportRegion` and `SupportResult` in support/mod.rs -- add derives
    7. `SupportConfig` in support/config.rs -- check if already has Serialize, add if not

    Add `serde_json` as a runtime dependency in slicecore-engine/Cargo.toml:
    ```toml
    serde_json = { workspace = true }
    ```

    For types with non-serializable fields (e.g., function pointers, OnceLock, complex borrowed types), use `#[serde(skip)]` on those fields and ensure the type derives `Default` if `serde(skip)` requires it for deserialization.

    After adding all derives, run `cargo build --workspace` to verify no compilation errors from transitive Serialize requirements. Fix any errors from nested types that also need Serialize -- propagate derives as needed.
  </action>
