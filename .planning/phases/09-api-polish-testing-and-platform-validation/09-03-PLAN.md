    The `wasm32-unknown-unknown` target fails because `getrandom` 0.3 (pulled in transitively by `boostvoronoi` -> `ahash` -> `getrandom`) requires explicit backend configuration for this target.

    1. Update `.cargo/config.toml` to add the `getrandom_backend` cfg flag for wasm32-unknown-unknown:
    ```toml
    [build]
    incremental = true

    [target.wasm32-unknown-unknown]
    rustflags = ["-C", "opt-level=s", "--cfg", "getrandom_backend=\"wasm_js\""]
    ```

    2. Add `getrandom` as a conditional dependency in the workspace Cargo.toml or in slicecore-engine/Cargo.toml for the wasm32 target:
    ```toml
    [target.'cfg(all(target_arch = "wasm32", target_os = "unknown"))'.dependencies]
    getrandom = { version = "0.3", features = ["wasm_js"] }
    ```

    This needs to be in whichever Cargo.toml actually pulls in `boostvoronoi` (slicecore-engine).

    3. Verify the fix:
    ```bash
    cargo build --target wasm32-unknown-unknown --workspace --exclude slicecore-plugin --exclude slicecore-plugin-api --exclude slicecore-ai --exclude slicecore-cli
    ```

<task type="auto">
  <name>Task 2: Expand CI to multi-platform matrix</name>
  <files>
    .github/workflows/ci.yml
  </files>
  <action>
    Rewrite `.github/workflows/ci.yml` to expand from single ubuntu-latest to a full multi-platform CI matrix. Keep existing jobs (check, test, clippy, fmt) but expand them.

    **New CI structure:**

    ```yaml
    name: CI

    on:
      push:
        branches: [main, master, "phase-*"]
      pull_request:
        branches: [main, master]

    env:
      CARGO_TERM_COLOR: always
      RUSTFLAGS: "-D warnings"

    jobs:
      fmt:
        name: Format
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: dtolnay/rust-toolchain@stable
            with:
              components: rustfmt
          - run: cargo fmt --all -- --check

      clippy:
        name: Clippy
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: dtolnay/rust-toolchain@stable
            with:
              components: clippy
          - uses: Swatinem/rust-cache@v2
          - run: cargo clippy --workspace -- -D warnings

      test:
        name: Test (${{ matrix.os }})
        strategy:
          fail-fast: false
          matrix:
            include:
              - os: ubuntu-latest
                target: x86_64-unknown-linux-gnu
              - os: macos-latest
                target: aarch64-apple-darwin
              - os: macos-13
                target: x86_64-apple-darwin
              - os: windows-latest
                target: x86_64-pc-windows-msvc
        runs-on: ${{ matrix.os }}
        steps:
          - uses: actions/checkout@v4
          - uses: dtolnay/rust-toolchain@stable
          - uses: Swatinem/rust-cache@v2
          - run: cargo test --workspace

      test-linux-arm:
        name: Test (Linux ARM64)
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: houseabsolute/actions-rust-cross@v0
            with:
              command: test
              target: aarch64-unknown-linux-gnu
              args: "--workspace"

      wasm:
        name: WASM Build (${{ matrix.target }})
        runs-on: ubuntu-latest
        strategy:
          matrix:
            target:
              - wasm32-unknown-unknown
              - wasm32-wasip2
        steps:
          - uses: actions/checkout@v4
          - uses: dtolnay/rust-toolchain@stable
            with:
              targets: ${{ matrix.target }}
          - uses: Swatinem/rust-cache@v2
          - run: cargo build --target ${{ matrix.target }} --workspace --exclude slicecore-plugin --exclude slicecore-plugin-api --exclude slicecore-ai --exclude slicecore-cli

      doc:
        name: Documentation
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: dtolnay/rust-toolchain@stable
          - uses: Swatinem/rust-cache@v2
          - run: cargo doc --no-deps --workspace
            env:
              RUSTDOCFLAGS: "-D warnings"
    ```

    Key points:
    - `fmt` and `clippy` stay on ubuntu-latest only (platform-independent checks)
    - `test` runs on ubuntu, macOS ARM (latest = M-series), macOS x86 (macos-13), Windows
    - `test-linux-arm` uses `actions-rust-cross` for aarch64-unknown-linux-gnu cross-compilation
    - `wasm` tests both wasm32-unknown-unknown and wasm32-wasip2
    - `doc` verifies zero rustdoc warnings with `-D warnings` flag
    - `fail-fast: false` so all platforms report even if one fails
    - Note: Windows ARM deferred (no GitHub Actions runner available) per research recommendation
  </action>
  <verify>
    The CI YAML is valid (no syntax errors).
    Run `cargo test --workspace` locally to verify tests still pass.
    Run `cargo doc --no-deps --workspace` with RUSTDOCFLAGS="-D warnings" locally.
  </verify>

  </done>
