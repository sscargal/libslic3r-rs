  <files>

  <action>
    **Event System (event.rs):**

    Create `crates/slicecore-engine/src/event.rs` with:
    1. `SliceEvent` enum (derive Debug, Clone, Serialize) with `#[serde(tag = "type")]`:
       - `StageChanged { stage: String, progress: f32 }` -- emitted at pipeline stage transitions
       - `LayerComplete { layer: usize, total: usize, z: f64 }` -- emitted after each layer
       - `Warning { message: String, layer: Option<usize> }` -- non-fatal issues
       - `Error { message: String }` -- fatal issues
       - `PerformanceMetric { stage: String, duration_ms: u64 }` -- timing data
       - `Complete { layers: usize, time_seconds: f64 }` -- final summary
    2. `EventSubscriber` trait: `fn on_event(&self, event: &SliceEvent)` with `Send + Sync` bounds
    3. `EventBus` struct with `Vec<Box<dyn EventSubscriber>>`:
       - `new()` constructor
       - `subscribe(sub: Box<dyn EventSubscriber>)` method
       - `emit(event: &SliceEvent)` method that dispatches to all subscribers
    4. `CallbackSubscriber<F>` struct wrapping `F: Fn(&SliceEvent) + Send + Sync`:
       - Implements `EventSubscriber`
    5. `NdjsonSubscriber` struct wrapping `Arc<Mutex<Box<dyn Write + Send>>>`:
       - Writes each event as a JSON line via serde_json

    **Engine Integration:**

    In engine.rs, add an `event_bus: Option<&EventBus>` parameter to the internal pipeline or create a `slice_with_events` method:
    - `Engine::slice_with_events(&self, mesh: &TriangleMesh, event_bus: &EventBus) -> Result<SliceResult, EngineError>`
    - Emit `StageChanged` at: mesh slicing start, per-layer processing start, gcode generation start
    - Emit `LayerComplete` after each layer is processed
    - Emit `Complete` at the end with final stats
    - The existing `Engine::slice()` continues to work without events (backward compatible)

    **Structured Output (output.rs):**

    Create `crates/slicecore-engine/src/output.rs` with:
    1. `SliceMetadata` struct (Serialize, Deserialize) containing:
       - `layer_count: usize`
       - `time_estimate: PrintTimeEstimate`
       - `filament_usage: FilamentUsage`
       - `config_summary: ConfigSummary` (layer_height, infill_density, infill_pattern, nozzle_diameter, etc.)
    2. `ConfigSummary` struct (Serialize, Deserialize) with key config fields
    3. `to_json(result: &SliceResult, config: &PrintConfig) -> Result<String, serde_json::Error>` -- pretty-printed JSON
    4. `to_msgpack(result: &SliceResult, config: &PrintConfig) -> Result<Vec<u8>, rmp_serde::encode::Error>` -- MessagePack
    5. `from_msgpack(data: &[u8]) -> Result<SliceMetadata, rmp_serde::decode::Error>` -- MessagePack decode

    Add `rmp-serde = "1"` to workspace Cargo.toml `[workspace.dependencies]` and to slicecore-engine/Cargo.toml `[dependencies]`.

    **Module registration:**

    In lib.rs, add:
    ```rust
    pub mod event;
    pub mod output;
    ```
    And re-export key types:
    ```rust
    pub use event::{SliceEvent, EventSubscriber, EventBus, CallbackSubscriber};
    pub use output::{to_json, to_msgpack, SliceMetadata};
    ```
  </action>

    3. If both flags set, prefer JSON (or error -- pick one).

    Implementation:
    ```rust
    #[derive(Subcommand)]
    enum Commands {
        Slice {
            input: PathBuf,
            #[arg(short, long)]
            config: Option<PathBuf>,
            #[arg(short, long)]
            output: Option<PathBuf>,
            /// Output slicing metadata as JSON to stdout
            #[arg(long)]
            json: bool,
            /// Output slicing metadata as MessagePack to stdout
            #[arg(long)]
            msgpack: bool,
        },
        // ... existing
    }
    ```

    In cmd_slice, after slicing and writing gcode:
    ```rust
    if json {
        let json_str = slicecore_engine::output::to_json(&result, &print_config).unwrap();
        println!("{}", json_str);
    } else if msgpack {
        let bytes = slicecore_engine::output::to_msgpack(&result, &print_config).unwrap();
        use std::io::Write;
        std::io::stdout().write_all(&bytes).unwrap();
    }
    ```

    Add `slicecore-engine` features and `rmp-serde` to CLI Cargo.toml if not already present.
    Add `serde_json` to CLI Cargo.toml dependencies.
