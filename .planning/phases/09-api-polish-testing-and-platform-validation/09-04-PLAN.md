
    Add benchmark configuration to `crates/slicecore-engine/Cargo.toml`:
    ```toml
    [dev-dependencies]
    criterion = { workspace = true }

    [[bench]]
    name = "slice_benchmark"
    harness = false

    [[bench]]
    name = "geometry_benchmark"
    harness = false
    ```

    Create `crates/slicecore-engine/benches/slice_benchmark.rs` with 5 synthetic test models:

    1. **Calibration cube** (20mm cube, ~12 triangles): Baseline simple model.
       Generate programmatically: 8 vertices, 12 triangles forming an axis-aligned box.

    2. **Cylinder** (~200 triangles, moderate complexity): Tests contour chaining with many small segments.
       Generate programmatically: create cylinder with 64 sides, caps on both ends.

    3. **Dense sphere** (~1280 triangles): Tests BVH and contour chaining with high triangle count.
       Generate programmatically: icosahedron with 3 levels of subdivision.

    4. **Thin-wall box** (walls at 0.8mm thickness, tests perimeter generation edge cases):
       Generate as outer cube minus slightly smaller inner cube (hollow box).

    5. **Multi-overhang model** (box with 45-degree overhang shelves): Tests support detection path.
       Generate as main box with offset shelf boxes at varying heights.

    Each model is built in-memory using `TriangleMesh::new(vertices, indices)` -- no external STL files needed. This ensures benchmarks are self-contained and reproducible.

    For each model, benchmark:
    ```rust
    fn bench_slice_cube(c: &mut Criterion) {
        let mesh = build_calibration_cube();
        let config = PrintConfig::default();
        let engine = Engine::new(config);

        c.bench_function("slice_calibration_cube", |b| {
            b.iter(|| engine.slice(&mesh).unwrap())
        });
    }
    ```

    Use `criterion_group!` and `criterion_main!` macros. Include a benchmark group for each model.

    Also add a "full config" benchmark for the cube that enables support, adaptive layers, and non-default infill (gyroid) to measure feature-rich slicing.

    Add a `bench_memory_estimate` function that measures allocation by slicing and recording the output gcode Vec<u8> size, layer count, and a rough memory estimate. (Criterion measures wallclock time; for memory, we can use peak RSS via `/proc/self/status` on Linux or document methodology for manual measurement.)
