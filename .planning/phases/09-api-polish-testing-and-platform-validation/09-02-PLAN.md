  <action>
    For each `pub mod` declaration in these 5 crates, find the corresponding module file and add a brief `//!` doc comment at the top if missing.

    Strategy:
    1. In each lib.rs, find all `pub mod foo;` declarations
    2. For each, open `src/foo.rs` or `src/foo/mod.rs`
    3. If the file lacks a `//!` doc comment at the top, add one

    Examples of good module-level docs:
    - `//! Ironing pass generation for top surfaces.`
    - `//! Triangle-plane intersection and contour chaining for mesh slicing.`
    - `//! Polygon boolean operations (union, intersection, difference, XOR).`
    - `//! BVH-accelerated spatial queries for triangle meshes.`

    Keep docs brief (1-2 sentences). Focus on WHAT the module provides, not HOW.

    The slicecore-engine crate is the largest -- it has modules for: toolpath, perimeter, infill, support, gcode_gen, engine, arachne, calibration, modifier, multimaterial, sequential, gap_fill, ironing, arc_fitting, seam, adaptive_layer, preview, polyhole, surface, planner, retraction, event, output. Each pub module file needs a `//!` line.

    Also add crate-level `//!` docs to each lib.rs itself if missing (e.g., `//! Core slicing engine for libslic3r-rs.`).

    Do NOT modify any code logic -- only add doc comments.
  </action>
  <verify>
    Run `grep -rn "pub mod" crates/slicecore-engine/src/lib.rs crates/slicecore-slicer/src/lib.rs crates/slicecore-mesh/src/lib.rs crates/slicecore-math/src/lib.rs crates/slicecore-geo/src/lib.rs` to list all public modules.
    For each module file, verify `//!` exists at top.
    Run `cargo build --workspace` succeeds (no syntax errors from docs).
  </verify>
  <done>
    All pub mod declarations in slicecore-engine, slicecore-slicer, slicecore-mesh, slicecore-math, and slicecore-geo have module-level doc comments.
  </done>
