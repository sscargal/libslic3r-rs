**JobDirError enum** (using thiserror):
```rust
#[derive(Debug, thiserror::Error)]
pub enum JobDirError {
    #[error("job directory '{}' is not empty (use --force to override)", .0.display())]
    NotEmpty(PathBuf),
    #[error("job directory '{}' is locked by PID {pid}", .path.display())]
    Locked { path: PathBuf, pid: u32 },
    #[error("I/O error: {0}")]
    Io(#[from] std::io::Error),
    #[error("JSON error: {0}")]
    Json(#[from] serde_json::Error),
}
```

pub struct JobDir {

**JobDir methods:**
- `pub fn create(path: PathBuf, force: bool) -> Result<Self, JobDirError>` -- mkdir -p via `std::fs::create_dir_all`, then acquire lock, then check emptiness (only .lock file should exist unless force=true). Lock FIRST, then check emptiness (per TOCTOU pitfall).
- `pub fn create_auto(base_dir: PathBuf, force: bool) -> Result<Self, JobDirError>` -- generates `Uuid::new_v4().to_string()` as directory name under base_dir, calls `Self::create`.
- `pub fn resolve_base(job_base: Option<&Path>) -> PathBuf` -- if job_base is Some, return it. Else check `std::env::var("SLICECORE_JOB_DIR")`. Else return `std::env::current_dir().unwrap_or_else(|_| PathBuf::from("."))`.
- `pub fn path(&self) -> &Path` -- returns &self.path
- `pub fn gcode_path(&self, model_name: &str) -> PathBuf` -- strips extension from model_name, appends ".gcode". E.g., "benchy.stl" -> "{dir}/benchy.gcode".
- `pub fn plate_gcode_path(&self) -> PathBuf` -- returns `self.path.join("plate.gcode")`
- `pub fn manifest_path(&self) -> PathBuf` -- returns `self.path.join("manifest.json")`
- `pub fn config_path(&self) -> PathBuf` -- returns `self.path.join("config.toml")`
- `pub fn log_path(&self) -> PathBuf` -- returns `self.path.join("slice.log")`
- `pub fn thumbnail_path(&self) -> PathBuf` -- returns `self.path.join("thumbnail.png")`
- `pub fn write_manifest(&self, manifest: &Manifest) -> Result<(), JobDirError>` -- serializes with `serde_json::to_string_pretty` and writes to manifest_path().
- `pub fn file_checksum(path: &Path) -> std::io::Result<String>` -- reads file, computes SHA-256, returns `format!("sha256:{:x}", hash)`. (Static/associated function, pub for reuse.)

**Locking functions (private):**
- `fn acquire_lock(dir: &Path) -> Result<std::fs::File, JobDirError>` -- checks if `.lock` exists, reads PID, checks if process alive with `process_exists()`, returns `Locked` error if alive, removes stale lock if dead. Creates new `.lock` with current PID (`std::process::id()`).
- `fn release_lock(dir: &Path)` -- removes `.lock` file, ignoring errors.
- `fn process_exists(pid: u32) -> bool` -- on unix: `unsafe { libc::kill(pid as i32, 0) == 0 }`. BUT to avoid adding libc dep, use `std::process::Command::new("kill").args(["-0", &pid.to_string()]).status()` and check success. On non-unix: always return true (conservative).

Note: Do NOT add a libc dependency. Use the `kill -0` command approach for process checking, or use `std::fs::read_link(format!("/proc/{pid}/exe"))` on Linux. Simplest cross-platform: use `Command::new("kill")` approach with `#[cfg(unix)]` and always-true fallback.

impl Drop for JobDir {
    fn drop(&mut self) {
        release_lock(&self.path);

}
