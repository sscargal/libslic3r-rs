<objective>
Implement accurate print time estimation using a trapezoid motion model and filament usage estimation (length, weight, cost).

Purpose: Print time estimation (GCODE-12) and filament usage estimation (GCODE-13) are essential user-facing features. The current naive time estimation (distance/feedrate) under-reports by 30-50%. The trapezoid model accounts for acceleration/deceleration ramps to produce more realistic estimates. Filament estimation enables cost tracking and material planning.

Note: The "within 15% of actual print time" accuracy target requires validation against real printer hardware and is outside the scope of automated testing. Automated tests verify the trapezoid model produces higher (more realistic) estimates than naive distance/feedrate calculation.

Output: Trapezoid-based time estimator, filament usage calculator, data included in SliceResult.
</objective>

<task type="auto">
  <name>Task 1: Trapezoid motion model for print time estimation</name>
  <files>
    crates/slicecore-engine/src/estimation.rs
    crates/slicecore-engine/src/lib.rs
  </files>
  <action>
1. Create estimation.rs with:

   - `trapezoid_time(distance: f64, entry_speed: f64, cruise_speed: f64, exit_speed: f64, acceleration: f64) -> f64` function implementing the trapezoidal velocity profile from the research doc:
     - Compute accel_dist = (cruise^2 - entry^2) / (2*accel)
     - Compute decel_dist = (cruise^2 - exit^2) / (2*accel)
     - If accel_dist + decel_dist > distance: triangular profile (cannot reach cruise speed)
     - Else: full trapezoidal profile with cruise phase
     - Return total time in seconds
     - Handle edge cases: distance <= 0 returns 0, acceleration <= 0 returns 0

   - `estimate_print_time(commands: &[GcodeCommand], print_acceleration: f64, travel_acceleration: f64) -> PrintTimeEstimate` that:
     - Iterates through GcodeCommand stream tracking current position (X, Y, Z) and feedrate
     - For each LinearMove/RapidMove, compute segment distance and apply trapezoid_time
     - Use print_acceleration for extrusion moves (has E parameter), travel_acceleration for travel moves (no E)
     - Simplification: use entry_speed = 0 and exit_speed = 0 for each segment (no junction speed lookahead -- this is sufficient for reasonable accuracy per research)
     - Actually, to improve accuracy: use min(feedrate, previous_feedrate) as entry speed for segments (simple lookahead approximation)
     - Add fixed time overhead per retraction (0.5s default) and per layer change (0.2s default)
     - Track total move time, retraction time, and travel time separately
     - For G2/G3 ArcMove commands: compute arc length from I/J/endpoint and treat as single move

   - `PrintTimeEstimate` struct:
     - `total_seconds: f64` -- total estimated print time
     - `move_time_seconds: f64` -- time spent extruding
     - `travel_time_seconds: f64` -- time spent traveling
     - `retraction_count: u32` -- number of retractions
     - Derive Debug, Clone, Serialize, Deserialize

2. Add `pub mod estimation;` to lib.rs.

3. Tests:
   - trapezoid_time with sufficient distance reaches cruise speed (verify known result)
   - trapezoid_time with short distance produces triangular profile (shorter time)
   - trapezoid_time with zero distance returns 0
   - estimate_print_time on a simple 3-move sequence produces reasonable time
   - estimate_print_time with retractions adds retraction overhead
   - Time estimate is greater than naive distance/feedrate (accounts for accel/decel)
  </action>
  <verify>
`cargo test -p slicecore-engine` -- all tests pass including estimation tests.
  </verify>
  <done>
Trapezoid motion model correctly computes segment times with acceleration ramps. Print time estimation processes GcodeCommand streams and produces PrintTimeEstimate with per-category breakdown.
  </done>

     - `length_mm: f64` -- total filament length in mm
     - `length_m: f64` -- total filament length in meters (convenience)
     - `weight_g: f64` -- total filament weight in grams
     - `cost: f64` -- total filament cost in configured currency
     - Derive Debug, Clone, Serialize, Deserialize

   - `estimate_filament_usage(commands: &[GcodeCommand], filament_diameter: f64, filament_density: f64, filament_cost_per_kg: f64) -> FilamentUsage` that:
     - Sums all E-values from LinearMove and ArcMoveCW/ArcMoveCCW commands (positive E only, skip retractions)
     - E-values are in mm of filament fed, so total_length = sum of E values
     - Weight = length * cross_section_area * density where cross_section = PI * (diameter/2)^2
