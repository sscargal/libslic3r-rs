**SC1: Firmware dialect validation**
- `test_phase_6_sc1_klipper_dialect`: Slice the calibration cube with `gcode_dialect: Klipper`. Validate the output with validate_gcode (must pass). Check that output contains:
  - "Klipper dialect" in start comment
  - "BED_MESH_CALIBRATE" (Klipper-specific)
  - "TURN_OFF_HEATERS" (Klipper-specific end command)
  - No M84 before TURN_OFF_HEATERS (Klipper ordering)
  If acceleration_enabled: check for SET_VELOCITY_LIMIT in output.

- `test_phase_6_sc1_reprap_dialect`: Slice with RepRapFirmware dialect. Validate passes. Contains:
  - "RepRapFirmware dialect" or "RepRap" in start comment
  - RepRap-specific end commands (M0 H1 or similar)

- `test_phase_6_sc1_bambu_dialect`: Slice with Bambu dialect. Validate passes. Contains:
  - "Bambu" in start comment
  - Bambu-specific sequence elements

**SC2: Multi-material tool changes**
- `test_phase_6_sc2_multi_material`: Configure multi_material with enabled=true, tool_count=2, two ToolConfig entries. Create a simple mesh and slice. Verify:
  - Output contains `T0` and/or `T1` tool change commands
  - Output contains purge tower extrusion moves (at purge_tower_position)
  - Valid G-code output (passes validation)
  Note: Full two-color model test requires modifier mesh tool assignment -- if that's too complex, test tool change sequence generation in isolation and verify the commands appear in output.

  If full engine integration is too complex for this test, test the modifier system in isolation: split_by_modifiers produces regions with different effective configs.

Add these tests inside engine.rs `#[cfg(test)] mod tests { ... }` using the existing calibration_cube_20mm() helper or similar fixtures.

**SC4: Print time and filament estimation accuracy**
- `test_phase_6_sc4_estimation`: Slice the calibration cube with default config. Verify:
  - SliceResult.time_estimate.total_seconds > 0
  - SliceResult.time_estimate.total_seconds > naive estimate (sum of distances / feedrates) -- proves acceleration modeling adds time
  - SliceResult.filament_usage.length_mm > 0
  - SliceResult.filament_usage.weight_g > 0
  - SliceResult.filament_usage.cost > 0
  - Filament length is reasonable for a 20mm cube (roughly 1000-5000mm depending on infill density)
  Note: The "within 15% of actual" criterion cannot be tested without real printer data. The test verifies the estimation machinery produces reasonable non-zero values and that trapezoid time > naive time.

- `test_phase_6_sc4_estimation_acceleration_impact`: Slice with two configs -- one with low acceleration (500 mm/s^2) and one with high acceleration (3000 mm/s^2). The low-acceleration estimate should be higher than the high-acceleration estimate (longer ramps).

**SC5: Arc fitting file size reduction**
- `test_phase_6_sc5_arc_fitting`: Create a test mesh with curved surfaces (or synthesize a circular toolpath -- a cylinder or sphere). Slice twice:
  1. With arc_fitting_enabled = false
  2. With arc_fitting_enabled = true
  Verify:
  - Arc-fitted output contains G2 and/or G3 commands
  - Arc-fitted output has fewer total commands than non-arc-fitted
  - Arc-fitted output is smaller in bytes
  - Both outputs pass G-code validation
  - Dimensional accuracy: The arc endpoints match the original endpoints (within tolerance)
  Note: The "20% reduction" target may not be achievable on a simple cube. If the calibration cube has no curves, synthesize a circular/cylindrical mesh for this test. A simple approach: create a cylinder by rotating a rectangle's edge around the Z axis.

If creating a curved mesh is too complex, test arc fitting in isolation: generate a series of G1 commands forming a semicircle, run fit_arcs, verify G2/G3 output and command count reduction.

Add tests to engine.rs `#[cfg(test)] mod tests`.
