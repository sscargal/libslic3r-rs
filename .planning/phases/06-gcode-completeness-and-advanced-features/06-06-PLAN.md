<objective>
Implement modifier meshes for region-specific setting overrides and polyhole conversion for dimensional accuracy of circular holes.

Purpose: Modifier meshes (ADV-03) allow users to apply different settings (infill density, wall count, speed) to specific regions of a model defined by a 3D volume. Polyhole conversion (ADV-07) replaces circular holes with regular polygons using the Nophead formula to improve dimensional accuracy -- a feature popularized by OrcaSlicer. Both features operate on layer contours before toolpath generation.

Output: Modifier mesh system with per-region config overrides, polyhole conversion with configurable enable/disable.
</objective>

<task type="auto">
  <name>Task 2: Polyhole conversion for dimensional accuracy</name>
  <files>
    crates/slicecore-engine/src/polyhole.rs
    crates/slicecore-engine/src/config.rs
    crates/slicecore-engine/src/engine.rs
    crates/slicecore-engine/src/lib.rs
  </files>
  <action>
1. Create polyhole.rs with:
   - `polyhole_sides(hole_diameter: f64, nozzle_diameter: f64) -> u32` using Nophead formula:
     - `ratio = nozzle_diameter / hole_diameter`
     - If ratio >= 1.0, return 3 (minimum polygon)
     - `sides = PI / acos(1.0 - ratio)`
     - Return sides.ceil().max(3.0) as u32

   - `polyhole_radius(desired_diameter: f64, sides: u32) -> f64`:
     - Inscribed circle radius of regular polygon: `R = desired_radius / cos(PI/n)`
     - This gives the circumradius that produces the desired inscribed circle diameter

   - `is_circular_hole(polygon: &ValidPolygon, min_diameter: f64) -> Option<((f64, f64), f64)>`:
     - Check if polygon is a hole (CW winding, negative area)
     - Fit a circle to the polygon vertices: compute centroid, then check if all vertices are within a tolerance of the mean radius (tolerance = 10% of mean radius)
     - If circular, return Some((center_x, center_y), diameter)
     - If not circular (irregular shape), return None
     - Skip if diameter < min_diameter

   - `convert_to_polyhole(center: (f64, f64), diameter: f64, nozzle_diameter: f64) -> ValidPolygon`:
     - Compute sides from polyhole_sides()
     - Compute circumradius from polyhole_radius()
     - Generate regular polygon vertices (CW winding for holes) at the computed radius
     - Return as ValidPolygon

   - `convert_polyholes(contours: &mut Vec<ValidPolygon>, nozzle_diameter: f64, min_diameter: f64)`:
     - Iterate through contours, identify circular holes, replace with polyholes
     - Only modify hole contours (CW winding), leave outer contours (CCW) unchanged

2. In config.rs, add to PrintConfig:
   - `polyhole_enabled: bool` -- default false
   - `polyhole_min_diameter: f64` -- default 1.0 mm (skip very small holes)
   Update Default impl.

3. In engine.rs, integrate polyhole conversion:
   - After slicing and before perimeter generation, if `config.polyhole_enabled` is true, run `convert_polyholes` on each layer's contours.

4. Add `pub mod polyhole;` to lib.rs.

5. Tests:
   - polyhole_sides(5.0, 0.4) returns a reasonable value (around 12)
   - polyhole_sides(1.0, 0.4) returns a smaller value (around 5)
   - polyhole_sides(0.3, 0.4) returns 3 (minimum)
   - polyhole_radius produces inscribed circle matching desired diameter
   - is_circular_hole correctly identifies a 16-vertex circle as circular
   - is_circular_hole rejects a square as non-circular
   - convert_to_polyhole produces correct vertex count and CW winding
   - TOML parsing of polyhole_enabled and polyhole_min_diameter
  </action>
