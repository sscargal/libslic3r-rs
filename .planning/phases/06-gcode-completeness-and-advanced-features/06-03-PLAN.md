<objective>
Implement TPMS-D (Schwarz Diamond) and TPMS-FK (Fischer-Koch S) infill patterns following the existing Gyroid implementation pattern.

Purpose: These two TPMS (Triply Periodic Minimal Surface) patterns are available in OrcaSlicer and provide additional structural options for functional parts. They reuse the same marching squares algorithm already proven with Gyroid, changing only the implicit surface function.

Output: Two new infill pattern modules, InfillPattern enum extended with TpmsD and TpmsFk variants, full dispatch wiring.
</objective>

  <name>Task 1: TPMS-D (Schwarz Diamond) infill pattern</name>
  <files>
    crates/slicecore-engine/src/infill/tpms_d.rs
    crates/slicecore-engine/src/infill/mod.rs
  </files>
  <action>
1. Create tpms_d.rs following the exact structure of gyroid.rs:
   - `schwarz_diamond(x: f64, y: f64, z: f64) -> f64` implicit surface function:
     `sin(x)*sin(y)*sin(z) + sin(x)*cos(y)*cos(z) + cos(x)*sin(y)*cos(z) + cos(x)*cos(y)*sin(z)`
   - `generate_tpms_d_infill(regions: &[ValidPolygon], density: f64, layer_z: f64, extrusion_width: f64) -> Vec<InfillLine>` function that:
     - Computes bounding box of regions (reuse compute_bounding_box from mod.rs)
     - Computes frequency from density and extrusion_width (same formula as Gyroid: `2*PI / (extrusion_width / density)`)
     - Evaluates Schwarz Diamond at grid points (grid step = extrusion_width for detail)
     - Runs marching squares to extract iso-contour line segments at the 0-level
     - Clips resulting segments against the infill regions using point-in-polygon
     - Returns clipped InfillLines
   - Copy/adapt the marching squares implementation from gyroid.rs (or factor it out into a shared helper -- judge based on whether gyroid.rs has it inline or factored). If gyroid.rs has the marching squares code inline, duplicate it into tpms_d.rs rather than refactoring gyroid.rs in this plan.

2. In mod.rs:
   - Add `pub mod tpms_d;`
   - Add `TpmsD` variant to InfillPattern enum
   - Add dispatch arm in `generate_infill()` for `InfillPattern::TpmsD` calling `tpms_d::generate_tpms_d_infill()`

3. Tests in tpms_d.rs:
   - schwarz_diamond evaluates to 0 at known zero-crossing points
   - generate_tpms_d_infill with a 20mm square region at 20% density produces non-empty infill lines
   - Lines are clipped within the region bounding box
  </action>
  <verify>
`cargo test -p slicecore-engine` -- all tests pass including TPMS-D tests.
  </verify>
  <done>
TPMS-D infill pattern generates Schwarz Diamond contours, dispatches via InfillPattern::TpmsD, and produces valid infill lines clipped to region boundaries.
  </done>
</task>
