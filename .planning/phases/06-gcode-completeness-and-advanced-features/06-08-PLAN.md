<objective>
Implement pressure advance calibration pattern generation that produces a test print with varying PA values.

Purpose: Pressure advance (ADV-06) requires empirical tuning per printer/filament combination. A calibration pattern prints alternating slow/fast sections at incrementally increasing PA values, allowing users to visually identify the optimal setting. This is a standalone G-code generator (not part of the normal slicing pipeline).

Output: PA calibration pattern generator producing complete G-code with dial-specific PA commands.
</objective>

  <action>
1. In config.rs, define:
   - `PaCalibrationConfig` struct:
     - `pa_start: f64` (default 0.0)
     - `pa_end: f64` (default 0.1)
     - `pa_step: f64` (default 0.005)
     - `slow_speed: f64` (default 20.0 mm/s -- reveals PA artifacts)
     - `fast_speed: f64` (default 100.0 mm/s -- reveals PA artifacts)
     - `line_width: f64` (default 0.5 mm)
     - `layer_height: f64` (default 0.2 mm)
     - `bed_center_x: f64` (default 110.0)
     - `bed_center_y: f64` (default 110.0)
     - `pattern_width: f64` (default 100.0 mm)
     - `nozzle_temp: f64` (default 200.0)
     - `bed_temp: f64` (default 60.0)
     - `filament_diameter: f64` (default 1.75)
     Derive Debug, Clone, Default, Serialize, Deserialize with `#[serde(default)]`.

2. Create calibration.rs with:
   - `generate_pa_calibration(config: &PaCalibrationConfig, dialect: GcodeDialect) -> Vec<u8>` that:
     - Generates a complete G-code file (start code + pattern + end code)
     - Start: home, heat bed+nozzle, prime line
     - For each PA step from pa_start to pa_end:
       - Emit dialect-specific PA command (using format_pressure_advance from gcode-io dialect module)
       - Print a line with alternating slow/fast sections:
         - 20mm at slow_speed (stable baseline)
         - 40mm at fast_speed (reveals over/under extrusion from PA mismatch)
         - 20mm at slow_speed (return to baseline)
       - Each line at a Y offset from the previous (line_width * 2 spacing)
       - Add a label comment with the PA value: `; PA = {value:.3}`
     - End: cool down, disable steppers
     - Use GcodeWriter for consistent output formatting with the specified dialect
     - Compute E-values using standard extrusion cross-section formula

   - `generate_pa_calibration_gcode(config: &PaCalibrationConfig, dialect: GcodeDialect) -> String`:
     - Convenience wrapper returning String instead of Vec<u8>

3. Add `pub mod calibration;` to lib.rs.

4. Tests:
   - Default PaCalibrationConfig has expected values
   - generate_pa_calibration produces non-empty output
   - Output contains PA commands at each step value
   - Output for Klipper dialect contains SET_PRESSURE_ADVANCE
   - Output for Marlin dialect contains M900 K
   - Number of PA command lines matches (pa_end - pa_start) / pa_step + 1
   - Output starts with start G-code and ends with end G-code
   - E-values are positive and non-zero
  </action>
