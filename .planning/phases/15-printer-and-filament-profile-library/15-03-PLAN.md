  <action>
Create `crates/slicecore-engine/tests/integration_profile_library.rs` with the following test structure:

1. **Synthetic tests (always run):**

   a. `test_batch_convert_empty_dir` -- Create a temp directory with no JSON files. Call `batch_convert_profiles`. Verify converted=0, skipped=0, errors empty.

   b. `test_batch_convert_skips_non_instantiated` -- Create temp dir with a single JSON profile where `"instantiation": "false"`. Call `batch_convert_profiles`. Verify converted=0, skipped=1.

   c. `test_batch_convert_single_profile` -- Create temp dir structure: `TestVendor/filament/Test_PLA.json` with `"instantiation": "true"`, `"type": "filament"`, `"name": "Test PLA"`, `"nozzle_temperature_initial_layer": ["210"]`, `"bed_temperature": ["60"]`. Call `batch_convert_profiles`. Verify:
      - converted=1
      - Output TOML file exists at `output/TestVendor/filament/Test_PLA.toml`
      - TOML file is valid (can be parsed by `PrintConfig::from_file`)
      - PrintConfig has nozzle_temp = 210.0 (not default 200.0)

   d. `test_batch_convert_inheritance` -- Create temp dir with two JSON files:
      - `TestVendor/filament/base_pla.json`: `"instantiation": "false"`, `"name": "base_pla"`, `"nozzle_temperature_initial_layer": ["215"]`, `"bed_temperature": ["60"]`
      - `TestVendor/filament/PLA_Variant.json`: `"instantiation": "true"`, `"name": "PLA Variant"`, `"inherits": "base_pla"`, `"flow_ratio": ["0.95"]`

      Call `batch_convert_profiles`. Verify:
      - converted=1 (only the instantiated variant)
      - The converted TOML contains nozzle_temp = 215.0 (inherited from parent)
      - The converted TOML contains flow_ratio related value from child override

   e. `test_index_entry_metadata` -- Create a synthetic profile named "0.20mm Standard @TestVendor X1" with `"type": "process"`, `"instantiation": "true"`. Convert and verify the index entry has:
      - profile_type = "process"
      - layer_height = Some(0.20)
      - quality = Some("Standard")
      - printer_model contains "TestVendor X1" or similar extraction

   f. `test_index_entry_filament_material` -- Create synthetic filament profile named "Generic PETG @TestVendor" with `"type": "filament"`, `"instantiation": "true"`. Convert and verify index entry has `material = Some("PETG")`.

   g. `test_sanitize_filenames_no_collision` -- Create two profiles with similar names (e.g., "PLA @BBL A1" and "PLA @BBL A1 0.2 nozzle"). Verify they produce distinct output files (no overwrite).

   h. `test_batch_convert_error_recovery` -- Create one valid JSON profile and one malformed JSON file (invalid JSON syntax). Call `batch_convert_profiles`. Verify:
      - converted >= 1 (valid profile succeeded)
      - errors has length >= 1 (malformed file recorded)
      - Batch did not abort

2. **Real profile tests (gated with `#[ignore]`):**

   i. `test_real_orcaslicer_batch_convert` -- Point `batch_convert_profiles` at actual `/home/steve/slicer-analysis/OrcaSlicer/resources/profiles/` with output to a temp dir. Verify:
      - converted > 100 (substantial number of profiles)
      - Index has correct structure (version=1, profiles non-empty)
      - At least 5 vendors present in index entries
      - At least one filament, process, and machine profile type present

   j. `test_real_profile_toml_loadable` -- Take 10 random converted TOML files from the real batch conversion and verify each can be loaded via `PrintConfig::from_file`. This proves round-trip fidelity at scale.

   k. `test_real_inheritance_produces_richer_profiles` -- For a known BBL PLA filament profile, compare the import result with and without inheritance resolution. The resolved version should have more mapped_fields than the raw child-only import.

Each test should use `tempfile::TempDir` for output directories. Tests touching real slicer-analysis data should be `#[ignore]` with a comment explaining the external dependency.
