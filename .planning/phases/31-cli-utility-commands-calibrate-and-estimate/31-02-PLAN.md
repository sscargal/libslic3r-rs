<task type="auto">
  <name>Task 1: Extend analyze-gcode with cost flags and model estimation</name>
  <files>crates/slicecore-cli/src/main.rs, crates/slicecore-cli/src/analysis_display.rs</files>
  <action>
    1. Add cost-related flags to the `AnalyzeGcode` variant in Commands enum:
       - `#[arg(long)] filament_price: Option<f64>` -- Filament price per kg
       - `#[arg(long)] printer_watts: Option<f64>` -- Printer power in watts
       - `#[arg(long)] electricity_rate: Option<f64>` -- Electricity rate per kWh
       - `#[arg(long)] printer_cost: Option<f64>` -- Printer purchase cost
       - `#[arg(long)] expected_hours: Option<f64>` -- Expected printer lifetime hours
       - `#[arg(long)] labor_rate: Option<f64>` -- Labor hourly rate
       - `#[arg(long, default_value = "5.0")] setup_time: f64` -- Setup time minutes
       - `#[arg(long)] markdown: bool` -- Output as markdown table
       - `#[arg(long)] model: bool` -- Treat input as model file (STL/3MF) for rough estimation

    2. Update `cmd_analyze_gcode` function to:
       - If `model` flag is set: load mesh via slicecore_fileio, compute MeshStats for volume, call `volume_estimate()`, build CostInputs from the rough estimate, call `compute_cost()`, display results with rough estimate disclaimer
       - If not model: after existing G-code analysis, build CostInputs from the analysis results (filament weight from statistics, print time from estimation) plus CLI cost flags, call `compute_cost()`, append cost section to output
       - Pass all new cost flags through to the CostInputs struct

    3. In analysis_display.rs add:
       - `pub fn display_cost_table(estimate: &CostEstimate, no_color: bool)` -- comfy-table with rows for each cost component. Show "N/A" with hint text for None values, format Some values as currency with 2 decimal places. Add total row at bottom
       - `pub fn display_cost_json(estimate: &CostEstimate)` -- serialize CostEstimate to JSON
       - `pub fn display_cost_csv(estimate: &CostEstimate)` -- CSV with component,amount,hint columns
       - `pub fn display_cost_markdown(estimate: &CostEstimate)` -- markdown table format
       - `pub fn display_volume_estimate(estimate: &VolumeEstimate, no_color: bool)` -- table showing rough filament, weight, time with disclaimer

    4. Wire display functions: after computing CostEstimate, call appropriate display based on json/csv/markdown/default flags.
  </action>
  <verify>
    <automated>cargo build -p slicecore-cli && cargo test -p slicecore-cli --lib 2>&1 | tail -10</automated>
  </verify>
  <done>analyze-gcode accepts cost flags and displays cost breakdown. --model flag provides rough volume-based estimation for mesh files.</done>
</task>
