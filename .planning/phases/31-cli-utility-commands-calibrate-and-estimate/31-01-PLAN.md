<task type="auto" tdd="true">
  <name>Task 1: Cost model module and MachineConfig watts field</name>
  <files>crates/slicecore-engine/src/cost_model.rs, crates/slicecore-engine/src/config.rs, crates/slicecore-engine/src/lib.rs</files>
  <behavior>
    - compute_cost with all inputs returns correct total (filament_weight * price/1000 + hours * watts/1000 * rate + printer_cost/expected_hours * hours + labor_rate * setup_mins/60)
    - compute_cost with only filament_price returns filament_cost=Some, others=None, total=filament_cost
    - compute_cost with zero expected_hours returns depreciation_cost=None (no division by zero)
    - compute_cost with no optional inputs returns all None costs and 4 missing_hints
    - volume_estimate returns rough filament length and weight from mesh volume (within order of magnitude for a 20mm cube)
    - MachineConfig.watts defaults to 0.0 and round-trips through TOML serialization
  </behavior>
  <action>
    1. Add `pub watts: f64` to `MachineConfig` in config.rs with doc comment "/// Printer power consumption in watts (for cost estimation, 0 = not set)." Default to 0.0 in Default impl.

    2. Create `crates/slicecore-engine/src/cost_model.rs` with:
       - `CostInputs` struct with `filament_weight_g: f64`, `print_time_seconds: f64`, plus Option fields for `filament_price_per_kg`, `electricity_rate`, `printer_watts`, `printer_cost`, `expected_hours`, `labor_rate`, `setup_time_minutes`
       - `CostEstimate` struct with `filament_cost: Option<f64>`, `electricity_cost: Option<f64>`, `depreciation_cost: Option<f64>`, `labor_cost: Option<f64>`, `total_cost: Option<f64>`, `missing_hints: Vec<String>`
       - `pub fn compute_cost(inputs: &CostInputs) -> CostEstimate` - compute each cost component independently; treat 0.0 and None as "not provided" for optional fields; build missing_hints vec with specific hints like "provide --filament-price to calculate filament cost"
       - `VolumeEstimate` struct with `filament_length_mm: f64`, `filament_weight_g: f64`, `rough_time_seconds: f64`, `disclaimer: String`
       - `pub fn volume_estimate(volume_mm3: f64, filament_diameter: f64, filament_density: f64) -> VolumeEstimate` - use formula: effective_volume = volume * 0.50 (combined infill+shell factor), filament_length = effective_volume / (PI * (d/2)^2), weight via filament_mm_to_grams, rough_time = filament_length / 40.0. Set disclaimer to "+/- 30-50% accuracy"

    3. Add `pub mod cost_model;` to `crates/slicecore-engine/src/lib.rs`

    Unit tests in cost_model.rs covering all behavior cases above.
  </action>
  <verify>
    <automated>cargo test -p slicecore-engine cost_model -- --nocapture</automated>
  </verify>
  <done>CostEstimate computes correctly with progressive disclosure. MachineConfig has watts field. Volume estimation produces reasonable rough values.</done>
</task>

       - `RetractionParams { start_distance: f64, end_distance: f64, step: f64, start_speed: f64, end_speed: f64 }` with `from_filament` deriving from filament_retraction_length +/- range
       - `FlowParams { baseline_multiplier: f64, step: f64, steps: usize }` with `from_config(config: &PrintConfig)` using extrusion_multiplier

       - `pub fn validate_bed_fit(width: f64, depth: f64, height: f64, machine: &MachineConfig) -> Result<(), String>` - check model fits with 10mm margin, return descriptive error with bed size and model size if too large

       - Follow the exact pattern from csg_command.rs for module structure

If no profiles specified, return PrintConfig::default()
       - `pub fn write_instructions(path: &std::path::Path, title: &str, sections: &[(String, String)]) -> Result<(), std::io::Error>` - write markdown instruction file with title, date, and key-value sections
       - `pub fn format_calibration_header(test_name: &str, params: &[(&str, String)]) -> String` - generate G-code header comment block with SliceCore version and calibration parameters
