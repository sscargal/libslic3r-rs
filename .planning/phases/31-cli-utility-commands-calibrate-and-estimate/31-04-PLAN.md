<task type="auto">
  <name>Task 1: Flow rate calibration command</name>
  <files>crates/slicecore-engine/src/calibrate.rs, crates/slicecore-cli/src/calibrate/flow.rs, crates/slicecore-cli/src/calibrate/mod.rs</files>
  <action>
    1. In calibrate.rs add `pub fn generate_flow_mesh(params: &FlowParams) -> TriangleMesh`:
       - Generate a simple hollow box (cube with thin walls) -- 40mm x 40mm x 25mm
       - Use primitive_box for outer shell, subtract inner box to create ~2mm wall thickness
       - Actually, since boolean difference may be complex, generate as a solid box. The slicer will create walls based on perimeter settings. The user measures wall thickness with calipers to verify extrusion multiplier
       - Multiple steps: generate N boxes side by side, each sliced with different extrusion multiplier. But since we slice as one model, we can't vary extrusion per box. Instead: generate ONE hollow cube, slice it, and the instructions tell the user to measure wall thickness and calculate the correction factor
       - Flow steps approach: generate a stepped tower (like temp tower) where each step represents a different flow rate. Post-process to inject M221 (flow rate override) commands at Z boundaries. M221 S{percent} sets flow rate as percentage
       - Number of blocks: params.steps (default 5)
       - Block dimensions: 30mm x 30mm x 5mm per step (thin walls visible for measurement)

    2. In calibrate.rs add `pub fn inject_flow_changes(commands: Vec<GcodeCommand>, schedule: &[(f64, f64)]) -> Vec<GcodeCommand>`:
       - At Z height boundaries, inject M221 S{flow_percent} commands
       - schedule is Vec of (z_height, flow_percent) where flow_percent = baseline * step_multiplier * 100
       - Also inject comments: "; Flow rate: {percent}% (multiplier: {multiplier})"
       - Note: M221 is a GcodeCommand::Raw or GcodeCommand::Comment -- check existing command types. If no M221 variant exists, use GcodeCommand::Raw format

    3. Create `crates/slicecore-cli/src/calibrate/flow.rs`:
       - `pub fn cmd_flow(args: FlowArgs) -> Result<(), Box<dyn std::error::Error>>`
       - Pipeline: resolve config -> build FlowParams from extrusion_multiplier +/- range -> generate mesh -> validate_bed_fit -> slice -> inject M221 flow changes at Z boundaries -> write G-code + instructions
       - Instructions explain: measure wall thickness of each section with calipers, calculate: correct_multiplier = expected_thickness / measured_thickness * section_multiplier
       - Summary: "Generated flow calibration: {start}% to {end}% in {step}% steps"

    4. Wire in mod.rs: replace Flow stub, add `mod flow;`

    Unit test: test_generate_flow_mesh produces valid mesh, test_inject_flow_changes inserts at correct Z.
  </action>

  </verify>
  <done>flow command generates stepped tower, injects M221 flow rate overrides, writes G-code + instructions with measurement guidance.</done>

  <action>
    1. In calibrate.rs add `pub fn generate_first_layer_mesh(params: &FirstLayerParams, bed_x: f64, bed_y: f64) -> TriangleMesh`:
       - Generate a flat plate (single layer height ~0.3mm) covering coverage_percent of bed area
       - For Grid pattern: generate a grid of thin rectangles (2mm wide, spaced 10mm apart) covering the bed. Use primitive_box for each strip, union them together
       - For Lines pattern: parallel strips across bed X axis
       - For Concentric pattern: nested rectangular frames
       - Use coverage_percent (default 80%) to determine how much of bed to cover, centering the pattern
       - Keep mesh simple -- a large flat plate with the layer height is sufficient. The slicer's infill will create the pattern. Use a single flat box covering 80% of the bed, 0.3mm tall
       - Actually simplest effective approach: generate a flat box (bed_x * 0.8) x (bed_y * 0.8) x 0.3mm. The slicer creates the pattern via infill settings. This tests Z-offset, adhesion, and flow across the bed

    2. Create `crates/slicecore-cli/src/calibrate/first_layer.rs`:
       - `pub fn cmd_first_layer(args: FirstLayerArgs) -> Result<(), Box<dyn std::error::Error>>`
       - Pipeline: resolve config -> build FirstLayerParams -> generate flat mesh at 80% bed coverage -> validate (always fits by construction) -> slice with layer_height set to first_layer_height -> write G-code + instructions
       - Override config: set infill to 100% rectilinear for solid first layer, top/bottom layers to 1, perimeters to 2
       - Instructions explain: what to look for (adhesion uniformity, elephant's foot, gaps between lines, over-squish), how to adjust Z-offset (too high: gaps between lines, too low: over-squish/rough surface)
       - Summary: "Generated first layer test: {width}mm x {depth}mm ({coverage}% bed coverage)"

    3. Wire in mod.rs: replace FirstLayer stub, add `mod first_layer;`

    Unit test: test_generate_first_layer_mesh produces flat mesh with correct approximate dimensions.
