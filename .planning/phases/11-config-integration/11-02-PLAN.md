   ```rust
   /// Finds connected components (disjoint sub-meshes) by vertex connectivity.
   ///
   /// Two triangles are connected if they share at least one vertex.
   /// Returns a list of vertex index sets, one per connected component.
   /// Each set contains the vertex indices belonging to that component.
   ///
   /// # Returns
   ///
   /// Vec of `(vertices, indices)` pairs where each pair represents a disjoint
   /// component. `vertices` are the vertex indices (into the original mesh) and
   /// `indices` are the triangle indices belonging to that component.
   pub fn connected_components(&self) -> Vec<(Vec<u32>, Vec<usize>)> {
   ```

2. **Algorithm:**
   - Initialize a `parent` array of size `self.vertices.len()` where `parent[i] = i`
   - For each triangle in `self.indices`, union the three vertex indices (union the first pair, then union the result with the third)
   - After processing all triangles, collect components by root
   - Group triangles by the root of any of their vertices (all three share the same root)
   - Return `Vec<(Vec<u32>, Vec<usize>)>` where each tuple is (vertex indices, triangle indices)

3. **Add unit tests:**
   - Single cube: 1 component
   - Two separated cubes (manually constructed): 2 components
   - Single triangle: 1 component

The union-find should use path compression and union by rank for efficiency, though for typical mesh sizes even a simple version is fine.

  <name>Task 2: Wire sequential check into Engine pipeline</name>
  <files>crates/slicecore-engine/src/engine.rs</files>
  <action>
Add a sequential printing check to `slice_to_writer_with_events()` after mesh validation and before mesh slicing:

1. **Insert sequential check block** after `if mesh.triangle_count() == 0 { return Err(EngineError::EmptyMesh); }` and before step 1 (slice mesh into layers):

   ```rust
   // 0. Sequential printing check (before slicing).
   if self.config.sequential.enabled {
       let components = mesh.connected_components();
       if components.len() <= 1 {
           // Single object -- sequential mode has no effect
           if let Some(bus) = event_bus {
               bus.emit(&crate::event::SliceEvent::Warning {
                   message: "Sequential printing enabled but mesh has only one object. \
                             Sequential mode has no effect for single objects.".to_string(),
                   layer: None,
               });
           }
       } else {
           // Multiple objects -- compute bounds and validate
           let object_bounds: Vec<crate::sequential::ObjectBounds> = components
               .iter()
               .enumerate()
               .map(|(comp_idx, (vert_indices, _tri_indices))| {
                   let mut min_x = f64::MAX;
                   let mut max_x = f64::MIN;
                   let mut min_y = f64::MAX;
                   let mut max_y = f64::MIN;
                   let mut max_z = f64::MIN;
                   let vertices = mesh.vertices();
                   for &vi in vert_indices {
                       let v = vertices[vi as usize];
                       if v.x < min_x { min_x = v.x; }
                       if v.x > max_x { max_x = v.x; }
                       if v.y < min_y { min_y = v.y; }
                       if v.y > max_y { max_y = v.y; }
                       if v.z > max_z { max_z = v.z; }
                   }
                   crate::sequential::ObjectBounds {
                       min_x, max_x, min_y, max_y, max_z,
                       object_index: comp_idx,
                   }
               })
               .collect();

           // Run collision detection and ordering.
           let _plan = crate::sequential::plan_sequential_print(
               &object_bounds,
               &self.config,
           )?;

           // Emit info about sequential order
           if let Some(bus) = event_bus {
               bus.emit(&crate::event::SliceEvent::StageChanged {
                   stage: format!("sequential_validation"),
                   progress: 0.0,
               });
               bus.emit(&crate::event::SliceEvent::Warning {
                   message: format!(
                       "Sequential printing validated: {} objects, no collisions detected",
                       components.len()
                   ),
                   layer: None,
               });
           }

           // Note: Full object-by-object slicing (slicing each component separately
           // and inserting safe-Z travels between them) requires API changes beyond
           // V1 scope. The current implementation validates that sequential printing
           // is feasible. The mesh is still sliced as one piece.
       }
   }
   ```

2. **Import note:** The `ObjectBounds` type is already available via `crate::sequential::ObjectBounds` (it is a pub type in the sequential module, re-exported in lib.rs).

3. **Run all tests** to ensure no regressions.
