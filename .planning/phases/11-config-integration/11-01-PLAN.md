<objective>
Wire plugin_dir config field into Engine auto-loading and update CLI to prevent double-loading.

Purpose: When a user sets `plugin_dir = "/path/to/plugins"` in their config TOML and uses the Engine as a library (not via CLI), plugins are currently never loaded. This plan makes the Engine self-sufficient by auto-loading plugins from config during construction.

Output: Engine auto-loads plugins from config.plugin_dir, stores warnings for deferred emission, CLI avoids double-loading.
</objective>

<task type="auto">
  <name>Task 1: Add plugin auto-loading infrastructure to Engine</name>
  <files>crates/slicecore-engine/src/engine.rs</files>
  <action>
Modify the `Engine` struct and constructor to support automatic plugin loading from `config.plugin_dir`:

1. **Add `startup_warnings` field** to `Engine` struct:
   ```rust
   pub struct Engine {
       config: PrintConfig,
       #[cfg(feature = "plugins")]
       plugin_registry: Option<slicecore_plugin::PluginRegistry>,
       startup_warnings: Vec<String>,
   }
   ```

2. **Update `Engine::new()`** to call auto_load_plugins after construction:
   ```rust
   pub fn new(config: PrintConfig) -> Self {
       let mut engine = Self {
           config,
           #[cfg(feature = "plugins")]
           plugin_registry: None,
           startup_warnings: Vec::new(),
       };
       #[cfg(feature = "plugins")]
       engine.auto_load_plugins();
       engine
   }
   ```

3. **Add `auto_load_plugins` private method** (cfg-gated behind `plugins` feature):
   - Check if `self.config.plugin_dir` is `Some(ref dir)`
   - Create a `slicecore_plugin::PluginRegistry::new()`
   - Call `registry.discover_and_load(Path::new(dir))`
   - On `Ok(loaded)` where `loaded.is_empty()`: push warning to `self.startup_warnings` saying "plugin_dir '{}' is set but contains no valid plugins"
   - On `Ok(loaded)` where `!loaded.is_empty()`: set `self.plugin_registry = Some(registry)`
   - On `Err(e)`: push warning to `self.startup_warnings` saying "Failed to load plugins from '{}': {}"

4. **Add `has_plugin_registry()` public accessor** (cfg-gated):
   ```rust
   #[cfg(feature = "plugins")]
   pub fn has_plugin_registry(&self) -> bool {
       self.plugin_registry.is_some()
   }
   ```

5. **Add `startup_warnings()` public accessor:**
   ```rust
   pub fn startup_warnings(&self) -> &[String] {
       &self.startup_warnings
   }
   ```

6. **Emit startup warnings in `slice_to_writer_with_events()`**: At the beginning of the method (after mesh validation, before slicing), if `event_bus` is `Some` and `self.startup_warnings` is non-empty, emit each warning as `SliceEvent::Warning { message, layer: None }`.

7. **Update `with_plugin_registry()`** to clear startup_warnings related to plugin loading (since manual registry replaces auto-loaded one):
   The existing method sets `self.plugin_registry = Some(registry)` and returns `self`. This is fine as-is -- the manual registry replaces whatever was auto-loaded.

Ensure all new code referencing `slicecore_plugin` is wrapped in `#[cfg(feature = "plugins")]`.

Run `cargo check` and `cargo check --features plugins` to verify both feature configurations compile.

2. **Modify plugin loading block** (currently lines ~233-252): Change the conditional to:
   - If `--plugin-dir` CLI flag was explicitly provided (i.e., the `plugin_dir` function argument is `Some`): always do manual loading and call `engine.with_plugin_registry()`. This override is intentional.
   - If no CLI flag but config has `plugin_dir`: check `engine.has_plugin_registry()`. If already loaded (Engine auto-loaded), skip manual loading and print info about auto-loaded plugins. If not loaded (shouldn't happen, but defensive), proceed with manual loading.

   Pseudocode:
   ```rust
   let cli_plugin_dir_provided = plugin_dir.is_some();

   if cli_plugin_dir_provided {
       // CLI flag override -- always load from specified dir
       if let Some(ref dir) = effective_plugin_dir {
           let mut registry = PluginRegistry::new();
           match registry.discover_and_load(std::path::Path::new(dir)) {
               Ok(loaded) => { /* existing print logic */ }
               Err(e) => { /* existing warning logic */ }
           }
           engine = engine.with_plugin_registry(registry);
       }
   } else if engine.has_plugin_registry() {
       // Engine auto-loaded from config.plugin_dir -- report what was loaded
       // (startup_warnings will be emitted during slicing via EventBus)
       eprintln!("Plugins auto-loaded from config plugin_dir");
   } else if let Some(ref dir) = effective_plugin_dir {
       // Fallback: config had plugin_dir but engine didn't load (shouldn't happen)
       let mut registry = PluginRegistry::new();
       match registry.discover_and_load(std::path::Path::new(dir)) {
           Ok(loaded) => { /* same as above */ }
           Err(e) => { /* same as above */ }
       }
       engine = engine.with_plugin_registry(registry);
   }
   ```
