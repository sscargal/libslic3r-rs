  <action>
This task runs the compiled CLI to generate the PrusaSlicer profile library. No source code is modified -- it exercises the code built in Plan 16-01 to produce the output artifacts.

Run the CLI to generate the PrusaSlicer profile library:

```bash
cargo run -p slicecore-cli -- import-profiles \
  --source-dir /home/steve/slicer-analysis/PrusaSlicer/resources/profiles \
  --source-name prusaslicer \
  --output-dir profiles
```

This will:
1. Parse all 33 FFF vendor INI files (skipping 2 SLA files)
2. Resolve inheritance for all concrete profiles
3. Convert to TOML and write to profiles/prusaslicer/vendor/type/
4. Merge PrusaSlicer entries into the existing index.json (preserving OrcaSlicer entries)

Verify the output:
- Check profiles/prusaslicer/ directory exists with vendor subdirectories
- Verify index.json contains entries from both "orcaslicer" and "prusaslicer" sources
- Count converted profiles (expect ~5000-9500 depending on mapping coverage)
- Check error count is reasonable (<5% of total)

If OrcaSlicer profiles don't exist yet (profiles/ is in .gitignore), first regenerate them:
```bash
cargo run -p slicecore-cli -- import-profiles \
  --source-dir /home/steve/slicer-analysis/OrcaSlicer/resources/profiles \
  --source-name orcaslicer \
  --output-dir profiles
```

Then run the PrusaSlicer import, and verify the merged index contains both sources.

Verify CLI discovery commands work with combined library:
```bash
cargo run -p slicecore-cli -- list-profiles --vendors --profiles-dir profiles
cargo run -p slicecore-cli -- search-profiles "PLA" --profiles-dir profiles | head -20
cargo run -p slicecore-cli -- search-profiles "prusaslicer" --profiles-dir profiles | head -20
```

- PrusaSlicer profiles have at least 3000 entries in the index

Create `crates/slicecore-engine/tests/integration_profile_library_ini.rs` with integration tests mirroring the Phase 15 pattern (8 synthetic + 3 real/ignored).

**Synthetic tests (always run, use tempfile TempDir):**

1. `test_parse_ini_sections` - Parse a multi-section INI string with vendor, printer_model, print, filament, printer sections. Verify section_type, name, is_abstract, field count.

2. `test_ini_inheritance_single_parent` - Create sections: `[print:*common*]` with base fields, `[print:0.20mm NORMAL]` with `inherits = *common*`. Verify child inherits parent's fill_density, perimeter_speed, etc., and child's own overrides are applied.

3. `test_ini_inheritance_multi_parent` - Create sections: `[print:*0.15mm*]` and `[print:*soluble_support*]` and `[print:0.15mm OPTIMAL SOLUBLE FULL]` with `inherits = *0.15mm*; *soluble_support*`. Verify left-to-right merge order (soluble_support overrides 0.15mm, child overrides both).

4. `test_prusaslicer_field_mapping_process` - Test import_prusaslicer_ini_profile with resolved fields: layer_height=0.2, perimeters=3, fill_density=15%, fill_pattern=cubic, first_layer_speed=20, seam_position=aligned. Verify PrintConfig values: wall_count=3, infill_density=0.15, infill_pattern=Cubic, first_layer_speed=20.0.

5. `test_prusaslicer_field_mapping_filament` - Test with: temperature=210, first_layer_temperature=215, bed_temperature=60, filament_density=1.24, filament_diameter=1.75, extrusion_multiplier=1. Verify: nozzle_temp=210, first_layer_nozzle_temp=215, bed_temp=60, filament_density=1.24.

6. `test_prusaslicer_field_mapping_machine` - Test with: nozzle_diameter=0.4,0.4 (multi-extruder), retract_length=0.8, retract_speed=35, retract_lift=0.2, gcode_flavor=marlin. Verify: nozzle_diameter=0.4 (first), retract_z_hop=0.2, gcode_dialect=Marlin.

7. `test_batch_convert_prusaslicer_synthetic` - Create a temp directory with a synthetic .ini file containing vendor section + abstract print + concrete print + abstract filament + concrete filament. Run batch_convert_prusaslicer_profiles. Verify: converted count, skipped abstract count, output TOML files exist, index entries have correct vendor/type.

8. `test_write_merged_index` - Create an initial index with 2 OrcaSlicer entries, write it. Create a PrusaSlicer index with 2 entries, call write_merged_index. Load the result and verify all 4 entries exist with correct sources.

**Real-data tests (gated with #[ignore], require slicer-analysis directory):**

9. `test_real_prusaresearch_conversion` - Load PrusaResearch.ini, parse, verify >5000 concrete sections found, convert a subset, verify TOML output is valid.

10. `test_real_small_vendor_conversion` - Load a small vendor file (e.g., Anker.ini), batch convert entire file, verify all concrete profiles converted without errors.

11. `test_real_combined_index` - After converting both OrcaSlicer and PrusaSlicer profiles, verify merged index contains entries from both sources with >6000 total profiles.

Add `tempfile` dev-dependency to slicecore-engine Cargo.toml if not already present (it was added in Phase 15-03).
