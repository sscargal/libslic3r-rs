  <action>
Create `crates/slicecore-engine/src/profile_import_ini.rs` with the INI parsing and inheritance resolution components:

**1. IniSection struct and INI parser (`parse_prusaslicer_ini`)**
- Parse `[type:name]` section headers (type = vendor/printer_model/print/filament/printer)
- Collect `key = value` pairs per section
- Skip comment lines starting with `#` or `;`
- Handle `##` section-level comments (treat as regular comments)
- Detect abstract profiles: name starts AND ends with `*` (e.g., `*common*`, `*PLA*`)
- Preserve `\n` escape sequences in values (G-code fields) -- do NOT interpret as newlines
- Return `Vec<IniSection>` where IniSection has: section_type, name, fields (HashMap), is_abstract

**2. Multi-parent inheritance resolver (`resolve_ini_inheritance`)**
- Parse `inherits` field: split on `; ` (semicolon-space) to get parent names
- Build a lookup: `HashMap<(String, String), usize>` mapping (section_type, name) -> index in Vec<IniSection>
- For each parent (left-to-right): recursively resolve ITS inheritance, merge fields
- Overlay child fields on top of merged parents
- MAX_INHERITANCE_DEPTH = 10 guard against cycles
- All inheritance scoped within the single vendor INI file (no cross-file resolution)
- Return flattened `HashMap<String, String>` of resolved key-value pairs

**3. Unit tests for parser and inheritance (in the same file, `#[cfg(test)]` module)**
- Test INI section parsing with a multi-section sample string (vendor + print + filament sections)
- Test abstract vs concrete detection (names with and without asterisks)
- Test multi-parent inheritance resolution (left-to-right merge, child override)
- Test comment handling (# and ;)
- Test `\n` escape in G-code values is preserved, not treated as newline

Add `pub mod profile_import_ini;` to lib.rs.

Note: Field mapping and conversion entry point are in Task 2 (next task).

**1. PrusaSlicer field mapping (`apply_prusaslicer_field_mapping`)**
- Map PrusaSlicer INI keys to PrintConfig fields. Use the mapping table from RESEARCH.md:
  - `perimeters` -> `wall_count`, `fill_density` -> `infill_density` (strip %, divide by 100)
  - `fill_pattern` -> `infill_pattern`, `temperature` -> `nozzle_temp`
  - `first_layer_temperature` -> `first_layer_nozzle_temp`, `bed_temperature` -> `bed_temp`
  - `first_layer_bed_temperature` -> `first_layer_bed_temp`
  - `retract_length` -> `retract_length`, `retract_speed` -> `retract_speed`
  - `retract_lift` -> `retract_z_hop`, `retract_before_travel` -> `min_travel_for_retract`
  - `skirts` -> `skirt_loops`, `default_acceleration` -> `print_acceleration`
  - `filament_cost` -> `filament_cost_per_kg`, `gcode_flavor` -> `gcode_dialect`
  - `nozzle_diameter` -> take first comma-separated value for multi-extruder printers
  - `machine_max_jerk_x/y/z` -> `jerk_x/y/z` (take first comma-separated value)
  - Plus all same-name fields: layer_height, first_layer_height, top_solid_layers, bottom_solid_layers, perimeter_speed, infill_speed, travel_speed, skirt_distance, brim_width, seam_position, filament_density, filament_diameter, extrusion_multiplier, retract_length, retract_speed, disable_fan_first_layers, fan_below_layer_time
- Skip percentage speed values (e.g., `first_layer_speed = 50%`) -- return false
- Map infill patterns: PrusaSlicer uses `cubic`, `gyroid`, `honeycomb`, `grid`, `rectilinear`, `monotonic`, `line`, `adaptivecubic`, `lightning` (same mapping as OrcaSlicer where possible)
- Map seam_position: PrusaSlicer uses `aligned`, `random`, `rear`, `nearest` (same as OrcaSlicer)
- Map gcode_flavor: `marlin` -> Marlin, `klipper` -> Klipper, `reprapfirmware`/`reprap` -> RepRapFirmware
- Return true/false for mapped/unmapped tracking

**2. Conversion entry point (`import_prusaslicer_ini_profile`)**

- Creates PrintConfig::default(), applies field mapping for each key-value
- Returns ImportResult (same struct as OrcaSlicer path) with config, mapped_fields, unmapped_fields, metadata
- Map section_type: `print` -> "process", `filament` -> "filament", `printer` -> "machine"

**3. Key-to-field name mapping (`prusaslicer_key_to_config_field`)**
- Parallel to `upstream_key_to_config_field` in profile_import.rs but for PrusaSlicer INI keys
- Returns Option<&'static str> mapping PrusaSlicer key name -> PrintConfig field name
- Needed for inheritance merge filtering (same pattern as Phase 15-03 fix)

**1. Add `batch_convert_prusaslicer_profiles` to profile_library.rs**

Add these imports at the top of profile_library.rs:
```rust
use crate::profile_import_ini::{parse_prusaslicer_ini, resolve_ini_inheritance, import_prusaslicer_ini_profile, IniSection};
```

- Signature: `pub fn batch_convert_prusaslicer_profiles(source_dir: &Path, output_dir: &Path, source_name: &str) -> Result<BatchConvertResult, EngineError>`
- Walk `*.ini` files in `source_dir` (NOT subdirectories -- PrusaSlicer INI files are directly in the profiles/ directory)
- Skip files containing "SLA" in filename (AnycubicSLA.ini, PrusaResearchSLA.ini)
- For each .ini file:
  a. Read file contents
  b. Call `parse_prusaslicer_ini(&contents)` to get Vec<IniSection>
  c. Build lookup HashMap for inheritance resolution
  d. Derive vendor name from filename stem (e.g., "PrusaResearch.ini" -> "PrusaResearch")
  e. For each concrete section (not is_abstract, section_type in ["print", "filament", "printer"]):
     - Call `resolve_ini_inheritance` to get flattened fields
     - Call `import_prusaslicer_ini_profile` to get ImportResult
     - Call `convert_to_toml(&import_result)` to get TOML
     - Map section type: print -> "process", filament -> "filament", printer -> "machine"
     - Write TOML to `output_dir/vendor/type/sanitized_name.toml`
     - Build ProfileIndexEntry using existing metadata extractors
  f. Collect into BatchConvertResult
- Individual profile errors collected but do not abort batch (same pattern as OrcaSlicer batch)

**2. Extend `write_index` to support merging**
- Add `pub fn write_merged_index(new_index: &ProfileIndex, output_dir: &Path) -> Result<(), EngineError>`
- If `output_dir/index.json` already exists, load it, merge new entries (append new, replacing entries with same `id`), write back
- If no existing index, write new_index as-is (fallback to current behavior)
- This prevents PrusaSlicer import from clobbering the OrcaSlicer index

**3. Extend quality name extraction**
- In `extract_quality_from_name`, add PrusaSlicer-specific quality terms:
  - "ultradetail" -> "Ultra Detail"
  - "detail" -> "Detail"
  - "optimal" -> "Optimal"
  - "normal" -> "Normal"
  - "speed" -> "Speed" (careful: must not match "first_layer_speed" etc. -- only match as standalone word or in profile name context)
  - "fast" -> "Fast"
- Check longest matches first: "ultradetail" before "detail", "super draft" before "draft"

**4. Extend sanitize_filename**
- Handle `&&` -> `_and_` (e.g., "MK3S && MK3S+" -> "MK3S_and_MK3S+")
- Handle `+` -> `+` (keep as-is, it's filesystem-safe)
