   - `maybe_par_iter!` macro: when `parallel` feature is enabled, expands to `$slice.par_iter()`; when disabled, expands to `$slice.iter()`. Use `pub(crate) use maybe_par_iter;` to make it available throughout the crate.

   - `AtomicProgress` struct wrapping `Arc<AtomicUsize>` with `new(total: usize)`, `increment()`, and `percent() -> f64` methods. This will be used by Plan 02 for parallel progress tracking.
   - Add `#[cfg(feature = "parallel")] use rayon::prelude::*;` at the top.

2. In `lib.rs`, add `mod parallel;` (no pub -- crate-internal only).

3. In `config.rs`, add two fields to `PrintConfig`:

   ```rust
   /// When false, layers are processed sequentially (useful for debugging
   /// or determinism verification). Default: true.

   /// Number of threads for parallel processing. None = auto-detect
   /// (rayon default: number of logical CPUs). Only effective when
   /// parallel_slicing is true and the `parallel` feature is enabled.
   pub thread_count: Option<usize>,
   ```
   Set `parallel_slicing: true` in the `Default` impl for `PrintConfig`. Set `thread_count: None` in the Default impl. Both fields should have `#[serde(default)]` behavior (already covered by PrintConfig's `#[serde(default)]` on the struct).
