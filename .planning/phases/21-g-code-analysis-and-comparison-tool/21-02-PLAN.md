    pub total_travel_delta_mm: f64,
    pub total_extrusion_delta_mm: f64,

}

**main.rs** -- Add two new subcommand variants to the `Commands` enum:

```rust
/// Analyze a G-code file and display structured metrics
AnalyzeGcode {
    /// Input G-code file path (use "-" for stdin)
    input: String,

    /// Output as JSON
    #[arg(long)]
    json: bool,

    /// Output as CSV
    #[arg(long)]
    csv: bool,

    /// Disable ANSI color output
    #[arg(long)]
    no_color: bool,

    /// Filament density in g/cm³ (default: 1.24 for PLA)
    #[arg(long, default_value = "1.24")]
    density: f64,

    /// Filament diameter in mm (default: 1.75)
    #[arg(long, default_value = "1.75")]
    diameter: f64,

    /// Filter output to specific feature types (comma-separated)
    #[arg(long)]
    filter: Option<String>,

    /// Summary only (no per-layer detail)
    #[arg(long)]
    summary: bool,
},

/// Compare multiple G-code files (first file is baseline)
CompareGcode {
    /// G-code files to compare (first is baseline, need at least 2)
    files: Vec<PathBuf>,

    /// Output as JSON
    #[arg(long)]
    json: bool,

    /// Output as CSV
    #[arg(long)]
    csv: bool,

    /// Disable ANSI color output
    #[arg(long)]
    no_color: bool,

    /// Filament density in g/cm³ (default: 1.24 for PLA)
    #[arg(long, default_value = "1.24")]
    density: f64,

    /// Filament diameter in mm (default: 1.75)
    #[arg(long, default_value = "1.75")]
    diameter: f64,
},
```

**Handler for AnalyzeGcode:**
1. Read input: if `input == "-"`, read from stdin via `BufReader::new(std::io::stdin().lock())`; otherwise open file with `BufReader::new(File::open(&input)?)`
2. Call `slicecore_engine::parse_gcode_file(reader, &filename, diameter, density)`
3. Apply feature filter if `--filter` is set: remove non-matching features from display (but keep the full GcodeAnalysis for JSON)
4. Dispatch to display format:
   - `--json`: call `analysis_display::display_analysis_json(&analysis)`
   - `--csv`: call `analysis_display::display_analysis_csv(&analysis, summary_only)`
   - default: call `analysis_display::display_analysis_table(&analysis, use_color, summary_only, filter)`
5. TTY detection for color: `use_color = !no_color && atty::is(atty::Stream::Stdout)`. If `atty` is not available, use a simple check: `!no_color && std::io::stdout().is_terminal()` (Rust 1.70+ has `IsTerminal` trait). Use `std::io::IsTerminal` since the project uses a recent Rust toolchain.

**Handler for CompareGcode:**
1. Validate at least 2 files provided, otherwise `eprintln!` error and `process::exit(1)`
2. Parse all files: `for file in &files { parse_gcode_file(BufReader::new(File::open(file)?), filename, diameter, density) }`
3. Call `slicecore_engine::compare_gcode_analyses(baseline, others)`
4. Dispatch to display format:
   - `--json`: call `analysis_display::display_comparison_json(&result)`
   - `--csv`: call `analysis_display::display_comparison_csv(&result)`
   - default: call `analysis_display::display_comparison_table(&result, use_color)`

**Functions to implement:**

`pub fn display_analysis_table(analysis: &GcodeAnalysis, use_color: bool, summary_only: bool, filter: &Option<Vec<String>>)`:
- Print summary header section using comfy-table:
  - File: {filename}
  - Slicer: {slicer} {version}
  - Layers: {count}
  - Total time (computed): {format_time}
  - Total time (header): {header_time} (delta: {delta})
  - Filament: {length_mm} ({volume_mm3} mm³, {weight_g} g)
  - Travel distance: {travel_mm}
  - Retractions: {count} ({distance_mm} mm)
  - Z-hops: {count} ({distance_mm} mm)
  - Unknown commands: {count}
- Print per-feature summary table:
  - Columns: Feature | Time | Time% | Filament | Moves | Speed min | Speed max | Speed avg
  - One row per feature type (sorted by time descending)
  - Apply filter if provided
- If not summary_only, print per-layer detail table:
  - Columns: Layer | Z | Height | Moves | Travel | Extrusion | Retractions | Time
  - One row per layer
- Color coding (when use_color):
  - Use `\x1b[1m...\x1b[0m` for bold headers
  - Use `\x1b[32m` green for improvements in comparison, `\x1b[31m` red for regressions
  - Use `\x1b[90m` dim for zero-value rows

`pub fn display_analysis_json(analysis: &GcodeAnalysis)`:
- `println!("{}", serde_json::to_string_pretty(&analysis).unwrap())`
- Hierarchical JSON structure (analysis struct is already hierarchical with nested layers/features)

`pub fn display_analysis_csv(analysis: &GcodeAnalysis, summary_only: bool)`:
- One row per layer: `layer,z_height,layer_height,moves,travel_mm,extrusion_mm,retraction_count,time_s`
- Header row first, then data rows
- If summary_only, output feature summary: `feature,time_s,time_pct,filament_mm,moves,speed_min,speed_max,speed_mean`

`pub fn display_comparison_table(result: &ComparisonResult, use_color: bool)`:
- Summary comparison table:
  - Columns: Metric | {baseline_filename} | {file2_name} | Delta | {file3_name} | Delta | ...
  - Rows: Total time, Header time, Filament, Layers, Retractions, Travel, etc.
  - Color-code deltas: green for less (improvement), red for more (regression), when absolute delta > 5%
- Per-feature comparison table:
  - Columns: Feature | {baseline} time | {file2} time | Delta | ...
  - One row per feature type (union of all features across files)

`pub fn display_comparison_json(result: &ComparisonResult)`:
- `println!("{}", serde_json::to_string_pretty(&result).unwrap())`

`pub fn display_comparison_csv(result: &ComparisonResult)`:
- Metric rows with columns for each file and deltas
- `metric,baseline,file2,delta2,delta2_pct,file3,delta3,delta3_pct,...`
