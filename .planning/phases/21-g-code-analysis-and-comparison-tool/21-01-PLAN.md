**metrics.rs** -- Define the core metric types:

```rust
#[derive(Debug, Clone, Default, Serialize, Deserialize)]
pub struct SpeedStats {
    pub min_mm_s: f64,
    pub max_mm_s: f64,
    pub mean_mm_s: f64,
    pub total_distance: f64,  // for weighted mean computation
    pub sample_count: u64,
}
```

- `LayerMetrics`: per-layer data (z_height, layer_height, move_count, travel_distance_mm, extrusion_distance_mm, retraction_count, layer_time_estimate_s, per-feature HashMap of FeatureMetrics)
- `FeatureMetrics`: per-feature data (move_count, travel_distance_mm, extrusion_distance_mm, extrusion_e_mm (filament used), time_estimate_s, speed_stats: SpeedStats)
- `HeaderMetadata`: parsed slicer header info (slicer_name: Option<String>, slicer_version: Option<String>, estimated_time_s: Option<f64>, filament_length_mm: Option<f64>, filament_volume_cm3: Option<f64>, filament_weight_g: Option<f64>, filament_density: Option<f64>, filament_diameter: Option<f64>, layer_count: Option<u32>, max_z_height: Option<f64>)
- `GcodeAnalysis`: the top-level result struct (filename: String, header: HeaderMetadata, slicer: SlicerType, layers: Vec<LayerMetrics>, features: HashMap<String, FeatureMetrics> (aggregated across all layers), total_time_estimate_s: f64, total_filament_mm: f64, total_filament_volume_mm3: f64, total_filament_weight_g: f64, total_travel_mm: f64, total_extrusion_mm: f64, total_moves: u64, retraction_count: u32, retraction_distance_mm: f64, zhop_count: u32, zhop_distance_mm: f64, unknown_command_count: u32, line_count: u64)
- All structs derive `Debug, Clone, Serialize, Deserialize`
- Implement `SpeedStats::update(&mut self, speed_mm_s: f64, distance: f64)` for incremental mean updates using weighted running average
- Implement `SpeedStats::merge(&mut self, other: &SpeedStats)` for combining across layers
- Implement `FeatureMetrics::merge(&mut self, other: &FeatureMetrics)` for combining across layers
- Filament weight computation: provide `pub fn filament_mm_to_weight_g(length_mm: f64, diameter_mm: f64, density_g_per_cm3: f64) -> f64` using cross-section area * length * density formula. This is the same formula as in statistics.rs but made public and standalone.
- Filament volume computation: `pub fn filament_mm_to_volume_mm3(length_mm: f64, diameter_mm: f64) -> f64` using PI * (d/2)^2 * length

**slicer_detect.rs** -- Slicer auto-detection:

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
pub enum SlicerType {
    BambuStudio,
    OrcaSlicer,
    PrusaSlicer,
    Slicecore,
    Unknown,
}
```

- `pub fn detect_slicer(lines: &[&str]) -> SlicerType` -- scan up to 100 lines for identification patterns
- BambuStudio: look for "BambuStudio" (case-insensitive) in comment lines
- OrcaSlicer: look for "OrcaSlicer" or "Orca Slicer" in comment lines
- PrusaSlicer: look for "PrusaSlicer" or "Prusa Slicer" in comment lines
- Slicecore: look for "slicecore" or "Generated by slicecore" in comment lines
- Priority order: BambuStudio > OrcaSlicer > PrusaSlicer > Slicecore > Unknown (check in this order, first match wins)
- `pub fn detect_feature_format(slicer: SlicerType) -> FeatureFormat` returning `FeatureFormat::BambuFeature` (for `; FEATURE:` prefix) or `FeatureFormat::PrusaType` (for `;TYPE:` prefix) or `FeatureFormat::Both` (for unknown -- try both)
- `FeatureFormat` enum with these 3 variants
- Unit tests: test detection for each slicer type, test unknown slicer returns Unknown, test feature format mapping

**parser.rs** -- Line-by-line state machine:

```rust
pub struct GcodeParserState {
    // Machine position
    x: f64, y: f64, z: f64, e: f64,
    feedrate_mm_min: f64,

    // Extrusion mode
    absolute_extrusion: bool,  // M82=true, M83=false, default true
    absolute_positioning: bool, // G90=true, G91=false, default true

    // Layer tracking
    current_layer_z: f64,
    current_layer_index: i32,  // -1 before first layer
    layer_height: f64,

    // Feature tracking
    current_feature: Option<String>,
    feature_format: FeatureFormat,

    // Slicer
    detected_slicer: SlicerType,

    // Counters
    unknown_command_count: u32,
    line_count: u64,
}
```

Core functions:
- `pub fn parse_gcode_file<R: BufRead>(reader: R, filename: &str, filament_diameter: f64, filament_density: f64) -> GcodeAnalysis` -- main entry point, reads line by line via BufReader, accumulates metrics
- Internal `fn parse_line(line: &str, state: &mut GcodeParserState, current_layer_metrics: &mut LayerMetrics)` -- dispatch based on first token
- Internal `fn parse_comment(line: &str, state: &mut GcodeParserState, metrics: &mut LayerMetrics)` -- handle feature annotations (`;TYPE:` and `; FEATURE:`), layer change markers (`;LAYER_CHANGE`, `; CHANGE_LAYER`), Z height annotations (`;Z:`, `; Z_HEIGHT:`), layer height annotations (`;HEIGHT:`, `; LAYER_HEIGHT:`)
- Internal `fn parse_header_comment(line: &str, header: &mut HeaderMetadata)` -- extract metadata from header comments (estimated time, filament usage, slicer name/version). BambuStudio format: `; model printing time:`, `; total filament length [mm] :`, `; total filament volume [cm^3] :`, `; total filament weight [g] :`, `; filament_density:`, `; total layer number:`. PrusaSlicer format: `; estimated printing time (normal mode) =`, `; filament used [mm] =`, `; filament used [cm3] =`, `; filament used [g] =`
- Internal `fn parse_move(parts: &[&str], is_rapid: bool, state: &mut GcodeParserState, metrics: &mut LayerMetrics)` -- parse G0/G1 parameters (X, Y, Z, E, F), compute distance, update position, track extrusion/travel. For extrusion: if `absolute_extrusion`, delta_e = new_e - state.e; if relative, delta_e = e_param directly. Positive delta_e = extrusion, negative = retraction. Update speed stats for current feature.
- Internal `fn parse_position_reset(parts: &[&str], state: &mut GcodeParserState)` -- handle G92 (especially G92 E0)
- Layer detection: detect new layer when Z increases. When Z changes: finalize current LayerMetrics, push to layers Vec, start new LayerMetrics. Also detect from slicer annotations (`;LAYER_CHANGE` / `; CHANGE_LAYER`).
- Z-hop detection: track when Z increases then decreases back to layer_z without a layer change (travel with z-hop pattern). Simpler: count Z-only moves (G0 with Z but no X/Y following a retraction) as z-hops.
- Time estimation: for each move, compute naive time as distance / (feedrate_mm_min / 60.0). Sum per layer and per feature. This is the feedrate-only estimate per user discretion recommendation.
- After all lines parsed: aggregate per-layer feature metrics into total feature metrics, compute total filament weight/volume from total filament_mm using diameter and density params, set independent time estimate.
- Strip inline comments before parsing parameters: `let code = line.split(';').next().unwrap_or("").trim()`
- Handle arc moves (G2/G3) simply by computing chord distance between start and end positions (not full arc path -- acceptable approximation for metrics)
- Unit tests in parser.rs:
  - Test M82 absolute extrusion tracking
  - Test M83 relative extrusion tracking
  - Test G92 E0 reset handling
  - Test G0 travel move distance computation
  - Test G1 extrusion move with E parameter
  - Test Z layer change detection
  - Test feedrate (F parameter) tracking and mm/min to mm/s conversion
  - Test inline comment stripping
