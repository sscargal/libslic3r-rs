Migrate the slice command to use CliOutput step indicators and progress bars.

The slice_workflow.rs warnings/info are also routed through CliOutput.

Output: cmd_slice fully migrated to CliOutput, step-based workflow visible in terminal, clean --quiet/--json behavior.

Migrate `cmd_slice` function (starts at line ~948) to use `CliOutput` for all progress, warnings, and informational output. The slice command should NOT use the old `SliceProgress` anymore -- remove the temporary backwards-compat re-export from cli_output.rs if it exists.

**1. Construct CliOutput at start of cmd_slice:**
Add `output: &cli_output::CliOutput` as a new parameter to `cmd_slice()`, or construct it inside using the quiet/json flags already passed. Preferred approach: construct inside cmd_slice since it has the per-command `json` flag:
```rust
let output = cli_output::CliOutput::new(quiet, json_output, color_mode);
```
Pass `color_mode` as a new parameter (parsed from `cli.color` in main). Update the call site in main() `Commands::Slice` match arm to pass `color_mode`.

**2. Define step count and replace eprintln! with step indicators:**

The slice workflow has these steps (5 total when using profile workflow, fewer for legacy --config):
- Step 1: Load mesh (read + parse + repair)
- Step 2: Resolve profiles / Load config
- Step 3: Validate config (when using profile workflow)
- Step 4: Slice
- Step 5: Write G-code

For legacy --config mode, steps are: Load mesh, Load config, Slice, Write G-code (4 steps).

Determine step count dynamically: `let total_steps = if use_profile_workflow { 5 } else { 4 };`

**Replace specific eprintln! patterns in cmd_slice:**

a) **Mesh loading + repair (currently lines ~990-1035):**
```rust
let step1 = output.start_step(1, total_steps, "Load mesh");
// ... existing mesh load/parse/repair code ...
if !report.was_already_clean {
    output.info(&format!("Mesh repaired ({} degenerates removed, {} edges stitched, {} holes filled, {} normals fixed)",
        report.degenerate_removed, report.edges_stitched, report.holes_filled, report.normals_fixed));
}
output.finish_step(&step1, "Load mesh");
```

b) **Config loading / profile resolution (lines ~1037-1079):**
For profile workflow:
```rust
let step2 = output.start_step(2, total_steps, "Resolve profiles");
// ... profile resolution code ...
output.finish_step(&step2, "Resolve profiles");

let step3 = output.start_step(3, total_steps, "Validate config");
// ... validation ...
output.finish_step(&step3, "Validate config");
```
For legacy --config:
```rust
let step2 = output.start_step(2, total_steps, "Load config");
// ... config loading ...
output.finish_step(&step2, "Load config");
```

c) **Plugin loading (lines ~1081-1144):**
Replace `eprintln!("Loaded {} plugin(s):", ...)` with `output.info(...)`.
Replace `eprintln!("Warning: Failed to load plugins...")` with `output.warn(...)`.
Replace `eprintln!("Plugins auto-loaded from config plugin_dir")` with `output.info(...)`.

d) **Slicing (find the `engine.slice()` or layer iteration):**
The slicing step should use start_step + add_progress_bar:
```rust
let slice_step_num = if use_profile_workflow { 4 } else { 3 };
let step_slice = output.start_step(slice_step_num, total_steps, "Slice");
// If there's a total_layers known:
// let bar = output.add_progress_bar(total_layers as u64);
// ... during slicing: bar.inc(1);
// bar.finish_and_clear();
output.finish_step(&step_slice, "Slice");
```

e) **Write G-code output:**
```rust
let write_step_num = if use_profile_workflow { 5 } else { 4 };
let step_write = output.start_step(write_step_num, total_steps, "Write G-code");
// ... write gcode ...
output.finish_step(&step_write, "Write G-code");
```

f) **Summary line at end:**
After all steps complete, print a summary (only when not quiet):
```rust
output.info(&format!("Output: {}", output_file.display()));
// Include layers, estimated time, filament usage if available
```

g) **Replace ALL remaining eprintln! in cmd_slice** that are progress/warning/info:
- `eprintln!("Error: ...")` -> `output.error_msg(...)` followed by `process::exit(1)`
- `eprintln!("Warning: ...")` -> `output.warn(...)`
- `eprintln!("Note: ...")` -> `output.info(...)`
- `eprintln!("Loaded N plugin(s):...")` -> `output.info(...)`

**Keep data output as direct println!/serde_json** -- JSON metadata, stats tables, G-code path output to stdout stay as-is. CliOutput only handles progress/status/warnings/errors to stderr.

**3. Remove old SliceProgress usage:**
If cmd_slice currently calls `progress::create_progress()` or uses `SliceProgress`, replace those calls with the new CliOutput methods. Remove the `SliceProgress` struct and `create_progress` function from cli_output.rs if they were kept temporarily.

The `run_slice_workflow()` function in slice_workflow.rs has its own `eprintln!` calls for warnings and errors (e.g., "Warning: --unsafe-defaults active", "Error: Failed to compose config"). These need to route through CliOutput.

**Option A (preferred): Pass CliOutput reference to run_slice_workflow:**
Update the function signature:
```rust
pub fn run_slice_workflow(options: &SliceWorkflowOptions, output: &crate::cli_output::CliOutput) -> Result<WorkflowResult, i32>
```

Replace all `eprintln!("Warning: ...")` with `output.warn(...)`.
Replace all `eprintln!("Error: ...")` with `output.error_msg(...)`.
Replace all `eprintln!(...)` for informational messages (profile names, override info) with `output.info(...)`.

Specifically scan for and replace these patterns in slice_workflow.rs:
- `eprintln!("Warning: --unsafe-defaults active, ...")` -> `output.warn("--unsafe-defaults active, using PrintConfig::default() with no profiles")`
- `eprintln!("Error: Failed to compose config: {e}")` -> `output.error_msg(&format!("Failed to compose config: {e}"))`
- Any `eprintln!("Error: ...")` -> `output.error_msg(...)`
- Any `eprintln!("Warning: ...")` -> `output.warn(...)`
- Any informational `eprintln!` (profile resolution messages, validation summaries) -> `output.info(...)`

Update the call site in cmd_slice (main.rs) to pass `&output` to `run_slice_workflow()`.

**Option B (if modifying signature is too invasive):**
Collect warnings in the WorkflowResult struct and let cmd_slice display them via CliOutput. Add `pub warnings: Vec<String>` and `pub infos: Vec<String>` to WorkflowResult. But Option A is cleaner.
