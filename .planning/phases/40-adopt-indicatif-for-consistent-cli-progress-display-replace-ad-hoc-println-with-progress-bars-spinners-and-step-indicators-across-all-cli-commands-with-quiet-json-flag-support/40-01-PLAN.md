Create `crates/slicecore-cli/src/cli_output.rs` — a new file that replaces `progress.rs`. This file provides the unified `CliOutput` struct.

**ColorMode enum:**
```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum ColorMode {
    Always,
    Never,
    Auto,
}
```

**CliOutput struct** with fields:
- `multi: MultiProgress` — from indicatif, manages all progress bars/spinners
- `quiet: bool` — effective quiet (true if --quiet OR --json)
- `json: bool` — whether JSON mode is active
- `is_tty: bool` — whether stderr is a terminal
- `color_enabled: bool` — resolved from ColorMode + TTY + NO_COLOR

**Constructor `CliOutput::new(quiet: bool, json: bool, color: ColorMode) -> Self`:**
- `is_tty = std::io::stderr().is_terminal()`
- `effective_quiet = quiet || json`
- `color_enabled`: Always=true, Never=false, Auto=(is_tty && env::var("NO_COLOR").is_err())
- If effective_quiet: `MultiProgress::with_draw_target(indicatif::ProgressDrawTarget::hidden())`
- Else: `MultiProgress::new()` (defaults to stderr)
- Call `console::set_colors_enabled_stderr(color_enabled)` and `console::set_colors_enabled(color_enabled)` once at construction

**Methods to implement:**

1. `pub fn start_step(&self, step: usize, total: usize, message: &str) -> ProgressBar` — Creates a numbered step spinner like `[1/5] Resolve profiles`. In non-TTY non-quiet mode: `eprintln!("[{step}/{total}] {message}...")` and return `ProgressBar::hidden()`. In TTY mode: create spinner via `self.multi.add(ProgressBar::new_spinner())`, set style template `"{spinner:.green} [{prefix}] {msg} {elapsed:.dim}"` with braille tick strings `&["⠋", "⠙", "⠹", "⠸", "⠼", "⠴", "⠦", "⠧", "⠇", "⠏", "✓"]`, set prefix to `"{step}/{total}"`, set message, call `enable_steady_tick(Duration::from_millis(80))`.

2. `pub fn finish_step(&self, spinner: &ProgressBar, message: &str)` — In non-TTY non-quiet: `eprintln!("[done] {message}")`. Otherwise: `spinner.finish_with_message(message.to_string())`.

3. `pub fn add_progress_bar(&self, total: u64) -> ProgressBar` — Nested progress bar. Non-TTY non-quiet: return `ProgressBar::hidden()`. TTY: `self.multi.add(ProgressBar::new(total))` with style `"  {bar:40.cyan/blue} {pos}/{len} {msg}"` and progress chars `"=>-"`.

4. `pub fn spinner(&self, message: &str) -> ProgressBar` — Simple spinner for medium-duration commands. Non-TTY non-quiet: `eprintln!("{message}...")` + return hidden. TTY: create spinner via multi.add, style `"{spinner:.green} {msg}"` with same braille ticks, enable_steady_tick(80ms).

5. `pub fn finish_spinner(&self, spinner: &ProgressBar, message: &str)` — `spinner.finish_with_message(message.to_string())`.

6. `pub fn warn(&self, msg: &str)` — If quiet, return early. Use `console::style(format!("Warning: {msg}")).yellow()` when color_enabled, else plain. Print via `self.multi.println(...)`.

7. `pub fn error_msg(&self, msg: &str)` — ALWAYS prints (never suppressed). Use `console::style(format!("Error: {msg}")).red().bold()` when color_enabled. Print via `self.multi.println(...)`.

8. `pub fn info(&self, msg: &str)` — If quiet, return early. Print via `self.multi.println(msg)`.

9. `pub fn is_json(&self) -> bool` — Returns self.json.

10. `pub fn is_quiet(&self) -> bool` — Returns self.quiet.
