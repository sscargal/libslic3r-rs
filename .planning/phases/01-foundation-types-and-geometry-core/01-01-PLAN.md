  <action>
    1. Create workspace root `Cargo.toml`:
       - `[workspace]` with members = ["crates/*"]
       - `[workspace.package]` with version = "0.1.0", edition = "2021", license = "MIT OR Apache-2.0", rust-version = "1.75"
       - `[workspace.dependencies]` for shared deps: serde = { version = "1", features = ["derive"] }, approx = "0.5", thiserror = "2", proptest = "1"

    2. Create `.cargo/config.toml` with any useful workspace settings (e.g., incremental compilation, target dir).

    3. Create `crates/slicecore-math/Cargo.toml`:
       - Dependencies: serde (workspace), approx (workspace)
       - Dev-dependencies: proptest (workspace)
       - No default features needed

    4. Implement `coord.rs`:
       - `pub type Coord = i64;`
       - `pub const COORD_SCALE: f64 = 1_000_000.0;` (1 mm = 1,000,000 internal units = nanometer precision)
       - `#[derive(Clone, Copy, Debug, PartialEq, Eq, Hash, PartialOrd, Ord, Serialize, Deserialize)]`
       - `#[repr(C)]` on IPoint2
       - `pub struct IPoint2 { pub x: Coord, pub y: Coord }` with:
         - `fn new(x: Coord, y: Coord) -> Self`
         - `fn from_mm(x: f64, y: f64) -> Self` (uses COORD_SCALE, rounds)
         - `fn to_mm(self) -> (f64, f64)` (divides by COORD_SCALE)
         - `fn zero() -> Self`
         - Implement Add, Sub, Neg for IPoint2
         - Implement Mul<Coord> and Div<Coord> for scalar operations
       - Document the precision strategy: "nanometer precision, range +/- 9.2e12 mm with i64, far beyond any 3D printer build volume"

    5. Implement `point.rs`:
       - `pub struct Point2 { pub x: f64, pub y: f64 }` -- for float-space operations (mesh vertices projected to 2D)
       - `pub struct Point3 { pub x: f64, pub y: f64, pub z: f64 }` -- for mesh vertex positions
       - Both: Clone, Copy, Debug, Serialize, Deserialize
       - PartialEq via approx comparison (implement manually with EPSILON)
       - Standard ops: Add, Sub, Neg
       - Methods: `fn new()`, `fn zero()`, `fn distance_to(&self, other: &Self) -> f64`, `fn midpoint(&self, other: &Self) -> Self`
       - Point3 additional: `fn to_point2(self) -> Point2` (drops z, for projection)
       - Conversion: `impl From<IPoint2> for Point2` (uses coord_to_mm)
       - Conversion: `impl Point2 { fn to_ipoint2(self) -> IPoint2 }` (uses mm_to_coord)

    6. Implement `vec.rs`:
       - `pub struct Vec2 { pub x: f64, pub y: f64 }` with: dot, cross (returns scalar for 2D), length, length_squared, normalize, perpendicular
       - `pub struct Vec3 { pub x: f64, pub y: f64, pub z: f64 }` with: dot, cross (returns Vec3), length, length_squared, normalize
       - Both: Clone, Copy, Debug, Serialize, Deserialize, Add, Sub, Neg, Mul<f64>, Div<f64>
       - Conversion: `impl From<Point2> for Vec2`, `impl From<Point3> for Vec3` (point-to-vector)
       - `Vec2::from_points(from: Point2, to: Point2) -> Vec2`
       - `Vec3::from_points(from: Point3, to: Point3) -> Vec3`

    7. Implement `bbox.rs`:
       - `pub struct BBox2 { pub min: Point2, pub max: Point2 }` with: `fn new(min, max)`, `fn from_points(points: &[Point2])`, `fn union(&self, other: &BBox2)`, `fn intersection(&self, other: &BBox2) -> Option<BBox2>`, `fn contains_point(&self, p: &Point2) -> bool`, `fn center(&self) -> Point2`, `fn width/height/area`, `fn expand(margin: f64)`
       - `pub struct BBox3 { pub min: Point3, pub max: Point3 }` with same operations plus: `fn depth()`, `fn volume()`, `fn from_points(points: &[Point3])`
       - Also create `pub struct IBBox2 { pub min: IPoint2, pub max: IPoint2 }` for integer-coordinate bounding boxes (used in polygon operations)
       - All: Clone, Copy, Debug, Serialize, Deserialize

    8. Implement `epsilon.rs`:
       - `pub const EPSILON: f64 = 1e-9;` (for coordinate comparison)
       - `pub const AREA_EPSILON: f64 = 1e-6;` (for area comparison)
       - `pub fn approx_eq(a: f64, b: f64, eps: f64) -> bool`
       - `pub fn approx_zero(a: f64, eps: f64) -> bool`

    9. Implement `convert.rs`:
       - `pub fn mm_to_coord(mm: f64) -> Coord` -- multiply by COORD_SCALE, round, cast to i64
       - `pub fn coord_to_mm(coord: Coord) -> f64` -- cast to f64, divide by COORD_SCALE
       - `pub fn points_to_ipoints(points: &[(f64, f64)]) -> Vec<IPoint2>`
       - `pub fn ipoints_to_points(points: &[IPoint2]) -> Vec<(f64, f64)>`

    10. Implement `lib.rs`:
        - Declare all modules: `pub mod coord; pub mod point; pub mod vec; pub mod bbox; pub mod epsilon; pub mod convert;`
        - Re-export key types at crate root: `pub use coord::{Coord, IPoint2, COORD_SCALE}; pub use point::{Point2, Point3};` etc.
        - Add module-level rustdoc: "Foundation math types for the slicecore 3D slicing engine."
  </action>

  </done>
