Implement the `slicecore-mesh` crate -- TriangleMesh data structure with arena+index pattern, custom SAH-based BVH for spatial indexing, and spatial query interfaces (plane intersection for slicing, ray intersection for support generation).

Purpose: Provide the 3D mesh representation that the slicing pipeline operates on. The mesh must be Send+Sync for parallel processing, use flat array storage for cache efficiency, and support BVH-accelerated spatial queries. These design choices are permanent -- all downstream mesh operations build on this representation.

    3. Implement `triangle_mesh.rs`:
       - `pub struct TriangleMesh`:
         - `vertices: Vec<Point3>` -- vertex positions (private, accessed via getter)
         - `indices: Vec<[u32; 3]>` -- triangle face indices into vertices array (private)
         - `normals: Vec<Vec3>` -- per-face normals, computed on construction (private)
         - `aabb: BBox3` -- mesh bounding box, computed on construction (private)
         - `bvh: OnceCell<BVH>` -- lazy spatial index. Use `std::cell::OnceCell` (Rust 1.70+) or `once_cell::sync::OnceCell` for thread-safe lazy init. Since TriangleMesh needs to be Send+Sync and we want interior mutability for lazy BVH, use `std::sync::OnceLock` (Rust 1.75+). This keeps the struct Send+Sync automatically.

       - Constructor: `pub fn new(vertices: Vec<Point3>, indices: Vec<[u32; 3]>) -> Result<Self, MeshError>`
         1. Validate: non-empty vertices, non-empty indices
         2. Validate: all indices < vertices.len()
         3. Compute per-face normals: for each triangle, compute cross product of two edges, normalize. Skip degenerate triangles (zero-area) -- store Vec3::ZERO for those and log/count them.
         4. Compute AABB from all vertices
         5. Initialize bvh as OnceLock::new() (empty, built lazily)

       - Getters: `pub fn vertices(&self) -> &[Point3]`, `pub fn indices(&self) -> &[[u32; 3]]`, `pub fn normals(&self) -> &[Vec3]`, `pub fn aabb(&self) -> &BBox3`, `pub fn vertex_count(&self) -> usize`, `pub fn triangle_count(&self) -> usize`

       - Triangle access: `pub fn triangle_vertices(&self, tri_idx: usize) -> [Point3; 3]` -- returns the 3 vertex positions for triangle at index

       - BVH access: `pub fn bvh(&self) -> &BVH` -- uses OnceLock::get_or_init to lazily build BVH on first access. Thread-safe.

       - IMPORTANT: Do NOT implement `unsafe impl Send for TriangleMesh` or `unsafe impl Sync for TriangleMesh`. With Vec<Point3>, Vec<[u32; 3]>, Vec<Vec3>, BBox3, and OnceLock<BVH>, the struct is automatically Send+Sync. Adding unnecessary unsafe is an anti-pattern (per research pitfall #6).

    4. Implement `bvh.rs` (~300-500 lines, SAH-based BVH per PBRT Ch4):
       - `pub struct BVH { nodes: Vec<BVHNode> }`
       - `pub(crate) enum BVHNode`:
         - `Leaf { aabb: BBox3, first_tri: u32, tri_count: u32 }`
         - `Interior { aabb: BBox3, left: u32, right: u32, split_axis: u8 }`
       - `pub struct RayHit { pub triangle_idx: usize, pub t: f64, pub u: f64, pub v: f64 }` (barycentric coords)

       - `impl BVH`:
         - `pub fn build(vertices: &[Point3], indices: &[[u32; 3]]) -> Self`:
           1. Compute AABB and centroid for each triangle
           2. Filter out degenerate triangles (zero-area AABB) before building
           3. Recursively partition using SAH cost metric:
              - For each axis (X, Y, Z), evaluate N split candidates (e.g., 12 buckets)
              - SAH cost = traversal_cost + (left_area * left_count + right_area * right_count) / parent_area
              - Choose axis and split with minimum cost
              - If best cost > leaf cost (just processing all triangles), create leaf
           4. Leaf threshold: 4 triangles per leaf (typical value)
           5. Store all nodes in flat Vec<BVHNode> for cache-friendly traversal
           6. Keep a parallel `tri_indices: Vec<u32>` that maps leaf ranges to original triangle indices

         - `pub fn query_plane(&self, z: f64) -> Vec<usize>`:
           - Traverse BVH tree. For each node, check if AABB spans the z height (aabb.min.z <= z <= aabb.max.z).
           - If not, prune entire subtree.
           - At leaf nodes, return triangle indices whose per-triangle AABB spans z.
           - This is THE critical query for slicing (called once per layer).

         - `pub fn intersect_ray(&self, origin: &Point3, direction: &Vec3) -> Option<RayHit>`:
           - Traverse BVH using ray-AABB intersection test (slab method).
           - At leaf nodes, test ray against each triangle (Moller-Trumbore algorithm).
           - Track closest hit (minimum t > 0).
           - Return closest RayHit or None.

         - `pub fn triangle_count(&self) -> usize` -- total triangles in BVH

       - Implement ray-AABB intersection (slab method, ~20 lines) as helper.
       - Implement Moller-Trumbore ray-triangle intersection (~30 lines) as helper.

    1. Implement `stats.rs`:
       - `#[derive(Clone, Debug, Serialize, Deserialize)]`
       - `pub struct MeshStats`:
         - `pub vertex_count: usize`
         - `pub triangle_count: usize`
         - `pub degenerate_count: usize` -- triangles with zero area
         - `pub aabb: BBox3`
         - `pub volume: f64` -- signed volume via divergence theorem (positive = outward-facing normals)
         - `pub surface_area: f64` -- sum of triangle areas
         - `pub is_manifold: bool` -- every edge shared by exactly 2 triangles
         - `pub is_watertight: bool` -- manifold + no boundary edges
         - `pub has_consistent_winding: bool` -- all triangle normals face same direction (outward)

       - `pub fn compute_stats(mesh: &TriangleMesh) -> MeshStats`:
         1. vertex_count and triangle_count from mesh
         2. Count degenerate triangles (zero-area normal)
         3. Volume: sum over all triangles of signed tetrahedron volume (vertex positions form tetrahedra with origin). Formula: sum of (v0 . (v1 x v2)) / 6.0 for each triangle.
         4. Surface area: sum of 0.5 * |edge1 x edge2| for each triangle
         5. Manifold check: build edge -> face count map. Each edge (sorted vertex pair) should have exactly 2 faces. Any edge with 1 face = boundary, >2 = non-manifold.
         6. Watertight: manifold + no boundary edges
         7. Consistent winding: for shared edges, adjacent triangles should have opposite vertex ordering on the shared edge (indicating consistent outward normals)

    2. Implement `transform.rs`:
       - `pub fn translate(mesh: &TriangleMesh, dx: f64, dy: f64, dz: f64) -> TriangleMesh` -- creates new mesh with translated vertices. Recomputes AABB. Normals unchanged.
       - `pub fn scale(mesh: &TriangleMesh, sx: f64, sy: f64, sz: f64) -> TriangleMesh` -- scale about origin. Recompute normals (may flip if negative scale) and AABB.
       - `pub fn rotate(mesh: &TriangleMesh, axis: &Vec3, angle_rad: f64) -> TriangleMesh` -- Rodrigues rotation. Recompute normals and AABB.
       - `pub fn mirror(mesh: &TriangleMesh, axis: MirrorAxis) -> TriangleMesh` -- mirror about a plane. Reverse triangle winding to maintain consistent normals.
       - `pub enum MirrorAxis { X, Y, Z }`
       - `pub fn transform(mesh: &TriangleMesh, matrix: &Matrix4x4) -> TriangleMesh` -- general affine transform via Matrix4x4. Apply to vertices, recompute normals (use inverse transpose of upper 3x3 for normal transformation), recompute AABB.
       - `pub fn center_on_origin(mesh: &TriangleMesh) -> TriangleMesh` -- translate so AABB center is at origin
       - `pub fn place_on_bed(mesh: &TriangleMesh) -> TriangleMesh` -- translate so AABB min Z = 0

       All transform functions return NEW meshes (immutable pattern -- original unchanged). This matches the research recommendation of immutable-after-construction.
