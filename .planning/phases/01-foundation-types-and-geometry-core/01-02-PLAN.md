    3. Implement `error.rs`:
       - `#[derive(Debug, thiserror::Error)]`
       - `pub enum GeoError`:
         - `#[error("polygon has fewer than 3 points ({0} given)")]` TooFewPoints(usize)
         - `#[error("polygon has zero area")]` ZeroArea
         - `#[error("polygon has collinear points that reduce it below 3 effective vertices")]` AllCollinear
         - `#[error("polygon has self-intersections")]` SelfIntersecting
         - `#[error("boolean operation failed: {0}")]` BooleanOpFailed(String)
         - `#[error("offset operation failed: {0}")]` OffsetFailed(String)

    4. Implement `area.rs`:
       - `pub fn signed_area_i64(points: &[IPoint2]) -> i64` -- shoelace formula using i64 arithmetic (no float conversion). Returns positive for CCW, negative for CW. Uses i128 intermediate to avoid overflow on large polygons.
       - `pub fn signed_area_f64(points: &[IPoint2]) -> f64` -- same but returns area in mm^2 (divides by COORD_SCALE^2)
       - `pub fn winding_direction(points: &[IPoint2]) -> Option<Winding>` -- returns None for zero area, Some(CCW) for positive area, Some(CW) for negative
       - Tests: square (known area), triangle, degenerate (collinear points = zero area), single point, two points

    5. Implement `polygon.rs`:
       - `#[derive(Clone, Debug, Serialize, Deserialize)]`
       - `pub struct Polygon { pub points: Vec<IPoint2> }` (unvalidated, public field for easy construction)
       - Methods: `fn new(points: Vec<IPoint2>) -> Self`, `fn from_mm(points: &[(f64, f64)]) -> Self`, `fn len()`, `fn is_empty()`, `fn iter()`, `fn reverse(&mut self)`
       - `pub fn validate(self) -> Result<ValidPolygon, GeoError>`:
         1. Check len >= 3 (else TooFewPoints)
         2. Remove consecutive duplicate points
         3. Remove collinear points (using cross product == 0 check)
         4. Check remaining >= 3 (else AllCollinear)
         5. Compute signed area, reject zero (ZeroArea)
         6. Determine winding from signed area sign
         7. Return ValidPolygon with cached area and winding

       - `#[derive(Clone, Debug)]`
       - `pub struct ValidPolygon`:
         - Private fields: points: Vec<IPoint2>, area: i64, winding: Winding
         - Methods: `fn points(&self) -> &[IPoint2]`, `fn area_i64(&self) -> i64`, `fn area_mm2(&self) -> f64`, `fn winding(&self) -> Winding`, `fn len(&self)`, `fn iter()`, `fn reversed(&self) -> ValidPolygon` (flips winding), `fn into_polygon(self) -> Polygon` (unwrap back to unvalidated)
         - `fn ensure_ccw(&self) -> ValidPolygon` -- returns self if CCW, reversed copy if CW
         - `fn ensure_cw(&self) -> ValidPolygon` -- opposite

       - `#[derive(Clone, Copy, Debug, PartialEq, Eq, Serialize, Deserialize)]`
       - `pub enum Winding { CounterClockwise, Clockwise }`

    6. Implement `polyline.rs`:
       - `pub struct Polyline { pub points: Vec<IPoint2> }` -- open path (not closed)
       - Methods: `fn new()`, `fn len()`, `fn is_empty()`, `fn iter()`, `fn length_i64() -> i64` (sum of segment lengths), `fn length_mm() -> f64`

    7. Implement `point_in_poly.rs` (winding number algorithm, ~30 lines):
       - `pub fn point_in_polygon(point: &IPoint2, polygon: &[IPoint2]) -> PointLocation`
       - `pub enum PointLocation { Inside, Outside, OnBoundary }`
       - Uses winding number test: for each edge, check if point is left/right/on the edge, accumulate winding number. Non-zero winding = inside.
       - Tests: point clearly inside, clearly outside, on vertex, on edge, at origin of a polygon centered at origin

    8. Implement `simplify.rs` (Ramer-Douglas-Peucker, ~50 lines):
       - `pub fn simplify(points: &[IPoint2], epsilon: i64) -> Vec<IPoint2>`
       - Recursive RDP algorithm. Epsilon is in integer coordinate units (use mm_to_coord to convert from mm tolerance).
       - Tests: straight line simplifies to 2 points, already-simple polygon unchanged, known simplification case

    9. Implement `convex_hull.rs` (Graham scan, ~80 lines):
       - `pub fn convex_hull(points: &[IPoint2]) -> Vec<IPoint2>`
       - Graham scan: find lowest point, sort by polar angle, scan with left-turn test.
       - Returns CCW-ordered hull points.
       - Tests: triangle (returns same), square (returns same), collinear points, points with interior points removed

    10. Implement `lib.rs`:
        - Declare all modules, re-export key types: Polygon, ValidPolygon, Winding, Polyline, GeoError, PointLocation
        - Re-export functions: signed_area_i64, signed_area_f64, winding_direction, point_in_polygon, simplify, convex_hull
        - Do NOT yet re-export boolean/offset (added in Task 2)

    5. Write comprehensive boolean operation tests (minimum 20 total, targeting 25+):

       **Basic correctness (5 tests):**
       - Union of two overlapping squares produces correct merged shape
       - Intersection of two overlapping squares produces correct overlap region
       - Difference (A - B) removes B from A correctly
       - XOR of two overlapping squares produces two separate regions
       - Union of identical squares produces single square with same area

       **Edge cases (5 tests):**
       - Union of non-overlapping squares produces two separate polygons
       - Intersection of non-overlapping squares produces empty result
       - Difference where B fully contains A produces empty result
       - Difference where A and B are identical produces empty result
       - Boolean op with single polygon (no clips) returns subject unchanged

       **Degenerate geometry (10+ tests):**

       - Collinear vertices: square with extra collinear points on edges -- should produce same result as without
       - Self-intersecting polygon (figure-8 shape) -- library should handle or ValidPolygon rejects it
       - Very thin polygon (width of 1 internal unit = 1 nanometer) -- boolean should handle
       - Polygon with duplicate consecutive vertices -- should handle after validation
       - Nearly-coincident edges (two polygons sharing an edge exactly) -- union should merge cleanly
       - Very large polygon (coordinates near +-1_000_000_000_000) -- should not overflow with i64
       - Very small polygon (1 micrometer triangle) -- should produce valid result
       - Polygon with hole: test that outer CCW + inner CW hole survives boolean ops
       - Many-vertex polygon (100+ vertices, like a fine circle approximation) -- should complete in reasonable time

       **Offset tests (5+ tests):**
       - Outward offset of square by 1mm -- result is larger square (area increases)
       - Inward offset of square by small amount -- result is smaller square (area decreases)
       - Inward offset past center -- result is empty (polygon collapses)
       - Offset of triangle with Round join -- approximates rounded triangle
       - Offset of concave polygon -- may split into multiple polygons
