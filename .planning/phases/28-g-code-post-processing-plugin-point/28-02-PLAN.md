  <name>Task 1: PostProcessConfig and 4 built-in post-processor plugins</name>
  <files>

  <behavior>
    - PostProcessConfig has enabled (bool), pause_at_layers (Vec of usize), pause_command (String, default "M0"), timelapse_enabled (bool), timelapse_park_x/y (f64), timelapse_dwell_ms (u32), timelapse_retract (f64), fan_override_enabled (bool), fan_override_rules (Vec of FanOverrideRule with start_layer, end_layer, fan_speed), custom_gcode_enabled (bool), custom_gcode_rules (Vec of CustomGcodeRule with trigger type, gcode string), plugin_order (Vec of String for explicit ordering)
    - PauseAtLayerPlugin inserts pause_command (M0 or M600) after LAYER_CHANGE comments at specified layers, returns unchanged if layers list empty
    - TimelapseCameraPlugin inserts retract + park move + dwell + unretract at every layer change, returns unchanged if not enabled
    - FanSpeedOverridePlugin overrides SetFanSpeed commands within specified layer ranges, returns unchanged if no rules
    - CustomGcodeInjectionPlugin injects raw G-code at triggers (every_n_layers, at_layers, before_retraction, after_retraction), returns unchanged if no rules
    - All built-ins implement PostProcessorPluginAdapter with PluginKind::Builtin
    - Default PostProcessConfig has all features disabled (backward compatible)
  </behavior>
  <action>
    In `config.rs`, add PostProcessConfig struct with serde(default) on all fields:
    - `enabled: bool` (default false -- master switch)
    - `pause_at_layers: Vec<usize>` (default empty)
    - `pause_command: String` (default "M0")
    - `timelapse: TimelapseConfig` sub-struct: enabled (false), park_x (0.0), park_y (0.0), dwell_ms (500), retract_distance (1.0), retract_speed (2400.0)
    - `fan_overrides: Vec<FanOverrideRule>` (default empty) where FanOverrideRule has start_layer, end_layer (Option, None=until end), fan_speed (u8)
    - `custom_gcode: Vec<CustomGcodeRule>` (default empty) where CustomGcodeRule has trigger (CustomGcodeTrigger enum: EveryNLayers(usize), AtLayers(Vec<usize>), BeforeRetraction, AfterRetraction), gcode (String)
    - `plugin_order: Vec<String>` (default empty -- uses priority ordering)
    - Add `pub post_process: PostProcessConfig` field to PrintConfig with `#[serde(default)]`.

    Create `postprocess_builtin.rs` with 4 structs implementing PostProcessorPluginAdapter:

    1. **PauseAtLayerPlugin**: ProcessingMode::All, priority 50. Iterates commands tracking current layer via LAYER_CHANGE comments, inserts Comment + Raw(pause_command) after LAYER_CHANGE at specified layers. Self-skips if layers empty.

    2. **TimelapseCameraPlugin**: ProcessingMode::All, priority 60. At each LAYER_CHANGE, inserts: retract, rapid move to park position, dwell, unretract, rapid move back. Tracks last XY position from LinearMove/RapidMove commands. Self-skips if not enabled.

    3. **FanSpeedOverridePlugin**: ProcessingMode::All, priority 70. Tracks current layer, replaces SetFanSpeed commands when within a rule's layer range with the rule's fan_speed. Self-skips if rules empty.

    4. **CustomGcodeInjectionPlugin**: ProcessingMode::All, priority 80. Handles triggers:
       - EveryNLayers(n): inject after LAYER_CHANGE every n layers
       - AtLayers(layers): inject after LAYER_CHANGE at specific layers
       - BeforeRetraction: inject Raw before each Retract command
       - AfterRetraction: inject Raw after each Unretract command
       Self-skips if rules empty.

    Each plugin has a `from_config(config: &PostProcessConfig) -> Self` constructor.

    Add `pub fn create_builtin_postprocessors(config: &PostProcessConfig) -> Vec<Box<dyn PostProcessorPluginAdapter>>` that creates enabled built-ins.

    Update `lib.rs` to add `pub mod postprocess_builtin;` and re-export `create_builtin_postprocessors`.

    Unit tests per plugin: self-skip on empty config, basic injection/modification behavior.
  </action>
  <verify>
    <automated>cargo test -p slicecore-engine --lib postprocess_builtin</automated>
  </verify>
  <done>PostProcessConfig in PrintConfig with serde defaults. 4 built-in post-processors implemented with self-skip behavior. All tests pass.</done>
</task>
