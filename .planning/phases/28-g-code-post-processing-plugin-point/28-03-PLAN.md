<task type="auto">
  <name>Task 1: CLI post-process subcommand</name>
  <files>crates/slicecore-cli/src/main.rs</files>
  <action>
    Add a `PostProcess` variant to the CLI's `Commands` enum (clap subcommand):

    ```
    slicecore post-process <input.gcode> [options]
    ```

    Arguments and flags:
    - `input`: Path to input G-code file (required)
    - `--output` / `-o`: Output file path (default: stdout)
    - `--config` / `-c`: Path to TOML config file with [post_process] section
    - `--pause-at-layer <N>`: Insert pause at layer N (repeatable: `--pause-at-layer 5 --pause-at-layer 10`)
    - `--pause-command <CMD>`: Pause command (default "M0", alternative "M600")
    - `--timelapse`: Enable timelapse mode
    - `--timelapse-park-x <X>`: Park X position for timelapse
    - `--timelapse-park-y <Y>`: Park Y position for timelapse
    - `--fan-override <START:END:SPEED>`: Fan speed override rule (repeatable, format "start_layer:end_layer:speed", use "*" for end_layer to mean until end)
    - `--inject-gcode <TRIGGER:GCODE>`: Custom G-code injection (repeatable, format "every_5:G-code" or "at_10:G-code" or "before_retract:G-code")

    Implementation:
    1. Read input G-code file to String
    2. Parse G-code lines into Vec<GcodeCommand> using existing GcodeParser or a simple line-by-line approach: for each line, try to parse as a known command; unrecognized lines become Raw(line)
    3. Build PostProcessConfig from CLI flags, merging with config file if --config provided (CLI flags override config file values)
    4. Create built-in post-processors via create_builtin_postprocessors
    5. Create FfiPrintConfigSnapshot with reasonable defaults (or from config file if provided)
    6. Call run_post_processors
    7. Write output G-code via GcodeWriter to output file or stdout

    Error handling: use eprintln + exit(1) matching existing CLI pattern (no anyhow).

    Note: For G-code parsing, use slicecore_gcode_io's GcodeCommand::from_line if available, or parse Raw lines since post-processors work on both typed and Raw commands. The simplest approach: read lines, wrap each as Raw(line) -- post-processors detect LAYER_CHANGE comments and specific commands by pattern matching. This avoids needing a full G-code parser for external files. However, if a parse_line function exists in gcode-io, prefer using it for typed commands.
  </action>

    - Assert total M0 count equals 2

    **Test 2: Timelapse camera insertion**
    - Create config with post_process.enabled=true, timelapse enabled, park at (0, 200), dwell 1000ms
    - Slice via Engine
    - Assert output G-code contains G0 moves to park position after each LAYER_CHANGE
    - Assert G4 P1000 dwell command present after park move

    **Test 3: Fan speed override**
    - Create config with post_process.enabled=true, fan_overrides=[{start_layer: 3, end_layer: None, fan_speed: 255}]
    - Slice via Engine
    - Assert M106 S255 appears after layer 3
    - Assert fan commands before layer 3 are unchanged

    **Test 4: Custom G-code injection (every N layers)**
    - Create config with post_process.enabled=true, custom_gcode=[{trigger: EveryNLayers(2), gcode: "M400\nG-code custom"}]
    - Slice via Engine
    - Assert custom G-code appears after every 2nd LAYER_CHANGE

    **Test 5: Post-processing disabled by default**
    - Slice with default config (post_process.enabled=false)
    - Assert no pause, timelapse, or custom commands in output
    - Assert output matches a slice without PostProcessConfig at all

    **Test 6: Multiple post-processors execute in order**
    - Enable both pause-at-layer and timelapse
    - Slice via Engine
    - Assert pause commands appear (priority 50) and timelapse commands appear (priority 60)
    - Both modifications present in final output

    **Test 7: Time estimation reflects post-processed output**
    - Slice with pause-at-layer enabled (adds M0 dwell)
    - Slice same mesh without post-processing
    - Assert estimated time with post-processing >= estimated time without (M0 adds time)
