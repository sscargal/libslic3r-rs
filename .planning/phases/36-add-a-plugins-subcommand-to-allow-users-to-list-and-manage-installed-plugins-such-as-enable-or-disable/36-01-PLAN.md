<objective>
Add plugin status management and status-aware discovery to the slicecore-plugin crate.

Purpose: Provides the foundation layer for the plugins CLI subcommand -- per-plugin .status files for enable/disable state, an error-collecting discovery function for listing broken plugins, status-aware discover_and_load to skip disabled plugins during slicing, and a hard-error path when a disabled plugin is explicitly referenced by name.

Output: New status.rs module, extended discovery.rs with discover_all_with_status(), updated registry.rs with enabled field on PluginInfo, require_infill_plugin() for disabled-plugin error path, and extended error.rs with StatusFileError + PluginDisabled variants.
</objective>
