   ```rust
   #[derive(Subcommand)]
   pub enum PluginsCommand {
       /// List installed plugins
       List {
           #[arg(long)] json: bool,
           #[arg(long)] category: Option<String>,  // "infill" or "postprocessor"
           #[arg(long)] status: Option<String>,     // "enabled", "disabled", "error"
       },
       /// Enable a plugin
       Enable { name: String },
       /// Disable a plugin
       Disable { name: String },
       /// Show detailed plugin information
       Info {
           name: String,
           #[arg(long)] json: bool,
       },
       /// Validate a plugin (health check)
       Validate { name: String },
   }
   ```

2. Implement `pub fn run_plugins(cmd: PluginsCommand, plugin_dir: &Path) -> Result<(), anyhow::Error>` that dispatches to handler functions.

3. Implement `fn cmd_list(plugin_dir: &Path, json: bool, category: Option<&str>, status_filter: Option<&str>) -> Result<(), anyhow::Error>`:
   - Call `slicecore_plugin::discovery::discover_all_with_status(plugin_dir)?`
   - Map each DiscoveredPlugin to a display row:
     - If manifest is Some: use metadata.name, metadata.version, plugin_type display, first capability as category
     - If manifest is None (broken): use dir name for Name, "???" for Version/Type, "???" for Category
     - Status: if error is Some -> "Error: {brief}", else if status.enabled -> "Enabled", else "Disabled"
   - Apply --category filter: match "infill" to InfillPattern, "postprocessor" to GcodePostProcessor
   - Apply --status filter: match "enabled"/"disabled"/"error" to actual state
   - If --json: serialize as flat JSON array using serde_json::to_string_pretty with a PluginListEntry struct that has fields: name, version, plugin_type, category, status, path, error
   - If not --json: use comfy_table with UTF8_FULL preset, headers ["Name", "Version", "Type", "Category", "Status"]

4. Implement `fn cmd_enable(plugin_dir: &Path, name: &str) -> Result<(), anyhow::Error>`:
   - Call discover_all_with_status to find the plugin by name (match manifest.metadata.name or fall back to dir name)
   - If not found, return error "Plugin '{name}' not found in {plugin_dir}"
   - Read current status; if already enabled, print "Plugin '{name}' is already enabled" and return Ok
   - Attempt validation: create a temporary PluginRegistry and try discover_and_load on the single plugin dir. If load fails, print error "Plugin '{name}' failed validation: {err}. Remains disabled." and return Err.
   - On success: write_status with enabled=true, print "Plugin '{name}' enabled successfully"

5. Implement `fn cmd_disable(plugin_dir: &Path, name: &str) -> Result<(), anyhow::Error>`:
   - Find plugin by name in discover_all_with_status results (match manifest.metadata.name or fall back to dir name)
   - If not found, return error "Plugin '{name}' not found in {plugin_dir}"
   - **Validate plugin identity before disabling (per user decision):** Check that the discovered plugin has `manifest.is_some()` -- this confirms the plugin.toml is parseable and the plugin is a real, identifiable plugin. If manifest is None (broken/unparseable plugin.toml), return error "Plugin '{name}' has an invalid manifest and cannot be managed. Run `slicecore plugins validate {name}` for details."
   - If already disabled, print "Plugin '{name}' is already disabled" and return Ok
   - Write_status with enabled=false, print "Plugin '{name}' disabled"

6. Implement `fn cmd_info(plugin_dir: &Path, name: &str, json: bool) -> Result<(), anyhow::Error>`:
   - Find plugin by name in discover_all_with_status results
   - If --json: serialize full details as JSON (name, version, description, author, license, plugin_type, capabilities, status, path, min_api_version, max_api_version, library_filename, error if any)
   - If not --json: print labeled fields one per line:
     ```
     Name:           zigzag-infill
     Version:        1.0.0
     Description:    Zigzag infill pattern
     Author:         Test Author
     License:        MIT
     Type:           native
     Capabilities:   infill_pattern
     Status:         Enabled
     Path:           /path/to/plugin/dir
     API Version:    0.1.0 - 0.2.0
     Library:        libzigzag_infill.so
     ```
   - For broken plugins, show whatever is available plus the error

7. Implement `fn cmd_validate(plugin_dir: &Path, name: &str) -> Result<(), anyhow::Error>`:
   - Find plugin by name
   - Check manifest parse (if None, report "Manifest parse failed: {error}")
   - Check version compatibility (reparse and validate)
   - Attempt to load: create PluginRegistry, call discover_and_load scoped to just this plugin's dir
   - Print results: "Manifest:    OK", "API Version: OK (0.1.0)", "Load Test:   OK" or FAIL with reason
   - Return Ok if all pass, Err if any fail

All functions use `anyhow::Error` for the return type (consistent with CLI pattern). Import `use slicecore_plugin::{discovery, status, PluginStatus, DiscoveredPlugin, PluginRegistry};` and `use slicecore_plugin_api::{PluginCapability, PluginManifest};`.
