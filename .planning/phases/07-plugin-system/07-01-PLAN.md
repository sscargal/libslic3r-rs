Purpose: This is the foundation of the three-crate plugin architecture. Both the host (slicecore-plugin) and individual plugin crates depend on this crate. Type layout agreement at load time prevents undefined behavior.

Output: A compilable `slicecore-plugin-api` crate with `#[sabi_trait]` InfillPatternPlugin, FFI-safe request/result types, plugin metadata, and FFI-safe error types.

    - name = "slicecore-plugin-api"
    - Use workspace version, edition, license, rust-version
    - Dependencies: abi_stable = "0.11", serde = workspace, semver = "1"
    - No dependency on slicecore-math, slicecore-geo, or slicecore-engine (plugins must not depend on internal crates)

    **src/types.rs:** Define FFI-safe types that mirror the internal infill types:
    - `FfiInfillLine`: `#[repr(C)] #[derive(StableAbi, Clone, Debug)]` with start_x, start_y, end_x, end_y (all i64)
    - `InfillRequest`: `#[repr(C)] #[derive(StableAbi, Clone, Debug)]` with:
      - `boundary_points: RVec<i64>` (flattened x,y pairs)
      - `boundary_lengths: RVec<u32>` (number of points per polygon)
      - `density: f64`, `layer_index: u64`, `layer_z: f64`, `line_width: f64`
    - `InfillResult`: `#[repr(C)] #[derive(StableAbi, Clone, Debug)]` with `lines: RVec<FfiInfillLine>`
    - Use `abi_stable::std_types::{RVec, RString}` -- NOT std Vec/String

    **src/error.rs:** Define FFI-safe error type:
    - `PluginError`: `#[repr(C)] #[derive(StableAbi, Clone, Debug)]` with `message: RString`
    - Implement Display and std::error::Error for PluginError

    **src/metadata.rs:** Define plugin metadata structures (plain serde, not FFI):
    - `PluginMetadata` with: name (String), version (String via semver), description (String), author (String), license (String), min_api_version (String), max_api_version (String)
    - `PluginCapability` enum: InfillPattern (other extension points deferred)
    - `PluginManifest` with: metadata (PluginMetadata), plugin_type (PluginType enum: Native/Wasm), library_filename (String), capabilities (Vec<PluginCapability>), resource_limits (Option<ResourceLimits>)
    - `ResourceLimits` with: max_memory_mb (u64), max_cpu_fuel (u64)
    - All derive Serialize, Deserialize, Debug, Clone

    **src/lib.rs:** Re-export all public types from types, error, metadata modules. Include module-level rustdoc describing the three-crate architecture.

  <name>Task 2: Define sabi_trait InfillPatternPlugin and RootModule</name>
  <files>

  <action>
    **src/traits.rs:** Define the FFI-safe plugin trait using abi_stable's sabi_trait macro:

    ```rust
    use abi_stable::prelude::*;
    use abi_stable::sabi_trait;
    use abi_stable::std_types::{RResult, RString};
    use crate::types::{InfillRequest, InfillResult};

    #[sabi_trait]
    pub trait InfillPatternPlugin: Send + Sync + Debug {
        fn name(&self) -> RString;
        fn description(&self) -> RString;
        #[sabi(last_prefix_field)]
        fn generate(&self, request: &InfillRequest) -> RResult<InfillResult, RString>;
    }
    ```

    This generates `InfillPatternPlugin_TO` (trait object wrapper) and `InfillPatternPluginBox` (boxed trait object) types automatically.

    Define the RootModule for native plugins:

    ```rust
    #[repr(C)]
    #[derive(StableAbi)]
    #[sabi(kind(Prefix))]
    pub struct InfillPluginMod {
        #[sabi(last_prefix_field)]
        pub new: extern "C" fn() -> InfillPatternPlugin_TO<'static, RBox<()>>,
    }
    ```

    Implement `RootModule` for `InfillPluginMod_Ref`:
    - BASE_NAME: "slicecore_infill_plugin"
    - NAME: "slicecore_infill_plugin"
    - VERSION_STRINGS: `package_version_strings!()`
    - `declare_root_module_statics! { InfillPluginMod_Ref }`

    Update **src/lib.rs** to re-export traits module and all generated types (InfillPatternPlugin_TO, InfillPatternPluginBox, InfillPluginMod, InfillPluginMod_Ref).
  </action>
