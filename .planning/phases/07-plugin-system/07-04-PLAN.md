<objective>
Integrate the plugin system into the slicing engine by adding InfillPattern::Plugin(String) variant, wiring generate_infill to dispatch to the PluginRegistry, and making the Engine hold and use an optional PluginRegistry.

Purpose: This connects the plugin infrastructure (plans 01-03) to the existing slicing pipeline. After this plan, a user can configure `infill_pattern = { plugin = "zigzag" }` in their TOML config and the engine will use a plugin-provided infill pattern.

Output: Modified engine that supports plugin infill patterns alongside all existing built-in patterns with zero regression.
</objective>

  <files>

  <action>
    **Cargo.toml updates:**
    - Add optional dependency: `slicecore-plugin = { path = "../slicecore-plugin", optional = true }`
    - Add feature: `plugins = ["dep:slicecore-plugin"]`
    - This keeps the engine compilable without the plugin system (important for WASM target)

    **src/infill/mod.rs changes:**

    1. Change `InfillPattern` from `#[derive(Copy)]` to just `#[derive(Clone)]` (Plugin(String) is not Copy):
    ```rust
    #[derive(Debug, Default, Clone, PartialEq, Eq, Serialize, Deserialize)]
    #[serde(rename_all = "snake_case")]
    pub enum InfillPattern {
        #[default]
        Rectilinear,
        Grid,
        Honeycomb,
        Gyroid,
        AdaptiveCubic,
        Cubic,
        Lightning,
        Monotonic,
        TpmsD,
        TpmsFk,
        /// A plugin-provided infill pattern, identified by registered name.
        #[serde(rename = "plugin")]
        Plugin(String),
    }
    ```

    IMPORTANT: Removing `Copy` from InfillPattern will cause compilation errors everywhere it's copied. Search for all uses and add `.clone()` where needed. Key locations:
    - `config.rs`: PrintConfig uses `infill_pattern` field -- may need `.clone()` on access
    - `engine.rs`: Multiple places call `generate_infill(self.config.infill_pattern, ...)` -- change to `self.config.infill_pattern.clone()`
    - `modifier.rs`: region_config.infill_pattern access
    - Any test files that copy InfillPattern values

    2. Update `generate_infill` signature to accept optional plugin registry:
    ```rust
    pub fn generate_infill(
        pattern: InfillPattern,  // Now Clone, not Copy
        infill_region: &[ValidPolygon],
        density: f64,
        layer_index: usize,
        layer_z: f64,
        line_width: f64,
        lightning_context: Option<&lightning::LightningContext>,
        #[cfg(feature = "plugins")]
        plugin_registry: Option<&slicecore_plugin::PluginRegistry>,
    ) -> Vec<InfillLine> {
    ```

    Actually, to avoid cfg complexity in the function signature, use a simpler approach:
    - Add a separate function `generate_plugin_infill` gated behind cfg(feature = "plugins")
    - In the main `generate_infill` match, the Plugin variant calls this helper
    - Without the plugins feature, Plugin variant returns empty vec with a warning or panics with a clear message

    Better approach for clean API:
    ```rust
    pub fn generate_infill(
        pattern: &InfillPattern,  // Take by reference to avoid clone at call sites
        infill_region: &[ValidPolygon],
        density: f64,
        layer_index: usize,
        layer_z: f64,
        line_width: f64,
        lightning_context: Option<&lightning::LightningContext>,
    ) -> Vec<InfillLine> {
        // ... existing match arms ...
        InfillPattern::Plugin(ref _name) => {
            // Plugin dispatch handled by engine directly, not here.
            // If we reach here, it means the engine didn't intercept the Plugin variant.
            // Return empty as fallback.
            Vec::new()
        }
    }
    ```

    The engine will check for Plugin variant BEFORE calling generate_infill, and route to the registry directly. This avoids threading the registry through the infill module.

    CRITICAL: Change `pattern: InfillPattern` to `pattern: &InfillPattern` in the generate_infill signature. This avoids the Copy issue entirely. Update ALL call sites to pass `&pattern` or `&self.config.infill_pattern`. This is cleaner than adding .clone() everywhere.

    3. Update `generate_rectilinear_infill` and any other functions that take InfillPattern by value.

    **src/error.rs:** Add Plugin error variant:
    ```rust
    #[error("Plugin error in '{plugin}': {message}")]
    Plugin { plugin: String, message: String },
    ```

    **src/engine.rs:**

    1. Add optional PluginRegistry to Engine struct:
    ```rust
    pub struct Engine {
        config: PrintConfig,
        #[cfg(feature = "plugins")]
        plugin_registry: Option<slicecore_plugin::PluginRegistry>,
    }
    ```

    2. Add builder method:
    ```rust
    #[cfg(feature = "plugins")]
    pub fn with_plugin_registry(mut self, registry: slicecore_plugin::PluginRegistry) -> Self {
        self.plugin_registry = Some(registry);
        self
    }
    ```

    3. In all places where `generate_infill` is called with `self.config.infill_pattern`:
    - Check if pattern is `InfillPattern::Plugin(name)` FIRST
    - If yes AND plugins feature enabled AND registry exists:
      - Use `slicecore_plugin::convert::regions_to_request()` to create FFI request
      - Call `registry.get_infill_plugin(name).generate(request)`
      - Use `slicecore_plugin::convert::ffi_result_to_lines()` to convert back
      - Convert `ConvertedInfillLine` to `InfillLine` (start/end fields match)
      - On error, return `EngineError::Plugin { plugin, message }`
    - If yes AND (no plugins feature OR no registry):
      - Return `EngineError::Plugin { plugin: name, message: "Plugin system not available" }`
    - If not Plugin, call `generate_infill()` as before

    Extract the plugin dispatch into a helper method on Engine to avoid duplicating the logic at each call site:
    ```rust
    fn generate_infill_for_layer(
        &self,
        pattern: &InfillPattern,
        regions: &[ValidPolygon],
        density: f64,
        layer_idx: usize,
        layer_z: f64,
        line_width: f64,
        lightning_ctx: Option<&lightning::LightningContext>,
    ) -> Result<Vec<InfillLine>, EngineError> {
        match pattern {
            InfillPattern::Plugin(name) => {
                self.generate_plugin_infill(name, regions, density, layer_idx, layer_z, line_width)
            }
            _ => Ok(generate_infill(pattern, regions, density, layer_idx, layer_z, line_width, lightning_ctx)),
        }
    }
    ```

    4. Update the `slice` method and `slice_with_preview` method to use `generate_infill_for_layer` and propagate the Result. Also update modifier mesh processing path.
