<objective>
Create the `slicecore-plugin` crate with the PluginRegistry, native plugin loader via abi_stable, plugin discovery via manifest scanning, and type conversion utilities between internal and FFI-safe types.

Purpose: This is the host-side plugin infrastructure. It discovers plugins on disk, validates compatibility, loads native dynamic libraries with full ABI verification, and presents a unified API for the engine to call plugin-provided infill patterns.

Output: A working `slicecore-plugin` crate that can discover, load, and call native infill plugins from a directory.
</objective>

    **src/convert.rs:** Conversion functions between internal slicecore types and FFI-safe plugin types:

    ```rust
    use slicecore_plugin_api::{FfiInfillLine, InfillRequest, InfillResult};
    use slicecore_geo::polygon::ValidPolygon;
    use slicecore_math::IPoint2;
    use abi_stable::std_types::RVec;

    /// Convert internal ValidPolygon regions to FFI-safe InfillRequest.
    pub fn regions_to_request(
        regions: &[ValidPolygon],
        density: f64,
        layer_index: usize,
        layer_z: f64,
        line_width: f64,
    ) -> InfillRequest {
        let mut boundary_points = RVec::new();
        let mut boundary_lengths = RVec::new();
        for poly in regions {
            let pts = poly.points();
            boundary_lengths.push(pts.len() as u32);
            for pt in pts {
                boundary_points.push(pt.x);
                boundary_points.push(pt.y);
            }
        }
        InfillRequest {
            boundary_points,
            boundary_lengths,
            density,
            layer_index: layer_index as u64,
            layer_z,
            line_width,
        }
    }

    /// Convert FFI-safe infill result back to internal InfillLine vec.
    pub fn ffi_result_to_infill_lines(result: &InfillResult) -> Vec<crate::InfillLine> {
        // Note: This returns a type that the engine can use.
        // The caller will need to construct InfillLine from IPoint2.
        result.lines.iter().map(|line| {
            // Return (start, end) pairs for the engine to construct InfillLine
            (IPoint2::new(line.start_x, line.start_y), IPoint2::new(line.end_x, line.end_y))
        }).collect()
    }
    ```

    Actually, since slicecore-plugin depends on slicecore-math but NOT slicecore-engine (to avoid circular deps), the convert module returns tuples of (IPoint2, IPoint2). The engine integration (plan 04) will handle constructing InfillLine from these.
