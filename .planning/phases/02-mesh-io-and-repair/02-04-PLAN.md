<objective>
Add 3MF and OBJ file parsers to slicecore-fileio, implement the unified load_mesh() function, and validate WASM compilation.

Purpose: 3MF is the modern successor to STL, used by Bambu Studio/OrcaSlicer. OBJ is widely used in CAD/3D modeling. Both are required for Phase 2 success criterion 1 (loading 4 formats). The unified load_mesh() function provides a single entry point for downstream consumers. WASM validation ensures these dependencies work in the browser target.

Output: Complete slicecore-fileio crate supporting STL (binary + ASCII), 3MF, and OBJ with unified loading.
</objective>

  <action>
    1. Update `crates/slicecore-fileio/Cargo.toml`:
       - Add `lib3mf = "0.1"` to dependencies
       - Add `tobj = { version = "4", default-features = false }` to dependencies
       - NOTE: tobj default-features may pull in image/texture loading; use `default-features = false` and enable only what's needed for mesh loading. Check if `load_obj_buf` is available without default features. If not, enable minimal features.

    2. Create `src/threemf.rs`:
       - `pub fn parse(data: &[u8]) -> Result<TriangleMesh, FileIOError>`
       - Use `lib3mf::Model::from_reader(Cursor::new(data))` to parse the 3MF
       - Iterate over `model.resources.objects`, extract meshes:
         a. For each object with a mesh, collect vertices (converting f32->f64 for Point3)
         b. Collect triangle indices with vertex_offset for multi-object merging
       - Return EmptyModel if no mesh data found
       - Construct `TriangleMesh::new(all_vertices, all_indices)`
       - Handle lib3mf errors by mapping to `FileIOError::ThreeMfError(e.to_string())`
       - NOTE: lib3mf API may differ from research examples. Read the actual lib3mf docs/source to confirm the API shape. The key operations are: parse from reader, iterate objects, get mesh vertices and triangles.
       - Tests:
         * Create a minimal 3MF test fixture (this is complex since 3MF is ZIP+XML -- may need to create one programmatically or use a known-good minimal 3MF bytes). Alternative: test with a hand-crafted 3MF created using the lib3mf write API if available, or skip fixture and test the parsing logic with mock data if the lib3mf API allows.
         * Test empty model returns EmptyModel error
         * Test multi-object 3MF correctly offsets vertex indices

    3. Create `src/obj.rs`:
       - `pub fn parse(data: &[u8]) -> Result<TriangleMesh, FileIOError>`
       - Use `tobj::load_obj_buf()` with a `Cursor` reader and `LoadOptions { triangulate: true, .. }` to auto-triangulate quads/n-gons
       - Extract vertices from model positions (groups of 3 f32 values)
       - Extract face indices from model indices (groups of 3 u32 values)
       - Convert to `Vec<Point3>` and `Vec<[u32; 3]>`
       - Handle tobj errors by mapping to `FileIOError::ObjError(e.to_string())`
       - If tobj's load_obj_buf requires a material reader, pass a closure that returns an empty material (OBJ material support is not needed for mesh geometry).
       - Tests:
         * Test with a minimal OBJ string (cube with 8 vertices and 12 triangular faces)
         * Test with quad faces (should be triangulated to 2 triangles each)
         * Test empty OBJ returns EmptyModel

    4. Update `src/lib.rs`:
       - Add module declarations: `pub mod threemf;` and `pub mod obj;`
       - Implement `pub fn load_mesh(data: &[u8]) -> Result<TriangleMesh, FileIOError>`:
         a. Call `detect_format(data)?`
         b. Match on format: StlBinary -> stl_binary::parse, StlAscii -> stl_ascii::parse, ThreeMf -> threemf::parse, Obj -> obj::parse
       - Implement `pub fn load_mesh_from_reader<R: std::io::Read>(reader: &mut R) -> Result<TriangleMesh, FileIOError>`:
         a. Read all bytes into Vec<u8>
         b. Call load_mesh(&data)
       - Re-export load_mesh and load_mesh_from_reader
       - Tests:
         * Test load_mesh dispatches STL binary correctly (same bytes as stl_binary tests)
         * Test load_mesh dispatches ASCII STL correctly
