<objective>
Create the slicecore-fileio crate with STL parsing (binary and ASCII) and magic-byte format detection.

Purpose: STL is the most common 3D model format. This plan establishes the file I/O crate structure and delivers the most critical import capability. Binary STL is the standard output of most CAD tools; ASCII STL is an alternative format. The parser must deduplicate vertices (binary STL has no shared vertex buffer) and handle the well-known "binary STL starting with 'solid'" ambiguity.

Output: New `crates/slicecore-fileio/` crate with working STL import and format detection.
</objective>

    3. Create `src/error.rs` with `FileIOError` enum:
       - `FileTooSmall` - file too small to identify format
       - `UnrecognizedFormat` - cannot detect format from content
       - `UnexpectedEof` - file truncated before expected end
       - `InvalidUtf8` - ASCII STL contains non-UTF8 bytes
       - `ParseError(String)` - parse failure with description
       - `ThreeMfError(String)` - 3MF-specific errors (placeholder for plan 02-04)
       - `ObjError(String)` - OBJ-specific errors (placeholder for plan 02-04)
       - `EmptyModel` - file contains no mesh data
       - `MeshError(slicecore_mesh::MeshError)` - upstream mesh construction error, with `#[from]`
       - `IoError(std::io::Error)` - I/O errors, with `#[from]`
       All variants derive thiserror::Error with descriptive messages.

    4. Create `src/detect.rs` implementing format detection:
       - `pub enum MeshFormat { StlBinary, StlAscii, ThreeMf, Obj }`
       - `pub fn detect_format(data: &[u8]) -> Result<MeshFormat, FileIOError>`
       - Detection logic (order matters):
         a. 3MF: starts with ZIP magic bytes `[0x50, 0x4B, 0x03, 0x04]`
         b. ASCII STL: starts with "solid" AND first ~1000 bytes contain "facet normal"
         c. Binary STL: length >= 84 AND file_size matches `84 + num_triangles * 50` (+-1 byte tolerance)
         d. OBJ: first non-comment line starts with "v " (vertex definition)
         e. If none match: `Err(FileIOError::UnrecognizedFormat)`
       - IMPORTANT: Handle the "binary STL starting with 'solid'" pitfall by checking for "facet normal" presence before classifying as ASCII STL. If "solid" is present but no "facet normal", fall through to binary size check.

    5. Create `src/stl_binary.rs`:
       - `pub fn parse(data: &[u8]) -> Result<TriangleMesh, FileIOError>`
       - Use `byteorder::ReadBytesExt` with `LittleEndian` for reading f32 values
       - Skip 80-byte header, read u32 triangle count
       - Validate file size: `data.len() >= 84 + num_triangles * 50`
       - Parse each triangle (50 bytes): skip 12-byte normal, read 3x (3x f32) vertices, skip 2-byte attribute
       - Deduplicate vertices using `HashMap<[i64; 3], u32>` with quantized key: `(coord * 1e5).round() as i64` for each x,y,z component
       - Construct `TriangleMesh::new(vertices, indices)`, map MeshError via `?`
       - TDD: Write tests FIRST:
         * Test known unit cube binary STL (hand-craft 12-triangle binary STL bytes)
         * Test vertex deduplication (shared vertices produce fewer vertices than 3 * triangle_count)
         * Test truncated file produces UnexpectedEof
         * Test empty file (< 84 bytes) produces FileTooSmall

    6. Create `src/lib.rs` with module declarations and re-exports:
       - `pub mod detect;`
       - `pub mod error;`
       - `pub mod stl_binary;`
       - Re-export: `pub use detect::{detect_format, MeshFormat};`
       - Re-export: `pub use error::FileIOError;`
