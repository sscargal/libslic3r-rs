       - Helper function: `make_ascii_stl_cube() -> Vec<u8>` that constructs a valid ASCII STL string of a unit cube
       - Helper function: `make_obj_cube() -> Vec<u8>` that constructs a valid OBJ file of a unit cube (8 vertices, 12 triangular faces)
       - Test `load_mesh_binary_stl`: load binary STL cube via load_mesh(), verify 8 vertices, 12 triangles
       - Test `load_mesh_ascii_stl`: load ASCII STL cube via load_mesh(), verify 8 vertices, 12 triangles
       - Test `load_mesh_obj`: load OBJ cube via load_mesh(), verify 8 vertices, 12 triangles
       - Test `load_mesh_unrecognized`: pass random bytes to load_mesh(), verify UnrecognizedFormat error
       - Test `format_detection_binary_stl_with_solid_header`: create binary STL with "solid" in header, verify it loads as binary (not ASCII)
       - NOTE: 3MF integration test may be limited since creating a valid 3MF programmatically requires ZIP creation. If lib3mf provides a write API, create a round-trip test.     2. Create `crates/slicecore-mesh/tests/repair_integration.rs`:
       - Helper function: `make_cube_with_degenerate() -> (Vec<Point3>, Vec<[u32; 3]>)` that creates a unit cube plus one degenerate triangle (two identical vertex indices)
       - Helper function: `make_cube_with_flipped_normal() -> (Vec<Point3>, Vec<[u32; 3]>)` that creates a unit cube with one triangle's winding reversed
       - Helper function: `make_cube_with_hole() -> (Vec<Point3>, Vec<[u32; 3]>)` that creates a unit cube with one face (2 triangles) removed
       - Test `repair_removes_degenerate_triangles`: pass cube_with_degenerate through repair(), verify degenerate_removed = 1, resulting mesh has 12 triangles
       - Test `repair_fixes_normals`: pass cube_with_flipped_normal through repair(), verify normals_fixed >= 1
       - Test `repair_fills_holes`: pass cube_with_hole through repair(), verify holes_filled >= 2 (at least 2 triangles added to fill one quad hole)
       - Test `repair_clean_mesh_reports_already_clean`: pass a valid unit cube through repair(), verify was_already_clean = true (or at least degenerate_removed + holes_filled == 0)
       - Test `repair_then_stats`: repair a defective mesh, then compute MeshStats on the result -- verify volume > 0 (positive volume indicates correct winding)

    3. Test the load-repair pipeline:
       - In fileio integration tests: load a binary STL cube, then call repair::repair(mesh.vertices().to_vec(), mesh.indices().to_vec()), verify the result is a valid mesh
       - This proves the two crates work together end-to-end
  </action>
