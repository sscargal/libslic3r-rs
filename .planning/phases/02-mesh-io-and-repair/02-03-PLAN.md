    3. Create `src/commands.rs` with structured command types:
       - `pub enum GcodeCommand`:
         * `Comment(String)` -- ; comment
         * `LinearMove { x: Option<f64>, y: Option<f64>, z: Option<f64>, e: Option<f64>, f: Option<f64> }` -- G1 with optional params
         * `RapidMove { x: Option<f64>, y: Option<f64>, z: Option<f64>, f: Option<f64> }` -- G0
         * `Home { x: bool, y: bool, z: bool }` -- G28 with optional axis selection
         * `SetAbsolutePositioning` -- G90
         * `SetRelativePositioning` -- G91
         * `SetAbsoluteExtrusion` -- M82
         * `SetRelativeExtrusion` -- M83
         * `SetExtruderTemp { temp: f64, wait: bool }` -- M104/M109
         * `SetBedTemp { temp: f64, wait: bool }` -- M140/M190
         * `SetFanSpeed(u8)` -- M106 S(0-255)
         * `FanOff` -- M107
         * `ResetExtruder` -- G92 E0
         * `Dwell { ms: u32 }` -- G4 P{ms}
         * `Retract { distance: f64, feedrate: f64 }` -- helper that emits G1 E-{d} F{f}
         * `Unretract { distance: f64, feedrate: f64 }` -- helper that emits G1 E{d} F{f}
         * `Raw(String)` -- pass-through for arbitrary G-code lines
       - Implement `Display` for `GcodeCommand` that formats each variant as its G-code text line
       - TDD: Write tests FIRST:
         * Test LinearMove formatting: `G1 X1.000 Y2.000 Z0.300 E0.500 F1800.0`
         * Test RapidMove formatting: `G0 X10.000 Y10.000 F6000.0`
         * Test SetExtruderTemp wait=true: `M109 S200`
         * Test Comment: `; this is a comment`
         * Test coordinates use 3 decimal places, feedrate uses 1 decimal place, temperature uses 0 decimal places
         * Test optional parameters omitted when None

    4. Create `src/dialect.rs`:
       - `pub enum GcodeDialect { Marlin, Klipper, RepRapFirmware, Bambu }`
       - Derive: `Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize`
       - `pub struct StartConfig { pub bed_temp: f64, pub nozzle_temp: f64, pub bed_x: f64, pub bed_y: f64 }`
       - `pub struct EndConfig { pub retract_distance: f64 }`

    5. Create `src/writer.rs`:
       - `pub struct GcodeWriter<W: std::io::Write>` with fields:
         * `writer: W`
         * `dialect: GcodeDialect`
         * `line_count: u32`
       - Methods:
         * `pub fn new(writer: W, dialect: GcodeDialect) -> Self`
         * `pub fn write_command(&mut self, cmd: &GcodeCommand) -> Result<(), GcodeError>` -- format and write one command
         * `pub fn write_commands(&mut self, cmds: &[GcodeCommand]) -> Result<(), GcodeError>` -- write multiple commands
         * `pub fn write_start_gcode(&mut self, config: &StartConfig) -> Result<(), GcodeError>` -- dialect-specific start
         * `pub fn write_end_gcode(&mut self, config: &EndConfig) -> Result<(), GcodeError>` -- dialect-specific end
         * `pub fn line_count(&self) -> u32`
         * `pub fn into_inner(self) -> W` -- consume writer, return inner writer
       - Start/end G-code dispatches to dialect-specific modules

    6. Create `src/marlin.rs`:

         * Actually use M83 (relative extrusion) per research recommendation -- simpler, avoids E-axis overflow, works with all firmware
         * M140 S{bed_temp} (set bed temp, no wait)
         * M104 S{nozzle_temp} (set extruder temp, no wait)
         * M190 S{bed_temp} (wait for bed temp)
         * M109 S{nozzle_temp} (wait for extruder temp)
         * G28 (home all axes)
         * G92 E0 (reset extruder)
       - `pub fn end_gcode(config: &EndConfig) -> Vec<GcodeCommand>`:

         * G1 E-{retract_distance} F2700 (retract)
         * G90 (absolute positioning)
         * M104 S0 (extruder off)
         * M140 S0 (bed off)
         * M107 (fan off)
         * G28 X Y (home X/Y)
         * M84 or Raw("M84") (disable steppers)

         * Test end_gcode contains M104 S0, M140 S0, M107
         * Test full round-trip: write start + commands + end to Vec<u8>, verify output is valid text
