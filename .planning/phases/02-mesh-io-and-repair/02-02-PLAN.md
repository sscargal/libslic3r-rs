Output: `repair` module in `crates/slicecore-mesh/` with a full repair pipeline and RepairReport.

       - BFS flood-fill from triangle 0:
         a. For each edge of current triangle, find neighbor triangles
         b. If neighbor has same edge direction (not reversed), flip the neighbor (swap indices[1] and indices[2])

  <action>
    1. Create `src/repair/stitch.rs`:
       - `pub fn stitch_edges(vertices: &mut Vec<Point3>, indices: &mut Vec<[u32; 3]>, tolerance: f64) -> usize`
       - Find boundary edges (edges that appear in only one triangle)
       - For each boundary edge vertex, find nearby vertices (within tolerance) using spatial hashing:
         a. Quantize vertex positions to grid cells of size `tolerance`
         b. For each boundary vertex, check neighboring grid cells for merge candidates
         c. Merge: update all index references from duplicate vertex to kept vertex
       - Remove resulting degenerate triangles (merged vertices may create zero-area triangles)
       - Return count of stitched edges
       - Default STITCH_TOLERANCE: `1e-4` (0.1 micron, well below print resolution)
       - TDD: Write tests FIRST:
         * Test with two separate triangles sharing an edge but with duplicated vertices at slightly different positions (within tolerance) -- stitched, vertex count reduced
         * Test with vertices beyond tolerance -- not stitched

    2. Create `src/repair/holes.rs`:
       - `pub fn fill_holes(vertices: &[Point3], indices: &mut Vec<[u32; 3]>) -> usize`
       - Find boundary edge loops:
         a. Build directed edge map from triangle winding
         b. Find edges with no matching reverse edge (boundary edges)
         c. Chain boundary edges into loops
       - For each boundary loop, triangulate using fan triangulation from loop centroid:
         a. Add centroid vertex (or use first vertex of loop for simple fan)
         b. Create triangles connecting consecutive boundary vertices to the fan center
         c. Add new triangles to indices
       - Return count of triangles added
       - TDD: Write tests FIRST:
         * Test with a cube missing one face (2 triangles) -- hole detected and filled
         * Test with a complete cube -- no holes found, count = 0

    3. Create `src/repair/intersect.rs`:
       - `pub fn detect_self_intersections(vertices: &[Point3], indices: &[[u32; 3]]) -> usize`
       - Build a temporary BVH from the triangle AABBs using `BVH::build(vertices, indices)`
       - For each triangle, query BVH for overlapping AABBs
       - For each candidate pair (i, j) where i < j and triangles don't share a vertex:
         a. Perform Moller triangle-triangle intersection test
         b. If intersecting, increment count
       - Return count of self-intersecting pairs
       - NOTE: This is detection only, not repair. Report the count in RepairReport.
       - Moller triangle-triangle intersection algorithm:
         a. Compute plane of triangle 2, check if triangle 1 vertices are on both sides
         b. Compute plane of triangle 1, check if triangle 2 vertices are on both sides
         c. If both tests pass, compute line of plane intersection, check overlap of projected intervals
       - TDD: Write tests FIRST:
         * Test with two non-intersecting triangles -- count = 0
         * Test with two intersecting triangles (crossing through each other) -- count = 1

    4. Update `src/repair.rs` (repair pipeline):
       - Wire in stitch_edges, fill_holes, and detect_self_intersections into the repair() function
       - Full pipeline order: (1) remove degenerates, (2) stitch edges, (3) fill holes, (4) fix normals, (5) recompute normals, (6) detect self-intersections
       - After all repairs, construct `TriangleMesh::new(vertices, indices)` and return with report
       - Set `was_already_clean = true` if all counts are zero
       - Add submodule declarations in repair.rs: `pub mod degenerate; pub mod normals; pub mod stitch; pub mod holes; pub mod intersect;`
