Build the ProfileComposer core: TOML value tree deep-merge with provenance tracking, --set parsing, and conflict detection.

Purpose: This is the foundational merge engine that combines 5 layers (Defaults -> Machine -> Filament -> Process -> User overrides -> --set) into a final PrintConfig with per-field provenance. Operates on toml::Value trees, not PrintConfig structs, to preserve the distinction between "not set" and "set to default."

Output: profile_compose.rs module with ComposedConfig, FieldSource, SourceType types, merge_layer function, --set value parsing, dotted key path insertion, and conflict detection.

  <action>
    Add `sha2 = "0.10"` and `strsim = "0.11"` to slicecore-engine Cargo.toml dependencies.

    Create `crates/slicecore-engine/src/profile_compose.rs` with:

    1. **Types:** `SourceType` enum (Default, Machine, Filament, Process, UserOverride, CliSet), `FieldSource` struct (source_type, file_path: Option<String>, overrode: Option<Box<FieldSource>>), `ComposedConfig` struct (config: PrintConfig, provenance: HashMap<String, FieldSource>, warnings: Vec<String>, profile_checksums: Vec<(String, String)>).

    2. **merge_layer():** Deep-merge a toml::map::Map layer into a base table, recording provenance at each leaf with dotted key paths. When both base and layer have a Table for the same key, recurse. Otherwise, leaf values override and record in provenance. If a field was already in provenance, set overrode to the previous source.

    3. **ProfileComposer** struct with a `compose()` method that:
       - Starts from PrintConfig::default() serialized to toml::Value::Table
       - Merges layers in order: machine TOML, filament TOML, process TOML, user overrides TOML, --set values
       - Records provenance at each step
       - Generates conflict warnings (same non-default field set by 2+ layers)
       - Computes SHA-256 checksums for each input profile file content
       - Deserializes final merged table back into PrintConfig
       - Returns ComposedConfig

    4. **parse_set_value():** Parse a string value as TOML literal -- try i64, f64, bool, fallback to String.

    5. **set_dotted_key():** Insert a value at a dotted key path (e.g., "speeds.perimeter") into a TOML table, creating intermediate tables as needed.

    6. **validate_set_key():** Check a dotted key path against valid PrintConfig field paths (built by recursively walking the default config's TOML table). If invalid, use strsim::jaro_winkler to suggest similar keys with "did you mean?" message.

    Add `pub mod profile_compose;` to lib.rs.

    Unit tests in the module: test merge ordering, provenance tracking, conflict detection, parse_set_value coercion for int/float/bool/string, set_dotted_key for nested paths, validate_set_key with typo suggestions.
