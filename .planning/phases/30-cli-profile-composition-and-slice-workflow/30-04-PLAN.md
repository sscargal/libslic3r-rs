<task type="auto">
  <name>Task 1: Add new CLI flags to Slice command and create slice_workflow orchestrator</name>
  <files>crates/slicecore-cli/Cargo.toml, crates/slicecore-cli/src/main.rs, crates/slicecore-cli/src/slice_workflow.rs</files>
  <action>
    Add `indicatif = "0.17"` to slicecore-cli Cargo.toml dependencies.

    **In main.rs**, add new fields to the Slice variant of Commands enum:

    ```
    // Profile composition flags (mutually exclusive with --config)
    -m/--machine: Option<String>     // Machine profile name or path
    -f/--filament: Option<String>    // Filament profile name or path
    -p/--process: Option<String>     // Process profile name or path
    --overrides: Option<PathBuf>     // TOML/JSON override file
    --set: Vec<String>               // Repeatable key=value overrides

    // Workflow flags
    --dry-run: bool                  // Resolve + merge + validate, skip slice
    --save-config: Option<Option<PathBuf>>  // Save merged config (optional path)
    --show-config: bool              // Print merged config with annotations
    --unsafe-defaults: bool          // Allow slicing without profiles
    --force: bool                    // Override safety validation errors
    --no-log: bool                   // Suppress log file creation
    --log-file: Option<PathBuf>      // Custom log file path
    --profiles-dir: Option<PathBuf>  // Profile library directory override
    ```

    Use clap conflicts_with to enforce --config vs -m/-f/-p mutual exclusion. The existing `config` field stays.

    Update `cmd_slice()` signature to accept the new flags. When -m/-f/-p are provided, delegate to `run_slice_workflow()` instead of the existing direct PrintConfig::from_file path. When only --config is provided, use the existing path (backwards compatible).

    **Create `crates/slicecore-cli/src/slice_workflow.rs`:**

    `SliceWorkflowOptions` struct with all the new flag values.

    `run_slice_workflow(options: &SliceWorkflowOptions) -> Result<ComposedConfig, i32>`:

    1. **Resolve profiles:** Create ProfileResolver with profiles_dir override. Resolve -m, -f, -p values. If -p not provided, use built-in "standard" process profile. If --unsafe-defaults and no -m/-f, use PrintConfig::default() with warning on stderr.

    2. **Read and parse TOML:** For each resolved profile, read file content, parse as toml::Value::Table. For --overrides file, auto-detect TOML vs JSON (use existing PrintConfig::from_file pattern: try TOML first, fallback to JSON).

    3. **Handle inheritance:** For each profile, call resolve_inheritance to get parent chain. Merge parents first, then child.

    4. **Parse --set values:** For each --set entry, split on first '=', validate key with validate_set_key, parse value with parse_set_value.

    5. **Compose:** Use ProfileComposer.compose() with all layers in order.

    6. **Resolve template variables:** Call resolve_template_variables on start_gcode and end_gcode fields.

    7. **Validate:** Call validate_config. Print warnings to stderr. If errors exist and --force not set, print errors to stderr and return exit code 4. If --force, print errors as warnings and continue.

    8. **Handle --dry-run:** Print profile resolution summary, merged config summary, validation results to stderr. Return early with exit code 0.

    9. **Handle --show-config:** Print final config with provenance annotations. If --json flag, output as JSON. Otherwise human-readable TOML-style with "# Source: ..." comments.

    10. **Handle --save-config:** Serialize merged config as TOML with provenance as section header comments. Write to specified path or default ({input_stem}.config.toml). Include reproduce command comment at top.

    11. **Return ComposedConfig** for the caller to proceed with slicing.

    Add `mod slice_workflow;` to main.rs.

    Ensure structured exit codes: 0=success, 1=general error, 2=config/profile error (ProfileError), 3=mesh error (existing), 4=safety validation error (new).

Existing --config path still works.</done>
