  <name>Task 1: Implement Engine orchestrator</name>
  <files>
    crates/slicecore-engine/src/engine.rs
    crates/slicecore-engine/src/lib.rs
  </files>
  <action>
    Implement the Engine that orchestrates the full slicing pipeline in `engine.rs`.

    **Core types:**
    ```rust
    /// Result of a slicing operation.
    pub struct SliceResult {
        pub gcode: Vec<u8>,             // Complete G-code output as bytes
        pub layer_count: usize,         // Number of layers sliced
        pub estimated_time_seconds: f64, // Total estimated print time
    }

    /// The slicing engine -- orchestrates the full pipeline.
    pub struct Engine {
        config: PrintConfig,
    }
    ```

    **Engine implementation:**
    ```rust
    impl Engine {
        pub fn new(config: PrintConfig) -> Self
        pub fn slice(&self, mesh: &TriangleMesh) -> Result<SliceResult, EngineError>
        pub fn slice_to_writer<W: Write>(&self, mesh: &TriangleMesh, writer: W) -> Result<SliceResult, EngineError>
    }
    ```

    **`slice()` pipeline** (each stage is a pure function call):

    1. **Slice mesh**: `slicecore_slicer::slice_mesh(&mesh, config.layer_height, config.first_layer_height)` -> `Vec<SliceLayer>`.
    2. **Generate per-layer data**: For each layer:
       a. **Perimeters**: `generate_perimeters(&layer.contours, &config)` -> `Vec<ContourPerimeters>`.
       b. **Surface classification**: `classify_surfaces(&layers, layer_idx, config.top_solid_layers, config.bottom_solid_layers)` -> `SurfaceClassification`.
       c. **Infill**: For solid regions, call `generate_rectilinear_infill(solid, 1.0, angle, width)`. For sparse regions, call `generate_rectilinear_infill(sparse, config.infill_density, angle, width)`. Use `alternate_infill_angle(layer_idx)` for the angle. The infill region is `inner_contour` from the perimeters. Intersect with solid/sparse classification.
       d. **Toolpath assembly**: `assemble_layer_toolpath(layer_idx, z, layer_height, &perimeters, &infill, &config)` -> `LayerToolpath`.
    3. **First-layer extras**: Generate skirt/brim from first layer contours. Prepend their toolpath segments to layer 0's toolpath.
    4. **G-code generation**: `generate_full_gcode(&layer_toolpaths, &config)` -> `Vec<GcodeCommand>`.
    5. **Write G-code**: Create `GcodeWriter` (Marlin dialect for Phase 3). Write start gcode, then all commands, then end gcode.
    6. Return `SliceResult` with the G-code bytes, layer count, and estimated time.

    **EngineError** enum (in lib.rs or a new error.rs):
    ```rust
    #[derive(Debug, thiserror::Error)]
    pub enum EngineError {
        #[error("Mesh has no geometry")]
        EmptyMesh,
        #[error("Slicing produced no layers")]
        NoLayers,
        #[error("Config error: {0}")]
        ConfigError(String),
        #[error("G-code write error: {0}")]
        GcodeError(#[from] slicecore_gcode_io::GcodeError),
        #[error("IO error: {0}")]
        IoError(#[from] std::io::Error),
    }
    ```

    **Tests:**
    - Engine::slice on a unit cube with default config produces non-empty G-code.
    - G-code output contains G28, M83, M104/M109, G1 commands.
    - layer_count matches expected value for the cube dimensions and layer height.
    - estimated_time_seconds > 0.
    - Slicing the same mesh twice with the same config produces identical output (determinism test).
    - Engine::slice on a mesh with default config and then with layer_height=0.1 produces roughly double the layer count.

  <name>Task 2: Create CLI binary</name>
  <files>

  <action>
    Create the `slicecore-cli` binary crate with clap-based argument parsing.

    **Cargo.toml**: Binary crate with dependencies:
    - `clap = { version = "4.5", features = ["derive"] }` for CLI parsing
    - `slicecore-engine` (path) for the slicing engine
    - `slicecore-fileio` (path) for mesh loading
    - `slicecore-mesh` (path) for repair

    **main.rs**: Implement CLI with clap derive macros.

    ```rust
    #[derive(Parser)]
    #[command(name = "slicecore", about = "3D model slicer")]
    struct Cli {
        #[command(subcommand)]
        command: Commands,
    }

    #[derive(Subcommand)]
    enum Commands {
        /// Slice an STL file to G-code
        Slice {
            /// Input STL file path
            input: PathBuf,
            /// Print config TOML file (optional -- uses defaults if not provided)
            #[arg(short, long)]
            config: Option<PathBuf>,
            /// Output G-code file path (default: input with .gcode extension)
            #[arg(short, long)]
            output: Option<PathBuf>,
        },
        /// Validate a G-code file
        Validate {
            /// G-code file to validate
            input: PathBuf,
        },
        /// Analyze a mesh file (print stats)
        Analyze {
            /// Mesh file to analyze
            input: PathBuf,
        },
    }
    ```

    **Slice command implementation:**
    1. Read input file bytes.
    2. Load mesh via `slicecore_fileio::load_mesh()`.
    3. Repair mesh via `slicecore_mesh::repair()`.
    4. Load config: if --config provided, read TOML file and parse via `PrintConfig::from_toml()`. Otherwise use `PrintConfig::default()`.
    5. Create engine: `Engine::new(config)`.
    6. Slice: `engine.slice(&mesh)`.
    7. Determine output path: if --output provided, use it. Otherwise, replace input extension with `.gcode`.
    8. Write G-code bytes to output file.
    9. Print summary: layer count, estimated time, output path.

    **Validate command implementation:**
    1. Read G-code file as string.
    2. Call `slicecore_gcode_io::validate_gcode()`.
    3. Print validation result (errors and warnings).
    4. Exit with code 0 if valid, 1 if errors found.

    **Analyze command implementation:**
    1. Read input file bytes.
    2. Load mesh via `slicecore_fileio::load_mesh()`.
    3. Compute stats via `slicecore_mesh::compute_stats()`.
    4. Print: vertex count, triangle count, bounding box, volume, surface area, manifold/watertight status.

    **Error handling:** Use `std::process::exit(1)` for errors. Print human-readable error messages to stderr. No anyhow/eyre dependency for Phase 3 -- just `eprintln!` and exit.

    **Tests:** Integration tests are deferred to plan 03-06. This plan verifies the binary compiles and help text works:
    - `cargo build -p slicecore-cli` succeeds.
    - The binary is created at `target/debug/slicecore-cli` (or configured binary name).
