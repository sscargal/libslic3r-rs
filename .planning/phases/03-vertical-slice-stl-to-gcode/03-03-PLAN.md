    **surface.rs**: Classify which layers need solid infill (top/bottom surfaces).

    **Core types:**
    ```rust
    /// Surface classification for a layer's infill regions.
    pub struct SurfaceClassification {
        pub solid_regions: Vec<ValidPolygon>,   // Regions needing 100% solid infill
        pub sparse_regions: Vec<ValidPolygon>,  // Regions using configured infill density
    }
    ```

    **Main function:** `classify_surfaces(layers: &[SliceLayer], layer_index: usize, top_layers: u32, bottom_layers: u32) -> SurfaceClassification`

    Algorithm (simplified for Phase 3):
    1. Get the current layer's contours.
    2. **Bottom detection**: For the first `bottom_layers` layers (layer_index < bottom_layers), the entire layer is solid. Return all contours as solid_regions, empty sparse_regions.
    3. **Top detection**: For the last `top_layers` layers (layer_index >= total_layers - top_layers), the entire layer is solid.
    4. **Interior layers**: Use polygon boolean operations to detect exposed surfaces:
       - Compute the "above region" = intersection of current layer contours with layer above (layer_index + 1). If no layer above, current layer is top.
       - "Top surface" = current layer region MINUS the above region (parts with nothing above).
       - Similarly for bottom: "below region" = intersection with layer below. "Bottom surface" = current MINUS below.
       - Repeat for N layers deep (check N layers above for top, N layers below for bottom).
       - For simplicity in Phase 3: just check 1 layer above/below. If the above/below layer has identical contours (same shape), mark as interior (sparse). If different shape or missing, mark regions as solid.
    5. Combine: solid_regions = union of top and bottom surface regions. sparse_regions = current contours MINUS solid_regions.
    6. If the boolean operations fail or produce empty results, fall back to treating the entire layer as sparse (safe default -- interior layers get pattern infill).

    **extrusion.rs**: Implement E-axis value computation.

    ```rust
    /// Compute the cross-sectional area of an extrusion bead (mm^2).
    /// Uses the Slic3r model: rectangle with semicircular ends.
    pub fn extrusion_cross_section(width: f64, height: f64) -> f64

    /// Compute E-axis value for a linear move (relative extrusion, M83).
    pub fn compute_e_value(
        move_length_mm: f64,
        extrusion_width: f64,
        layer_height: f64,
        filament_diameter: f64,
        extrusion_multiplier: f64,
    ) -> f64

    /// Compute the distance between two points in mm.
    pub fn move_length(from: (f64, f64), to: (f64, f64)) -> f64
    ```

    Formulas (from research):
    - `cross_section = (width - height) * height + PI * (height/2)^2`
    - `volume = cross_section * move_length`
    - `filament_area = PI * (filament_diameter/2)^2`
    - `e = volume / filament_area * extrusion_multiplier`

    **Tests for surface.rs:**
    - A 5-layer stack where all layers have identical contours: layers 0,1 are bottom-solid (bottom_layers=2), layers 3,4 are top-solid (top_layers=2), layer 2 is sparse.
    - A single-layer slice: entire layer is solid (both top and bottom).
    - Two layers: both are solid (overlapping top and bottom classification).

    **Tests for extrusion.rs:**
    - `extrusion_cross_section(0.4, 0.2)` produces expected area (~0.071 mm^2).
    - `compute_e_value(10.0, 0.44, 0.2, 1.75, 1.0)` produces a reasonable E value (~0.0295).
    - E value scales linearly with move_length.
    - E value scales with extrusion_multiplier.
    - Zero move_length produces zero E.
    - `move_length((0.0, 0.0), (3.0, 4.0))` returns 5.0.

    Define the intermediate representation between geometry (contours, infill lines) and G-code.

    **toolpath.rs:**

    ```rust
    /// The type of feature being printed (affects speed, extrusion settings).
    #[derive(Debug, Clone, Copy, PartialEq, Eq)]
    pub enum FeatureType {
        OuterPerimeter,
        InnerPerimeter,
        SolidInfill,
        SparseInfill,
        Skirt,
        Brim,
        Travel,  // Non-extrusion move
    }

    /// A single extrusion or travel segment.
    #[derive(Debug, Clone)]
    pub struct ToolpathSegment {
        pub start: Point2,          // Start position in mm
        pub end: Point2,            // End position in mm
        pub feature: FeatureType,   // What is being printed
        pub e_value: f64,           // E-axis value (0.0 for travel)
        pub feedrate: f64,          // mm/min
        pub z: f64,                 // Z height in mm
    }

    /// All toolpath segments for a single layer, in print order.
    pub struct LayerToolpath {
        pub layer_index: usize,
        pub z: f64,
        pub layer_height: f64,
        pub segments: Vec<ToolpathSegment>,
    }

    impl LayerToolpath {
        /// Estimate the time to print this layer in seconds.
        pub fn estimated_time_seconds(&self) -> f64
    }
    ```

    **Assembly function:** `assemble_layer_toolpath(layer_index: usize, z: f64, layer_height: f64, perimeters: &[ContourPerimeters], infill: &LayerInfill, config: &PrintConfig) -> LayerToolpath`

    This function converts geometric perimeters and infill into ordered ToolpathSegments:
    1. Determine speed for this layer: if layer_index == 0, use first_layer_speed for everything; otherwise use per-feature speeds from config.
    2. For each contour's perimeters (in shell order per wall_order):
       a. For each shell, convert polygon points to sequential ToolpathSegments.
       b. Mark outer shells as `OuterPerimeter`, inner shells as `InnerPerimeter`.
       c. Compute E-axis value for each segment using `compute_e_value()`.
       d. Insert Travel segments between disconnected paths.
    3. For infill lines:
       a. Convert each InfillLine to a ToolpathSegment with `SolidInfill` or `SparseInfill` feature type.
       b. Order infill lines by nearest-neighbor (start from the end of the last line, find the closest next-line start). This simple heuristic minimizes travel moves.
       c. Insert Travel segments between infill lines that are not connected.
    4. Convert IPoint2 coordinates to Point2 (mm) using `coord_to_mm()`.

    **estimated_time_seconds**: Sum of `segment_length / feedrate_mm_per_sec` for all segments (feedrate is stored in mm/min, so divide by 60).

    **Tests:**
    - assemble_layer_toolpath with known perimeters and infill produces segments in correct order (perimeters first, then infill).
    - Travel segments are inserted between disconnected paths.
    - E-values are non-zero for extrusion segments, zero for travel.
    - First layer uses first_layer_speed.
    - Subsequent layers use per-feature speeds.
    - estimated_time_seconds returns a positive value for a layer with segments.
