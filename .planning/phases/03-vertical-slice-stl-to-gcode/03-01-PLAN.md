    **Cargo.toml**: Add `slicecore-slicer` as a workspace member (already handled by `crates/*` glob pattern). Create `crates/slicecore-slicer/Cargo.toml` with dependencies on `slicecore-math` (path), `slicecore-mesh` (path), `slicecore-geo` (path), and `serde` (workspace).

    **contour.rs**: Implement two core functions:

    1. `intersect_triangle_z_plane(v0: Point3, v1: Point3, v2: Point3, z: f64) -> Option<(Point2, Point2)>`:
       - Classify each vertex as above/on/below the Z-plane using signed distance `d = v.z - z`.
       - Find edges that cross the plane (sign changes in d values between endpoints).
       - For crossing edges, linearly interpolate: `t = d_a / (d_a - d_b)`, then `x = va.x + t * (vb.x - va.x)`, `y = va.y + t * (vb.y - va.y)`.
       - Handle vertex-on-plane cases (d.abs() < 1e-12): treat the vertex as an intersection point.
       - Return None for degenerate cases (coplanar triangle, single-vertex touch). Return Some((start, end)) when exactly 2 distinct intersection points are found.
       - Use `Point2` (float) for the intersection points since they come from float vertex coordinates.

    2. `chain_segments(segments: Vec<(IPoint2, IPoint2)>) -> Vec<Vec<IPoint2>>`:
       - Convert input segments from float to integer coordinates immediately using `IPoint2::from_mm()` for exact endpoint matching (absorbs floating-point jitter).
       - Build an adjacency map: `HashMap<IPoint2, Vec<(IPoint2, usize)>>` mapping start points to (end point, segment index).
       - Walk chains: for each unused segment, follow the adjacency graph until the chain closes (current == first point) or no unused continuation exists.
       - Collect contours with >= 3 points. Log/skip open chains (mesh defects) rather than panicking.
       - IMPORTANT: Use `HashMap` here (not BTreeMap) since the iteration order for segment selection doesn't affect the output -- each segment has a unique start->end mapping from the mesh topology. The chains themselves are deterministic because the adjacency walk is deterministic.

    3. `slice_at_height(mesh: &TriangleMesh, z: f64) -> Vec<ValidPolygon>`:
       - Use `query_triangles_at_z(mesh, z)` to get candidate triangle indices from BVH.
       - For each candidate triangle, call `intersect_triangle_z_plane()`.
       - Collect all segments, convert to IPoint2 pairs.
       - Call `chain_segments()` to assemble closed contours.
       - For each contour, create a `Polygon` from the IPoint2 points, then call `.validate()` to get `ValidPolygon`.
       - Classify winding: CCW = outer boundary, CW = hole (per established convention).
       - Return Vec<ValidPolygon>.

    **layer.rs**: Define the SliceLayer type and the main slicing function:

    ```rust
    pub struct SliceLayer {
        pub z: f64,                          // Z height in mm
        pub layer_height: f64,               // Height of this layer in mm
        pub contours: Vec<ValidPolygon>,      // Outer boundaries and holes
    }

    pub fn compute_layer_heights(aabb: &BBox3, layer_height: f64, first_layer_height: f64) -> Vec<(f64, f64)>
    ```
    - `compute_layer_heights`: Returns Vec of (z_height, this_layer_height) tuples. First layer at z = first_layer_height / 2 (midpoint of first layer). Subsequent layers increment by `layer_height`. Continue until z exceeds `aabb.max.z`. Each tuple records both the Z height and the layer thickness.

    ```rust
    pub fn slice_mesh(mesh: &TriangleMesh, layer_height: f64, first_layer_height: f64) -> Vec<SliceLayer>
    ```
    - Compute layer heights from mesh AABB.
    - For each height, call `slice_at_height()`.
    - Collect into Vec<SliceLayer>, sequential (no parallelism).

    **lib.rs**: Re-export `SliceLayer`, `slice_mesh`, `compute_layer_heights`, and `slice_at_height`.

    **Tests**: Include unit tests:

    - `intersect_triangle_z_plane` with triangle fully above/below returns None.

    - `chain_segments` with 4 segments forming a square produces 1 contour with 4 points.

    - `compute_layer_heights` for a 1mm tall object at 0.2mm layer height produces 5 layers.
    - `slice_mesh` on a unit cube produces layers with square contours at each height.
