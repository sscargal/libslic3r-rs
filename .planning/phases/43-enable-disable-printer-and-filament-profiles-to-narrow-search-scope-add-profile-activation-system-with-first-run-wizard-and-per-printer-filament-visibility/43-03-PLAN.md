**2. Create `crates/slicecore-cli/src/profile_wizard.rs`:**

This module contains all interactive wizard logic, separated from command routing for testability.

```rust
//! Interactive profile setup wizard and pickers.
//!
//! Provides the first-run wizard (vendor -> printer -> filament flow),
//! interactive enable/disable pickers, and slicer detection for import suggestion.
```

**Key functions to implement:**

**`run_setup_wizard(profiles_dir: Option<&Path>, reset: bool) -> Result<(), anyhow::Error>`**
- The main wizard entry point
- If `reset`: create fresh `EnabledProfiles::default()`. Otherwise: load existing via `EnabledProfiles::load()` or default if missing
- Create `ProfileResolver::new(profiles_dir)` and get index via `resolver.index()`
- If index is None or index has no profiles: call `suggest_import()` and return early
- Call `wizard_select_printers(&index, &mut enabled)` for vendor -> model flow
- Call `wizard_select_filaments(&index, &enabled)` for filament selection with compatibility
- Call `wizard_auto_enable_process(&index, &mut enabled)` for auto-enabling process profiles
- Save to `EnabledProfiles::default_path()`
- Print summary: "Setup complete! {N} machines, {N} filaments, {N} process profiles enabled."

**`run_setup_noninteractive(machines: &[String], filaments: &[String], processes: &[String], profiles_dir: Option<&Path>, reset: bool) -> Result<(), anyhow::Error>`**
- Non-interactive path for `profile setup --machine X --filament Y`
- Load or create EnabledProfiles (reset if --reset)
- For each machine/filament/process ID: resolve via ProfileResolver to validate it exists, then enable
- If processes is empty and machines is non-empty: auto-enable process profiles for those machines
- Save and print summary

**`wizard_select_printers(index: &ProfileIndex, enabled: &mut EnabledProfiles) -> Result<(), anyhow::Error>`**
- Extract unique vendors from machine profiles: `index.profiles.iter().filter(|p| p.profile_type == "machine").map(|p| p.vendor.clone()).collect::<HashSet<_>>()`
- Sort vendors alphabetically
- Use `dialoguer::MultiSelect::new().with_prompt("Select printer vendor(s)").items(&vendors).interact()?`
- For each selected vendor, get machine profiles for that vendor
- Use `dialoguer::MultiSelect::new().with_prompt(&format!("Select {vendor} printer(s)")).items(&machine_names).interact()?`
- For each selected machine, call `enabled.enable("machine", &machine.name)`
- Pre-check currently enabled machines in the multi-select if re-running setup

**`wizard_select_filaments(index: &ProfileIndex, enabled: &EnabledProfiles) -> Result<EnabledProfiles, anyhow::Error>`**
- Get enabled machine IDs from `enabled.machine.enabled`
- Build `CompatibilityInfo::from_index_entries(&machine_ids, index)` to find compatible filaments
- Get filament entries from index, filter by compatibility
- If compatible filaments exist: use `dialoguer::MultiSelect` with all pre-selected (Enter for all). Prompt: "Select filaments (Enter to accept all compatible)"
- For each selected: `enabled.enable("filament", &filament.name)`
- Return modified enabled

**`wizard_auto_enable_process(index: &ProfileIndex, enabled: &mut EnabledProfiles) -> Result<(), anyhow::Error>`**
- Get process profiles from index (profile_type == "process")
- For enabled machines, find process profiles matching by printer_model or vendor
- Auto-enable matching process profiles
- If no matches, enable all "generic" process profiles (vendor-neutral ones)
- Print to stderr: "Auto-enabled {N} process profiles for your printers."

**`suggest_import() -> Result<(), anyhow::Error>`**
- Call `detect_installed_slicers()` to find installed slicers
- If found: print suggestion with exact command: `"Found {slicer} at {path}. Run:\n  slicecore import-profiles --source-dir {path}"`
- If none found: print generic message: `"No profile library found. Import profiles first:\n  slicecore import-profiles --source-dir /path/to/OrcaSlicer/resources/profiles"`
- Return Ok(()) -- caller handles the early exit

**`detect_installed_slicers() -> Vec<(&'static str, PathBuf)>`**
- Check common paths for OrcaSlicer, PrusaSlicer, BambuStudio
- Linux: `~/.config/OrcaSlicer/system`, `~/.config/PrusaSlicer/vendor`
- macOS: `~/Library/Application Support/OrcaSlicer/system`, etc.
- Return vec of (name, path) pairs for found installations

**`run_enable_picker(profile_type: Option<&str>, profiles_dir: Option<&Path>) -> Result<Vec<String>, anyhow::Error>`**
- Load index, get all profiles (optionally filtered by type)
- Load EnabledProfiles, filter out already-enabled profiles
- Use `dialoguer::MultiSelect` to let user pick from disabled profiles
- Return list of selected profile names (caller does the enable + save)

**`run_disable_picker(profile_type: Option<&str>, profiles_dir: Option<&Path>) -> Result<Vec<String>, anyhow::Error>`**
- Load EnabledProfiles
- Show enabled profiles in a `dialoguer::MultiSelect` (optionally filtered by type)
- Return list of selected profile names to disable (caller does the disable + save)

**CRITICAL: Guard all dialoguer calls** with `std::io::stdin().is_terminal()` check (use `std::io::IsTerminal`). If not a TTY, return an error: `anyhow::bail!("Interactive mode requires a terminal. Use --machine and --filament flags for non-interactive setup.")`

All public functions should have rustdoc comments.

// Check if profile setup is needed (first-run wizard trigger)
if use_profile_workflow {
    let enabled_path = slicecore_engine::enabled_profiles::EnabledProfiles::default_path();
    if let Some(ref path) = enabled_path {
        if !path.exists() && !force {
            if std::io::IsTerminal::is_terminal(&std::io::stdin()) {
                eprintln!("No profiles enabled yet. Starting setup wizard...");
                eprintln!("(Use --force to skip, or run 'slicecore profile setup' manually)\n");
                if let Err(e) = crate::profile_wizard::run_setup_wizard(profiles_dir, false) {
                    eprintln!("Setup wizard failed: {e}");
                    eprintln!("Run 'slicecore profile setup' manually or use --force to skip.");
                    std::process::exit(1);
                }
            } else {
                eprintln!("Error: No enabled profiles found.");
                eprintln!("Run: slicecore profile setup --machine <id> --filament <id>");
                eprintln!("Or use --force to proceed without profile activation.");
                std::process::exit(1);
            }
        }
    }
}
