```rust
/// Enable one or more profiles by ID.
///
/// Auto-detects profile type from library index metadata.
/// Omit IDs to launch interactive picker (requires terminal).
Enable {
    /// Profile IDs to enable (omit for interactive picker)
    ids: Vec<String>,

    /// Profile type filter for interactive picker
    #[arg(long)]
    r#type: Option<String>,

    /// Output as JSON
    #[arg(long)]
    json: bool,

    /// Override profiles directory
    #[arg(long)]
    profiles_dir: Option<PathBuf>,
},

/// Disable one or more profiles.
///
/// Omit IDs to launch interactive picker showing enabled profiles.
Disable {
    /// Profile IDs to disable (omit for interactive picker)
    ids: Vec<String>,

    /// Profile type filter for interactive picker
    #[arg(long)]
    r#type: Option<String>,

    /// Output as JSON
    #[arg(long)]
    json: bool,

    /// Override profiles directory
    #[arg(long)]
    profiles_dir: Option<PathBuf>,
},

/// Show enabled profile summary.
///
/// Displays count of enabled profiles by type (machine, filament, process).
Status {
    /// Output as JSON
    #[arg(long)]
    json: bool,

    /// Override profiles directory
    #[arg(long)]
    profiles_dir: Option<PathBuf>,
},
```

**3. Modify the existing `List` variant** to add activation filter flags:

Add these three fields to the `List` variant:
```rust
/// Show only enabled profiles (default when enabled-profiles.toml exists)
#[arg(long)]
enabled: bool,

/// Show only disabled profiles
#[arg(long)]
disabled: bool,

/// Show all profiles regardless of activation status
#[arg(long)]
all: bool,
```

**4. Add match arms in `run_profile_command`** for the three new variants:

**cmd_enable implementation:**
- Load EnabledProfiles from default path (or create new if file doesn't exist, using `EnabledProfiles::load(path)?.unwrap_or_default()`)
- For each ID in `ids`:
  - Resolve via `ProfileResolver::new(profiles_dir.as_deref())` to find the profile and determine its type
  - Try resolving against each type ("machine", "filament", "process") if `--type` not specified. Use the first successful resolution's `profile_type` field
  - If `--type` is specified, resolve only against that type
  - Call `enabled.enable(&resolved.profile_type, &resolved.name)`
  - Print to stderr: `"Enabled {type} profile: {name}"` (or JSON object if --json)
- If `ids` is empty: print to stderr `"Interactive picker not yet implemented. Specify profile IDs."` and `process::exit(1)` (Plan 03 adds the interactive path)
- Save EnabledProfiles to default path
- If --json: print `{"enabled": [...list of enabled IDs...]}` to stdout

**cmd_disable implementation:**
- Load EnabledProfiles from default path. If file doesn't exist, print "No profiles are enabled." and return Ok(())
- For each ID in `ids`:
  - Call `enabled.disable("machine", id)`, `enabled.disable("filament", id)`, `enabled.disable("process", id)` (disable from all sections -- safe since disable is no-op on missing)
  - Alternatively if `--type` given, only disable from that section
  - Print to stderr: `"Disabled profile: {name}"`
- If `ids` is empty: print to stderr `"Interactive picker not yet implemented. Specify profile IDs."` and `process::exit(1)`
- Save EnabledProfiles to default path
- If --json: print `{"disabled": [...list of disabled IDs...]}` to stdout

**cmd_status implementation:**
- Load EnabledProfiles from default path
- If None: print `"No profiles enabled. Run 'slicecore profile setup' to get started."` and return
- Get counts via `enabled.counts()`
- Plain output to stdout:
  ```
  Profile activation status:
    Machines:  {N} enabled
    Filaments: {N} enabled
    Process:   {N} enabled
  ```
- If --json: print `{"machine_count": N, "filament_count": N, "process_count": N, "machine": [...], "filament": [...], "process": [...]}` to stdout

**5. Modify the existing List match arm:**
- After getting profiles from resolver, apply activation filtering:
  - Determine filter mode: if `--all` flag: pass `None` to filter_enabled. If `--enabled`: load EnabledProfiles and pass Some. If `--disabled`: load EnabledProfiles, filter to those NOT in enabled. If no flag: default to `--enabled` when `enabled-profiles.toml` exists, `--all` when it doesn't
  - When filtering is active and no profiles exist, show hint: `"No enabled profiles. Run 'slicecore profile setup' or use --all to see everything."`
  - Apply `ProfileResolver::filter_enabled(&all_profiles, enabled_opt.as_ref())` before displaying

**6. Update module doc comment** at the top of the file to list the new subcommands.
